Architectural Overview (as of STAGE-001-011)

Single-process FastAPI backend with pluggable async collectors, AES-GCM encrypted secrets, cookie+token auth, and SQLite persistence. React UI and external sidecars (VictoriaMetrics, VictoriaLogs, Alertmanager) are scaffolded but not yet active.

1. High-level architecture

A single Python process built on FastAPI hosts the kernel and all collectors. Five kernel modules (auth scheduler plugins db api) compose the backend. State lives in one SQLite database via aiosqlite + SQLAlchemy. Future sidecars (VictoriaMetrics, VictoriaLogs, Alertmanager, Grafana) are contracted via MetricsWriter / LogsWriter Protocols, today implemented in-memory.

2. Boot sequence (lifespan)

The FastAPI lifespan context manager (kernel/api/lifespan.py) runs a hybrid abort-or-degrade bootstrap. Critical failures (no master key, pending migrations) call os._exit(1); recoverable failures (e.g., one collector failing to register) are logged and the process continues with that collector marked degraded.

Shutdown is the reverse: scheduler.stop() → cancel TTL refresh task → close httpx.AsyncClient → dispose_engine().

3. Request flow & middleware

Three middlewares wrap every request. Registration order is reversed by Starlette: the last add_middleware call is the outermost layer (fires first). The actual runtime flow is:

Auth precedence (locked, see middleware.py):
  1. Authorization: Bearer <token> → API token (CSRF-immune)
  2. Else, homelab_monitor_session cookie → session (CSRF enforced on POST/PUT/PATCH/DELETE)
  3. Else, unauthenticated. Route's Depends decides 401.
Both present? Token wins; the session cookie is ignored.

Auth-exempt paths (no resolution attempted, even if cookies/headers present): /api/healthz, /api/version, /api/openapi.json, /api/docs, /api/redoc, /api/auth/login. Notably /api/auth/logout is NOT exempt — it needs the resolved session to know which row to delete.

4. Auth model

Session cookie

Cookie value format: <session_id>.<hmac_hex> (32 hex chars + . + 32 hex chars = 65 chars). HMAC-SHA256 truncated to 16 bytes; key derived from the master key via HKDF-SHA256 with info=b"homelab.session.v1.hmac". Verification is constant-time. Session row carries csrf_token, expires_at, created_ip.

CSRF (double-submit cookie)

At login the server sets homelab_monitor_csrf alongside the session cookie (httponly=False so JS can read it). Browser echoes it in X-CSRF-Token on state-changing requests. The require_session / require_user_or_token dependencies compare the header to session.csrf_token in constant time.

API tokens

Format: homelab_<env_prefix>_<40-char base64url body> (raw body is 30 random bytes from secrets.token_bytes). Only the SHA-256 hex hash is stored (api_tokens.hash with a UNIQUE index, added in migration 0004). Plaintext is shown to the operator once at create time and never persisted. last_used_at bumped best-effort per request.

Per-route dependencies

DependencyAcceptsNotesUsed on

require_user_or_token(scopes) Either cookie session OR Bearer token with ANY of the scopes Session path enforces CSRF; token path checks scope intersection (no production routes yet — staged for hybrid ingest endpoints)

Login rate limiter

InProcessLoginRateLimiter implements the LoginRateLimiter Protocol: sliding window of 5 attempts per 5 minutes per source IP, in-memory only (lost on restart — acceptable per locked decision D4). Only failed logins consume budget; the same limiter is shared by the password-reverify path of change-password.

Password storage

bcrypt with cost factor 12 by default (configurable via HOMELAB_MONITOR_BCRYPT_COST, range 4–20; tests use 4). Minimum length 12 chars enforced at hash time. verify_user_password is constant-time and never leaks the cause of failure.

5. Scheduler tick lifecycle

The scheduler runs one asyncio.Task per collector as a tick loop with absolute deadlines. After each scheduled tick the deadline advances by interval; if the loop has fallen more than one interval behind it skips forward to now + interval instead of firing back-to-back catch-up ticks. An immediate-run queue (driven by POST /api/collectors/{name}/retry) is polled every 50 ms, so retries feel interactive without spinning.

The 6 self-metrics

  • homelab_collector_run_success_total — counter, label name
  • homelab_collector_run_failure_total — counter, labels name, reason (timeout / exception / result_error)
  • homelab_collector_run_shutdown_total — counter, label name (CancelledError during stop())
  • homelab_collector_run_skipped_total — counter, labels name, reason (quarantined / group_busy)
  • homelab_collector_run_duration_seconds — summary, label name
  • homelab_collector_run_last_error_age_seconds — gauge, label name (set to 0 at error, then rises)

FailureBudget & quarantine

Tracks consecutive failures per collector (default threshold 5; per-collector override via CollectorConfig.quarantine_after). At threshold the collector is marked quarantined in the collectors table (3 columns: consecutive_failures, quarantined_at, quarantine_reason — added by migration 0003) and an audit_log row collector.quarantine_entered is written in the same transaction. Successful runs reset the in-memory counter; they do not auto-clear quarantine. Operators clear via POST /api/collectors/{name}/retry, which also enqueues an immediate run.

Concurrency groups

Collectors with a non-default concurrency_group share a single asyncio.Lock. Lock acquisition has an interval/2 deadline; on timeout the tick is skipped with reason group_busy.

6. Plugin model

A collector is anything satisfying the Collector Protocol — six ClassVars (name, interval, timeout, concurrency_group, run_kind, trust_level) plus an async run(ctx) → CollectorResult. Most collectors subclass the BaseCollector ABC for the ergonomic defaults (run_kind=ASYNC, trust_level=BUILTIN, concurrency_group="default").

The 4 RunKinds

ASYNC default

Awaited directly on the FastAPI event loop. Best for I/O-bound work (HTTP, async DB drivers). Shares http_client + DB handle with other collectors. Fastest, but a slow collector blocks no one (event-loop semantics) except via the concurrency group lock.

THREAD

Driven by a worker thread via loop.run_in_executor(thread_pool, …). For sync libraries (paramiko, certain SNMP libs). The collector instance is shared with the loop side, so any self-state survives across ticks. ThreadPoolExecutor bounded to 4 workers (thread_pool_size).

PROCESS

Runs in a worker subprocess from a ProcessPoolExecutor with mp_context="forkserver" (default size min(2, os.cpu_count())). The worker receives only a pickle-safe ProcessCollectorContext (config, secrets, BufferingMetricsWriter); no DB / HTTP / SSH / log handle. Buffered metrics are replayed parent-side after the future resolves.

SUBPROCESS

Spawned OS process per tick driven by subprocess_runner.run_subprocess. The plugin lives outside Python: any executable described by a plugin.yaml manifest. Communication via JSON-line stdout (one event per line). Secrets injected as env vars; tick context propagated as HOMELAB_TICK_ID, HOMELAB_TRIGGER_KIND, HOMELAB_REQUEST_ID. Manifest declares which secrets the plugin sees (secrets: allowlist) and which trust level it runs at.

Trust tiers

  • BUILTIN — ships in the monorepo; full kernel access.
  • TRUSTED — third-party but vetted; full kernel access.
  • UNTRUSTED — forced into subprocess execution; no DB writes; only declared secrets visible. (Manifest restricts to TRUSTED|UNTRUSTED — UNTRUSTED is the default for unvetted manifests.)

Loader pipeline

PluginLoader is an in-process registry. register(cls, overrides) validates a CollectorConfig via Pydantic and appends a LoadedCollector(collector, config) to the list. load_subprocess_plugins(plugins_dir) walks the tree for plugin.yaml files, validates each SubprocessManifest, verifies the command is executable, and registers a generated BaseCollector subclass per manifest. After all collectors are registered, persist_to_db(repo) upserts a row per collector into the collectors table so FailureBudget UPDATEs have a target.

7. Secrets layer

Each secret is encrypted with AES-256-GCM using a per-row key derived from the master key via HKDF-SHA256 (info = b"homelab-monitor/secrets/v1/" + row_id, 16-byte random salt, 12-byte random nonce). Storage shape per row: id, name (unique), created_at, kdf_salt (BLOB), ciphertext (base64 of nonce || aead_payload), rotated_at.

Master key bootstrap. Loaded at process start: env var HOMELAB_MONITOR_MASTER_KEY first, then the file at /run/secrets/master-key (override via HOMELAB_MONITOR_MASTER_KEY_FILE in tests). Both inputs must be base64-encoded and decode to exactly 32 bytes. master_key_fingerprint(key) exposes a stable HMAC-SHA256 identifier for rotation detection without leaking key bytes.

Audit atomicity. Every mutating call (set, rotate, delete, rotate_master) writes data + audit row inside a single repo.transaction(). The audit INSERT is issued directly against that connection (NOT via the standalone audit_write), so a failed audit rolls back the secret op.

SyncSecretsResolver. A frozen dataclass that wraps a decrypted snapshot ({name → plaintext}) behind a MappingProxyType. Pickle-safe via a custom __reduce__ so it can cross subprocess boundaries. filtered(declared_names) returns a narrower resolver scoped to a plugin's manifest declarations.

TtlCachingSecretsResolver. Decrypts a fresh snapshot at startup and on a 60-second loop, swapping atomically. Collectors call ttl_resolver.current() synchronously each tick — no await, no DB hit. Operator rotations propagate on the next 60 s tick without restart.

8. Database schema

Async SQLAlchemy + aiosqlite driver, single file at HOMELAB_MONITOR_DB_URL. Alembic manages schema; migrations run automatically at boot when HOMELAB_MONITOR_AUTO_MIGRATE=true (default). 9. API surface

MethodPathAuthDescription

GET /api/events session Server-Sent Events stream of collector.tick events. Replays the last 50 from the ring buffer on connect. Slow subscribers (queue full at 64) get an error event and are dropped.
GET /api/openapi.json, /api/docs, /api/redoc none FastAPI-generated OpenAPI schema + Swagger / Redoc UIs.

Error envelope

All non-2xx responses use the uniform shape {"error": {"code": "...", "message": "...", "details": {...}}}. Domain exceptions (HttpProblem, UnauthenticatedProblem, InsufficientScopeProblem, CsrfMismatchProblem, RateLimitedProblem, WrongPasswordProblem, WeakPasswordProblem, NotFoundProblem, DependencyUnavailableProblem) carry the status code; a generic exception handler converts MigrationsPendingError, MasterKeyError, and SecretIntegrityError into 503 dependency_unavailable.

SSE broker

SseBroker implements the EventSink Protocol. Per-subscriber asyncio.Queue(maxsize=64); ring buffer of last 50 events for replay-on-connect. Each event carries a monotonic seq (used as the SSE id: field) and a SchedulerTickEvent payload. publish() is contract-bound to never raise; failures are logged and swallowed.

Logging

structlog with contextvars merge + log-level + ISO timestamp. Renderer: JSONRenderer(sort_keys=True) in production, ConsoleRenderer with rich tracebacks when HOMELAB_MONITOR_LOG_FORMAT=pretty. Stdlib loggers (uvicorn, uvicorn.error, uvicorn.access, httpx, alembic) are piped through ProcessorFormatter so all output is uniform JSON. request_id and tick_id are bound on contextvars and merged into every log line during their respective scopes.

12. What's NOT yet built

  • Real metrics writer — InMemoryMetricsWriter is the only implementation today; vmagent/VictoriaMetrics arrives in STAGE-015.
  • Real logs writer — InMemoryLogsWriter placeholder; vector → VictoriaLogs in STAGE-016.
  • Alertmanager + vmalert — alert ingestion endpoint, alerting pipelines, deduplication. Reserved for STAGE-017+.
  • Heartbeat ingestion + alert ingestor — token scopes heartbeat:write and alerts:ingest:write exist but have no live route. STAGE-013.
  • SSH client factory — _StubSshFactory.open raises NotImplementedError. EPIC-017.
  • Home Assistant client — ctx.ha is always None today. Future stage.
  • Auto-fix subsystem — runbooks, runbook_runs tables are stubs; safety-model decisions are locked in project_autofix_safety_model.md but no execution path yet.
  • React UI — apps/ui is scaffolded (Vite + TS strict) but no real components. STAGE-014 lights up the bootstrap + collectors page.
  • PROCESS-pool resilience — a worker hard-crash breaks the executor until scheduler restart. FailureBudget quarantine bounds the blast radius; pool rebuild is deferred.
  • Pool size dynamic reconfig — SchedulerConfig snapshots configs at __init__; collector reload not yet wired.

13. Key invariants

  • Audit atomicity. Every state-changing kernel op writes its data row AND an audit_log row in the same transaction via insert_audit(conn, …). audit_write() (one-shot) exists for boot-time / non-transactional callers but is the exception. A failed audit rolls the whole op back.
  • UTC everywhere. utc_now_iso() is the single source of timestamps. The display layer converts to America/New_York (configurable).
  • Slots ordering. CollectorContext is a slots=True dataclass — optional fields (only ha) MUST come last to satisfy the field-default rule.
  • Structured logging is universal. structlog JSON in prod; stdlib loggers (uvicorn, httpx, alembic) are piped through the same processor chain.
  • Contextvar propagation. request_id (RequestId middleware) and tick_id (Scheduler _tick) are bound on structlog.contextvars for the duration of the scope and explicitly reset in a finally block to prevent cross-task leakage.
  • Streaming endpoints MUST have an auth dep. GET /api/events uses require_session at the route, NOT just middleware (middleware never enforces).
  • Plugins are observable about themselves. Six homelab_collector_run_* metrics are emitted unconditionally per tick by the scheduler — collectors don't get to opt out.
  • Master-key fingerprint never reveals key bytes. HMAC-SHA256 of a domain-separator under the key; safe to log and display to operators.
  • Token plaintext is shown once. Only the SHA-256 hash hits the DB; api_tokens.hash has a UNIQUE index.
  • Scheduler never reuses a stopped instance. start() after stop() raises RuntimeError — the lifespan context manager owns the single instance.
