## Conventions

### Authentication

Two cookies are set by the backend on successful login:

- `homelab_monitor_session` — httpOnly session token (never readable by JS)
- `homelab_monitor_csrf` — httpOnly=false, readable by JS

The `apiClient` in `api/client.ts` always sends `credentials: 'include'`. A CSRF middleware reads `homelab_monitor_csrf` and injects `X-CSRF-Token` on every POST, PUT, PATCH, and DELETE.

### Error Handling

`unwrap(result)` in `api/client.ts` converts openapi-fetch's `{ data, error, response }` tuple into either the unwrapped data or a thrown `ApiError`. `ApiError` carries `status`, `code`, `message`, `details`, and `retryAfterSeconds`.

Exception: `useCurrentUser` and `ensureAuthenticated` special-case HTTP 401 to return `null` rather than throwing, so route guards can redirect cleanly without a try/catch.

### Routing

Routes are defined code-first in `router.tsx`. All authenticated screens must be children of `protectedLayoutRoute`, whose `beforeLoad` calls `ensureAuthenticated(queryClient)`. That function checks the TanStack Query cache before hitting the network, so navigating between protected routes does not re-fetch.

`/` redirects to `/overview` via `indexRoute`.

Public routes (currently only `/login`) are direct children of `rootRoute`.

### TanStack Query Hooks

All hooks live in `api/queries.ts`. Available hooks:

| Hook | Query key | Endpoint |
|---|---|---|
| `useCurrentUser()` | `['auth', 'me']` | `GET /api/auth/me` |
| `useVersion()` | `['version']` | `GET /api/version` |
| `useCollectors()` | `['collectors']` | `GET /api/collectors` |
| `useMetricsSnapshot()` | `['metrics', 'snapshot']` | `GET /api/metrics/snapshot` |
| `useAlerts(params?)` | `['alerts', params]` | `GET /api/alerts` |
| `useLogin()` | mutation | `POST /api/auth/login` |
| `useLogout()` | mutation | `POST /api/auth/logout` |

`queryKeys` is exported for use in `invalidateQueries` calls.

### SSE

```ts
const { value, status, failureCount, reconnect } = useSSE<MyType>({
  topic: 'my-event-name',   // matches the SSE event type the server emits
  parser: (event) => JSON.parse(event.data) as MyType,
  // endpoint defaults to '/api/events'
  // factory: optional EventSource constructor injection (tests only)
})
```

- `status`: `'connecting' | 'open' | 'error'`
- Reconnects with exponential backoff starting at 500 ms, capped at 30 s.
- `reconnect()` resets backoff and forces an immediate reopen.
- The server emits SSE comment-line keepalives every 15 s to prevent proxy timeouts.

### Theme

Dark mode is the default. `index.html` contains an inline script that reads `localStorage['homelab-monitor:theme']` and sets `document.documentElement.dataset.theme` before React mounts, preventing flash of unstyled content. After mount, `AppShell` takes over persistence via `localStorage`.

CSS classes use the `[data-theme='light']` selector for light overrides; `@theme` tokens are defined in `index.css`.

### Testing

- Environment: jsdom via vitest config.
- Setup file: `src/test/setup.ts` imports `@testing-library/jest-dom/vitest` and polyfills `window.matchMedia`.
- Call `cleanup()` in `afterEach` (auto-imported from `@testing-library/react`).
- Coverage thresholds: **80% lines / 75% branches / 80% functions / 80% statements** (not 100%; the backend kernel holds 100%).
- `src/api/schema.ts` and `src/components/ui/**` are excluded from coverage.
- Mirror the `Login.test.tsx` pattern: mock `useCurrentUser`/`useVersion` as needed, render, assert.

## How to Add a New Screen

### 1. Create the page component

```
apps/ui/src/routes/MyScreen.tsx
```

Export a named component, e.g. `MyScreenPage`. Use existing query hooks or add new ones in `api/queries.ts`.

### 2. Register the route

In `src/router.tsx`, create a route and add it to the tree:

```ts
const myScreenRoute = createRoute({
  getParentRoute: () => protectedLayoutRoute,  // or loginRoute if public
  path: '/my-screen',
  component: MyScreenPage,
})

// Add myScreenRoute to protectedLayoutRoute.addChildren([...])
```

### 3. Add a sidebar entry

In `src/components/SidebarNav.tsx`, add a `<Link to="/my-screen">` item. For a placeholder not yet navigable, use `<button aria-disabled>` instead of `<Link>`.

### 4. Add a query hook (if needed)

In `src/api/queries.ts`, add a `queryKeys` entry and a `useMyThing()` function following the existing pattern. Call `unwrap(result)` to surface errors.

### 5. Write a test

```
apps/ui/src/routes/MyScreen.test.tsx
```

```ts
import { cleanup, render, screen } from '@testing-library/react'
import { afterEach, describe, expect, it, vi } from 'vitest'

afterEach(cleanup)

vi.mock('@/api/queries', () => ({
  useCurrentUser: () => ({ data: { username: 'admin' }, isLoading: false }),
  useVersion: () => ({ data: { version: '0.0.0' }, isLoading: false }),
  // mock any hooks your screen uses
}))

describe('MyScreenPage', () => {
  it('renders heading', () => {
    render(<MyScreenPage />)
    expect(screen.getByRole('heading', { name: /my screen/i })).toBeInTheDocument()
  })
})
```

## Dev Server

```bash
make ui-dev          # vite on 0.0.0.0:5173, proxies /api → localhost:9090
```

Override with environment variables (`.env.local` or shell):

| Variable | Default | Purpose |
|---|---|---|
| `VITE_DEV_PORT` | `5173` | Dev server port |
| `VITE_DEV_HOST` | `0.0.0.0` | Dev server bind address |
| `VITE_API_PROXY_TARGET` | `http://localhost:9090` | Backend proxy target |

LAN access works out of the box because `VITE_DEV_HOST` defaults to `0.0.0.0`.

## OpenAPI Codegen

The type chain runs automatically via pre-commit hooks:

```
apps/monitor (FastAPI)
  → packages/shared-types/openapi.json   (export-openapi.sh, on kernel/api change)
      → apps/ui/src/api/schema.ts        (generate-ui-types.sh, on openapi.json change)
```

To regenerate manually:

```bash
scripts/export-openapi.sh     # regenerates openapi.json
scripts/generate-ui-types.sh  # regenerates schema.ts
# or combined:
cd apps/ui && pnpm run generate-types
```

`schema.ts` is gitignored. When modifying Python API files, the full chain takes two commits (or one re-stage cycle) because the pre-commit hook for `schema.ts` runs after `openapi.json` is already staged.
