## [STAGE-005-001] HA REST client + secret + lifespan wiring

**Summary:** Implements the Home Assistant REST client kernel package with async methods for configuration, entity state queries, error logs, and service calls. Replaces the empty `HomeAssistantClient` Protocol scaffold with concrete typed implementations. Uses a token-provider callback for secret rotation and post-startup recovery (token never stored). Integrates with the kernel lifespan to construct the client once at startup and inject it into request context. Error handling mirrors the existing `FetchError` pattern: all methods return `HaError` on failure (return-not-raise) with discriminated failure reasons (unreachable/timeout/auth/http_error/bad_response). Validated in Refinement 3b against the real HA instance at `http://192.168.2.148:8123`.

**Key changes:**

**Kernel:**
- New `kernel/ha/` package: `errors.py` defines `HaError` (frozen/slots discriminated union), `client.py` implements `HomeAssistantRestClient` with async methods: `get_config()`, `get_states()`, `get_error_log()`, `call_service(domain, service, data=None)`. Reuses shared `httpx.AsyncClient`; token-provider invoked per-request.
- `kernel/config.py`: Added `HaConfig` dataclass + `load_ha_config()` reading `HOMELAB_MONITOR_HA_URL` (default `http://192.168.2.148:8123`, trailing slash stripped).
- `kernel/api/lifespan.py`: HA client constructed once at startup, injected as `ctx.ha`. No-token case yields `HaError(reason="auth")` at call time (enables no-restart recovery).
- `kernel/plugins/io.py`: Updated `HomeAssistantClient` Protocol with 4 async methods (matching concrete implementation).

**Error Mapping:**
- ConnectError/ConnectTimeout → unreachable
- ReadTimeout/TimeoutException → timeout
- 401/403 → auth
- Other non-2xx → http_error (with status)
- JSON/shape failures → bad_response
- Bearer token never appears in error messages or logs.

- UPDATED: `tests/test_plugins_context.py`, `tests/test_plugins_io.py` — updated `_FakeHa` stubs to satisfy now-non-empty Protocol.

**Files:**

- MOD: apps/monitor/homelab_monitor/kernel/plugins/io.py, apps/monitor/homelab_monitor/kernel/config.py, apps/monitor/homelab_monitor/kernel/api/lifespan.py, apps/monitor/tests/test_plugins_context.py, apps/monitor/tests/test_plugins_io.py

## [STAGE-005-002] HA websocket client + intent-replay reconnect + metrics

**Summary:** Implements a persistent, supervised Home Assistant `/api/websocket` client with multiplexed command/event handling, auto-reconnect with exponential backoff, and intent-replay (auto-resubscribe active subscriptions on reconnect). Provides `send_command()` one-shot RPC and `subscribe()` async-iterator handles. Includes observability: connection state, message timestamps, reconnect counts, and error reasons. Validates against the real HA instance; discovers constraint: bulk `get_states` exceeds HA's 1 MB frame limit—REST client required for entity sync. Fixes two production hangs found during Build (sequential-replay deadlock on reconnect; event-loop starvation in test mock).

**Key changes:**

**Kernel:**
- New `kernel/ha/websocket.py`: `HomeAssistantWebsocketClient` — persistent supervised client (start_task/stop_task mirroring BuildSourcesLoader) with async-iterator `Subscription` and injectable `connect` seam (minimal `_WsConnection` Protocol; single library-boundary cast; no `Any`).
- Auth handshake: `auth_required` → `auth` → `auth_ok`/`auth_invalid`; bearer token NEVER logged.
- Multiplexed protocol: single receive loop demuxes HA frames — `result` frames resolve command `Future`, `event` frames enqueue to matching subscription's bounded queue (drop-oldest + log on full), keyed by HA's monotonic per-connection `id`.
- `send_command(type_, **fields) -> dict | HaError` (one-shot RPC with timeout); `subscribe(type_, **fields) -> Subscription | HaError` (intent + bounded queue).
- Reconnect: exponential backoff (1s→60s); **intent-replay auto-resubscribes** active subscriptions CONCURRENTLY with receive loop (fixed sequential deadlock). Pending command futures failed on disconnect (no hangs). Per-connection subscription clear prevents stale-id routing. Transition-deduped auth_invalid logging.
- Metrics: `homelab_ha_websocket_connected` (gauge), `homelab_ha_websocket_last_message_timestamp_seconds` (gauge epoch), `homelab_ha_websocket_reconnect_total` (counter, RE-connections only), `homelab_ha_websocket_error_total{reason}` (counter).
- `kernel/api/lifespan.py`: WS client constructed after metrics writer; `start_task()` guarded on `ha_config.base_url`; injected as `app.state.ha_ws_client`; `stop_task()` in finally block.
- `websockets>=16.0` added as direct dependency (was transitive via uvicorn[standard]; declared explicit for honesty; zero install delta; `uv lock` run).

**Tests:**
- NEW: `apps/monitor/tests/test_ha_websocket.py` (48 test functions, injected fake WS, no real network/sleeps; **100% line+branch coverage** of websocket.py). Coverage: auth handshake/invalid/no-token, send_command (success/failure/not-connected/timeout/send-failure), subscribe/event/unsubscribe, reconnect+intent-replay, per-connection subscription-clear, unmapped-id drop, replay-subscribe-failure logging, disconnect-fails-pending, queue-full drop-oldest, backoff increment+reset, default-connect seam, log-transition dedup, fail-all-pending done-future skip.
- UPDATED: `tests/test_api_lifespan.py` — HA-disabled lifespan branch test (websocket start skipped).
- Build fixes: (1) Sequential-replay reconnect deadlock — moved intent-replay concurrent to receive loop; (2) `record_sleep` test mock not yielding to event loop — added `asyncio.sleep(0)` to progress concurrent tasks.

**Files:**
- NEW: apps/monitor/homelab_monitor/kernel/ha/websocket.py, apps/monitor/tests/test_ha_websocket.py
- MOD: apps/monitor/homelab_monitor/kernel/ha/__init__.py (exports), apps/monitor/homelab_monitor/kernel/api/lifespan.py (client construct/wiring), apps/monitor/tests/test_api_lifespan.py (HA-disabled branch), apps/monitor/pyproject.toml (+websockets>=16.0), apps/monitor/uv.lock
