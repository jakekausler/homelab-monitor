## [STAGE-004-018B] Configurable visible columns in logs results

**Summary:** Lets the user choose which fields show as COLUMNS in the Logs Explorer results, beyond the default timestamp + message. A "Columns" control (a dropdown) in the Explorer header lists toggleable fields grouped Common (service/host/severity, always shown) + Discovered (scope-aware fields from `GET /api/logs/fields`). Toggling adds/removes a result column rendered between the timestamp and the message. The severity column reuses the existing severity tint. A line missing a configured field renders an empty cell (no crash). Selection persists in the URL (`?columns=service,severity`) and round-trips on reload. Mobile = horizontal scroll (no column hiding).

**Key changes:**

**Frontend:**
- **Backward-compatible (load-bearing):** the shared `<LogViewer>`/`LogLineList` gain an OPTIONAL `columns?` prop. The existing `LogRow` renderer is unchanged; a SEPARATE `LogRowWithColumns` handles the multi-column path. Callers passing no columns (docker logs viewer, cron run log viewer, RecentCrashesSection, RecentHealthcheckIncidentsSection, SignatureDetailPage) render exactly as before (timestamp + message only). NO backend change (columns resolve frontend-side from promoted LogLine fields or the `fields` bag).
- New `apps/ui/src/routes/logs/ColumnsControl.tsx` (DropdownMenu + checkbox items; shows union of discovered ∪ selected so stale columns stay removable); exported pure `getColumnValue(line, field)` in LogLineList.tsx (promoted→fields-bag→'', objects→compact JSON).
- `parseColumnsParam`/`columnsToCsv` URL helpers (router.tsx / api/logs.ts) threaded through validateSearch + writeUrl (all call sites).
- LogsExplorerBody renders the control in the header (reusing the existing useLogsFieldsQuery field list) and passes `columns` to LogViewer.

**Tests:**
- Back-compat structural guard (LogLineList with no columns → `log-row-columns` testid ABSENT, timestamp+message only).
- getColumnValue pure tests (promoted/fields-bag/missing→''/number+bool+object coercion).
- Multi-column render + severity tint + missing-field empty cell + gridTemplateColumns.
- ColumnsControl picker (Common always shown, discovered listed, toggle add/remove, stale-selected-column still removable).
- parseColumnsParam/columnsToCsv URL round-trip.
