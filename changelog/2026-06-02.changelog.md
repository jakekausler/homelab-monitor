## [STAGE-004-013] Saved Queries (+ Refinement-driven Explorer redesign)

**Summary:** Users can save the current Logs Explorer state (LogsQL/plain text + selected services + time range + mode) as a named query in SQLite and recall it later. Refinement expanded scope to add Update/Duplicate, a load-state bugfix, sidebar/number-format polish, and a full control-bar + sidebar redesign.

**Backend (Saved Queries API):**
- **New:** `log_saved_queries` SQLite table (alembic migration 0031; UNIQUE constraint on name per-user). Columns: `id` (PK), `user_id` (FK), `name` (TEXT, UNIQUE per user), `preset` (ENUM: 'last_1h'|'last_24h'|'last_7d'|'custom_range'), `range_start` (ISO-8601 UTC, nullable), `range_end` (ISO-8601 UTC, nullable), `logsql` (TEXT, nullable), `plain_text` (TEXT, nullable), `services` (TEXT CSV), `mode` ('plain'|'advanced'), `created_at` (UTC), `updated_at` (UTC).
- **New:** `SavedQueriesRepository` (kernel/logs/saved_queries_repo.py): `create()`, `list_sorted()`, `get()`, `rename()`, `delete()`, `update()`. Methods accept user_id; returns `SavedQuery` domain model. Raises `DuplicateNameError` (→ 409) on name conflict.
- **New:** REST endpoints (kernel/api/routers/logs.py):
  - `GET /api/logs/saved-queries` — lists user's queries sorted by updated_at DESC (limit 50)
  - `POST /api/logs/saved-queries` — create new query, body is `SavedQueryCreateRequest` (exactly-one-of preset-vs-custom-range + both-or-neither partial-range validation)
  - `PATCH /api/logs/saved-queries/{query_id}` — rename only (body: `{name}`)
  - `PUT /api/logs/saved-queries/{query_id}` — update payload (ignores body.name; body accepts new preset/range/logsql/plain_text/services/mode)
  - `DELETE /api/logs/saved-queries/{query_id}` — soft-delete (no-op if already deleted)
  - All: session-required, CSRF on mutations, returns 201/204/404/409/422
- **Pydantic schemas:** `SavedQueryCreateRequest`, `SavedQueryUpdateRequest`, `SavedQueryResponse` with field-level validation (not both logsql and plain_text; not neither; preset XOR custom-range). Schemas use ISO-8601 UTC strings for timestamps.
- **New CLI:** `hm dev seed-saved-queries [--clear]` (kernel/cli/dev.py): creates 3 idempotent sample queries (Last Hour Plain, Last 24H Advanced, Last 7D Filtered). Skips existing names. `--clear` removes all sample queries by name pattern.

**Frontend (Saved Queries UI + Explorer redesign):**
- **New hooks & helpers** (api/savedLogQueries.ts):
  - `useSavedQueries()` (Query hook, 30s refetch)
  - `useSaveQuery()` (Mutation: POST)
  - `useUpdateSavedQuery()` (Mutation: PUT, for payload updates)
  - `useRenameSavedQuery()` (Mutation: PATCH)
  - `useDeleteSavedQuery()` (Mutation: DELETE)
  - Pure helpers: `computeCopyName()` (smart (copy)/(copy 2)/(copy N) increment for Duplicate), `savedRowToCreateRequest()` (convert SavedQuery → CreateRequest for load/duplicate workflow)
- **New components** (components/logs/):
  - `SaveQueryModal.tsx` — modal to save current state, inline 409 conflict handling, disabled-state for required fields, optional description field. Validation: name required, preset XOR range validation, at-least-one-mode required
  - `SavedQueriesPanel.tsx` — sidebar tab showing user's saved queries; each row: query name, button trio (Load / Update [confirm modal] / Duplicate [clone to new name]), Rename [inline edit], Delete [confirm]. Two-line rows (title + "Last modified X ago" + mode/preset badge). Sorted by updated_at DESC
  - `LogsServiceChip.tsx` — extracted from STAGE-004-012; unchanged
- **Control-bar + sidebar redesign (breaking layout change):**
  - **Changed:** `LogsExplorerPage.tsx` (new sidebar-open state, toggle handler, layout grid)
  - **Changed:** `LogsExplorerBody.tsx` (new control-bar layout: icon-only buttons for search-submit, advanced-toggle, refresh, filter, time-picker, wrap-mode; all collect into a flex container with gap; sidebar toggled by new Filter button)
  - **New:** `components/ui/sheet.tsx` (Radix Sheet primitive for mobile drawer) — desktop: sidebar always-visible / mobile: overlay drawer opened by Filter button
  - **Control-bar buttons:** search-submit icon (magnifying glass, click-or-Enter triggers search), advanced-toggle icon (</> or ABC icon), refresh icon (circular arrow, clears cache), filter icon (funnel, toggles sidebar), time-picker trigger (clock icon), wrap-toggle icon (wrap-text or unwrap), save icon (bookmark or disk, opens SaveQueryModal)
  - **UTC toggle relocation:** moved from control-bar to inside time-picker dropdown (less clutter)
  - **Sidebar polish:** count-before-name aligned columns (using new `formatCompactCount` helper), long service names truncate with Radix Tooltip (no horizontal scroll)
- **New helper** (lib/formatCount.ts): `formatCompactCount()` — tiered truncated abbreviations: 1.1k / 11k / 1.1m / 1.1b (matches design system)
- **Bug fix (mode-toggle):** fixed stale LogsQL resurfacing after loading a plain-mode saved query — inactive buffer now cleared (`plainText` / `logsQl` state zeroed when swapping modes)

## [STAGE-004-014] Query History (last 20 executed queries)

**Summary:** The Logs Explorer auto-records the last 20 executed queries to localStorage and surfaces them in a 3rd "Recent" sidebar tab. Click a recent entry to re-run it (restores full Explorer state). Disposable, separate from Saved Queries. Frontend-only (no backend/migration).

**Implementation (Option A — choke-point recorder + pure core + thin hook):**
- `apps/ui/src/lib/queryHistory.ts` — pure core: HistoryEntry type (with stable `id`), STORAGE_KEY `homelab-monitor:logs-query-history`, readHistory (SSR-guard + try/catch + per-entry shape validation → drops malformed entries), writeHistory, equalityKey (excludes id/timestamp/resultCount; order-insensitive services), appendWithDedupeAndCap (most-recent-first; consecutive-identical collapses by updating the existing entry's timestamp, preserving its id; cap 20 drops oldest), recordQuery, clearHistory, module-level pub-sub (subscribe/emit) for same-tab reactivity.
- `apps/ui/src/lib/useQueryHistory.ts` — hook { entries, clear }; subscribes to the pub-sub, unsubscribes on unmount.
- `apps/ui/src/routes/logs/QueryHistoryPanel.tsx` — "Recent" tab: most-recent-first rows (relative timestamp + query preview), click-to-load, "Clear history" button, empty state. Keys on stable entry.id.
- `apps/ui/src/routes/logs/LogsExplorerPage.tsx` — DRY refactor: extracted serializeCurrentExplorerState (buildSavePayload spreads it; saved-query behavior unchanged) and reconstructFromPayload (shared by handleLoadSavedQuery + new handleLoadHistoryEntry). writeUrl() now records a HistoryEntry built from its ARGS (about-to-commit values, avoiding stale closure) with a crypto.randomUUID() id + Date.now() timestamp.
- `apps/ui/src/routes/logs/LogsExplorerBody.tsx` — sidebarTab extended to services|saved|history; 3rd "Recent" tab + render branch; onLoadHistoryEntry threaded.

**Behavior:** auto-record every committed query; consecutive-dedupe; cap 20; click-to-rerun restores LogsQL/plain + services + range + mode; persists across reload (localStorage); independent from Saved Queries.

**Tests:** queryHistory.test.ts (append/dedupe/cap/equalityKey-ignores-id/corrupt-JSON/pub-sub) + QueryHistoryPanel.test.tsx (empty/rows/relative-time/preview/click-load/clear/reactivity). make verify GREEN (UI 836 tests, functions 81.29%; backend 100% coverage maintained).
