## STAGE-003-005: Docker discoverer + suggestions data + per-container UX hardening

### Summary

Lands the Docker discoverer plugin with event-stream + 5-min periodic scan producing suggestions for missing containers. Adds `suggestions` + `suggestions_docker` tables with cursor-paginated `GET /api/integrations/docker/suggestions` API endpoint and `useListDockerSuggestions` UI hook. Introduces PendingSuggestionsPanel + SuggestionCard components to surface actionable intelligence. Refinement expanded scope with 4 migrations (0020–0022) addressing compose-file-path capture, logical-key re-keying of targets (fixing container recreation duplicates), per-container grid columns, and per-container restart metrics. Mobile parity added: ContainerGridCard shows status + restarts + healthcheck badge. Server-side filtering suppresses non-homelab-monitor vendor labels. Hide-missing toggle on container grid (default shows all, toggle hides `status='missing'` rows with count). All refinement feedback incorporated; code review 7 must-address findings fixed.

### Changes

**Backend (new modules):**
- `kernel/docker/discoverer.py` — `DockerDiscoverer(BaseCollector)` with 5-min interval, event-stream async generator `events()`, background tick. Yields `SuggestionDelta` when containers appear/disappear. Resolves both running and exited containers via `inspect_container()` lookup. Preserves suggestion state across container lifecycle (resurrects from `container_gone` → `pending` when same logical-key re-appears; keeps terminal states `accepted`/`ignored`).
- `kernel/db/repositories/suggestions_repository.py` — `SuggestionsRepository` with `upsert_suggestion()`, `list_with_cursor_pagination()`, `update_state()`, `mark_orphaned()` methods. Raw SQL via `SqliteRepository`. Filters suggestions by state, accepts filters dict for extensibility.
- `kernel/api/routers/docker.py` (extended) — `GET /api/integrations/docker/suggestions` with cursor pagination (limit, after_id), returns `SuggestionsListResponse` with `extra="ignore"`.
- `kernel/api/lifespan.py` (extended) — registers `DockerDiscoverer` with scheduler; injects into route handlers via dependency.

**Database migrations:**

## STAGE-003-006: Label-based probe auto-config

### Summary

Implements automatic per-container probe creation from Docker labels of the form
`homelab-monitor.<kind>.<name>=<value>` (kind ∈ {http, tcp, exec, metrics}). When
the discoverer sees such a label on a container, it upserts a `probe_targets`
row; the new `ProbeSupervisor` collector then spawns a per-container asyncio
task that executes the probe on its configured interval and writes outcomes back
to the DB. UI exposes the probes at a new route `/integrations/docker/containers/{name}/probes`.

### Changes

**Backend (new modules):**
- `apps/monitor/homelab_monitor/kernel/db/repositories/probe_targets_repository.py` — new repository with `upsert_probe_target_conn`, `set_enabled_conn`, `update_run_outcome_conn`, `mark_missing_except_conn`, `summarize_by_container` methods.
- `apps/monitor/homelab_monitor/kernel/docker/label_parser.py` — `parse_homelab_labels` returning `ParseResult` with descriptors, collisions, malformed, exec_authorized.
- `apps/monitor/homelab_monitor/kernel/docker/probe_resolver.py` — `resolve_probe` with `host`/`container` sentinel substitution + exec gating.
- `apps/monitor/homelab_monitor/kernel/docker/probe_executor.py` — executors for http/tcp/exec/metrics + dispatcher; exec command capped at 4096 chars, rejects newlines / null bytes.
- `apps/monitor/homelab_monitor/kernel/metrics/probe_supervisor.py` — registered Collector + per-container `asyncio.Task` lifecycle, mirroring DockerDiscoverer's hybrid pattern; live `container_ip` lookup via socket inspect.

**Backend (extended modules):**
- `apps/monitor/homelab_monitor/kernel/docker/socket_client.py` — extended with `exec_in_container` (POST `/containers/{id}/exec` + start + inspect for exit code).
