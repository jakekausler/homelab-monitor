# 2026-05-22

## STAGE-003-005: Docker discoverer + suggestions data + per-container UX hardening

### Summary

Lands the Docker discoverer plugin with event-stream + 5-min periodic scan producing suggestions for missing containers. Adds `suggestions` + `suggestions_docker` tables with cursor-paginated `GET /api/integrations/docker/suggestions` API endpoint and `useListDockerSuggestions` UI hook. Introduces PendingSuggestionsPanel + SuggestionCard components to surface actionable intelligence. Refinement expanded scope with 4 migrations (0020–0022) addressing compose-file-path capture, logical-key re-keying of targets (fixing container recreation duplicates), per-container grid columns, and per-container restart metrics. Mobile parity added: ContainerGridCard shows status + restarts + healthcheck badge. Server-side filtering suppresses non-homelab-monitor vendor labels. Hide-missing toggle on container grid (default shows all, toggle hides `status='missing'` rows with count). All refinement feedback incorporated; code review 7 must-address findings fixed.

### Changes

**Backend (new modules):**
- `kernel/docker/discoverer.py` — `DockerDiscoverer(BaseCollector)` with 5-min interval, event-stream async generator `events()`, background tick. Yields `SuggestionDelta` when containers appear/disappear. Resolves both running and exited containers via `inspect_container()` lookup. Preserves suggestion state across container lifecycle (resurrects from `container_gone` → `pending` when same logical-key re-appears; keeps terminal states `accepted`/`ignored`).
- `kernel/db/repositories/suggestions_repository.py` — `SuggestionsRepository` with `upsert_suggestion()`, `list_with_cursor_pagination()`, `update_state()`, `mark_orphaned()` methods. Raw SQL via `SqliteRepository`. Filters suggestions by state, accepts filters dict for extensibility.
- `kernel/api/routers/docker.py` (extended) — `GET /api/integrations/docker/suggestions` with cursor pagination (limit, after_id), returns `SuggestionsListResponse` with `extra="ignore"`.
- `kernel/api/lifespan.py` (extended) — registers `DockerDiscoverer` with scheduler; injects into route handlers via dependency.

**Database migrations:**
- `alembic/versions/0019_suggestions_tables.py` — creates `suggestions` + `suggestions_docker` tables; `suggestions_docker` has `compose_file_path`, `compose_project`, `compose_service`.
- `alembic/versions/0020_suggestions_docker_compose_path.py` — adds `compose_file_path` column if not present (no-op in 0019-fresh installs).
- `alembic/versions/0021_targets_logical_key_rekey.py` — **DESTRUCTIVE ONE-WAY**: consolidates duplicate container rows (created during recreation events) by introducing logical-key derivation (`kind:source:name` with leading slash stripped); renames `container_id` to `current_container_id`, adds `previous_container_id` + `recreated_at`; deduplicates on logical-key; migration downgrade emits UserWarning + module docstring warns of one-way nature.
- `alembic/versions/0022_targets_docker_compose_columns.py` — adds `compose_project`, `compose_service`, `compose_file_path`, `restart_count_24h_cached` to `targets_docker`; backfills `restart_count_24h_cached` from VictoriaMetrics `increase()` query during migration.

**Lifespan + app wiring:**
- `kernel/metrics/docker_discoverer.py` → metric `homelab_collector_run_docker_discoverer` (gauge, 1.0 per tick; renamed from `_total` suffix which was incorrect for gauge).

**UI (new + modified):**
- `apps/ui/src/routes/integrations/PendingSuggestionsPanel.tsx` (new) — displays suggested containers in card grid; filters by state; shows action buttons (accept/ignore/etc.).
- `apps/ui/src/routes/integrations/SuggestionCard.tsx` (new) — renders single suggestion: image, tags (filtered to `homelab-monitor.*` prefix only), compose file path (truncated + tooltip), state badge, action buttons.
- `apps/ui/src/routes/integrations/composeBasename.ts` (new) — helper extracting parent-dir basename from compose-file-path with fallback.
- `apps/ui/src/api/docker.ts` (extended) — `useListDockerSuggestions(cursorPaginationParams)` TanStack Query hook; 30s refetch.
- `apps/ui/src/routes/integrations/ContainerGrid.tsx` (modified) — added "Hide Missing" toggle (default shows all); toggle counts rows with `status='missing'`; server-side ORDER BY compose_file_path NULLS LAST, name.
- `apps/ui/src/routes/integrations/ContainerGridRow.tsx` (extended) — new "Compose" first column showing parent-dir basename + tooltip of full path; "Restarts (24h)" header + formatted restart count from VictoriaMetrics `increase()` (reset-aware).
- `apps/ui/src/routes/integrations/ContainerGridCard.tsx` (extended) — mobile parity: shows StatusBadge, restart count, healthcheck badge; matches desktop visual hierarchy.
- `apps/ui/src/routes/integrations/DockerIntegrationPage.tsx` (modified) — renders PendingSuggestionsPanel; wires "Hide Missing" toggle state; passes filter params to server.
- `apps/ui/src/routes/integrations/types.ts` (extended) — adds `compose_file_path`, `restart_count_24h_cached` to `ContainerRow` type.

**Logging (warnings added during Build):**
- Warning logs emitted when a compose container lacks `service` or `config_files` labels (early signal for Compose orchestration misconfigs).
