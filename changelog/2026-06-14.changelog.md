Each drill-list renders the actual rows behind the existing count tiles (which entities are unavailable, which batteries are low, etc.) so the user can drill from "962 unavailable" into the specific entities. ## [STAGE-005-029] Home Assistant persistent-notification detail (live re-query + safe render)

**Summary:** Added a backend endpoint that live-queries Home Assistant for persistent-notification bodies (deliberately kept out of VictoriaMetrics for privacy) and a frontend widget that renders them safely. The HA panel's Status tab now shows the actual current notifications (title + body + relative time), sourced by a live HA WebSocket re-query, rendered text-only/escaped with no HTML-injection surface.

**New files:**
- `apps/monitor/homelab_monitor/kernel/ha/notifications.py` — shared notification parsing logic: `extract_notifications(data)` defensively coerces HA's persistent_notification/get response into a list of dicts with `notification_id` (required), `title` (null if missing), `message` ("" if missing), `created_at` (null if missing). Skips malformed notifications (non-dict, missing/empty notification_id).
- `apps/monitor/tests/test_api_ha_notifications.py` — endpoint tests (200 with live HA data, 502 on HA unreachable/timeout/http_error/bad_response/auth, defensive parsing skips malformed, bodies + token never logged — sentinel test confirms absence via grep).
- `apps/ui/src/routes/integrations/HaNotificationsDrill.tsx` — container component owning `useHomeAssistantNotifications` TanStack Query hook + independent loading/empty/error/502 state; renders active notifications (notification_id key, title fallback "(untitled)", relative time via `formatAge`, message rendered plain-text).
- `apps/ui/src/routes/integrations/HaNotificationsDrill.test.tsx` — hook-mocked render test (notification list, title/message/time rendering, safe-render malicious-string test: `<script>`/`<img onerror>`/`[x](javascript:)`/`**bold**` render as literal inert text).

**Modified files:**
- `apps/monitor/homelab_monitor/kernel/api/dependencies.py` — added `get_ha_ws_client` dependency (injected `ha_ws_client` from app.state).
- `apps/monitor/homelab_monitor/kernel/api/routers/integrations_home_assistant.py` — added `GET /api/integrations/home-assistant/notifications` endpoint (`Depends(require_session())` + `Depends(get_ha_ws_client)`). Live-queries HA via `ha_ws_client.send_command("persistent_notification/get")`. Returns `HaNotificationsResponse {rows: list[HaNotificationRow], total, returned}`. Maps `HaError` (unreachable/timeout/http_error/bad_response/auth) to HTTP 502 (upstream_unavailable); auth errors are 502 NOT 401 (bad HA token is a server config problem). Uses shared `extract_notifications` parser.
- `apps/monitor/homelab_monitor/plugins/collectors/integrations/homeassistant/ha_persistent_notification.py` — refactored to use shared `extract_notifications` from `kernel.ha.notifications` (single parsing-truth).
- `apps/ui/src/api/home_assistant.ts` — added `useHomeAssistantNotifications` TanStack Query hook (mirrors 028 hooks: queryKey under `haQueryKeys`, `unwrap<HaNotificationsResponse>`, `refetchInterval: 30_000`).
- `apps/ui/src/routes/integrations/types.ts` — re-exported `HaNotificationRow` / `HaNotificationsResponse`.
- `apps/ui/src/routes/integrations/HomeAssistantStatusTab.tsx` — wired `<HaNotificationsDrill>` into the Status tab, reusing 028 `<DrillList<HaNotificationRow>>` shell.
- `apps/ui/src/routes/integrations/HomeAssistantStatusTab.test.tsx` — added a default `beforeEach` mock for `useHomeAssistantNotifications` so the auto-mocked new hook does not render `undefined` in the existing tab tests.
- `packages/shared-types/openapi.json` — regenerated with new models: `HaNotificationRow`, `HaNotificationsResponse`.

**Privacy & Safety (hard contracts, validated end-to-end):**
- **Bodies never logged.** Notification bodies + titles are returned only to the authenticated session and are NEVER logged. Verified on prod rig: `docker logs homelab-monitor | grep -iE 'ha_token|bearer|eyJ'` → 0 hits; `grep -i <distinctive-body-substring>` → 0 hits. Enforced by backend sentinel test.
- **Safe render.** Message + title render as plain-text React auto-escape (NO markdown lib, NO sanitizer, NO `dangerouslySetInnerHTML`). Enforced by frontend safe-render test: `<script>`, `<img onerror>`, `[x](javascript:)`, `**bold**` render as literal inert text — no DOM node, no handler fires.

**Design decisions:** D-NOTIF-SOURCE = WS `persistent_notification/get` via `app.state.ha_ws_client` (reuse the already-wired client + the shared `extract_notifications` helper; not REST, not VM). D-NOTIF-PRIVACY = full title+message+created_at to the authenticated session with enforced no-log/no-metric/no-error-leak (the STAGE-012 bodies-out decision was about METRICS, not the authenticated UI). D-NOTIF-SAFE-RENDER = plain-text React auto-escape + `whitespace-pre-wrap break-words` (no markdown lib, no sanitizer, no `dangerouslySetInnerHTML`). D-NOTIF-WIDGET-PLACEMENT = reuse the 028 `<DrillList<HaNotificationRow>>` shell with a rich renderRow in a new "Notifications" PanelSection on the Status tab. Degradation: HaError → 502 (incl. auth → 502, not 401).

**Verification:** `make verify` GREEN — ruff clean, pyright 0, pytest 4838 pass / 2 skip, kernel coverage 100%, tsc 0, eslint 0 errors / 94 warnings (baseline), vitest 1521 pass, UI build OK. Refinement 3a (frontend Desktop + Mobile approved) + 3b (prod rig real data: 2 live HA notifications rendered safely; token + bodies never logged — both greps returned 0).
