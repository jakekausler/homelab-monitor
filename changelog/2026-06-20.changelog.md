2026-06-20 5660516 EPIC-007/STAGE-007-016A: vector config auto-reload on re-render (`--watch-config`)

Added `--watch-config` to the prod vector service command so the `homelab-vector` container auto-reloads its configuration when the monitor re-renders `vector.toml`, eliminating the manual `docker restart homelab-vector` step after any vector template/config change. Closes the deploy-propagation gap found during STAGE-007-016 Refinement (a fixed-and-valid config on disk did not take effect until vector was manually restarted).

Design: Selected Approach 1 (`--watch-config`) over Approach 2 (SIGHUP-on-render). SIGHUP is structurally redundant because render runs only at monitor boot and `vector depends_on monitor: service_healthy`, so vector starts (and reads fresh config) AFTER the render. The `--watch-config` flag enables file-watch-based reloads, ensuring any `vector.toml` swap (via atomic rename during re-render) triggers an in-process graceful reload without stopping the container.

Build: `deploy/compose/docker-compose.yml` vector `command:` updated to `["--config", "/etc/vector/vector.toml", "--watch-config"]`. Single edit covers prod, `make dev`, and `make dev-prod` (dev override only adds `ports:`; compose deep-merges command from base). Corrected build-phase assumption: vector 0.41.1 has only `--watch-config` (no `--watch-config-method poll` — that flag arrived in 0.43.0). The 0.41.1 default inotify watcher is already atomic-rename-safe (it re-arms its inode watch before reloading), so it correctly catches the monitor's `os.replace` write. No `poll` fallback needed. Updated `apps/monitor/homelab_monitor/kernel/cron/render.py` docstring to reflect auto-reload (no logic change).

Refinement validation (prod rig): 3 successive atomic-rename config swaps each produced `Vector has reloaded.` logs with the vector PID unchanged (in-process graceful reload, not a restart). No restart loop, no event loss, no dropped metrics. Reload latency ~12-21s (inotify on the overlayfs-backed named volume). Documentation consistency: updated `docs/admin/redaction.md` (operator runbook — restart monitor only; vector auto-reloads) and `epics/EPIC-003-docker/regression.md`; added STAGE-007-016A forward-pointers to historical "no hot-reload" notes in `epics/EPIC-003-docker/STAGE-003-002.md`, `epics/EPIC-002-heartbeat-cron/STAGE-002-012.md`, and `epics/EPIC-004-logs-pipeline/{regression.md,STAGE-004-004.md}`.
