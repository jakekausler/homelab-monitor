## [STAGE-001-015] VictoriaMetrics + vmagent integration

Replaced the in-memory MetricsWriter with a multiplex writer that fans out to both an in-memory latest-value map AND a prometheus_client registry. vmagent scrapes the new unauthenticated `/metrics` endpoint and remote-writes to VictoriaMetrics; `/api/metrics/range` proxies range queries to VM. HostCpuTile optimistically backfills its sparkline with real history.

### Foundation epic progress: 14/22 → 15/22

### Backend

New `MultiplexMetricsWriter` (`kernel/metrics/multiplex.py`) fans out every `write_*`/`replace_family` call to a list of writers (in-memory + prometheus_client). New `PrometheusRegistryWriter` (`kernel/metrics/prometheus_writer.py`) backed by `prometheus_client>=0.21`. Caches metric objects keyed by `(name, frozenset(label_names))`. Different label-name sets log warning + skip. Thread-safe via `threading.RLock`.

New unauthenticated `GET /metrics` endpoint (`kernel/api/routers/observability.py`) serves Prometheus exposition format. Mounted at root path (NOT `/api/...`). New authenticated `GET /api/metrics/range` (`kernel/api/routers/metrics.py`) proxies to VM's `/api/v1/query_range`. Validates `expr <= 4096 chars` and `step` regex `/^\d+(ms|s|m|h|d)$/`. Surfaces VM `status="error"` and HTTP errors as 502. Uses shared `app.state.http_client`.

Lifespan attaches three new state attributes: `metrics_writer` (multiplex, for writes), `in_memory_metrics_writer` (for snapshot + inspection), `prom_registry` (for `/metrics`). `collectors.py` and `health.py` now inject `get_in_memory_metrics_writer` (resp. `_optional`) for inspection method calls (`last_tick_at_for`, `failures_in_window`, etc.) — multiplex no longer needs to forward those non-Protocol methods.

Pre-existing `AccessLogMiddleware` bug fixed: `exception=True` (boolean) tripped structlog's special exception-rendering path. Renamed to `had_exception=True`.

### Deployment

New `deploy/compose/docker-compose.yml` with monitor + VM v1.107.0 + vmagent v1.107.0. Monitor port bound to `127.0.0.1` by default (external access via upstream reverse proxy; `/metrics` exposure unauthenticated for in-network scraping only). New `deploy/compose/vmagent/scrape.yaml` — single static target (one job, one `/metrics` endpoint). New `deploy/compose/.env.example` documenting required env vars + security model.

New multi-stage `apps/monitor/Dockerfile` (uv sync from workspace lockfile → slim runtime with UI bundle from `apps/ui/dist`). New `.dockerignore` at repo root. New `make compose-build`, `compose-up`, `compose-down`, `compose-logs` targets. `make verify-ci` now includes `docker compose -f deploy/compose/docker-compose.yml config -q` for syntax validation.

### Frontend

New `useMetricsRange(expr, start, end, step)` TanStack Query hook in `apps/ui/src/api/queries.ts`. `HostCpuTile` optimistically backfills its sparkline with 600s of historical data from `/api/metrics/range`. On VM unreachable: silently retains synthetic baseline. SSE-driven appends continue regardless. Added `data-values` attribute to `Sparkline` for testable assertions.

### Refinement-phase fix

HostCpuTile race condition: SSE-tick effect was setting `seededFromHistoryRef.current = true`, killing the range backfill. Removed the SSE side effect on the ref. Renamed ref to `historyBackfilledRef` for clarity.

### Tests

22 new backend tests across 4 new files (`test_metrics_multiplex.py`, `test_metrics_prometheus_writer.py`, `test_api_metrics_endpoint.py`, `test_api_metrics_range.py`). 6 new frontend tests in `HostCpuTile.test.tsx` covering range backfill success, failure, empty, padding, race-with-SSE. 100% kernel coverage maintained. Backend: 901 tests passing. UI: 16 HostCpuTile tests + full UI suite passing.

### Code review

Two-pass code review with code-reviewer-Opus. Pass 1: 19 findings (3 Critical, 7 Important, 9 Minor) — all addressed. Pass 2 caught a deadlock introduced by Pass 1 fixes (`threading.Lock` was non-reentrant; `replace_family` calls `write_gauge` which also acquired the lock). Switched to `threading.RLock` (reentrant). Lesson: Pass 2 catches what Pass 1 misses.

## [STAGE-001-015A] Backup hook + disk budget + minimal test rig

Layered operational concerns onto the VM sidecar from STAGE-001-015: a backup endpoint that triggers SQLite + VictoriaMetrics snapshots with audit logging, a disk-budget self-monitor with auto-shrink decision v1, and a minimal integration test rig that's reusable for future stages.

### Foundation epic progress: 15/22 → 16/22

### Key changes

**Backup endpoint:**
- New `POST /api/admin/backup` (cookie session OR API token with new `Scope.ADMIN_BACKUP_WRITE`). Orchestrates SQLite `.backup()` snapshot + VM `/snapshot/create` + `cp -al` snapshot files to backup root. Falls back to `cp -r` when backup is on a different filesystem than VM data. Two-row audit pattern: `requested` row written before work, `completed`/`failed` row written after.
- New `BackupService` at `kernel/backup/service.py`. Best-effort semantics: SQLite + VM steps independent; failures collected into `errors` list rather than raising. Pre-checks disk space (2x source DB size) before SQLite backup; cleans up partial files on failure. Validates VM snapshot ID against `^[A-Za-z0-9._-]+$` regex (path-traversal hardening). 5s VM HTTP timeout.
- New CLI: `hm backup {run, list, retention --keep N}`. Each subcommand bootstraps its own engine + repository for audit; disposes on exit. CLI runs write `who="cli:<euid>"` audit rows.
- Apply-retention: filters glob through name regex (`sqlite-YYYYMMDD-HHMMSS.sqlite` for SQLite, alphanumeric for VM dirs) to skip foreign files. Rejects `keep < 1`.

**Disk budget:**
- New `SelfDiskCollector` (60s interval) emits `homelab_self_disk_used_bytes`, `_budget_bytes` per slot (vm/vl/sqlite/runbook_transcripts), `_used_pct`, and `_shrink_total {tier}`. Uses `os.walk(followlinks=False)` for safety.
- Auto-shrink v1: at >95% used_pct, emits shrink counter + writes critical-severity audit row. 5-minute cooldown prevents flooding during sustained incidents (instead of 60 audit rows/hour, max 12). No actual data eviction in v1; deferred to future stage.

**Config loader:**
- New `kernel/config.py` — minimal `yaml.safe_load` helper. Loads `HOMELAB_MONITOR_CONFIG` (default `/config/homelab-monitor.yaml`). Returns `DiskBudgetConfig` with default ratios (60% VM / 30% VL / 10% SQLite) when YAML absent. `HOMELAB_MONITOR_DISK_BUDGET_GB` env overrides total. Validates ratios sum ~1.0.

**vmalert rules:**
- New `deploy/vmalert/metrics/self_disk.yaml`: SelfDiskWarn (70%/5m), SelfDiskError (85%/5m), SelfDiskCritical (95%/0s). Static YAML; vmalert deployment lands in STAGE-001-018.

**Test rig:**
- New `Dockerfile.test` (multi-stage with dev deps).
- New `deploy/compose/docker-compose.test.yml` (VM + integration-tests services on shared network).
- New `scripts/run-integration.sh` — `docker compose up --abort-on-container-exit --exit-code-from integration-tests`.
- New `apps/monitor/tests/integration/` with smoke test asserting VM `/health` reachable. Marked `@pytest.mark.integration`; excluded from default test runs via `--ignore=tests/integration` in addopts.
- Makefile: new `make integration` target.
- Compose: monitor service binds VM data dir RO at `/var/vm-data`.

### Tests

- 100% kernel coverage maintained.
