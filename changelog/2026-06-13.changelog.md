## [STAGE-005-023] Entity-health + battery count widgets (HA panel Health tab)

**Summary:** Fill the HA panel's Health tab (built in 022) with two live COUNT widgets consuming the 021 `/summary` endpoint. A new shared `useHomeAssistantSummary()` hook (reused by 024), two pure presentational widget components, and a tab-level connection banner. Frontend-only. COUNTS-ONLY by design: the 021 contract returns counts, not per-entity detail, so the card's drill-lists + `stale` count were dropped and the drill-down work was tracked as new stages STAGE-005-027 (backend detail endpoints) + STAGE-005-028 (frontend drill-down widgets).

**New files:**
- `apps/ui/src/api/home_assistant.ts` — `useHomeAssistantSummary()` react-query hook (mirrors `docker.ts::useListContainers`): queryKey `['integrations','home-assistant','summary']`, `apiClient.GET('/api/integrations/home-assistant/summary', {})`, `unwrap<HaSummaryResponse>`, 30s refetch. Shared — STAGE-005-024 reuses it for the Status tab.
- `apps/ui/src/routes/integrations/HaEntityHealthWidget.tsx` — pure presentational 3-cell `<dl>` (Total / Available / Unavailable; Unavailable amber-tinted when > 0).
- `apps/ui/src/routes/integrations/HaBatteryWidget.tsx` — pure presentational 2-cell `<dl>` (Low amber / Critical red when > 0); "All batteries healthy" EmptyState when both 0.
- `apps/ui/src/routes/integrations/HaEntityHealthWidget.test.tsx`, `HaBatteryWidget.test.tsx`, `HomeAssistantHealthTab.test.tsx` — render tests (widget counts + tint + EmptyState; tab states via a `vi.mock('@/api/home_assistant')` hook mock).

**Modified files:**
