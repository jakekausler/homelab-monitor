## [STAGE-001-011] Local auth (bcrypt + signed cookie sessions + CSRF + scoped API tokens)

- Replaces the `X-Auth: dev` placeholder from STAGE-010 with production-grade auth. After this stage every API call (except `/api/healthz`, `/api/version`, `/api/auth/login`, and the future heartbeat/alert-ingest endpoints) requires either a valid session cookie or a valid API token.
- New `kernel/auth/` package: `scopes.py` (StrEnum + parse/serialize helpers), `passwords.py` (bcrypt cost 12, MIN_PASSWORD_LENGTH=12, `HOMELAB_MONITOR_BCRYPT_COST` env override), `csrf.py` (uuid4 hex + constant-time compare), `api_tokens.py` (`homelab_<HOMELAB_MONITOR_TOKEN_PREFIX>_<base64url-30>` format; SHA-256 hash storage), `sessions.py` (HMAC-SHA256-truncated-128-bit signature over session_id; HKDF-derived key with info=b"homelab.session.v1.hmac"), `models.py` (frozen Pydantic), `errors.py` (9 HttpProblem subclasses), `rate_limit.py` (LoginRateLimiter Protocol + InProcessLoginRateLimiter; sliding-window 5/5min/IP, only consumes budget on FAILED login attempts), `repository.py` (AuthRepository with 17 atomic methods).
- Migration `0004_api_tokens_hash_unique.py` adds UNIQUE INDEX `api_tokens_hash_idx` on `api_tokens(hash)` for O(log n) token lookup + duplicate-insert protection.
- AuthMiddleware (replaces DevAuthMiddleware): resolves `request.state.user` from session cookie OR `Authorization: Bearer ...` header. Sets `request.state.auth_kind` (`"session" | "token" | "unauthenticated"`). Empty Bearer plaintext short-circuits before DB hit. Token-takes-precedence-over-cookie order documented (CSRF-immune by design).
- Per-route auth dependencies: `require_session()` (CSRF check on state-changing methods built in), `require_token_scope(scope)`, `require_user_or_token(scopes)`. The previous `require_dev_auth` is removed; `HOMELAB_MONITOR_DEV_AUTH` env var no longer recognized; `X-Auth: dev` header has no effect.
- Auth router (`/api/auth/login`, `/api/auth/logout`, `/api/auth/me`, `/api/auth/change-password`):
  - Login: validates credentials, deletes ALL prior sessions for the user (session-fixation defense), creates new session + CSRF token, sets `homelab_monitor_session` (HttpOnly + SameSite=Lax) and `homelab_monitor_csrf` (HttpOnly=false + SameSite=Strict) cookies. Rate limiter consumes budget only on failed attempts; 6th failed attempt within 5 min triggers 429.
  - Logout: requires session via `Depends(require_session())` (so CSRF is enforced); deletes session row + clears cookies; returns 204.
  - Change-password: requires session + valid current password; deletes ALL user sessions and creates a fresh one; rate-limited on failed current-password verification.
- AccessLogMiddleware emits `auth=session(user_id=N)` or `auth=token:<token-name>` per request for post-incident grep + audit-driven token rotation.
- `/api/version` exposes new `users_configured: bool` field (NOT cached — re-checked per request) so the React frontend (STAGE-001-014) can detect pre-bootstrap state without authenticating.
- Lifespan emits structured `lifespan.bootstrap_required` warning when zero users are configured at boot.
- `/api/collectors`, `/api/collectors/{name}/retry`, `/api/events` all now require `Depends(require_session())`. The `/api/events` SSE endpoint previously had no auth, causing the test suite to hang indefinitely on unauthenticated requests; closed.
- New CLI commands: `hm user create USERNAME` (twice-prompted password, MIN length 12), `hm user list`, `hm user passwd USERNAME`, `hm user delete USERNAME`. `hm api-token create --scope SCOPE --name NAME` (--name and at least one --scope are required; plaintext token printed once with a clearly-marked block; SHA-256 hash stored). `hm api-token list`, `hm api-token revoke ID`. All state-change methods write atomic audit_log rows.
