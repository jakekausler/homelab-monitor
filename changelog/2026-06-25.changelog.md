2026-06-25 751bdf0 EPIC-006/STAGE-006-025: Tier-3 query-feed shipper (/api/queries → VictoriaLogs) + query_feed_streaming overview field + Logs-tab stream selector

The Tier-3 forensic capability: a toggle-gated log shipper that streams Pi-hole's per-query feed into VictoriaLogs, searchable in the embedded LogViewer. Default-OFF (PII: every domain every device resolves); the user enables it per-host.

Backend: a new `PiholeQueryFeedCollector` (`plugins/collectors/integrations/pihole/query_feed.py`, interval 60s) pages `/api/queries` (cursor ID-based), transforms each record into a structured JSON VL log line (query_id, domain, client_ip/name/kind via Tier-2 `classify_one` loopback→host attribution, status, query_type, dnssec, reply_type/time, ede_code, upstream, cname, list_id; `_time` = the real query time), and ingests into a dedicated `pihole-queries` VL stream. Durable cursor dedup via `app_settings` (`pihole.query_feed.last_id`, defensively parsed); first run records a baseline and ships nothing; subsequent ticks ship only `id > last_id`. Per-UTC-day byte cap (default 500 MiB, env `HOMELAB_MONITOR_PIHOLE_QUERY_FEED_MAX_BYTES_PER_DAY`); cap-hit drops + advances the cursor + emits a self-observability counter. Gated by `pihole_stream_query_feed` (env `HOMELAB_MONITOR_PIHOLE_STREAM_QUERY_FEED`, default false) — the collector no-ops when off. The stream inherits global VL retention (30d); per-stream retention isn't enforceable in VL today (documented).

Added `query_feed_streaming: bool` to `PiholeOverviewResponse` (config-derived). Fixed a Build gap caught in production Refinement: `HOMELAB_MONITOR_PIHOLE_STREAM_QUERY_FEED` (and the byte-cap knob) are now wired through `deploy/compose/docker-compose.yml`'s monitor env block (default false), so the flag actually reaches the prod container.

When the flag is off (public default) the selector is hidden and the tab is unchanged.
