# 2026-05-15

## STAGE-002-007A ‚Äî Cron auto-soft-delete + auto-restore

### Added

- **Migration `0011_cron_soft_deleted_at.py`** ‚Äî adds `soft_deleted_at` column (TEXT NULL, UTC ISO-8601) to `crons` table; idempotent both directions.
- **Cron reconciliation on discovery** ‚Äî `CronRepo.reconcile_soft_deletes()` method; called from cron-discoverer scheduled tick (300s interval) and from `POST /api/crons/discover-now`. When a clean scan no longer finds a cron's fingerprint in a source file, that cron is soft-deleted (`soft_deleted_at` set, `crons.soft_delete` audit row with `who=system`). When the fingerprint reappears on a later clean scan, it auto-restores (`crons.restore` audit row).
- **Reconciliation semantics** ‚Äî per-host and per-source-file scoping; remote-only crons (`source_path IS NULL`) are never auto-soft-deleted; unreadable source files are never treated as clean empty scans (tracked via `unreadable_source_paths` on `CronScanResult`).
- **Register-time auto-restore** ‚Äî `POST /api/hb/{fingerprint}/register` now auto-restores a soft-deleted cron with two audit rows in one transaction (`crons.restore` then `crons.register`).
- **API response changes** ‚Äî `GET /api/crons` hides soft-deleted rows by default; `?include_soft_deleted=true` query param includes them; `GET /api/crons/{fingerprint}` always returns the row (even if soft-deleted); `CronOut` schema gains `soft_deleted_at` field.
- **UI inventory toolbar** ‚Äî "Show soft-deleted" toggle; soft-deleted crons rendered with "Soft-deleted" badge + dimmed (opacity-60) styling.
- **UI detail panel** ‚Äî cron-detail header badge + Disk source panel field display `soft_deleted_at`.
- **Heartbeat unaffected** ‚Äî `POST /api/hb/{fingerprint}/start`, `/ok`, `/fail` are unaffected by `soft_deleted_at` state.

### Bug fixes from 3b prod-rig refinement

1. **compose build broken for `make dev-prod`** ‚Äî monitor service lacked `build:` section; fixed by adding `build: .` to service definition.
2. **IMAGE_TAG shadowing in dev.env** ‚Äî removed `IMAGE_TAG` from `deploy/dev/dev.env.example` to prevent override of `deploy/compose/.env` value during `docker compose up --build`.
3. **Data corruption on unreadable source files** ‚Äî cron-discoverer treated unreadable files as cleanly-scanned empty files, wrongly soft-deleting every cron under them. Fixed: discoverer now tracks `unreadable_source_paths` on `CronScanResult` and excludes them from reconciliation.
4. **Crontab ACL lost after `crontab -e`** ‚Äî user re-editing via `crontab -e` rewrites the file and loses the read ACL set by `host-setup.sh`. Fixed: systemd `.path` unit watches `/var/spool/cron/crontabs/` and auto-re-applies ACL via `refresh-crontab-acl.sh` on any write.
5. **SPA deep-link 404 in prod** ‚Äî requests to client-side routes (e.g., `/overview`) returned 404 from static file serving. Fixed: FastAPI app now has SPA catch-all route (`@app.get("/{path:path}")`) that serves `index.html` for unknown paths, enabling client-side router to handle them.
