## [STAGE-004-023] Backend SSE endpoint for live log tail

**Summary:** New `GET /api/logs/tail?expr=<LogsQL>&services=<csv>` Server-Sent Events endpoint that streams new VictoriaLogs lines to the client in real time (`tail -f`-like), replacing the click-refresh pattern. Backend half only (frontend tail UI is STAGE-004-024). Polls VL ~1s, pushes `event: line` SSE events, with per-connection + global connection caps and backpressure.

**Key changes:**
- New `TailSession` poll loop + `ForwardCursor` (forward/newer cursor with same-ns boundary dedup) + `TailRegistry` (global connection counter) + `TailEvent` union, in `kernel/logs/tail_service.py`.
- `GET /api/logs/tail` handler with strict ordering: global cap → HTTP 503 + `Retry-After: 60`; pre-flight LogsQL probe → 422 `invalid_logsql` (VL structural 4xx) / 502 `upstream_unavailable` (VL down); then a `text/event-stream` StreamingResponse.
- Per-connection backpressure (drop surplus over `max_lines_per_sec`, emit `event: dropped`), per-connection duration cap, VL-error retry (emit `event: error`, retry ~10s, close after 5 consecutive failures), keepalive comments.
- Config `load_tail_config()` (`HOMELAB_MONITOR_TAIL_POLL_MS=1000`, `_MAX_CONNECTIONS=5`, `_MAX_LINES_PER_SEC=200`, `_MAX_DURATION_S=3600`; foot-gun values floor-clamped to >=1).
- `VictoriaLogsClientError.status_code` added so the probe can distinguish VL 4xx from 5xx.
- `ServiceUnavailableProblem` + handler (503 + Retry-After) in `kernel/api/errors.py`.
- Mandatory metrics: `homelab_log_tail_active_connections` (gauge, published by a new `TailMetricsCollector` reading the registry), `homelab_log_tail_lines_streamed_total`, `homelab_log_tail_lines_dropped_total`, `homelab_log_tail_errors_total{kind}` (counters, written from the session).
- Validated against real VictoriaLogs v0.30.0 (`make integration`): confirmed VL returns 200+empty for phrase-garbage LogsQL and 4xx only for structural errors; the 422 probe maps VL's structural 4xx (criterion corrected to match VL's real contract — no client-side LogsQL validation, consistent with /logs/query and /logs/export).

## [STAGE-004-024] Frontend live tail + bidirectional windowed log pager

**Summary:** Add a "Live tail" toggle to the Logs Explorer that consumes the STAGE-004-023 SSE endpoint, plus a general bidirectional windowed log pager. Toggle turns green when live; tail keeps already-loaded historical logs and appends live lines below with sticky auto-scroll. Stopping freezes the streamed lines on screen and pins the custom end time to the stop moment. A 1000-line render-capped FIFO trims from either end: "Load older" trims newest, a new "Load newer" trims oldest, with trimmed-lines banners at each end. (Redesigned mid-Refinement per user feedback — replaced the original status-bar/pause design.)

**Key changes:**
- New `useWindowedLogs` reducer hook (`apps/ui/src/lib/useWindowedLogs.ts`): single rendered line list + 1000-cap FIFO with `reset`/`prependOlder`/`appendNewer`/`trimFrontTo` ops, composite-key dedup, trimmedOlder/trimmedNewer flags. Single source of truth fed by historical query + tail + load-newer.
- `useLogsTail` (`apps/ui/src/lib/logsTail.ts`) — EventSource SSE hook for live lines: rAF-batched `onLines` callback (re-entrancy-safe scheduling), 503/422/502 failure classification via an AbortController-guarded fetch (no double-open of the slot-consuming stream), exponential-backoff reconnect, status/error surface.
- `fetchNewerLogs` (`apps/ui/src/api/logs.ts`) — window-shift query `[newestShown, end]` for "Load newer" (no backend change; VL returns latest-N in any window). Known limitation: if >~500 new lines accumulated, the oldest of the gap are skipped (converges via repeat load-newer or tail).
- LogsExplorerBody: windowed buffer drives render; historical-query→buffer effect (memoized flatLines + early-return guard to avoid a render loop); on-Stop freeze via deterministic pinned-queryKey seeding (frozen lines survive the range-settle refetch); green aria-pressed toggle; inline `role="alert"` error surface for tail + load-newer failures; sticky auto-scroll (re-keyed on the buffer length; `el.scrollTo` guarded); "Load older" stops tailing.
- LogViewer + UseLogsResult: a "Load newer" button (bottom) + top/bottom trimmed-lines banners.
- Removed the interim `LiveTailStatusBar` component (superseded by the green toggle).
