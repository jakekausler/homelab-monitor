## [STAGE-004-023] Backend SSE endpoint for live log tail

**Summary:** New `GET /api/logs/tail?expr=<LogsQL>&services=<csv>` Server-Sent Events endpoint that streams new VictoriaLogs lines to the client in real time (`tail -f`-like), replacing the click-refresh pattern. Backend half only (frontend tail UI is STAGE-004-024). Polls VL ~1s, pushes `event: line` SSE events, with per-connection + global connection caps and backpressure.

**Key changes:**
- New `TailSession` poll loop + `ForwardCursor` (forward/newer cursor with same-ns boundary dedup) + `TailRegistry` (global connection counter) + `TailEvent` union, in `kernel/logs/tail_service.py`.
- `GET /api/logs/tail` handler with strict ordering: global cap → HTTP 503 + `Retry-After: 60`; pre-flight LogsQL probe → 422 `invalid_logsql` (VL structural 4xx) / 502 `upstream_unavailable` (VL down); then a `text/event-stream` StreamingResponse.
- Per-connection backpressure (drop surplus over `max_lines_per_sec`, emit `event: dropped`), per-connection duration cap, VL-error retry (emit `event: error`, retry ~10s, close after 5 consecutive failures), keepalive comments.
- Config `load_tail_config()` (`HOMELAB_MONITOR_TAIL_POLL_MS=1000`, `_MAX_CONNECTIONS=5`, `_MAX_LINES_PER_SEC=200`, `_MAX_DURATION_S=3600`; foot-gun values floor-clamped to >=1).
- `VictoriaLogsClientError.status_code` added so the probe can distinguish VL 4xx from 5xx.
- `ServiceUnavailableProblem` + handler (503 + Retry-After) in `kernel/api/errors.py`.
- Mandatory metrics: `homelab_log_tail_active_connections` (gauge, published by a new `TailMetricsCollector` reading the registry), `homelab_log_tail_lines_streamed_total`, `homelab_log_tail_lines_dropped_total`, `homelab_log_tail_errors_total{kind}` (counters, written from the session).
- Validated against real VictoriaLogs v0.30.0 (`make integration`): confirmed VL returns 200+empty for phrase-garbage LogsQL and 4xx only for structural errors; the 422 probe maps VL's structural 4xx (criterion corrected to match VL's real contract — no client-side LogsQL validation, consistent with /logs/query and /logs/export).
