## [STAGE-004-007] Cursor pagination

**Summary:** "Load older" cursor pagination across all 3 log endpoints + the shared `<LogViewer>`. Pagination surfaces only when content exceeds the page size (500). Fixes the deferred STAGE-003-011 D-DEFER-PAGINATION.

**Backend (Python/FastAPI):**
- Added `apps/monitor/homelab_monitor/kernel/logs/pagination.py` (new) ‚Äî `LogCursor` model + base64url `encode_cursor`/`decode_cursor` (`InvalidCursorError`), `_iso_to_ns`/`_ns_to_iso` (ns-precision, total ‚Äî malformed `_time` ‚Üí 0, never raises), and the shared A1 `paginate_older`: collision-safe cursor `(t=oldest_ns, n=count_at_t)`, inclusive `end` re-include + skip-n, a `[GROUP-COMPLETE]` branch (a page never splits a same-ns group ‚Äî re-queries the full group at the boundary ns), `has_more` from VL's `result.truncated`, and a per-call timestamp-string-keyed ns memo. Returns lines oldest‚Üínewest.
- Updated `apps/monitor/homelab_monitor/kernel/logs/victorialogs_client.py` ‚Äî `with_limits()` per-page limit override.
- Updated `apps/monitor/homelab_monitor/kernel/api/routers/{logs,docker,crons}.py` (+ schemas.py, cron/schemas.py) ‚Äî all 3 endpoints accept `cursor`, return `next_cursor`/`has_more`, call the single `paginate_older`; malformed cursor ‚Üí 400 `invalid_cursor` (consistent HttpProblem envelope across all 3); `/api/logs/query` page-size default 100‚Üí500; cron paginates within its finite run window.

**Frontend (React/Vite):**
- Updated `UseLogsResult` with optional `hasMore`/`isLoadingOlder`/`loadOlder`.
- Updated `<LogViewer>` to render a top-of-list "Load older" button (gated, spinner, `aria-busy`, error preserves lines).
- Migrated `useContainerLogs`/`useCronRunLog` ‚Üí `useInfiniteQuery`, new `useLogsQuery` (api/logs.ts, scaffolding for STAGE-004-010) ‚Äî mirror `useListDockerSuggestions`.
- Viewers flatten `pages.slice().reverse().flatMap(p => p.lines)` so older pages render on top (oldest‚Üínewest).
- **CronRunLogViewer migrated to render via `<LogViewer>` (headerSlot)** preserving badges/anomaly/run-state/running-refresh/expired/banner.
- OpenAPI + UI types regenerated for the new params/fields.

**Tests:**

- Updated 3 endpoints' tests + frontend viewer tests (useInfiniteQuery `{pages:[...]}` mock shape, multi-page order older-above-newer).

**Refinement (real-VL + manual UI) found + fixed 2 real bugs unit tests couldn't catch:**
