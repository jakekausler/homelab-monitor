## [STAGE-001-007] In-process plugin loader + scheduler

**Commit:** 32e036cbc34d2a26f45fef740f2206081870661c

Implements the runtime backbone for periodic collector execution: an async per-collector tick scheduler with absolute-deadline precision, three execution modes (ASYNC / THREAD / PROCESS), uniform self-metrics emission, and graceful shutdown semantics. First stage where the system "runs" anything — after STAGE-001-007, starting the monitor wires up the noop collector and ticks fire on schedule.

### New modules

- `apps/monitor/homelab_monitor/kernel/scheduler/scheduler.py` — `Scheduler` + `SchedulerConfig`. Per-collector `asyncio.Task` tick loops with `asyncio.get_running_loop().time()` monotonic clock. Drift skip-forward (no catch-up storms). Hash-based initial offset for ≥1s intervals. Three RunKind dispatch arms; PROCESS via `ProcessPoolExecutor` with module-level `_process_runner` and pure-result worker pattern (returns `(CollectorResult, list[MetricEntry])`; parent replays buffered metrics through real `MetricsWriter`).
- `apps/monitor/homelab_monitor/kernel/plugins/loader.py` — `LoadedCollector` (frozen dataclass) + `PluginLoader` (in-memory registry; `register()` validates via `CollectorConfig.model_validate`, `load_all()` returns defensive copy). Filesystem + entry-point scans deferred to STAGE-001-009 / EPIC-002 with scaffolding comments at insertion points.
- `apps/monitor/homelab_monitor/kernel/plugins/process_context.py` — `BufferingMetricsWriter` (frozen, slots, picklable, `drain()` returns + clears) + `ProcessCollectorContext` (narrow picklable context for PROCESS workers; fields: `config`, `secrets`, `metrics`).

### Self-metrics (5 mandatory, scheduler-owned)

- `homelab_collector_run_success_total{name}` — counter
- `homelab_collector_run_failure_total{name, reason}` — counter; reason ∈ {"timeout", "exception", "result_error"}
- `homelab_collector_run_shutdown_total{name}` — counter (NEW; distinct from failure_total so alerting on `rate(failure_total[5m])>0` doesn't page on graceful restart)
- `homelab_collector_run_duration_seconds{name}` — summary
- `homelab_collector_run_last_error_age_seconds{name}` — gauge; pattern (ii): emit 0 on failure, age on subsequent success, never before first failure

### Tests added

- `apps/monitor/tests/test_scheduler.py` — unit tests for tick precision, multi-interval, timeout, all 3 RunKinds, offset decorrelation (with deterministic hash-collision skip guard), shutdown, in-flight cancellation, failure classification, last-error-age decay, drift, double-start guard, plus a no-emission-before-first-failure test.
- `apps/monitor/tests/test_plugin_loader.py` — register/validate/defensive-copy/frozen-dataclass tests.
- `apps/monitor/tests/test_process_context.py` — parametrized over write_gauge/counter/summary; drain semantics; pickle roundtrip; defensive label copy.
- `apps/monitor/tests/test_scheduler_e2e.py` — 6 e2e scenarios with REAL kernel types (no mocks): real `SqliteRepository`, `httpx.AsyncClient`, `SyncSecretsResolver`. Covers 30s tick precision, mixed RunKind concurrency, PROCESS worker hard-crash isolation (`os._exit(1)`), restart independence, high-cardinality hash-offset spread.
- New shared helper `apps/monitor/tests/_helpers.py` (count_metric).

D1 per-collector asyncio.Task with absolute-deadline ticks. D2 `loop.time()` (monotonic). D3 pure-result PROCESS workers + BufferingMetricsWriter (single pickle hop in/out). D4 scheduler-owned uniform self-metrics. D5 distinct `shutdown_total` counter (alerting hygiene).

## [STAGE-001-008] Concurrency groups + failure budget + quarantine

**Commit:** ed4cd9f5dfde5690b6429e8ddcd9572b5ae5d591

Adds the two pieces of behavioral discipline that make the scheduler safe under load: **concurrency groups** prevent thundering-herd against shared backends (the UDM controller, the HA API, etc.), and **failure budgets with quarantine** prevent a chronically-failing plugin from running every tick forever.

### New / modified modules

- `apps/monitor/homelab_monitor/kernel/scheduler/failure_budget.py` (NEW) — `FailureBudget` class + `QuarantineState` frozen dataclass. In-memory authoritative consecutive-failure counter; persists on failure / quarantine entry / clear (NEVER on success — eliminates ~99% of steady-state DB writes). Default threshold 5; per-collector override via `CollectorConfig.quarantine_after`. Audit event names: `collector.quarantine_entered`, `collector.quarantine_cleared`. Defensive guard at top of `record_failure` prevents double quarantine_entered audit rows if the scheduler's gate is bypassed. WARNING log on quarantine entry. TODO scaffolding for STAGE-001-013 alert dispatch.
- `apps/monitor/homelab_monitor/kernel/scheduler/scheduler.py` (modified) — `_tick` body refactored: quarantine gate at TOP (early return + `skipped_total{reason="quarantined"}`), then group lock acquisition with `interval/2` deadline OUTSIDE the inner `asyncio.timeout()` block (`skipped_total{reason="group_busy"}` on TimeoutError), then dispatch in try/finally that releases the lock unconditionally. Each failure path (timeout, exception, result_error) calls `failure_budget.record_failure` if a budget is present; success path calls `record_success`. New ctor parameter `failure_budget: FailureBudget | None = None`. New `_group_locks: dict[str, asyncio.Lock]`, `_configs: dict[str, CollectorConfig]`, helper `_threshold_for(c)`. New public method `clear_quarantine(name, by="operator")` that delegates to FailureBudget. `start()` calls `failure_budget.load_state()` if budget present.
- `apps/monitor/homelab_monitor/kernel/db/audit.py` (modified) — appended public `insert_audit(conn, *, who, what, before, after, ip, when)` helper + `AUDIT_INSERT` SQL constant. Designed for the in-transaction pattern (UPDATE + audit_log INSERT atomic in same connection). Legacy `audit_write` refactored to use the same `AUDIT_INSERT` constant for SQL deduplication.
- `apps/monitor/alembic/versions/0003_collector_quarantine.py` (NEW) — additive migration. Adds 3 columns to `collectors` (`quarantined_at TEXT NULL`, `quarantine_reason TEXT NULL`, `consecutive_failures INTEGER NOT NULL DEFAULT 0`). Adds `CREATE UNIQUE INDEX ix_collectors_name_unique ON collectors(name)`. Online-mode-only defensive duplicate-name check raises if any duplicates exist before adding the unique index.
- `apps/monitor/homelab_monitor/kernel/plugins/types.py` (modified) — `CollectorConfig.quarantine_after: int | None = Field(default=None, ge=1)` added.

### New 6th self-metric

`homelab_collector_run_skipped_total{name, reason}` — emitted by scheduler when ticks are SKIPPED (not just failed). Reasons: `"group_busy"` (lock acquire timeout), `"quarantined"` (collector currently quarantined). Joins the existing 5 self-metrics to give operators a complete picture of tick outcomes.
