## [STAGE-001-004] SQLite + Alembic + first migration

### Design (commit `47f4c016940f58dbe2d9ea12d804078863e3152a`)
- Locked decisions: schema strategy = all-19-tables-up-front with minimal stubs (option a from spec); engine = async sqlalchemy[asyncio]+aiosqlite; migrations = Alembic with async env.py; UUIDv7 PKs via `uuid-utils` PyPI package (except `users.id` which is INTEGER autoincrement); thin generic `SqliteRepository` facade; audit_write helper at `kernel/db/audit.py`; pragmas (WAL, FK ON, busy_timeout=5000) applied via `engine.sync_engine` connect listener; `HOMELAB_MONITOR_AUTO_MIGRATE` env var (default `true`) gates boot-time migration.

### Build (commit `ae8d2c66b88258b21fe5748ae87dbd824c683d41`)
- Added kernel/db package: `engine.py`, `repository.py`, `migrations.py`, `audit.py`, `ids.py`, `time.py`
- Alembic infrastructure: `alembic.ini`, `alembic/env.py` (handles both standalone and pytest-asyncio event-loop contexts via ThreadPoolExecutor), `alembic/script.py.mako`, `alembic/versions/0001_initial_schema.py`
- 0001 migration creates 19 tables: 4 fully defined (`users`, `sessions`, `audit_log`, `api_tokens`) + 15 stubs (`targets`, `collectors`, `crons`, `heartbeats_state`, `alerts`, `alert_outcomes`, `runbooks`, `runbook_runs`, `secrets`, `channels`, `routing_rules`, `digest_configs`, `maintenance_windows`, `suggestions`, `tool_scorecards`)
- 2 indexes: `idx_alerts_fingerprint`, `idx_targets_name`
- CLI subcommands: `hm migrate`, `hm migrate status`, `hm migrate history`
- 8 new test files + conftest.py — 40 backend tests, 100% coverage on `homelab_monitor.kernel.*` and `homelab_monitor.cli.migrate`
- Added `uuid-utils>=0.10` as runtime dep
