## [STAGE-001-007] In-process plugin loader + scheduler

**Commit:** 32e036cbc34d2a26f45fef740f2206081870661c

Implements the runtime backbone for periodic collector execution: an async per-collector tick scheduler with absolute-deadline precision, three execution modes (ASYNC / THREAD / PROCESS), uniform self-metrics emission, and graceful shutdown semantics. First stage where the system "runs" anything ŌĆö after STAGE-001-007, starting the monitor wires up the noop collector and ticks fire on schedule.

### New modules

- `apps/monitor/homelab_monitor/kernel/scheduler/scheduler.py` ŌĆö `Scheduler` + `SchedulerConfig`. Per-collector `asyncio.Task` tick loops with `asyncio.get_running_loop().time()` monotonic clock. Drift skip-forward (no catch-up storms). Hash-based initial offset for Ōēź1s intervals. Three RunKind dispatch arms; PROCESS via `ProcessPoolExecutor` with module-level `_process_runner` and pure-result worker pattern (returns `(CollectorResult, list[MetricEntry])`; parent replays buffered metrics through real `MetricsWriter`).
- `apps/monitor/homelab_monitor/kernel/plugins/loader.py` ŌĆö `LoadedCollector` (frozen dataclass) + `PluginLoader` (in-memory registry; `register()` validates via `CollectorConfig.model_validate`, `load_all()` returns defensive copy). Filesystem + entry-point scans deferred to STAGE-001-009 / EPIC-002 with scaffolding comments at insertion points.
- `apps/monitor/homelab_monitor/kernel/plugins/process_context.py` ŌĆö `BufferingMetricsWriter` (frozen, slots, picklable, `drain()` returns + clears) + `ProcessCollectorContext` (narrow picklable context for PROCESS workers; fields: `config`, `secrets`, `metrics`).

### Self-metrics (5 mandatory, scheduler-owned)

- `homelab_collector_run_failure_total{name, reason}` ŌĆö counter; reason Ōłł {"timeout", "exception", "result_error"}

## [STAGE-001-008] Concurrency groups + failure budget + quarantine

**Commit:** ed4cd9f5dfde5690b6429e8ddcd9572b5ae5d591

Adds the two pieces of behavioral discipline that make the scheduler safe under load: **concurrency groups** prevent thundering-herd against shared backends (the UDM controller, the HA API, etc.), and **failure budgets with quarantine** prevent a chronically-failing plugin from running every tick forever.

### New / modified modules

- `apps/monitor/homelab_monitor/kernel/scheduler/failure_budget.py` (NEW) ŌĆö `FailureBudget` class + `QuarantineState` frozen dataclass. In-memory authoritative consecutive-failure counter; persists on failure / quarantine entry / clear (NEVER on success ŌĆö eliminates ~99% of steady-state DB writes). Default threshold 5; per-collector override via `CollectorConfig.quarantine_after`. Audit event names: `collector.quarantine_entered`, `collector.quarantine_cleared`. Defensive guard at top of `record_failure` prevents double quarantine_entered audit rows if the scheduler's gate is bypassed. WARNING log on quarantine entry. TODO scaffolding for STAGE-001-013 alert dispatch.
- `apps/monitor/homelab_monitor/kernel/scheduler/scheduler.py` (modified) ŌĆö `_tick` body refactored: quarantine gate at TOP (early return + `skipped_total{reason="quarantined"}`), then group lock acquisition with `interval/2` deadline OUTSIDE the inner `asyncio.timeout()` block (`skipped_total{reason="group_busy"}` on TimeoutError), then dispatch in try/finally that releases the lock unconditionally. Each failure path (timeout, exception, result_error) calls `failure_budget.record_failure` if a budget is present; success path calls `record_success`. New ctor parameter `failure_budget: FailureBudget | None = None`. New `_group_locks: dict[str, asyncio.Lock]`, `_configs: dict[str, CollectorConfig]`, helper `_threshold_for(c)`. New public method `clear_quarantine(name, by="operator")` that delegates to FailureBudget. `start()` calls `failure_budget.load_state()` if budget present.
- `apps/monitor/homelab_monitor/kernel/db/audit.py` (modified) ŌĆö appended public `insert_audit(conn, *, who, what, before, after, ip, when)` helper + `AUDIT_INSERT` SQL constant. Designed for the in-transaction pattern (UPDATE + audit_log INSERT atomic in same connection). Legacy `audit_write` refactored to use the same `AUDIT_INSERT` constant for SQL deduplication.
- `apps/monitor/alembic/versions/0003_collector_quarantine.py` (NEW) ŌĆö additive migration. Adds 3 columns to `collectors` (`quarantined_at TEXT NULL`, `quarantine_reason TEXT NULL`, `consecutive_failures INTEGER NOT NULL DEFAULT 0`). Adds `CREATE UNIQUE INDEX ix_collectors_name_unique ON collectors(name)`. Online-mode-only defensive duplicate-name check raises if any duplicates exist before adding the unique index.
- `apps/monitor/homelab_monitor/kernel/plugins/types.py` (modified) ŌĆö `CollectorConfig.quarantine_after: int | None = Field(default=None, ge=1)` added.

## [STAGE-001-009] Subprocess plugin runner + JSON line protocol

**Commit:** 6a5173028ec0a1884ba461d839b8371ca6efc4e1

Adds the second plugin execution path: subprocess plugins. Any-language scripts/binaries described by a `plugin.yaml` manifest. Runner spawns the subprocess on schedule, reads line-delimited JSON events from stdout, captures stderr to structured logs, enforces timeout via SIGTERM/SIGKILL, and interprets the exit code. Unlocks bash/Go/Rust/whatever plugins, untrusted/sandboxed plugins, and a clean path for community plugins.

After this stage, a hello-world bash plugin emits metrics that flow through the same `MetricsWriter` as in-process collectors.

### Locked design decisions

D1 SubprocessCollector class wrapper + new RunKind.SUBPROCESS enum (zero-cost dispatch via existing async path; class-factory pattern for ClassVar Protocol satisfaction). D2 "DB writes blocked" architectural ŌĆö JSON protocol has no db_write line type; trust differences are env scrubbing + secrets narrowing + workdir confinement. D3 hybrid context delivery (manifest env ŌåÆ env vars + allowlist; secrets + runtime metadata ŌåÆ JSON on stdin one-shot, then EOF). D4 strict line-by-line stdout parsing with warn-and-skip on errors; partial-line at EOF discarded; unknown type forward-compat; line-after-result discarded; no-result synthesizes failure. D5 concurrent stderr task; info-level real-time + warning summary on non-zero exit; no content duplication. D6 two-phase loader: sync `register()` + new async `persist_to_db(repo)` (closes STAGE-008's loader-INSERT gap). D7 `runbooks/_examples/hello-subprocess-plugin/` bash plugin (drains stdin + emits 1 metric + 1 result).
