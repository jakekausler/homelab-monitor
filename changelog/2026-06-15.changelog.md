## [STAGE-005-042] HA cadence detail API + Status-tab idle cadence widget

**Summary**

The in-app HA cadence surface (split off from STAGE-005-038): a new `GET /api/integrations/home-assistant/cadence` endpoint + a Status-tab widget listing idle (>24h or never-triggered) automations and idle scripts, mirroring the `/entities` endpoint + `HaEntitiesDrill` pattern. Final stage of EPIC-005 (Home Assistant integration) â€” completes the epic at 42/42.

**Backend** â€” `GET /cadence` returns two named collections (`automations`, `scripts`) from VictoriaMetrics per-series gauges. Automation universe = `homelab_ha_automation_enabled` (LEFT JOIN `homelab_ha_automation_last_triggered_seconds`; absent â†’ `last_triggered_age_seconds: null` = never-triggered). Include iff `enabled AND (age is None OR age > 86400)`. Scripts: universe = `homelab_ha_script_last_triggered_seconds`, include iff `age > 86400`. Sorted most-idle-first (null age first, then age DESC), top-N capped, `*_total`/`*_returned` per collection, `filtered_to="idle_24h"`. friendly_name enriched via one live HA `get_states()` (HA-down â†’ null, never 502; VM-down â†’ 502).

**Frontend** â€” `useHomeAssistantCadence` hook; `HaCadenceAutomationsDrill` + `HaCadenceScriptsDrill` (both consuming the one shared hook, TanStack-deduped), mounted as "Idle automations" + "Idle scripts" `PanelSection` cards in the HA Status tab. Null age renders "never".

**Refinement scope change (user)** â€” disabled automations EXCLUDED entirely (the card's "idle + disabled" became "idle only"): the predicate dropped the `not enabled` inclusion clause, `filtered_to` â†’ `"idle_24h"`, the frontend "disabled" badge removed, the card retitled "Idle automations".
