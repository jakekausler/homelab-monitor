## [STAGE-005-041] Grafana dashboard review & readability pass (System + Home Assistant) + new metric tabs

**Summary**

Interactive readability pass over the Grafana observability surface that expanded (by user direction) from a Grafana-only review into a backend collector fix, a host-overview rework, four brand-new dashboards each embedded as its own Metrics-page tab, and a full Home Assistant dashboard rework. Dual sign-off (System dashboard + Home Assistant dashboard) plus Desktop/Mobile review of the new tabs.

**Host disk collector fix** ‚Äî `homelab_host_disk_bytes` was emitting ~19 meaningless container-internal mountpoints (the collector runs inside the container, so `psutil.disk_partitions()` returns the container mount namespace). Added a `disk_mountpoint_relabel` allowlist+relabel config field (default `{"/config": "/", "/host-compose": "/storage"}`) so the dashboard shows only the real `/` and `/storage` filesystems with friendly labels. Empty map preserves emit-all behavior.

**System (host-overview) dashboard** ‚Äî CPU split into "all cores" + per-core (numeric legend via `sort_by_label_numeric`); Load average split into "1m" + "1m/5m/15m"; Memory & swap timeseries; Disk bargauge percent-full on the relabeled `/`+`/storage` series; Network split into separate Download + Upload charts (each rate[1m]/[5m]/[15m]); Uptime `dtdurations`.

**Four new dashboards + Metrics tabs** ‚Äî `containers`, `collectors`, `heartbeats`, `storage-logs` (auto-provisioned JSON), each embedded as a tab inserted between System and Home Assistant. Timeseries-first; introduced the reusable binary Up/Down stat pattern (green "Up"/red "Down" word, no digit) on the Storage & Logs VictoriaLogs panel.

**Home Assistant dashboard rework** ‚Äî "Status" + "WebSocket" binary Up/Down; new "Registered entities" + "Integrations not loaded" stats; fixed the `sum(homelab_ha_entity_available == 0)` always-0 bug ‚Üí `count(... == 0)` (now correctly shows ~198, matching the in-app Status page); "Entities Unavailable" / "Stale Entities" converted to timeseries; "Low/Critical Batteries" bargauge ‚Üí "Batteries below 20%" timeseries; "Collector Run Health" (always-empty error queries) ‚Üí "HA collector run rate"; count stats threshold-colored (green 0 / amber <10 / red ‚â•10; Config Entry Errors + Integrations not loaded green 0 / red ‚â•1); full grid reflow.

**Mobile fix** ‚Äî the Metrics tab nav now scrolls horizontally (`overflow-x-auto`) instead of forcing page-wide overflow when the 6 tabs exceed viewport width.

**Design decisions**

- Binary up/down panels use `color.mode: "thresholds"` (steps null‚Üíred, 1‚Üígreen) + value-mappings for the "Up"/"Down" text. (`color.mode: "fixed"` shows the text but NOT the green/red background ‚Äî caught in code review.)
- Disk relabel map baked as the `HostCollectorConfig` default (no live per-collector config injection path exists yet; matches the existing `extra_mountpoints`/`exclude_disk_devs` default pattern).
- `count(metric == 0)` not `sum(metric == 0)` for "how many match" ‚Äî `sum(==0)` always yields 0.
- Grafana parser-safe regex `[0-9]` (not `\d`, which 422s) + `sort_by_label_numeric` for numeric legend order.
- New dashboards auto-provision from `deploy/grafana/dashboards/*.json`; tabs reuse the `GrafanaDashboardEmbed` kiosk-iframe pattern.
