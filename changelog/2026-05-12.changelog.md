## STAGE-002-005 — /api/hb/{fingerprint}/register endpoint + hidden semantic shift + wrapper_installed_at rename

### Added

- New endpoint: `POST /api/hb/{fingerprint}/register` (token-auth, `Scope.HEARTBEAT_WRITE`). Idempotent upsert: 201 on first create, 200 on re-register with `wrapper: true`. Updates `wrapper_last_seen_at` on every successful re-register and writes an audit row (`crons.register`, D10 Path 3). Returns `CronOut` in response body (D1). Rejects URL-vs-body fingerprint mismatch with 422 + `fingerprint_mismatch` detail flag (D6). Never overwrites policy fields (name, expected_grace_seconds, enabled, hidden_at) on re-register (D7).
- New Pydantic schema `RegisterCronBody` in `apps/monitor/homelab_monitor/kernel/cron/schemas.py` — fields: `host`, `command`, `schedule`, `cadence_seconds`, `expected_grace_seconds`, `enabled`, `wrapper` (boolean, controls whether to refresh `wrapper_last_seen_at` and audit). Reuses wrapper-aware validation from `CronCreate`.
- Alembic migration 0009 (`apps/monitor/alembic/versions/0009_wrapper_last_seen_at_rename.py`) — renames `crons.wrapper_installed_at` → `crons.wrapper_last_seen_at` (no data loss; single ALTER TABLE + renaming index).
- Helper `cron_record_to_out()` in `kernel/cron/repository.py` — shared extraction of `CronRecord` → `CronOut` logic; used by both new `register_cron` method and `/api/crons/{fingerprint}` GET route (avoids duplication).
- Helper `_cron_to_audit_after()` in `kernel/cron/repository.py` — isolates audit row construction for register flow; accepts `action` ('register'), `who` (token name), `wrapper` (flag).
- New route handler `receive_register()` in `kernel/api/routers/heartbeat.py` — FastAPI POST with full OpenAPI responses dict: 201 Created, 200 OK, 401 Unauthorized, 403 Forbidden, 422 Unprocessable Entity, 429 Too Many Requests. Helper `_cron_json_response()` factored to emit (status_code, CronOut) for both register and legacy observe paths.
- New method `CronRepo.register_cron()` in `kernel/cron/repository.py` — 3-branch logic (compute FP → query existing → [branch 1] INSERT if new, [branch 2] UPDATE wrapper_last_seen_at if wrapper=true + existing, [branch 3] no-op if not-new AND wrapper=false). Atomic transaction; includes audit on INSERT and on wrapper UPDATE (D10 Path 3: audit every wrapper handshake; wrapper=false re-register writes no audit).
- 20+ new backend tests in `apps/monitor/tests/test_api_register.py` — comprehensive coverage of D1–D10 decisions, status code matrix (201/200/401/403/422/429), fingerprint-mismatch rejection, policy-field immutability, wrapper-flag semantics, rate limiting, audit invariants, invalid schedule/cadence rejection (422 + detail flags per D9).
- Updated tests in `test_api_heartbeat.py`, `test_heartbeat_repo.py`, `test_cron_repo.py`, `test_db_migrations.py`, `test_api_crons.py` — refactored for column rename (wrapper_last_seen_at) + shared helper (cron_record_to_out) usage + audit verb expectations on register flow.

### Refinement summary

e2e-tester ran 18 live-backend scenarios against running monitor:

1. **201 on first register** — new fingerprint + valid payload → 201 + CronOut + audit `crons.register` row
2. **200 on re-register wrapper=true** — same fingerprint, wrapper=true → 200 + CronOut + audit `crons.register` row (wrapper refresh)
3. **200 on re-register wrapper=false** — same fingerprint, wrapper=false → 200 + CronOut + no audit row (D10 Path 3)
4. **422 fingerprint mismatch** — URL FP ≠ body FP → 422 + `fingerprint_mismatch` detail flag (D6)
5. **422 invalid schedule** — malformed cron expr → 422 + `invalid_schedule` flag
6. **422 invalid cadence** — cadence=−1 → 422 + `invalid_schedule` flag
7. **401 missing token** — no auth header → 401
8. **403 wrong scope** — token with wrong scope → 403
9. **429 rate limit** — 61 requests in 60s → first 60 succeed, 61st returns 429 + `Retry-After` header
10. **Policy field immutability** — PATCH updates name; re-register with different name → name unchanged (D7)
11. **Hidden cron data capture** — hidden_at IS NOT NULL; POST /ok → 204 + heartbeats_state upserted + audit row written (D5)
12. **Hidden cron from /register** — /register with hidden_at in body (rejected, not whitelisted in RegisterCronBody)
13. **Wrapper timestamp refresh** — first register sets wrapper_last_seen_at=T1; re-register with wrapper=true → timestamp updates to T2 > T1 (D2)
14. **Wrapper=false skips timestamp** — register with wrapper=false → wrapper_last_seen_at unchanged (D10)
15. **Audit `who` field** — register audit row has who = token name (not user, not 'system'; D3)
16. **CronOut response shape** — 201 + 200 both return same CronOut with all fields (fingerprint, name, host, schedule, cadence_seconds, expected_grace_seconds, enabled, hidden_at, source_path, wrapper_last_seen_at, last_seen_state, updated_at; D1)
17. **Re-register from same host-command pair** — deterministic fingerprint → same result both times (idempotent upsert)
18. **GET /api/crons/{fp}** — uses new `cron_record_to_out()` helper; same response shape as /register (consistency)

All 18 PASS.
