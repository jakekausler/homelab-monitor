### Decisions locked (recorded in stage tracking)

- D1 Hybrid schedule storage: raw `schedule TEXT` + denormalized `schedule_canonical TEXT` (deviates from brainstormer's "string-only" recommendation; preserves round-trip fidelity while enabling future query-ability).
- D2 Soft-delete via existing `archived_at` column (no new `deleted_at`). List/get filters `WHERE archived_at IS NULL` by default; `?include_archived=1` opts in. Restore via `PATCH {archived_at: null}` (no separate /restore endpoint).
- D2a Receiver 404s on archived crons via `_SELECT_CRON_SQL` filter + new regression test.
- D3 Changed-fields-only audit on PATCH: `before`/`after` JSONs contain only diffs. Empty-diff PATCH returns 200 with NO audit row. Audit verbs: `crons.create`, `crons.update`, `crons.delete`, `crons.restore`.
- D3a Schedule preview endpoint: both saved (`GET /api/crons/{id}/preview-runs`) and unsaved (`POST /api/crons/preview-runs`) variants; single croniter source.
- D4 Separate route `/inventory/crons/{id}` for detail (bookmarkable + deep-linkable); filter state preserved via TanStack Router `search` params.
- D4a Shared `<CronForm>` component with `mode=create|edit|promote` props (reused by AddCronModal, CronDetail, future STAGE-002-007 promote-suggestion flow).
- Pagination: offset/limit, default 100, max 500; `CronListQuery(extra='forbid')` rejects rogue keys.
- Xor loosened mid-Build: "at-least-one" between schedule and cadence_seconds (`schedule != '' OR cadence_seconds > 0`). Backend mirrors cadence from schedule via croniter average-interval helper, so the form ships values that satisfy both invariants.

### Refinement summary

Six UI production files modified during Refinement to address user-tested behaviors:

- **Mode-swap correctness** (`CronForm.tsx`): React reconciliation reusing DOM nodes across mutually-exclusive ternary branches caused `type="number"` from cadence input to persist on schedule input, coercing cron expressions to "0". Fix: explicit `key="schedule-input"` and `key="cadence-input"` force remount. Also: Zod superRefine silent-block fix (`cadence_seconds=0` forced when scheduleMode='schedule'); RHF mode-swap handlers now `setValue('scheduleMode',...)` with `{shouldDirty,shouldTouch,shouldValidate}`; `lastScheduleRef`/`lastCadenceRef` (and post-review fixes seeding them from raw `defaultValues` to capture backend-mirrored cadence/schedule) preserve mode-swap defaults; `titleCase` used in mode dropdown options.
- **Save flow** (`CronDetail.tsx`): `navigate({to:'/inventory/crons'})` after successful PATCH so the user lands back on the list with their changes visible. Post-code-review: post-archive nav now sets `include_archived: true` so the just-archived row stays visible with its `archived` badge instead of vanishing.
- **Title-Case display** (`badges.tsx`, `CronsToolbar.tsx`, `CronForm.tsx`): badge text + dropdown labels capitalize the first character via a new shared `capitalize` helper at `apps/ui/src/lib/text.ts` (re-exported from `badges.tsx` as `titleCase` for backward compatibility). `aria-label` retains the lowercase enum value for screen readers and tests.
- **Archive terminology**: "Soft-delete" button + "Soft-delete cron?" heading renamed to "Archive" / "Archive cron?" / "Archiving...". Server-side action unchanged (`archived_at` column). Placeholder name â€” STAGE-002-006 renames to "Hide" with the derived-state model.
- **Mobile dialog overflow** (`components/ui/dialog.tsx`): `DialogContent` gets `max-h-[calc(100vh-2rem)] overflow-y-auto`.
