## [STAGE-005-017] HAPushChannel (alerts → HA mobile push)

**Summary:** First Wave-D channel. A dispatcher `Channel` that delivers alert events to Home Assistant mobile push (`notify.mobile_app_<device>`) — the outbound half of the bidirectional integration. Implements the standard `Channel.deliver(AlertEvent)` contract; registered in the AlertDispatcher; reuses the shared HA REST client's `call_service`. WHICH alerts reach it is STAGE-005-018's job (minimal severity routing); this stage makes the channel exist + deliver.

**Files — Backend (new):**
- `apps/monitor/homelab_monitor/kernel/dispatch/channels/ha_push.py` — `HAPushChannel` (kind="ha_push"). Constructor injects the shared `HomeAssistantRestClient` + the `notify_service` string + `public_url_provider: Callable[[], str|None] = get_public_url`. `async deliver(event)`: EMPTY notify_service → no-op return (open-source-safe; the channel does nothing until configured). On `AlertFiringEvent` → title `f"[{severity.value.upper()}] {summary}"`, message = description, data `{tag: fingerprint, severity, group, url?}`; on `AlertResolvedEvent` → title `f"[RESOLVED] {summary}"`, message `f"{summary} resolved"`, data `{tag: fingerprint, severity}`. Fallback chain: alertname = labels["alertname"] or "alert"; summary = annotations["summary"] or alertname; description = annotations["description"] or summary. Deep-link `data["url"]` only when `get_public_url()` is set (public_url + annotations["explorer"], else public_url + "/alerts/active"); omitted when public_url is None (never a bare relative path). Calls `call_service("notify", notify_service, {message, title, data})`; on `HaError` raises a token-safe `RuntimeError(f"ha_push delivery failed: {reason} status={status}")` (reason+status only — structurally cannot carry the token) so the dispatcher records the per-channel failure; on `HaServiceResult` returns. Uses `tag=fingerprint` so a re-firing alert UPDATES the existing HA notification and a resolved push REPLACES the firing card.
- `apps/monitor/tests/test_ha_push_channel.py` — 10 tests covering every branch (firing full/empty-annotations/no-alertname; deep-link with-explorer / fallback-to-/alerts/active / omitted-when-no-public-url; resolved [RESOLVED] format; empty-notify-service no-op; HaError → token-safe RuntimeError; HaServiceResult success). 100% branch coverage on ha_push.py.
