# 2026-05-13

## STAGE-002-006 — Cron detail page 4-panel redesign + list-view filter additions + sonner toasts

### Added

- **Cron detail page rewrite** — 4-panel layout (Heartbeat state / Disk source / Monitoring policy / Actions) in 2×2 grid at `lg:` (1024px+), 1-col stack below. Mobile order: Heartbeat state → Disk source → Monitoring policy → Actions.
- **Header redesign** — cron name + StateBadge + "Remote" badge (when `source_path` is null) + "Hidden" badge (when `hidden_at` is set). Subtitle: host · command.
- **Disk source panel** — host, source_path, schedule (with canonical tooltip), command. Blue info banner when `source_path` is null: "Remote cron on `<host>`. The monitor doesn't have direct file access to this host. Wrapper-based heartbeats are the only signal."
- **Heartbeat state panel** — current_state, current_streak, last_ok_at, last_fail_at, expected_next_at, last_duration_seconds, last_exit_code, wrapper-last-seen display (timestamp or "No wrapper installed").
- **Monitoring policy panel** — CronForm (name, expected_grace_seconds, enabled). Save triggers sonner success/error toast.
- **Actions panel** — Hide/Unhide button + disabled "Install heartbeat wrapper" placeholder. Hide button click → silent action (no confirmation modal) → toast.success → stays on detail page; query refetches and shows "Hidden" badge in header. "Unhiding…" in-flight text on Restore button.
- **Cron list view updates** — Remote badge in Host column, Wrapper column (✓ Badge when `wrapper_last_seen_at` is set), Hidden column. New "Wrapper installed" filter dropdown in toolbar (Any wrapper / Wrapper installed / No wrapper). Updated empty state: "No crons yet. Crons will appear here once they are discovered or have registered a heartbeat."
- **sonner toast primitive** — added `sonner` npm dep, `<Toaster position="top-right" richColors />` mounted at app root.
- **Backend wrapper_installed filter** on `GET /api/crons` — accepts `wrapper_installed: true|false|null` (or omitted = no filter); maps to `wrapper_last_seen_at IS [NOT] NULL` query.
- **Audit payload fingerprint backfill** — `update_cron` and `register_cron` wrapper-refresh path now add `fingerprint` to before/after audit JSON payloads. Reserved for future audit surfaces (STAGE-002-007A and later).

### Refinement-phase scope reduction

- **Removed wrapper handshake history panel** — user's mental model of "history" was "schedule/script change history," but those fields are part of fingerprint identity (changes manifest as new cron rows). Only `wrapper_last_seen_at` refresh is meaningful; user judged this not a change worthy of history display.
- **Removed first-registered subtitle** in detail header (audit-query-driven; deferred to future stage).
- **Removed navigation after Hide** — toast remains, user stays on detail page; cron query refetch shows "Hidden" badge.
- **Renamed in-flight text** "Restoring…" → "Unhiding…".
