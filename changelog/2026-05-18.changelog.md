# 2026-05-18

## STAGE-002-009 — Wrapper helpers: embed fingerprint at install time, call /register first, Install button local-host only

### Added

- **Heartbeat wrapper install feature** — bash wrapper template (`apps/monitor/homelab_monitor/data/cron-with-heartbeat.sh.tmpl`) wraps a host cron command and POSTs `/start` then `/ok`|`/fail` heartbeats to the monitor. The wrapper has the fingerprint, heartbeat URL base, and token file path baked in at install time.
- **Three install paths:**
  - UI "Install heartbeat wrapper" button on the cron detail page (`InstallHeartbeatModal.tsx` — dry-run preview → confirm flow)
  - CLI `hm cron install-wrapper`
  - Standalone remote-host CLI `install_wrapper_remote.py` for hosts not running the monitor stack
- **Host-side cron-apply executor architecture** — the monitor container has ZERO host-write capability. It writes only an IPC request file into `/var/lib/homelab-monitor/cron-apply/{requests,results}/`. A root systemd oneshot service (`scripts/hm-cron-apply.sh`, triggered by `homelab-monitor-cron-apply.path` watching the requests dir, with a `homelab-monitor-cron-apply.timer` 60s safety-net) performs all privileged host writes: the crontab rewrite, the wrapper script at `/usr/local/bin/cron-with-heartbeat.sh`, and the shared token at `/etc/homelab-monitor/heartbeat.token` (mode 0644, world-readable so the cron-runner user can read it). The executor only wraps an already-present crontab line, cross-checks the supplied wrapped line against its own re-derivation, writes to fixed paths only, and snapshots + rolls back on any failure.
- **Crontab-snapshot discovery fix** — fixed a latent STAGE-007 bug: the per-file user-crontab read ACL that discovery relied on broke vixie-cron (it rejects a crontab carrying an ACL with an "INSECURE MODE" error). Replaced with a root systemd timer + .path watcher (`hm-crontab-snapshot.service`/`.path`/`.timer`) that runs `crontab -l -u <user>` and writes a world-readable snapshot directory; the discoverer reads the snapshot instead of the 0600 spool. The old crontab-acl systemd units (`homelab-monitor-crontab-acl.path`/`.service`) and `scripts/refresh-crontab-acl.sh` were retired and deleted.
- **New API endpoint `GET /api/crons/wrapper-template`** (auth: `HEARTBEAT_WRITE` token) serves the raw wrapper template; the standalone remote CLI fetches the template from this endpoint rather than embedding its own copy.
- **New `is_local` computed field on `CronOut` schema** — gates the UI Install button: local-host crons enabled, remote crons disabled with an EPIC-017 tooltip.
- **New env var `HOMELAB_MONITOR_PUBLIC_URL`** (no default; wrapper install fails with HTTP 400 if unset) provides the heartbeat URL base baked into the wrapper.
- **Fingerprint convergence** — a wrapped crontab line is unwrapped before fingerprinting, so a cron keeps its original fingerprint across install (no spurious soft-delete).
- **Single system-managed `HEARTBEAT_WRITE` token** — shared across all wrapped crons (`ensure_heartbeat_wrapper_token`).
- **`host-setup.sh` updated** — installs the cron-apply + crontab-snapshot systemd units (including the new `cron-apply.timer`), idempotently retires the old crontab-acl units, strips stale ACLs.
- **Documentation** — new `docs/cron/install-heartbeat.md` and `docs/cron/remote-install.md` (install guides); updates to `docs/architecture/cron-identity.md` and `docs/deploy/host-prerequisites.md`.

## STAGE-002-009A — Wrapper REMOVAL helpers: uninstall-wrapper API, modal toggle, CLI, and remote script mode

### Added

- **Heartbeat wrapper uninstall feature** — the inverse of STAGE-009's install. Reverts a wrapped crontab line `<schedule> /usr/local/bin/cron-with-heartbeat.sh -- <original>` back to `<schedule> <original>` (byte-exact original command).
- **Three uninstall paths** (mirroring the three install paths):
  - UI cron detail page's Actions panel now has an Install/Remove TOGGLE (new `RemoveHeartbeatModal` component with dry-run un-wrap diff preview → confirm flow)
  - CLI `hm cron uninstall-wrapper`
  - Standalone remote-host CLI `install_wrapper_remote.py` gained an `--uninstall` mode (with its own snapshot + rollback)
- **New shared `CrontabLineDiff` component** — adopted by both install and remove modals to display side-by-side wrapped/unwrapped diff.
- **New API endpoint `POST /api/crons/{fingerprint}/uninstall-wrapper`** with `{confirm: bool}` parameter (dry-run + confirm, mirroring the install endpoint).
- **Host-side uninstall executor** — uninstall routes through the same host-side cron-apply executor as install. New `unwrap-crontab` operation in `scripts/hm-cron-apply.sh`; executor re-derives the unwrapped line itself, refuses a not-wrapped line, snapshots + rolls back on failure. The shared wrapper script and shared token are NEVER deleted on a per-cron uninstall.
- **New typed error `NotWrappedError`** → HTTP 409 (mirrors `AlreadyWrappedError`).
- **New `wrapper_installed` boolean column on crons table** (alembic migration 0013) — derived by the discoverer from the crontab line's actual wrapped state, set by install/uninstall, exposed on `CronOut`. The UI Install/Remove toggle keys on this field, NOT on `wrapper_last_seen_at` (which is a wrapper-health signal that is NULL for an installed-but-never-fired wrapper, causing toggle state misalignment).
- **Executor inline snapshot-refresh** — `scripts/hm-cron-apply.sh` now refreshes the crontab snapshot directory immediately after every wrap/unwrap via new `refresh_user_snapshot` helper. This eliminates stale snapshot state and fixes a real production bug where toggling the wrapper twice within 5 minutes failed.
- **Startup cron-discovery** — the monitor now requests a one-shot cron-discovery run on startup (lifespan.py), so `wrapper_installed` converges within seconds of a restart/upgrade instead of up to 300s. Gated by `HOMELAB_MONITOR_DISABLE_STARTUP_CRON_DISCOVERY` (set by test fixtures to avoid startup-discovery vs test-seeded-row races).
- **Fingerprint stability** — a cron's fingerprint is unchanged across install→uninstall (the discoverer unwraps before fingerprinting).
- **Documentation** — updated `docs/cron/install-heartbeat.md` (added wrapper-removal section), `docs/cron/remote-install.md` (added `--uninstall` mode), `docs/architecture/cron-identity.md`, and `docs/deploy/host-prerequisites.md`.
