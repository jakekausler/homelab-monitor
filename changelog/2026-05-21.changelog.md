## STAGE-003-002: Vector container-log ingestion fix + opt-out wiring

### Summary

STAGE-001-016 originally claimed to ship per-container log ingestion to VictoriaLogs but actually shipped `include_containers = []` (blocking ALL containers). STAGE-003-002 fixes this regression. Per D-OPT-OUT-MODEL, vector now tails ALL containers by default; users opt out specific ones via `VECTOR_DOCKER_EXCLUDE` env var (CSV format). Refinement validated against the live prod stack: BEFORE state showed 0 container logs in VL; AFTER showed 7 containers from the user's real ecosystem (homelab-monitor, homelab-karma, homelab-victorialogs, zigbee2mqtt, rtlamr2mqtt, homeassistant, podcast-feed). Opt-out via `VECTOR_DOCKER_EXCLUDE=homelab-karma` correctly suppressed karma logs while other containers continued flowing.

### Changes

- **Vector template** (`deploy/vector/vector.toml.template`): removed `include_containers = []` (omission means tail ALL containers); replaced `exclude_containers = []` with `exclude_containers = ${VECTOR_DOCKER_EXCLUDE}` (templated, substituted at monitor boot).
- **Render-on-boot** (`apps/monitor/homelab_monitor/kernel/cron/render.py`): refactored to dict-based substitution map; added `VectorRenderContext` dataclass holding `cron_events_token` + `docker_exclude_csv`; added `csv_to_toml_array()` helper with defensive handling (entries containing `"` or `\` are skipped with WARNING; vector still starts).
- **Env wiring**: `VECTOR_DOCKER_EXCLUDE` added to `deploy/dev/dev.env.example`, `deploy/compose/.env.example` (commented out by default per project convention), and passed through monitor service in `deploy/compose/docker-compose.yml`.
- **Tests**: new `apps/monitor/tests/test_vector_render.py` covers CSV edge cases + render integration + env-var reading; `apps/monitor/tests/test_cron_render.py` migrated 8 call sites to new `render_config(context=...)` signature; `apps/monitor/tests/test_vector_template.py` extended with populated-exclude-list test.

### Operational notes

- **Vector restart is REQUIRED when changing `VECTOR_DOCKER_EXCLUDE`**: `docker compose up -d monitor vector` does NOT restart vector if its image/env hasn't changed; explicit `docker compose restart vector` is needed. This is documented in `epics/EPIC-003-docker/regression.md`.
- **Structlog event names** for the render pipeline renamed from `cron_events.render.*` to `vector.render.*` to reflect that the renderer now handles the full vector.toml, not just the cron-events token. No operator-facing consumers (grep across runbooks/vmalert/grafana/docs returned zero hits).

## STAGE-003-004: Docker socket collector + container inventory API

### Summary

Lands the Docker socket collector that gathers signals cadvisor cannot see — container inventory, status, restart counts, exit codes, healthcheck — and exposes them via `GET /api/integrations/docker/containers`. Populates the existing UI grid skeleton from STAGE-003-003 with real container data. Cadvisor's cpu/mem metrics are cached into a sidecar table on the same collector tick (30s) for graceful degradation when VictoriaMetrics is unavailable.

This is the first stage in EPIC-003 to actually flow real container data to the user-visible UI. The grid populated with ~38 containers on the dev rig and ~39 in the prod stack VM after Refinement.

### Changes

**Backend (new modules):**
- `kernel/docker/socket_client.py` — `DockerSocketClient` using manual httpx over UDS transport (per T-DOCKER-SDK). Constructor accepts optional injected `httpx_client` for test injection. Methods: `list_containers()`, `inspect_container()`, `aclose()`. Returns `cast<TypedDict>(...)` after runtime shape validation (trust boundary documented in code).
- `kernel/metrics/docker_socket_collector.py` — `DockerSocketCollector(BaseCollector)`. 30s interval, `docker.socket` concurrency group. Per tick: list containers → inspect each → normalize state + healthcheck → upsert `targets`+`targets_docker` → mark missing → query VictoriaMetrics for cpu/mem → upsert sidecar cache → emit gauges (`homelab_container_status` with state label, always 1.0 per kube_pod_status_phase convention; `homelab_container_restart_count`, `homelab_container_last_exit_code`, `homelab_container_healthcheck`). VM unreachable → log + skip, cache untouched.
- `kernel/db/repositories/targets_repository.py` — `TargetsRepository.upsert_docker_container_conn`, `mark_missing_except_conn`, `list_docker_containers`, `upsert_cpu_mem_cache`. Raw SQL via `SqliteRepository`. INSERT-OR-REPLACE pattern with FK CASCADE on `targets_docker`.
- `kernel/api/routers/docker.py` — `GET /api/integrations/docker/containers` returns `ContainerListResponse` with `extra="ignore"` for forward-compat (STAGE-003-005+ adds fields). Session auth via `require_session()`.

**Database migrations:**
- `alembic/versions/0017_targets_generic_columns.py` — adds `kind`, `status`, `first_seen`, `last_seen`, `hidden_at`, `labels`, `source` columns to `targets` + index on `kind`.
- `alembic/versions/0018_targets_docker_sidecar.py` — creates `targets_docker` sidecar table with FK CASCADE on `targets.id`.

**Lifespan + app wiring:**
- `kernel/api/lifespan.py` — registers `DockerSocketCollector` with the scheduler; constructs `DockerSocketClient` from `HOMELAB_MONITOR_DOCKER_SOCKET` env (default `/var/run/docker.sock`); closes client cleanly on shutdown.
- `kernel/api/app.py` — includes `docker.router` under `/api` prefix.

**OpenAPI schema regenerated:**
- `packages/shared-types/openapi.json` (regenerated via `make openapi-export`)
- `apps/ui/src/api/schema.ts` (regenerated via `pnpm --filter ui run generate-types`)
