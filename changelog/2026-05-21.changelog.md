## STAGE-003-002: Vector container-log ingestion fix + opt-out wiring

### Summary

STAGE-001-016 originally claimed to ship per-container log ingestion to VictoriaLogs but actually shipped `include_containers = []` (blocking ALL containers). STAGE-003-002 fixes this regression. Per D-OPT-OUT-MODEL, vector now tails ALL containers by default; users opt out specific ones via `VECTOR_DOCKER_EXCLUDE` env var (CSV format). Refinement validated against the live prod stack: BEFORE state showed 0 container logs in VL; AFTER showed 7 containers from the user's real ecosystem (homelab-monitor, homelab-karma, homelab-victorialogs, zigbee2mqtt, rtlamr2mqtt, homeassistant, podcast-feed). Opt-out via `VECTOR_DOCKER_EXCLUDE=homelab-karma` correctly suppressed karma logs while other containers continued flowing.

### Changes

- **Vector template** (`deploy/vector/vector.toml.template`): removed `include_containers = []` (omission means tail ALL containers); replaced `exclude_containers = []` with `exclude_containers = ${VECTOR_DOCKER_EXCLUDE}` (templated, substituted at monitor boot).
- **Render-on-boot** (`apps/monitor/homelab_monitor/kernel/cron/render.py`): refactored to dict-based substitution map; added `VectorRenderContext` dataclass holding `cron_events_token` + `docker_exclude_csv`; added `csv_to_toml_array()` helper with defensive handling (entries containing `"` or `\` are skipped with WARNING; vector still starts).
- **Env wiring**: `VECTOR_DOCKER_EXCLUDE` added to `deploy/dev/dev.env.example`, `deploy/compose/.env.example` (commented out by default per project convention), and passed through monitor service in `deploy/compose/docker-compose.yml`.
- **Tests**: new `apps/monitor/tests/test_vector_render.py` covers CSV edge cases + render integration + env-var reading; `apps/monitor/tests/test_cron_render.py` migrated 8 call sites to new `render_config(context=...)` signature; `apps/monitor/tests/test_vector_template.py` extended with populated-exclude-list test.

### Operational notes

- **Vector restart is REQUIRED when changing `VECTOR_DOCKER_EXCLUDE`**: `docker compose up -d monitor vector` does NOT restart vector if its image/env hasn't changed; explicit `docker compose restart vector` is needed. This is documented in `epics/EPIC-003-docker/regression.md`.
- **Structlog event names** for the render pipeline renamed from `cron_events.render.*` to `vector.render.*` to reflect that the renderer now handles the full vector.toml, not just the cron-events token. No operator-facing consumers (grep across runbooks/vmalert/grafana/docs returned zero hits).

## STAGE-003-001 — cadvisor sidecar + scrape config + first container metrics in VM

First stage of EPIC-003 (Docker integration). Deploys cadvisor as a sidecar in prod, dev, and integration test rigs. Adds the curated scrape config + file_sd_configs extension point for STAGE-003-005+. Ships a dev-only `hm dev seed-container-metrics` CLI for synthetic container metrics during UI Refinement.

### Added

- **cadvisor sidecar** across all compose stacks:
  - `deploy/compose/docker-compose.yml` (prod)
  - `deploy/dev/docker-compose.dev.yml` (dev)
  - `deploy/compose/docker-compose.test.yml` (integration test)
  - Pinned to `gcr.io/cadvisor/cadvisor:v0.49.1` (D-CADVISOR-VERSION)
  - Non-privileged with no extra capabilities (D-CADVISOR-LEAST-PRIV — verified on Linux 5.15 cgroup v2)
  - Flags: `--docker_only=true`, `--store_container_labels=false`, `--whitelisted_container_labels=homelab-monitor.*,org.label-schema.*,com.docker.compose.*` (D-CADVISOR-FLAGS)
  - Dev rig published port `18081` (D-DEV-PORT-18081)
- **Curated scrape config** (`deploy/compose/vmagent/scrape.yaml`):
  - 13 metric families allowlist per D-CADVISOR-RELABEL-CURATED (CPU, memory, network, disk I/O essentials)
  - Filesystem-noise drop rule to exclude tmpfs/sysfs metrics
  - `file_sd_configs: ['targets.d/*.yml']` extension point for STAGE-003-005+ (D-SCRAPE-HYBRID)
- **Scrape config for test rig** (`deploy/compose/test-fixtures/vmagent-scrape.yaml`) — mirrors prod cadvisor job setup
- **Extension point directory** `deploy/compose/vmagent/targets.d/` with README documenting the contract for future stages
- **New `hm dev seed-container-metrics` CLI subcommand** (`apps/monitor/homelab_monitor/cli/dev.py`):
  - Syntax: `hm dev seed-container-metrics [--containers N] [--clear] [--force] [--vm-url URL]`
  - Deterministic seeding via RNG keyed on `(N, current_minute)` (D-SEED-CLI-DETERMINISTIC)
  - Labels all series with `homelab_synthetic="true"` for targeted cleanup via `--clear`
  - 5 shape variants (cpu-sawtooth, memory-step, network-bursts, idle, spiky) cycling on `idx%5`
  - Local-only (hostname gate via `socket.gethostname()` comparison)
