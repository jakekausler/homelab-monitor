## STAGE-003-013 — Container drill-down unification + integration-page simplification

**Epic:** EPIC-003 (Docker collector + per-container probes)
**Date:** 2026-05-27
**Commit:** 97da570

### Summary

Slimmed the Docker integration page (`/integrations/docker`) to a compose-grouped list with badges and CPU/RAM metrics. All container-level actions (Pull & Restart, probe management, log viewing, image-update review) moved into a unified per-container drill-down page at `/integrations/docker/containers/$name/{overview|probes|logs|actions}` with a shared sticky-free header and horizontal tab strip.

### What changed

**New parent route** `/integrations/docker/containers/$name` (ContainerPage) with 4 child routes (overview / probes / logs / actions). Index redirects to `/overview`.

**New components:** ContainerPage, ContainerOverviewTab, ContainerProbesTab, ContainerLogsTab, ContainerActionsTab, ContainerTabsNav, ContainerList, ContainerListRow, DockerContainerLogsViewerBody.

**Refactored badges:** ImageUpdateBadge + ProbesBadge converted from clickable `<Link>` styled pills to canonical `<Badge>` components matching StatusBadge/HealthcheckBadge pattern; render `null` instead of `—` for loading/no-data states. StatusBadge + HealthcheckBadge gained leading lucide icons (Play/Square/RefreshCw/Pause/Circle/HelpCircle for status; Heart/HeartCrack/HeartPulse for healthcheck).

**Integration list rows** now stack vertically on mobile (name → image → badges → metrics, each wrappable) and use side-by-side on `sm:`+ viewports.

**Container drill-down page** is non-sticky; header + tabs scroll naturally with content.

**CPU/RAM 15-min sparklines** on the Overview tab via the existing `useMetricsRange` proxy.

**Pull & Restart**: button on the Overview tab when `update_available === true`; "Pull & Restart" for registry source, "Rebuild & Restart" for local_build. Uses `useRef`+`useEffect` cleanup so the 30s post-action timeout doesn't fire after unmount.

### Architecture decisions (locked in STAGE-003-013.md)

**Drill-down parent route:**
- D-PARENT-ROUTE-CONTAINERS — single route captures all container-level views; avoids repeating container resolution per tab
- D-INDEX-REDIRECTS-OVERVIEW — `/containers/$name` index redirects to `/overview`
- D-TABS-HORIZONTAL-STRIP — tabs render as clickable buttons in horizontal strip (matches Karma UI pattern)
- D-SCROLL-NATURAL — removed sticky header; entire page scrolls together

**Badge refactor:**
- D-BADGE-CANONICAL — ImageUpdateBadge + ProbesBadge use canonical Badge + BadgeIcon pattern (no Link styling)
- D-BADGE-NULL-LOADING — badges render `null` instead of `—` for loading/no-data; cleaner whitespace
- D-ICONS-LUCIDE — all badges + status chips gain lucide icons (Play/Square/RefreshCw/Pause for container status; Heart/HeartCrack/HeartPulse for healthcheck; InfoIcon for probe status)

**Integration list (slim grid):**
- D-LIST-ROWS-COMPOSE-GROUP — containers grouped by compose service/project at the top level
- D-LIST-STACK-MOBILE — each row stacks vertically on mobile (name, image tag, badges, CPU/RAM) with natural wrapping
- D-LIST-SIDE-SM-PLUS — on `sm:`+ (640px) viewports, rows go side-by-side for compact display
- D-SPARKLINES-PROXY — CPU/RAM metrics via existing `useMetricsRange` proxy (no new backend endpoint)

**Pull & Restart on Overview:**
- D-PULL-RESTART-TAB — moved from integration-page button to Overview tab; appears only when `update_available === true`
- D-LABEL-REGISTRY-VS-LOCAL — "Pull & Restart" for `source="registry"`, "Rebuild & Restart" for `source="local_build"`
- D-CLEANUP-ON-UNMOUNT — `useRef` + `useEffect` cleanup prevents 30s timeout callback firing after tab switch

### Code review

Round 1: 18 findings (2 Critical, 7 Important, 5 Minor, 4 Suggestion) — ALL addressed.

Notable correctness fixes:
- **C1 (route param type):** `$name` param was `string | undefined`; wrapped route-level resolver to validate and 404 on undefined.
- **C2 (metrics range clamping):** sparkline range calculation was off-by-one on end time; fixed to `Math.floor(endAt / interval) * interval` for aligned bucket boundaries.
- **I1 (tab active class):** Link activeClass wasn't firing because route was nested; changed to `useLocation().pathname.includes(tabPath)` for accurate styling.
- **I4 (image tag truncation):** long image tags (e.g., sha256 digests) overflowed the row; added `text-ellipsis overflow-hidden whitespace-nowrap` + title attribute.
- **M2 (null vs undefined):** some code checked `badge !== null`, others `badge === undefined`; standardized to `!badge` for null/undefined both.
