## [STAGE-004-012] Stream picker sidebar

**Summary:** Added a left-side "Services" sidebar to the Logs Explorer (`/logs`) listing distinct `service` values with line counts (window-scoped, sorted descending) for the current time range. Click to filter; selected services show as removable chips above the search box. Multiple services OR together and AND with the user's search/LogsQL query (works in both plain and advanced modes; the user's expression is never mutated). Selection is reflected in the URL as `?services=a,b` (shareable/reload-safe). Mobile: sidebar becomes an overlay dialog (shared Radix Dialog primitive) toggled by a "Services" button.

**Backend (VictoriaLogs stats + caching):**
- **New:** `GET /api/logs/services?start&end&limit=100` endpoint returns `{services:[{service,count}], truncated}` powered by VictoriaLogs `stats by (service) count()` with 30s cache and top-100 with truncated flag.
- **New:** Optional `services` CSV param on `GET /api/logs/query` that AND-composes `(service:"a" OR ...) AND (<expr>)` server-side via the canonical `logsql_quote_phrase` helper.
- **Tests:** Backend e2e validated against real VictoriaLogs (including special-char service-name quoting).

**Frontend (sidebar + dialog layout + URL state):**
- **New:** `LogsServicesSidebar.tsx` (desktop) and `LogsServicesDialog.tsx` (mobile overlay) showing service chips with line counts. `LogsServiceChip.tsx` (removable chip component with count badge).
- **Changed:** `LogsExplorerPage.tsx` (independent `selectedServices` state; `services` URL param). `LogsExplorerBody.tsx` (sidebar/dialog layout management; conditional rendering by viewport). `router.tsx` (`services` in `validateSearch`).
- **State:** Counts are window-scoped (refresh on range change only), independent of search/selection.
- **Mobile UX:** Sidebar toggle becomes a "Services" button; clicking opens overlay dialog (shared Radix Dialog primitive).

**Verification:** User Desktop+Mobile approval.

**Deferred:** Service grouping by type (STAGE-004-012A).
