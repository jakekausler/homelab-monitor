## [STAGE-004-011] LogsQL advanced mode + syntax highlighting

**Summary:** Added an "Advanced (LogsQL)" toggle to the Logs Explorer (`/logs`). When enabled, the plain-text search box swaps for a CodeMirror 6 editor with basic LogsQL syntax highlighting; the raw LogsQL is sent to VictoriaLogs as the query `expr` (plain mode still translates to `_msg:"…"`). Toggling preserves both modes' text. Deep-linkable via `?logsql=<expr>` (mutually exclusive with `?q=`).

**Frontend (CodeMirror 6 integration + state management):**
- **New:** `apps/ui/src/components/logs/logsQlLanguage.ts` — CodeMirror `StreamLanguage` tokenizer + `HighlightStyle` for LogsQL keywords/fields/strings/numbers/operators. `LogsQlEditorImpl.tsx` (hand-wired CM6 EditorView; lazy-loaded chunk). `LogsQlEditor.tsx` (shared shell: plain `<textarea>` on narrow viewports / Suspense-wrapped CM6 on desktop). `AdvancedToggle.tsx` (toggle control).
- **Changed:** `LogsExplorerPage.tsx` (independent `advancedMode` / `plainText` / `logsQl` state; `logsql` URL param). `LogsExplorerBody.tsx` (toggle + conditional editor). `router.tsx` (`logsql` in `validateSearch`). `vitest.config.ts` (exclude CM6 EditorView impl from coverage — non-functional in jsdom, validated via textarea-fallback tests + manual Refinement).
- **Deps:** `@codemirror/state`, `@codemirror/view`, `@codemirror/language`, `@codemirror/commands`, `@lezer/highlight` (lazy-loaded; editor chunk loads only when advanced mode is first enabled on desktop). First lazy/Suspense usage in the UI.
- **Editor UX:** visible caret + click-anywhere-to-position (CM theme fills the box); Enter submits, Shift+Enter newline.

**Backend (time-window clock-skew fix + test infrastructure hardening):**
- **Fixed (clock-skew, backend):** `parse_and_validate_window` now allows a 5-second future grace (`FUTURE_SKEW_GRACE`) so an absolute `end == browser-now` is not rejected as `range_in_future` under browser/container clock skew; genuinely-future ranges still rejected. (`apps/monitor/.../kernel/logs/time_window.py` + 4 tests.)
- **Fixed (test infra):** `tests/_uvicorn_fixture.py` overrides `HOMELAB_MONITOR_DOCKER_SOCKET` to a nonexistent path so the SSE smoke test's startup collectors don't do real Docker/registry I/O (was flaking when a real Docker daemon is reachable).

**Docs:** `docs/admin/logs-explorer.md` extended with an "Advanced LogsQL mode" section.

**Out of scope (unplanned):** LogsQL autocomplete / query builder, syntax error highlighting.

**Tests:** logsQlLanguage.test.ts (tokenizer coverage for keywords/fields/strings/numbers/operators), LogsQlEditor.test.tsx (textarea fallback on narrow viewports, Suspense boundary, enter/shift+enter wiring), AdvancedToggle.test.tsx, LogsExplorerPage.test.tsx extended (toggle state preservation, `logsql` URL param hydration, mode-exclusive param behavior), time_window.test.py (clock-skew grace + boundary cases), _uvicorn_fixture.test.py (Docker socket override validation).
