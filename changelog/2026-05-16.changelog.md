# 2026-05-16

## STAGE-002-008 ŌĆö B-mode cron log-scrape (Vector PUSH ingest of cron run evidence)

### Added

- **New endpoint `POST /api/internal/cron-events`** ŌĆö ingests structured cron run events from Vector. Accepts a JSON array of `CronEventItem` objects (host, cmd, timestamp, optional exit code). Idempotent delivery via journald cursor dedup (`cron_log_cursors` table); when cursor is unavailable (production), uses deterministic content-derived `synthesize_cursor()` key.
- **Vector config rendering on boot** ŌĆö `kernel/cron/render.py` renders `deploy/vector/vector.toml.template` at API startup, substituting the cron-events ingest token (mirrors Alertmanager config rendering pattern). Wired into API lifespan via `homelab_monitor.kernel.cron.render.render_on_boot()`.
- **Cron-events ingest auth** ŌĆö new `kernel/cron/log_ingest_token.py` module; generates a random token at boot; new auth scope `cron:events:ingest:write` for the internal endpoint.
- **D1 ŌĆö neutral "observed run" semantic** ŌĆö bare cron command lines (no exit code, vanilla cron stdout) recorded as neutral observed runs via new `HeartbeatRepo.record_observed_run()` ŌĆö increments `observed_runs_total`, sets `last_observed_run_at`, does NOT modify `current_state`, `last_ok_at`, or `last_fail_at`. Only wrapper-tagged `exit=N` lines trigger `record_ok()` or `record_fail()`. B-mode log evidence asserts "ran", never implicitly "succeeded".
- **D4 ŌĆö ambiguity handling** ŌĆö events matching 2+ cron fingerprints skipped (no fan-out); increments `homelab_cron_logscrape_ambiguous_total` counter.
- **D5 ŌĆö command matching via log key** ŌĆö new `kernel/cron/log_match.py` module with shared `canonical_log_key()` function: scrub secrets, collapse whitespace, strip one wrapping `(...)` layer. New indexed column `crons.log_match_key` (TEXT NULL) populated at discovery; endpoint matches via `(host, log_match_key)`. Empty-canonicalized commands stored as NULL (never match).
- **At-most-once delivery semantics** ŌĆö documented limitation; journald cursor used when available; `synthesize_cursor()` content-derived key used in production when journald omits cursor.
- **Schema migration `0012_logscrape_columns.py`** ŌĆö adds `crons.log_match_key` (TEXT NULL, indexed), `heartbeats_state.observed_runs_total` (INTEGER, default 0), `heartbeats_state.last_observed_run_at` (TEXT NULL, UTC ISO-8601), and new `cron_log_cursors` dedup table (journal_cursor PRIMARY KEY, processed_at).
- **Architecture documentation** ŌĆö new `docs/architecture/cron-logscrape.md` (Vector PUSH model, event shape, matching & dedup semantics, production limitations); `docs/architecture/cron-identity.md` updated with B-mode log-scrape integration.

### Bug fixes from 3b prod-rig refinement

1. **Ingest token chicken-and-egg** ŌĆö Vector needed token before monitor started. Solved: render Vector config on boot with token embedded; similar pattern to Alertmanager.
2. **Fresh docker named volume seeded root:root** ŌĆö non-root monitor user could not write to config dir. Fixed: added one-shot `config-init` root compose service to chmod/chown dirs.
3. **`config-init` originally destructive** ŌĆö initial `rm -rf` was dangerous. Fixed: `config-init` now idempotent (chmod/chown only; no deletion).
4. **`config-init` group ownership** ŌĆö setgid-inherited group ownership not working. Fixed: `config-init` now explicitly chowns render dirs to monitor user.
5. **Vector journald source read empty journal** ŌĆö container machine-id ŌēĀ host's; journald journal not shared. Fixed: bind-mount host `/etc/machine-id` read-only into Vector container.
6. **VRL field-name errors** ŌĆö `.MESSAGE` should be `.message`; unit filter too narrow. Fixed: corrected field names; broadened SYSLOG_IDENTIFIER filter to `== "CRON"`; prefer `.timestamp`.
7. **422 request shape** ŌĆö Vector's `http` sink default buffering didn't emit JSON array. Fixed: added `framing.method = "character_delimited"` + `delimiter = ","` to Vector config; removed incorrect `.host = ._HOSTNAME` mapping (journald source already normalizes hostname to top-level `.host`).
8. **Dockerfile / docker-compose.yml adjustments** ŌĆö multiple bind-mount, entrypoint, and service-dependency updates required to support the above.
