## [STAGE-001-013] Alert ingestor + first `inproc-dashboard` channel

New `kernel/alerts/` package with alert repository and fingerprinting logic. Module `types.py` defines `Severity` and `AlertStatus` StrEnums, `AlertOutcome` StrEnum (acknowledged/dismissed/resolved/escalated), `Alert` and `AlertmanagerV2Payload` Pydantic models. Module `fingerprinting.py` implements `compute_fingerprint(labels_dict) -> str` (deterministic JSON hash) and `quarantine_fingerprint(reason) -> dict` for structured audit. Module `events.py` defines `AlertFiringEvent` and `AlertResolvedEvent` as Pydantic BaseModel subclasses extending new `BaseEvent` base class (refactored from Protocol to BaseModel). Module `repository.py` implements `AlertRepository` with full CRUD: `insert(alert)`, `update_last_seen(alert_id)`, `mark_resolved(alert_id, resolved_at, decided_by)`, `set_ack(alert_id, ack_at, ack_by)`, `find_by_id(alert_id)`, `find_active_by_fingerprint(fingerprint)`, `find_active_quarantine_alert(fingerprint)`, `list_paginated(status/severity/source_tool/fingerprint, cursor, limit)`, `find_outcome_history(alert_id)`. Race-safe dedup via `ux_alerts_fingerprint_firing` UNIQUE PARTIAL INDEX on `(fingerprint) WHERE status='firing'` + IntegrityError handling on insert.

New `kernel/dispatch/` package with pluggable alert channels. Module `types.py` defines `Channel` Protocol (async `deliver(alert, severity) -> DeliveryResult`) and `DeliveryResult` dataclass. Module `dispatcher.py` implements `AlertDispatcher` with per-channel exception isolation, sequential delivery, and per-channel failure counter for observability. Module `channels/inproc_dashboard.py` implements `InprocDashboardChannel` publishing `AlertFiringEvent` and `AlertResolvedEvent` to `SseBroker`.

`BaseEvent` refactored from Protocol to Pydantic `BaseModel` with `kind: Literal[...]` and `ts: datetime` fields. `EventSink.publish()` now accepts any `BaseEvent` subclass. `SseBroker.publish()` uses event polymorphism with documented lock-contention caveat. `SchedulerTickEvent`, `AlertFiringEvent`, and `AlertResolvedEvent` all inherit `BaseEvent`; Pydantic v2 Literal kind overrides require `# pyright: ignore[reportIncompatibleVariableOverride]` per-line idiom (Protocol-style stub `model_dump(self) -> dict: ...` was silently overriding Pydantic's real implementation, returning None — caught by test failures).

New POST `/api/alerts/ingest` (Alertmanager v2 webhook receiver). Auth via `require_user_or_token({Scope.ALERTS_INGEST_WRITE})` — cookie path enforces CSRF, token path validates scope. Per-alert: compute fingerprint, `find_active_by_fingerprint(status='firing')` lookup; firing alerts insert OR update_last_seen; resolved alerts call `mark_resolved()` (idempotent); resolved-without-prior silently ignored. Returns 202 with `{received, ingested}` counts. Privacy: INFO logs include fingerprint + alert_id only, never full payload.

Four new API endpoints: `GET /api/alerts` (paginated, filterable by status/severity/source_tool/fingerprint), `GET /api/alerts/{id}` (detail view + outcome history), `POST /api/alerts/{id}/ack` (session-only, CSRF enforced), `POST /api/alerts/{id}/dismiss` (session-only, CSRF enforced). Malformed cursor raises `ValueError("invalid cursor format")` → mapped to 400 HTTPException.

Migration `0005_add_alerts_tables.py` adds 13 columns to `alerts` (source_tool, severity, status, opened_at, last_seen_at, resolved_at, ack_at, ack_by, runbook_id, payload_json, fingerprint) and 3 to `alert_outcomes` (outcome, decided_at, decided_by). Three indexes on `(source_tool, opened_at)`, `(status, opened_at)`, and `(alert_outcomes.alert_id)`. UNIQUE PARTIAL INDEX `ux_alerts_fingerprint_firing` on `(fingerprint) WHERE status='firing'` for race-safe dedup.

`FailureBudget` and `Scheduler` constructors widened with kw-only `alert_repo` and `dispatcher` parameters (default None for backward compat). Quarantine integration: insert/update active alert row + dispatch `AlertFiringEvent`. `Scheduler.clear_quarantine()` reordered: resolve alert FIRST (via fingerprint lookup using `_extract_last_reason()` helper) THEN clear budget. Ensures resolved-without-cleared-budget remains operator-recoverable since `mark_resolved()` is idempotent.

Lifespan constructs `AlertRepository`, `InprocDashboardChannel`, `AlertDispatcher`; threads through `Scheduler` + `FailureBudget`; sets `app.state.alert_repo` and `app.state.alert_dispatcher` for router access.
