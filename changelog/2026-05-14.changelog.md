# 2026-05-14

## STAGE-002-007 — Cron auto-discovery + register-time secret-scrub + LAN-accessible prod stack

### Added

- **Cron auto-discoverer plugin** at `apps/monitor/homelab_monitor/plugins/discoverers/cron_discoverer.py` + `cron_parser.py` — scans `/etc/crontab`, `/etc/cron.d/*`, `/var/spool/cron/crontabs/*` on 300s interval (configurable via `HM_CRON_DISCOVERY_INTERVAL_SECONDS`); emits `homelab_collector_run_*` metrics per plugin contract; returns `CronScanResult` with `partial` flag for 007A soft-delete reconciliation.
- **Cron line parser** with edge-case handling: blank lines, comments, env-var assignments skip; `@reboot` tracked with `cadence_seconds=0`; `@hourly`/`@daily`/`@weekly`/`@monthly` supported; 5-field user crontabs via filename; malformed schedules logged, scan continues with `partial=True`.
- **CronRepo.upsert_discovered()** — fingerprint-keyed upsert; diffs all four fields (schedule_canonical, cadence_seconds, command, name) to detect auto-heal cases; `last_discovered_at` bumped idempotently on every scan that sees the fingerprint.
- **Migration 0010** — `ADD COLUMN crons.last_discovered_at TEXT NULL` (UTC ISO-8601); idempotent both directions.
- **POST /api/crons/discover-now** endpoint — admin-only, 10s in-memory throttle between calls, returns 202 Accepted + `{found_count, partial, errors}` summary.
- **CLI command** `hm cron discover` — calls discoverer service directly (no HTTP).
- **CLI command** `hm collector unquarantine` — clears stale quarantine state (auto-called in lifespan on startup).
- **Host setup script** `scripts/host-setup.sh` — idempotent, sudo-required; creates `homelab-monitor` host user (GID extracted from UID map), adds to crontab group, sets read ACLs on `/var/spool/cron/crontabs/*` (rx on dir, r on existing files, default for new).
- **Frontend "Discover now" button** — icon-only in CronsToolbar; throttle-aware toast feedback ("Discovery scan triggered" / "Try again in N seconds").
- **Frontend "Last discovered" field** — in CronDetail Disk source panel; UTC timestamp with relative-time tooltip.
- **Container static UI serving** — FastAPI now serves built UI via `StaticFiles(html=True)` mount at `/` (configurable via `HOMELAB_MONITOR_UI_DIR`, default `/app/ui`).
- **Dev/prod port separation** — dev 1xxxx (19090 backend, 5180 UI, 18428 metrics, etc.); prod 2xxxx (29090 backend only, sidecars container-internal). New env var `HOMELAB_MONITOR_BIND_HOST` (default `127.0.0.1`, set to `0.0.0.0` for LAN access in trusted homelab networks).
- **Compose bind-mounts** — `/etc:/host/etc:ro` + `/var/spool/cron/crontabs:/host/var/spool/cron/crontabs:ro` (Debian/Ubuntu; document override for other distros).
- **Secret-scrub patterns** at storage time (command field, not fingerprint): `-pSECRET` (with `(?<![A-Za-z0-9_-])` lookbehind), `--password=`, `MYSQL_PWD=`, `PGPASSWORD=`, `--api-key=`, `--token=`.
- **Smarter `derive_name`** — skips shell wrappers (bash, sh, sudo, test, [, command, env), operators (&&, ||), flags, and env-var assignments; 10/16 real-host rows auto-renamed via bug B fix on next discovery scan.

### Bug fixes from 3b prod-rig refinement

1. **Quarantine state survived container restarts** — fixed via auto-clear in `lifespan.py` + new `hm collector unquarantine` CLI.
2. **`upsert_discovered` only diffed schedule_canonical + cadence_seconds** — missed command + name diffs; prevented auto-heal. Now diffs all four fields, auto-heals corrupted rows.
3. **`host-setup.sh` only set default ACL on dir** — existing files never granted read. Fixed: applies rx on dir, r on each existing crontab file, plus default for new files.
4. **Secret-scrub patterns for `-p`, `--password=`, MYSQL_PWD, PGPASSWORD, --token=, --api-key=** applied at storage time (fingerprint stays raw for wrapper convergence). Tightened `-p` regex with `(?<![A-Za-z0-9_-])` lookbehind to prevent run-parts false-positive.
5. **Container missing built UI serve** — FastAPI now serves UI via `StaticFiles(html=True)` mount. `HOMELAB_MONITOR_UI_DIR` env var (default `/app/ui`).
6. **Dev/prod port conflicts** — clean separation: dev 1xxxx (19090), prod 2xxxx (29090). Backend bind host now configurable via `HOMELAB_MONITOR_BIND_HOST`.
7. **`derive_name` too naive** — returned `system` for wrapper-guarded commands. Now skips shell wrappers + operators + flags; 10/16 real-host rows auto-named on next scan.

### Locked Design Decisions (D1–D9)

- D1: Dedicated host user + idempotent `scripts/host-setup.sh` (non-root container).
- D2: Best-effort partial scans; per-file errors don't gate 007A soft-delete reconciliation.
- D3: Periodic 300s poll (configurable).
- D4: `crons.discover` on first-insert; `crons.discover.update` on non-bump field change; no audit on bump-only.
- D5: `@reboot` tracked with `cadence_seconds=0`; lateness rule deferred to STAGE-002-010.
- D6: API endpoint + CLI + frontend button for on-demand discover scan.
- D7: Raw schedule in fingerprint; `schedule_canonical` for display (wrapper convergence).
- D8: Hand-rolled parser + croniter validation (see edge-case table in STAGE-002-007.md).
- D9: Skip "Stale (no longer discovered)" badge in this stage (007A's soft-delete is the signal).
