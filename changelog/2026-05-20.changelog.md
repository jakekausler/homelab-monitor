# 2026-05-20

## STAGE-002-015 ‚Äî Cron run-log frontend + Refinement fixes

Fifth and final of the five appended EPIC-002 stages. Frontend stage that completes the cron-run-logs arc with three new UI surfaces (recent runs teaser, paginated runs list, log viewer) and a developer seed CLI. Plus several backend Refinement fixes for production correctness.

### Added

- **New route `/inventory/crons/$fp/runs`** ‚Äî paginated cron runs list with:
  - Desktop table layout (run ID, started, state, duration, exit code, anomalies)
  - Mobile card layout (state badge, duration, recent context)
  - CSS-gated responsive layout (no `useMediaQuery`)
  - State filter dropdown (`running`/`ok`/`fail`/`unknown`)
  - Keyset cursor pagination with previous/next buttons
- **New route `/inventory/crons/$fp/runs/$runId`** ‚Äî run-log viewer with:
  - Response-shape handling for all 5 states: `available` (200 with log), `empty` (200, no output), `running` (200, in-flight), `discarded` (expired, 200), `enrichment-pending` (200, not yet queried)
  - Auto-refresh (5s) when `running`
  - Monospace log display with line-count summary
  - Run metadata sidebar (fingerprint, started, ended, duration, exit code, anomaly flags)
- **New component `RecentRunsPanel`** ‚Äî 5-row teaser shown on CronDetail page, linked to full runs list
- **New component `RunStateBadge`** in `badges.tsx` ‚Äî color-coded badge for `running`/`ok`/`fail`/`unknown` states
- **New utility `formatDuration`** ‚Äî human-readable duration formatter:
  - 3-decimal precision for durations <10s (e.g., `0.123s`)
  - 1-decimal precision for 10-59s (e.g., `1.5s`)
  - `Xm Ys` format for 60-3599s (e.g., `2m 30s`)
  - `Xh Ym` format for ‚â•1h (e.g., `1h 30m`)
- **New hooks `useListCronRuns` and `useCronRunLog`** ‚Äî React Query wrappers for the new API endpoints
- **Shared `RunSearchSchema` type** at `apps/ui/src/routes/inventory/types.ts` ‚Äî search params for runs list (state filter, cursor)
- **New dev CLI `hm dev seed-cron-runs <fp>`** ‚Äî seeds 50 OK + 5 fail + 3 unknown + 5 running cron_runs rows for UI exercise. Local-only (hostname gate via `gethostname()` comparison).
- **New `CronRunReconcilerConfig` fields**:
  - `enrich_max_per_tick` (default 200) ‚Äî maximum runs to enrich per 30s tick
  - `enrich_window_slack_seconds` (default 30) ‚Äî adds slack to VL query window to bridge ingest lag
  - Environment variable loaders: `HOMELAB_MONITOR_CRON_RUN_ENRICH_MAX_PER_TICK`, `_ENRICH_WINDOW_SLACK_SECONDS`
- **VL query window widening (wrapper-only, STAGE-002-012 crons only)**:
  - Applied in BOTH reconciler `_enrich` and API `_query_run_log` ‚Äî bridges ~1.2-1.4s observed journald‚ÜíVector‚ÜíVL ingest lag
  - Logscrape-sourced runs (B-mode) do NOT get slack (log timestamp already present in VL)
  - Configurable via `enrich_window_slack_seconds` (default 30 seconds added to window end)
- **Wrapper script enhancement** (`cron-with-heartbeat.sh.tmpl`):
  - Now uses `date +%s.%N` + `awk` for fractional duration with 3-decimal precision (was whole-second `$SECONDS`)
  - Adds microsecond-precision to run-timing telemetry
- **New `HM_DEV_BIND_HOST` env knob** ‚Äî lets the dev rig listen on `0.0.0.0` for LAN access (trusted homelab networks only, disabled by default).

### Changed

- **`CronRunReconciler._enrich` phase** ‚Äî now respects `enrich_max_per_tick` to prevent unbounded loop from exceeding 20s collector timeout (observed on prod with 13K-row backlog, FailureBudget quarantine now active).
- **`CronRunRepository.list_runs`** ‚Äî improved pagination cursor handling for edge cases (high-cardinality timestamps, runs created in burst).
- **Wrapper health computation** ‚Äî verified against both legacy and current wrapper formats during refinement.
- **Dev rig documentation** (`docs/dev/local-environment.md`) ‚Äî added `HM_DEV_BIND_HOST` section and troubleshooting for Vite API proxy misconfiguration.

### Fixed

- **Prod image stale during Refinement 3a** ‚Äî previous docker image was masking the slack window fix from STAGE-002-014; Refinement 3b with fresh build confirmed fix works on real data.
- **Wrapper duration precision** ‚Äî whole-second precision in legacy wrapper lost sub-second variance (e.g., 0.5s and 1.4s both became 1s); wrapper now captures 3 decimals via `date +%s.%N`.
- **Reconciler enrich loop unbounded** ‚Äî 13K-row backlog on prod was exceeding 20s collector timeout; bounded via `enrich_max_per_tick`. FailureBudget quarantine prevents runaway retries.
- **VL window too tight by ~1.2s** ‚Äî journald‚ÜíVector‚ÜíVL ingest lag not accounted for in original window (STAGE-002-011 carry-forward); slack widening now applied to both reconciler and API paths.
- **Host setup scripts missing tee for journald** ‚Äî `/storage/scripts/rtlamr-watchdog.sh` and `/storage/scripts/cron/backup.sh` updated to pipe stdout through `tee` for journald capture (B-mode log-scrape now has access to these runs' logs).
