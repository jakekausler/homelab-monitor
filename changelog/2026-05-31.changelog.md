## [STAGE-004-009] Local-time timestamp rendering with UTC toggle

**Summary:** Log timestamps now render in the configured local timezone (America/New_York) by default with a per-viewer UTC toggle, applied once in the shared `<LogViewer>` and propagated to the docker, cron, and (future) Explorer surfaces. Fixes the local-time/UTC-toggle item deferred from STAGE-003.

**Frontend (shared utility extended — no parallel impl):**
- `apps/ui/src/lib/relativeTime.ts` `formatLogTimestamp(raw, opts?: { timezone?: 'local'|'utc'; tz?: string })` — back-compatible: no opts / `timezone:'utc'` → unchanged UTC regex fast-path (`YYYY-MM-DD HH:MM:SS UTC`); `timezone:'local'` → `Intl.DateTimeFormat('en-CA', { timeZone })` → `YYYY-MM-DD HH:MM:SS EDT/EST` (seconds only; DST-aware zone label; explicit Intl timeZone, not browser `toLocaleString`; midnight `24:→00:` guard). Added `export const DEFAULT_DISPLAY_TZ = 'America/New_York'` (env-var seam) and `formatLogTimestampParts(raw, opts?) -> { display, tooltip }` (display + the other-zone tooltip; defaults display to local).
- New `apps/ui/src/lib/useTimezonePreference.ts` — `[timezone, toggle]`, localStorage `homelab-monitor:timezone`, default 'local', SSR-safe.
- New `apps/ui/src/components/logs/TimezoneToggle.tsx` — "UTC" checkbox (clone of WrapToggle).
- `<LogViewer>` forwards its `timezone` prop → `LogLineList` → per-row render via `formatLogTimestampParts(line.timestamp, { timezone })` with the other-format in a native `title=` tooltip. Both docker + cron viewers call `useTimezonePreference()`, render `<TimezoneToggle>` in their header, and pass `timezone` to `<LogViewer>`. The docker header "Last:" timestamp converts with the same timezone (both per-row AND header convert together).

**Behavior:** default local time; UTC toggle flips all timestamps (rows + header); hover shows the other format; preference persists across navigation and all log viewers; both formats seconds-only (consistent).

**Docs:** extended `docs/admin/log-time-ranges.md` with a "Local time vs UTC" section.

**Tests:** relativeTime.test.ts (local EDT/EST DST + midnight + invalid + parts), useTimezonePreference.test.ts, TimezoneToggle.test.tsx, LogViewer/LogLineList/DockerContainerLogsViewer tests (incl. toggle-click flips both row + header), cron test (mock + toggle wiring).

**Verification:** `make verify` GREEN — backend 3399 passed @ 100% kernel coverage; frontend 658 passed, coverage gates met (Stmts ~85.3 / Branch ~79.3 / Funcs ~81.9 / Lines ~86.9); pyright 0; eslint 0 errors; vite build OK. Frontend-only — no backend change, no OpenAPI regen.

**Refinement note:** local display originally showed milliseconds while UTC didn't; user requested removing ms from local for consistency — both now seconds-only.
