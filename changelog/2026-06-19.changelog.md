2026-06-19 25058c0 EPIC-007/STAGE-007-011: DHCP config + DNS-steering collector (rest/networkconf)

New `UnifiNetworkconfCollector` (`plugins/collectors/integrations/unifi/networkconf.py`, `rest/networkconf`, 5m, `concurrency_group="unifi"`, stateless): per DHCP-enabled non-WAN network emits `homelab_unifi_dhcp_pool_size{network}` (address count), `homelab_unifi_dhcp_pool_start{network}` / `_pool_end{network}` (int(ip) as float), and `homelab_unifi_dhcp_dns_primary{network, dns=<dhcpd_dns_1>}=1.0` (info-gauge). ALWAYS emits `homelab_unifi_dhcp_enabled_network_count` (0.0 when none) + `homelab_unifi_api_took_seconds{endpoint="rest/networkconf"}`.

Pool gauges gated on both start+stop parsing to same-family IPs with stop >= start (inverted/mixed-family/malformed ranges skip pool gauges but the network still counts + emits its dns gauge — field-by-field graceful degrade).

`homelab_unifi_dhcp_dns_primary` is the DNS-steering signal: a `dns` label != 192.168.2.148 (the Pi-hole) means clients are no longer steered to filtered DNS. The expected value lives in STAGE-007-015's vmalert rule (the sole DNS-protection signal after the firewall DNS-force rules were deleted).

The always-present `dhcp_enabled_network_count` lets STAGE-007-015 avoid the absent() trap.

Tracked deferrals: `dhcp_pool_used`/occupancy -> STAGE-007-015 PromQL join (homelab_unifi_client_count_by_network / dhcp_pool_size); authoritative leases -> STAGE-007-012; `reservation_count{network}` -> STAGE-007-019 (added as a deliverable on that card).

Registered in the unifi bundle. No protocol/client/config changes (`rest_networkconf()` already existed).

~24 unit tests (all branches incl. `_parse_ip` 5 branches, WAN/disabled filtering, inverted + mixed-family pool ranges, missing DNS). Live-validated against the real UDM (populated path: pool_size=249, dns=192.168.2.148 confirmed).
