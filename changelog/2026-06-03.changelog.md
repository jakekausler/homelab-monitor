## [STAGE-004-016A] LogsQL structured field filters (+ single-line LogsQL editor fix)

**Summary:** The Field Inspector's "Add to filter" now composes STRUCTURED LogsQL `field:"value"` clauses for queryable fields instead of imprecise `_msg` substring matches. Also fixed (in-Refinement, user-reported) the advanced LogsQL editor inserting newlines on Enter.

**Key changes:**
- Add-to-filter mapping upgraded: `host`, and `fields`-bag entries (e.g. `compose_service`, `container_name`) → structured `field:"value"` LogsQL clauses (advanced mode). `severity` → structured `severity:"<raw>"` using the RAW stored token from `fields["severity_raw"]` (VictoriaLogs stores the raw severity, not the normalized display value, so filtering on the display value would match nothing). `service` → identity chip (unchanged). `message` → `_msg:"<substring>"` (unchanged; substring is correct for free text). `stream` → `_msg` substring (stream maps from the VL builtin `_stream_id`, not a queryable field). `timestamp` → Copy-only. The `severity_raw` bag row is Copy-only (it is a backend-only field, not queryable in VL).
- New `fieldFilterClause(field, value)` helper (sibling of `msgFilterClause`, reuses `escapeLogsQlPhrase`) and `appendFieldFilter(field, value)` Explorer handler mirroring `appendMsgFilter` — composes the clause, switches to advanced mode, routes through the `writeUrl` choke-point (STAGE-015 persistence + history). Multiple field filters AND together.
- Backend contract verified: all VictoriaLogs fields are queryable via `field:"value"` (Vector flattens every field to JSON; backend passes the `expr` verbatim). Raw LogsQL field filters already worked in advanced mode; this stage just composes them via the inspector.
- Opt-in preserved: Docker/Cron log viewers pass no add-to-filter callbacks → no buttons (unchanged).
- LogsQL advanced editor single-line enforcement: pressing Enter now executes the search WITHOUT inserting a newline (or a stray space, or oscillating). Root cause was a CodeMirror transactionFilter returning an inconsistent (changes, selection) pair; fixed by atomically rejecting multi-line transactions (`EditorState.transactionFilter.of(tr => tr.docChanged && tr.newDoc.lines > 1 ? [] : tr)`). Multi-line paste into the editor is blocked (trade-off accepted for a single-line query box). Extracted a pure `sanitizeSingleLineInsert` helper (CodeMirror-free module `logsQlEditorUtils.ts`) used by the value-resync effect and unit-tested.

**Files:** apps/ui/src/lib/logsQlTranslate.ts, apps/ui/src/routes/logs/LogsExplorerPage.tsx, apps/ui/src/routes/logs/LogsExplorerBody.tsx, apps/ui/src/components/logs/FieldInspectorPanel.tsx, apps/ui/src/components/logs/LogsQlEditorImpl.tsx, apps/ui/src/components/logs/logsQlEditorUtils.ts (new), plus tests (logsQlTranslate.test.ts, LogsExplorerPage.test.tsx, FieldInspectorPanel.test.tsx, LogsQlEditor.test.tsx).

**Tests:** 961 frontend tests pass; `make verify` green (backend 100% coverage unchanged). New tests cover fieldFilterClause, appendFieldFilter, the severity raw-value routing (incl. numeric journald '4' and absent-fallback), severity_raw button suppression, and the editor single-line helper.
