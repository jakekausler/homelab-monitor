## [STAGE-004-020] Log-line export (download matching logs as .txt or .json)

**Summary:** New `GET /api/logs/export` endpoint + frontend Export button in Logs Explorer control bar that downloads the current query's matching log lines as `.txt` or `.json` via a true-streaming backend endpoint with an enforced line cap.

**Key changes:**
- Backend (Option A — true streaming, chosen over buffering and batched pagination): `VictoriaLogsClient.stream_query()` (httpx `client.stream()` + `aiter_lines()`, reuses `_parse_one`, enforces local `limit` cap, O(1) memory). New `kernel/logs/export.py` — `format_txt_line` (line formatter: `<timestamp> [<severity>] <service>: <message>`), `stream_export_lines` (incremental JSON-array framing with `[]` for empty). Pre-flight first-line probe maps VL errors to 502 before the 200 commits. `GET /api/logs/export` route (auth, window validation via `parse_and_validate_window`, services compose via `_compose_services_expr`, format validated via `Literal` for txt|json, max clamped 1..100000, expr > 4096 → 400, `Content-Disposition: attachment; filename="logs_<UTC>Z.{txt,json}"`). `ExportRequest`/`ExportResponse` schemas.
- Frontend: new `ExportButton.tsx` (control bar, positioned after Save Query button) — Radix Dialog with format radio (txt/json), max-lines input (default 10000, clamped 1–100000 via input constraints), Download/Cancel buttons; pure `buildExportUrl` helper; hidden-anchor cookie-auth download; no react-query hook. Integrated into `LogsExplorerBody` control bar.
- Decisions: D-EXPORT-TXT-AND-JSON, D-EXPORT-CAPS-ENFORCED (100k line cap), D-EXPORT-STREAMED (true streaming, O(1) memory footprint).
- Tests: backend `test_logs_export.py` (stream formatting, empty/single/multi-line, JSON array framing) + `test_api_logs_export.py` (route, auth, window validation, format literal, max clamp, expr size, Content-Disposition); `stream_query` unit tests in `test_victorialogs_client.py`; frontend `ExportButton.test.tsx` (dialog state, radio/input binding, URL construction, download link). 100% kernel coverage; `make verify` green.
- Refinement: frontend 3a; user approved Desktop + Mobile; integration check passed.
- Forward-compat: independent of 021.

**Files:** apps/monitor/homelab_monitor/kernel/logs/export.py (new), apps/monitor/homelab_monitor/kernel/api/routers/logs.py, apps/monitor/homelab_monitor/kernel/logs/victorialogs_client.py (stream_query), apps/monitor/tests/test_logs_export.py (new), apps/monitor/tests/test_api_logs_export.py (new), apps/monitor/tests/test_victorialogs_client.py, apps/ui/src/routes/logs/ExportButton.tsx (new), apps/ui/src/routes/logs/LogsExplorerBody.tsx, plus tests (ExportButton.test.tsx).
