New `kernel/logs/export.py` — `format_txt_line` (line formatter: `<timestamp> [<severity>] <service>: <message>`), `stream_export_lines` (incremental JSON-array framing with `[]` for empty). Pre-flight first-line probe maps VL errors to 502 before the 200 commits. `GET /api/logs/export` route (auth, window validation via `parse_and_validate_window`, services compose via `_compose_services_expr`, format validated via `Literal` for txt|json, max clamped 1..100000, expr > 4096 → 400, `Content-Disposition: attachment; filename="logs_<UTC>Z.{txt,json}"`). `ExportRequest`/`ExportResponse` schemas.

- Tests: backend `test_logs_export.py` (stream formatting, empty/single/multi-line, JSON array framing) + `test_api_logs_export.py` (route, auth, window validation, format literal, max clamp, expr size, Content-Disposition); `stream_query` unit tests in `test_victorialogs_client.py`; frontend `ExportButton.test.tsx` (dialog state, radio/input binding, URL construction, download link). 100% kernel coverage; `make verify` green.
