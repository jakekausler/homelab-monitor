## [STAGE-004-022] Global retention settings UI

**Summary:** Added a Settings → Logs page showing VictoriaLogs retention, disk usage, and warn/crit thresholds, with a control to change retention (persisted; signals restart-required since VL retention is startup-only). Per-store, no cross-store coordinator.

**Key changes:**
- Backend: new `GET /api/settings/logs/retention` + `PATCH /api/settings/logs/retention {retention_days}` (require_session). Response: `{retention_days (effective), pending_retention_days|null, disk_used_pct, disk_used_gb, disk_budget_available, warn_pct, crit_pct, retention_source (env|runtime|default), restart_required}`.
- VL retention is startup-only (`-retentionPeriod` compose flag; no runtime API) → PATCH persists a runtime override to new `app_settings` kv table (alembic 0032) and signals restart-required; does NOT live-apply. Effective-vs-pending reconciliation in `kernel/logs/vl_retention.py` (`reconcile_retention`/`resolve_retention`/`persist_retention`); `retention_source` distinguishes env-set vs default.
- Disk usage computed against VL disk BUDGET (`load_disk_budget_config` vl_ratio, GiB) via `compute_vl_disk_usage`; `disk_budget_available` flag distinguishes config error / zero budget from genuinely-empty-but-configured VL. Thresholds `HOMELAB_MONITOR_VL_DISK_WARN_PCT=70` / `_CRIT_PCT=85` (`load_vl_disk_warning_config`, normalized so warn≤crit). Graceful degrade (no 500) on malformed disk-budget config incl. yaml.YAMLError.
- Frontend: new `/settings` route shell + `/settings/logs` page (`SettingsLogsPage`) — retention card (current + source + pending + restart-required banner + numeric input clamped 1–365 + Save) + disk-usage card (used GiB + % vs budget, warn/crit coloring) + per-stream-caps info card; enabled previously-disabled "Settings" sidebar entry. New `useLogsRetention`/`useUpdateLogsRetention` hooks.
- Decisions: D-NO-COORDINATOR, D-VL-RETENTION-UI, D-RESTART-REQUIRED-IF-NEEDED (Option A — persist-to-SQLite + effective-vs-pending + explicit restart_required; rejected read-only-display and auto-recreate-via-docker-socket).
- Tests: backend `test_vl_retention.py` + `test_api_settings_logs.py` + `test_db_migrations.py` (app_settings); frontend `SettingsLogsPage.test.tsx` + `settingsLogs.test.ts` + `SettingsLayout.test.tsx`. `make verify` green (100% kernel coverage; UI func coverage 80.51%; 0 eslint errors).
- Out of scope: per-stream/per-signature retention, VM/SQLite threshold UIs, cross-store coordinator, retention preview, auto-apply.
