## [STAGE-004-019] Histogram of line counts (time-series bar chart above Logs Explorer)

**Summary:** New `GET /api/logs/histogram` endpoint + stacked-by-severity bar chart (recharts, first in repo) above the Logs Explorer showing line density over time; click a bar narrows the range to that bucket.

**Key changes:**
- Backend (Option A — native VL `/select/logsql/hits?field=severity`, ONE call): `VictoriaLogsClient.hits()` + tolerant `_parse_hits` + `HitsSeries`. New `kernel/logs/histogram.py` — `coarse_bucket` (raw severity → error/warn/info via PUBLIC `normalize_severity`), `compute_step_ms`/`bucket_count`/`assign_bucket` (re-bins VL's epoch-aligned `/hits` onto START-aligned buckets; `end` inclusive in v0.30.0), `HistogramCache` (30s), `fetch_histogram` (exactly n buckets, all 3 coarse keys, empty→zero-filled). `GET /api/logs/histogram` route (auth, window validation, services compose, buckets clamp 1..500→422, 502 on VL error). `HistogramBucket`/`LogsHistogramResponse` schemas. Renamed `_normalize_severity`→`normalize_severity` (public, cross-module reuse).
- Frontend: `useLogsHistogramQuery` (30s staleTime); `severityBarColor` (red/yellow/gray); `HistogramChart.tsx` (recharts stacked `<BarChart>`; loading/error/empty; bar click → narrow range via writeUrl); mounted in `LogsExplorerBody` between controls and `<LogViewer>`.
- Tests: unit (bucket math, coarse-map, cache, hits parse), endpoint, `HistogramChart.test.tsx` (recharts jsdom vi.mock), integration test (`test_histogram_hits.py`) confirming v0.30.0 `/hits` field=severity.
- Refinement: frontend 3a; user approved Desktop + Mobile; integration check passed.
- Forward-compat: independent of 020/021.
