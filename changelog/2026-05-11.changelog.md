## STAGE-002-001 — Heartbeat receiver + crons/heartbeats_state schema

### Added
- POST /api/hb/{cron_id}/{start,ok,fail} — three Healthchecks-style endpoints (token-only via Scope.HEARTBEAT_WRITE; 204 on success). Optional run-context query params: `?duration=<seconds>` (0–86400), `?exit_code=N` on /fail (0–255). Unknown query params are rejected with 422 via the `extra='forbid'` Pydantic schemas plus a custom `query_model()` dependency factory (FastAPI's stock `Depends(BaseModel)` bypasses model_validate, so the factory wraps `model_validate(dict(request.query_params))` and re-raises ValidationError as HTTPException(422) with structured `details.errors`).
- `crons` table extended (alembic 0006) with: name, host, schedule, cadence_seconds, expected_grace_seconds, integration_mode CHECK ('observe'|'heartbeat'|'both'), enabled, last_seen_state CHECK ('unknown'|'running'|'ok'|'failed'|'late'), updated_at, archived_at. ADD COLUMN guarded by `inspect()` for idempotency.
- `heartbeats_state` table dropped + recreated (stub had wrong PK 'id' instead of 'cron_id'): cron_id PK FK ON DELETE CASCADE, current_state CHECK enum, last_{start,ok,fail}_at, current_streak, expected_next_at, last_duration_seconds REAL, last_exit_code INTEGER, updated_at. Migration includes a working downgrade.
- `HeartbeatRepo`: all state-changing writes route through one private `_record_state_transition` chokepoint that atomically (a) UPSERTs `heartbeats_state`, (b) UPDATEs `crons.last_seen_state` mirror, (c) INSERTs `audit_log` (single transaction). Streak math: same state → +1, transition → reset to 1. expected_next_at computed only on /ok when cadence_seconds > 0; cleared on every non-OK transition (prevents phantom 'late' alerts in STAGE-002-006).
- `CronRateLimiter`: per-cron in-memory token bucket (default 60/min capacity 60, refill 1/sec, injectable clock for tests). 429 with structured ErrorEnvelope body + `Retry-After` header.
- 60 new backend tests (1129 → 1189): 23 in test_api_heartbeat.py (auth, 404, 422 incl. unknown-query-param + invalid-value, parametrized 429 across all three verbs, observe-mode warning, audit invariants, atomicity rollback under simulated insert_audit failure, streak math, expected_next_at branches), 16 in test_heartbeat_repo.py, 9 in test_heartbeat_rate_limiter.py, 3 in test_db_migrations.py (migration 0006 columns, heartbeats_state shape, idempotent round-trip with downgrade-shape assertion).
- EPIC-002 regression.md seeded with 6 STAGE-002-001 regression checks.
