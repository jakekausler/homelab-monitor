## STAGE-003-007: Per-service Docker config-file override

**Epic:** EPIC-003 (Docker)
**Date:** 2026-05-23
**Commit:** `d41f20e`

### Summary

Adds a higher-precedence probe-config source: per-service YAML files at `/config/plugins/docker/<container_name>.yaml` override label-derived probes. The file_override path takes total ownership of a container's probes; an independent 30s asyncio loader scans the directory, validates against a pydantic schema, and writes to `probe_targets` with `config_source='file_override'`. UI surfaces the source per-probe (Source column / mobile badge) and a red "Config error" row badge when validation fails. Validation errors for orphan files (no matching container) flow to the existing `PendingSuggestionsPanel` via a new `docker_file_override_malformed` suggestion kind.

### Behavior changes

- **NEW**: Operators can define probe configurations in YAML files; they take precedence over `homelab-monitor.*` labels.
- **NEW**: `exec_authorized: bool` is persisted per-probe in `probe_targets` (migration 0025). Source-agnostic gate ŌĆö both label and file_override paths populate it; `ProbeSupervisor` reads it from the probe row instead of consulting labels at runtime. This unblocks file_override exec probes for label-free containers.
- **NEW**: `GET /api/integrations/docker/probes/summary` response extended (backward compatible) with per-container `source_breakdown: dict[str, int]` and `config_errors: list[str] | null`.
- **NEW**: `GET /api/integrations/docker/containers/{name}/probes` response includes `exec_authorized: bool` per probe.
- **NEW**: Suggestion kind `docker_file_override_malformed` for orphan validation errors (container doesn't exist).
