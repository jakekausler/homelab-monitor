# STAGE-008-008: Utilization collector (CPU / mem / disk IO / NIC / NFS)

## Status: Complete

## Overview

Collect live resource utilization from `SYNO.Core.System.Utilization` `get`: CPU load, memory + swap
percent, per-disk IO, per-NIC rx/tx, and NFS OPS + latency (the Resource Monitor surface). This is the
"how hard is the NAS working" signal and the NFS-serving-health signal (relevant to the containers that
mount `/rackstation`).

**Vertical slice:** after this stage `homelab_synology_cpu_*`, `_mem_usage_percent`, `_swap_usage_percent`,
`_disk_io_*{device}`, `_net_*{iface}`, `_nfs_*` are emitted for the live NAS at 60s.

**Spec references:** §6.2. EPIC-008 "Verified deployment reality" (`System.Utilization` shape).

## Concrete deliverables

### 1. Utilization collector

`utilization.py`: calls `client.system_utilization()`, emits:
- CPU: `homelab_synology_cpu_user_percent`, `_system_percent`, `_other_percent`, load 1/5/15min.
- Memory: `homelab_synology_mem_usage_percent` (`real_usage`), `_swap_usage_percent`, total/avail/cached.
- Disk IO: `homelab_synology_disk_utilization_percent{device}`, `_disk_read_bytes{device}`,
  `_disk_write_bytes{device}` (per the 8 devices).
- Network: `homelab_synology_net_rx_bytes{iface}`, `_net_tx_bytes{iface}` (total + eth0..ethN).
- NFS: `homelab_synology_nfs_read_ops`, `_write_ops`, `_total_ops`, `_max_latency_seconds` (the
  NFS-serving-health signal).

### 2. Tests

3a fixture (recon `System.Utilization` shape: cpu/memory/disk[8]/network[6]/nfs); 3b live emission.

## Design Phase

**Status**: [x] Design (Complete)

**Design Notes** (selected 2026-06-23; brainstormer used for the multi-series + units + scope choices, recommendations adopted):

Grounded against the live-captured `SYNO.Core.System.Utilization get` payload (`/tmp/synology_utilization_raw.json`). Card field-name guesses corrected against ground truth: cpu `1min_load`/`5min_load`/`15min_load` are INTS = load-avg ×100 (88 = 0.88), NOT percent; `user_load`/`system_load`/`other_load` ARE percent ints (no idle_load, no per-core); memory sizes are KB; `disk.disk[]` is the per-disk list (8 drives) with `disk.total` a sibling aggregate; `network` is a LIST whose element [0] is the `device:"total"` aggregate (no separate total key); `nfs` is a LIST-of-one with UPPERCASE `_OPS` and `_max_latency` (ms); and the payload ALSO carries a `space.volume[]` (per-LVM-volume IO) section not named in the card.

SINGLE fetch (`system_utilization()`), MULTI-SERIES (per-disk/per-NIC/per-volume lists) → mirrors pool.py's `as_list_of_dicts` per-item emit. Decisions:

- **CPU:** `homelab_synology_cpu_load1`/`_load5`/`_load15` = load-avg as a true FLOAT (÷100 in the collector — store the natural unit, not the ×100 footgun; matches node_exporter `node_load1`); `homelab_synology_cpu_user_percent`/`_system_percent`/`_other_percent` = raw int percent.
- **Memory:** `homelab_synology_mem_usage_percent` (`real_usage`) + `_swap_usage_percent` (raw int); `_mem_total_bytes`/`_mem_available_bytes`/`_mem_cached_bytes`/`_mem_buffer_bytes` = KB→bytes (×1024, Prometheus base-unit convention; from total_real/avail_real/cached/buffer).
- **Disk IO (per-disk + aggregate):** `homelab_synology_disk_read_bytes_per_second{device}`/`_disk_write_bytes_per_second{device}` (renamed from the card's `_disk_read_bytes` — these are DSM-computed RATES, the `_per_second` suffix prevents a consumer from wrongly `rate()`-ing them), `_disk_read_iops{device}`/`_disk_write_iops{device}` (read_access/write_access), `_disk_utilization_percent{device}`. The `disk.total` aggregate is emitted into the SAME families with `{device="total"}` (uniform label; DSM's total utilization% is not sum-derivable).
- **Network (per-NIC + aggregate):** `homelab_synology_net_rx_bytes_per_second{iface}`/`_net_tx_bytes_per_second{iface}` (renamed for the same rate-clarity reason). The `total` aggregate is naturally element [0] of the list → emitted as `{iface="total"}`.
- **Volume IO (`space.volume[]`, the LVM/dm layer — distinct from physical disk IO and from 005's volume SIZE):** `homelab_synology_vol_io_read_bytes_per_second{volume}`/`_vol_io_write_bytes_per_second{volume}`/`_vol_io_utilization_percent{volume}` (volume label = `display_name` e.g. "volume1"); `space.total` aggregate as `{volume="total"}`. Included now (free from the same payload, distinct signal, supports the NFS/container story).
- **NFS (LIST-of-one, iterated for graceful len≠1):** `homelab_synology_nfs_read_ops`/`_write_ops`/`_total_ops` (counts, NO label — singleton aggregate); `homelab_synology_nfs_read_max_latency_seconds`/`_write_max_latency_seconds`/`_total_max_latency_seconds` (ms→seconds, ÷1000, all three read/write/total).
- **Cap-routing:** unchanged `capped_emitter`/`cap_for_synology`/`family(name,obs)` closure (8 disks + 6 NICs + volumes + aggregates ≈ <20 series/family, far under 500).
- **Helpers:** import `as_dict`/`as_list_of_dicts`/`as_float` + `client_unconfigured_result`/`fetch_or_result`/`cap_for_synology`/`capped_emitter` from `_shared` (NOT `bool_to_gauge`/`nested` — no bools, shallow nesting). One NEW local `_scaled(v, factor) -> float|None` (as_float×factor, None-propagating) for the KB→bytes / ms→s / load÷100 conversions.

SCOPE-OUT (explicit YAGNI, NOT silent deferral — no future stage owns these; they are deliberately not built): `smb` (all-empty lists on this NAS → zero series, untestable) and `lun` (empty list → no iSCSI) utilization sections are NOT emitted; `memory.si_disk`/`so_disk` (swap-in/out IO) are NOT emitted (`swap_usage_percent` already flags swap pressure). These are documented scope decisions, revisitable only if a concrete future requirement names them — no tracked stage is created because there is no downstream consumer. All 5 card metric families (cpu/mem/disk/net/nfs) ARE emitted; nothing the card requested is dropped.

Files: NEW `apps/monitor/homelab_monitor/plugins/collectors/integrations/synology/utilization.py`; EDIT `synology/__init__.py` (import + add `SynologyUtilizationCollector` to `_SYNOLOGY_COLLECTORS`); NEW `apps/monitor/tests/test_synology_utilization_collector.py` (mirror pool/system tests, 100% BRANCH coverage — real-payload fixture from the captured JSON + synthetic edge rows: missing/non-numeric scalars, empty/absent disk/network/nfs/space lists, non-dict entries, `_scaled` None branch, client-unconfigured ok=False, SynologyError ok=False, non-dict payload ok=True-no-metrics); EDIT `test_synology_integration_bundle.py` (add `test_register_all_registers_utilization`, interval=60 via its own constant). Cadence 60s (`timedelta(seconds=60)`), timeout 30s, `concurrency_group="synology"`. BACKEND stage → no user prompt in Refinement.
