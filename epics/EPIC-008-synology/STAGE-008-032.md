# STAGE-008-032: CollectorConfig YAML-loading mechanism (populate synology_mounts + general collector config)

## Status: Complete

## Overview

The plugin loader currently constructs only the BASE `CollectorConfig` (see `kernel/plugins/loader.py`); collector subclasses' extra fields (e.g. `HostCollectorConfig.extra_mountpoints`, `SynologyMountHealthCollectorConfig.synology_mounts`) are read via `getattr(ctx.config, "<field>", <default>)` and CANNOT currently be overridden from any file or env — there is NO YAML config-loading for collector config fields (only a `TODO(STAGE-014)` in lifespan.py + scattered subclass docstrings that reference it; no stage actually owns it). This blocks every per-host collector override in the project.

Build the YAML config-loading mechanism so per-host collector config (starting with `synology_mounts` for the STAGE-008-018 mount-health collector) can be populated. This is the hard PREREQUISITE that makes the EPIC-008 mount features actually function on real data.

**Vertical slice:** after this stage, a host-override YAML (e.g. `/config/plugins/collectors/<name>.yaml`) populates `synology_mounts` so `SynologyMountHealthCollector` probes the real `/rackstation/*` mounts and emits `homelab_synology_mount_up` with real series — which in turn lets `SynologyMountDown` (STAGE-008-019) fire and the inhibit rule suppress dependent ContainerDown alerts on real data.

**Spec references:** STAGE-008-018 (mount-health collector — the first consumer), STAGE-008-019 (inhibit rule — blocked on real mount_up series), the `TODO(STAGE-014)` markers in `kernel/api/lifespan.py`, the subclass docstrings in `builtin/host.py` + `builtin/synology_mount_health.py`.

## Concrete deliverables

### 1. YAML config-loading for CollectorConfig subclass fields

A loader (extending `kernel/config.py` / the plugin loader) that reads per-collector config (e.g. from a `/config/plugins/collectors/*.yaml` mounted volume or the existing config mechanism) and constructs the correct CollectorConfig SUBCLASS (not just the base), so `getattr(ctx.config, "synology_mounts", [])` returns the configured value. Resolve the `TODO(STAGE-014)` markers in lifespan.py.

### 2. Wire synology_mounts (+ host extra_mountpoints) through

Make `SynologyMountHealthCollector` actually probe a configured mount list in prod, and `HostCollector` honor a configured `extra_mountpoints`, via the new mechanism.

### 3. Tests / validation

3a: unit tests for the config loader (absent file → defaults; present file → subclass fields populated; malformed → graceful). 3b (instance A): real host-override populates `synology_mounts` with the real `/rackstation/*` paths; verify `homelab_synology_mount_up` emits real series and `SynologyMountDown` can fire.

## Design Phase

**Status**: [x] Design (Complete)

**Design Notes** (recorded 2026-06-26):

**Context:** the user prioritized this prerequisite ahead of STAGE-008-019 (which is paused at Design Complete) because it unblocks BOTH 018's real mount probing AND 019's real-data inhibit firing. The mechanism is GENERAL (any CollectorConfig subclass), not synology-specific.

**THE BLOCKER (verified):** `kernel/plugins/loader.py::register()` validates overrides against the BASE `CollectorConfig` only (`CollectorConfig.model_validate(overrides)`), and base CollectorConfig has `model_config = ConfigDict(extra="forbid")`. So a subclass field like `synology_mounts` passed to the base validator RAISES. Three collectors define CollectorConfig SUBCLASSES with extra fields the loader can't currently populate: `HostCollectorConfig` (extra_mountpoints, top_n_processes, exclude_disk_devs, disk_mountpoint_relabel, exclude_temp_sensors), `SynologyMountHealthCollectorConfig` (synology_mounts), `WatchedDirSizeCollectorConfig` (watched_directories). Consumers read via `getattr(ctx.config, "<field>", <default>)` (host.py:111, synology_mount_health.py:157) — after this stage those return CONFIGURED values.

**Selected approach** (mirrors TWO existing precedents — the STAGE-003-007 docker `OverrideLoader` at `kernel/docker/override_loader.py` AND the `kernel/config.py` yaml-load idiom — minimal novel surface):

**DECISION 1 — Subclass-awareness via a ClassVar:** add `config_class: ClassVar[type[CollectorConfig]] = CollectorConfig` to `BaseCollector` (fits the existing ClassVar convention alongside `name`/`interval`/`timeout`). The 3 subclass collectors override it (`config_class = HostCollectorConfig`, etc.). `register()` validates the merged dict against `collector_cls.config_class` instead of the hardcoded base. `extra="forbid"` interaction is CORRECT: the subclass DECLARES its field so validation succeeds; base-only collectors still forbid extras (catches operator typos).

**DECISION 2 — Per-collector YAML at `/config/plugins/collectors/<name>.yaml`** (matches the literal `TODO(STAGE-014)` comment in lifespan.py AND the docker `OverrideLoader`'s `/config/plugins/docker/*.yaml` precedent — two signals agree). Loader seam: a small helper `load_collector_overrides(name) -> dict[str, object]` called INSIDE `register()` keyed on `overrides["name"]`, so lifespan.py call sites stay UNCHANGED (still pass the flat ClassVar-derived `{name, interval_seconds, timeout_seconds}` dict). Merge precedence: register base dict < YAML file, then `config_class.model_validate(merged)` (YAML wins so an operator can also bump interval/timeout). NO env-per-field layer (YAGNI). Absent file → empty dict → subclass validates with its `Field` defaults → behavior IDENTICAL to today (open-source-safe). The loader mirrors `load_disk_budget_config`'s idiom: `if path.is_file()` → `yaml.safe_load(f) or {}` → `isinstance(dict)` type-check (else ValueError) → cast.

**DECISION 3 — env/dir convention:** `HOMELAB_MONITOR_COLLECTOR_OVERRIDES_DIR`, default `/config/plugins/collectors`, mirroring `HOMELAB_MONITOR_DOCKER_OVERRIDES_DIR`. NO new compose mount — the existing `${HOMELAB_MONITOR_OVERRIDES_DIR:-/var/lib/homelab-monitor/overrides}:/config:ro` bind already covers `/config/plugins/collectors/*` (empty by default = no-op). Document in `.env.example` (a commented stanza + one example).

**DECISION 4 — Scope (YAGNI):** build the GENERAL mechanism; WIRE + TEST only the two collectors with live consumers — `synology_mount_health` (synology_mounts — THE point, unblocks 018) and `host` (extra_mountpoints — natural second). `WatchedDirSizeCollectorConfig` works for free via the general mechanism (supported-but-not-configured; no shipped YAML, no dedicated wiring test). Remove the `TODO(STAGE-014)` markers in lifespan.py that this resolves.

**DECISION 5 — Test matrix (100% branch):** inject via `tmp_path` + `monkeypatch.setenv("HOMELAB_MONITOR_COLLECTOR_OVERRIDES_DIR", str(tmp_path))` (mirror test_config.py). Branches: (1) dir absent → defaults; (2) file absent → defaults; (3) file present+valid → field flows through; (4) malformed (non-dict root) → ValueError; (5) unknown YAML key → ValidationError via subclass extra=forbid (catches typos); (6) collector with no config_class → base used; (7) collector WITH config_class → subclass used. pyright-strict: `config_class: ClassVar[type[CollectorConfig]]`, loaded YAML typed `dict[str, object]`/`cast` after the isinstance guard.

**Files touched:** `kernel/plugins/base.py` (add `config_class` ClassVar), `kernel/plugins/loader.py` (subclass-aware validate + the `load_collector_overrides` seam — OR a sibling `kernel/plugins/collector_config.py` for the loader; Build decides, mirroring override_loader.py), `builtin/{host,synology_mount_health}.py` (set `config_class`; watched_dir_size.py optionally sets it too for free), `kernel/api/lifespan.py` (remove the resolved TODO(STAGE-014) markers; call sites unchanged), `deploy/compose/.env.example` (document HOMELAB_MONITOR_COLLECTOR_OVERRIDES_DIR), new test file(s).

**REFINEMENT INCLUDES THE STAGE-008-018 e2e (user-mandated this stage):** 032's 3b must ALSO run the 018 validation that was previously blocked: write `/config/plugins/collectors/synology_mount_health.yaml` with the real `synology_mounts: ["/rackstation/..."]` paths, recreate the prod monitor (`docker compose up -d --force-recreate monitor` with HM_RACKSTATION_SRC=/rackstation), and verify the `synology_mount_health` collector emits REAL `homelab_synology_mount_up{mount}` series (=1 for responsive real mounts, with free/total bytes) against the live NAS via the monitor's metrics — the real-data probing 018 could not perform. This proves the WHOLE chain: config-loading → collector reads synology_mounts → real mount_up series exists (which in turn unblocks 019's SynologyMountDown firing).

**Scope-outs (YAGNI):** no env-per-collector-field overrides; no WatchedDirSize YAML wiring/test (works for free, no consumer this epic); no interval/timeout-float widening (the OTHER TODO(STAGE-014) about sub-second intervals is a SEPARATE concern — leave that TODO if present, only remove the YAML-loading TODO this stage resolves).
