# STAGE-008-023: vmalert-logs rules over the `service="synology-*"` syslog→VL stream

## Status: Not Started

## Overview

Author the Synology LOG alert rules (`deploy/vmalert/logs/synology.yaml`) over the `service="synology-*"`
stream that STAGE-008-020 forwards into VL. Per Amendment 5, all login/event signals are VL-derived (NOT
API-polled): failed-login bursts, abnormal/geo-anomalous logins, SMART-event log lines, package/update event
lines, and a general DSM error-rate. Rules must validate against the REAL forwarded stream (the EPIC-006/007
"rules validate against real data" discipline) — so this stage runs AFTER the syslog source exists.

**Vertical slice:** after this stage the Synology log rules are deployed and validated against the real
`service="synology-*"` lines (the exact patterns finalized against what DSM actually forwards).

**Spec references:** EPIC-008 "Alert catalog" (log rows) + Amendment 5 (events→syslog, VL-derived; single
source). EPIC-005 STAGE-005-014 (log rules) + EPIC-007 STAGE-007-017 (CEF log rules) — precedent (DSM is
RFC, not CEF; patterns differ). EPIC-008 STAGE-008-020 coverage map (which categories actually forward).

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- **VL-derived, single source** — these signals come from syslog→VL, NOT API polling. If a geo-anomalous-
  login enrichment (city/country) is only available via the API and NOT in the forwarded lines, the Design
  decides whether to accept the coarser syslog signal or note the enrichment as a documented gap (per the
  coverage map) — but does NOT re-introduce API log polling.
- Patterns finalized against the REAL forwarded lines (rules validate against real data).
- Severity: failed-login-burst + abnormal-login + SMART-event = warning; package-event + general-error-rate
  = info.

## Concrete deliverables

### 1. `deploy/vmalert/logs/synology.yaml`

Rules: `SynologyFailedLoginBurst` (failed-login lines burst — DSM/SSH/SMB), `SynologyAbnormalLogin`
(geo/abnormal-login lines), `SynologySmartEventLog` (SMART warning/error lines), `SynologyPackageEventLog`
(package install/update/failure lines), `SynologyLogErrorRate` (general DSM error-rate). Exact LogsQL patterns
derived from the real forwarded content.

### 2. Tests

3a LogsQL rule golden tests over captured line fixtures (firing/non-firing); 3b validate against the real
`service="synology-*"` stream (instance-A if the live stream lands there).
