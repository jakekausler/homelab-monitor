# STAGE-008-013: Security Advisor status + active-connections collector

## Status: Not Started

## Overview

Collect the structured security STATUS signals (not log events ‚Äî those come via syslog per Amendment 5):
Security Advisor result (`SYNO.Core.SecurityScan.Status` method **`system_get`** ‚Äî recon: `sysStatus=risk`
currently, per-category fail counts) and the active-connection count (`SYNO.Core.CurrentConnection list` ‚Äî
how many sessions, who/from-where is a view-time re-query on the panel, not a stored series). Per Amendment 5:
do NOT API-poll anything that arrives via syslog; Security Advisor scan-status is a periodic scan result (not
a log event) so it is API-sourced; login *events* are VL-derived (STAGE-008-023).

**Vertical slice:** after this stage `homelab_synology_security_status` +
`homelab_synology_security_findings{category,severity}` + `homelab_synology_active_connections` are emitted
for the live NAS.

**Spec references:** ¬ß6.2. EPIC-008 Amendment 5 (events‚Üísyslog, status‚Üímetric; single source per signal).
EPIC-008 "Verified deployment reality" (SecurityScan.Status `system_get`; CurrentConnection shape).

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- `synology` concurrency group; self-metrics + API latency. Security 1h (scans infrequent), connections 60s.
- **Single source per signal** ‚Äî Security Advisor STATUS is API (a scan result, not a log); login EVENTS are
  syslog‚ÜíVL (STAGE-008-023), NOT API-polled here.
- The "who is connected" detail is a panel view-time re-query (STAGE-008-027), not a stored metric series
  (privacy + cardinality).

## Concrete deliverables

### 1. Security + connections collector

`security.py`: calls `client.security_scan_status()` (`system_get`) + `client.current_connection_list()`,
emits:
- `homelab_synology_security_status` (0=safe / 1=warning / 2=risk, from `sysStatus`),
  `homelab_synology_security_findings{category,severity}` (per-category fail counts:
  malware/network/systemCheck/update/userInfo √ó danger/warning/risk/outOfDate/info),
  `homelab_synology_security_last_scan_age_seconds`.
- `homelab_synology_active_connections` (count from `CurrentConnection total`). (The who/from-IP/service list
  is NOT emitted as series ‚Äî surfaced live on the panel in STAGE-008-027.)

### 2. Tests

3a fixtures: SecurityScan `system_get` (risk with a userInfo finding; safe); CurrentConnection list. 3b live
emission.
