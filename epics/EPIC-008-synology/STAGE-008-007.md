# STAGE-008-007: System info + temps collector

## Status: Complete

## Overview

Collect DSM system identity + thermal + reboot signals: `SYNO.Core.System` v3 `info` (model/serial/firmware/
uptime/`sys_temp`/fan status), plus `SYNO.Core.Hardware.NeedReboot` (a pending-reboot bool). Fan speed on
this box is status-strings-only (no numeric RPM ŌĆö recon), so we emit the fan status enum, not RPM.

**Vertical slice:** after this stage `homelab_synology_system_*` (uptime, `sys_temp`, temp-warning flag,
firmware/model as labels) + `homelab_synology_need_reboot` are emitted for the live NAS.

## Concrete deliverables

### 1. System collector

`system.py`: calls `client.system_info()` (v3) + `client.need_reboot()` + (optionally)
`client.hardware_fanspeed()`, emits:
- `homelab_synology_system_uptime_seconds` (parse `up_time` HH:MM:SS), `homelab_synology_sys_temp_celsius`,
  `homelab_synology_sys_temp_warning` (1/0 from `sys_tempwarn`/`temperature_warning`),
  `homelab_synology_info{model,serial,firmware,cpu_series}` (gauge=1, identity labels).
- `homelab_synology_fan_status{state}` (from FanSpeed status strings; `all_disk_temp_fail`, `cool_fan`).
- `homelab_synology_need_reboot` (1/0).

### 2. Tests

3a fixture (recon `Core.System` v3 + FanSpeed + NeedReboot shapes) covering uptime-parse + temp + warn-flag +
reboot bool; 3b live emission.

## Design Phase

**Status**: [x] Design (Complete)

**Design Notes** (selected 2026-06-23; brainstormer used for the multi-fetch + extraction choices, recommendations adopted):

Grounded against live-captured payloads (`/tmp/synology_system_info_raw.json`, `/tmp/synology_fanspeed_raw.json`, `/tmp/synology_need_reboot_raw.json`). Card field-name guesses corrected against ground truth: `firmware_ver` (NOT `firmware`); `up_time` is a STRING "1205:13:7" (HHH:MM:S), not a seconds int; THREE temp-warn variants exist (`sys_tempwarn`/`systempwarn`/`temperature_warning`); fan payload is a FLAT dict with yes/no STRINGS (`cool_fan`/`all_disk_temp_fail`) + a mode string `dual_fan_speed` ("quietfan") ŌĆö NO RPM, NO list; `need_reboot` is a native bool in `{"need_reboot": false}`.

THREE-API fetch (the key difference from 005/006's single-fetch): `system_info()` (SYNO.Core.System v3 info ŌĆö PRIMARY), `hardware_fanspeed()` (SYNO.Core.Hardware.FanSpeed v1 get), `need_reboot()` (SYNO.Core.Hardware.NeedReboot v1 get).

Decisions:
- **Multi-fetch partial-failure = degraded-but-useful (Option B):** a local `_fetch(ctx, resp, start, emitted, errors)` wraps `fetch_or_result` ŌĆö on a `SynologyError` it records the error string into `errors` and returns None (does NOT early-return the whole collector); on success it emits `api_took` (via the inner `fetch_or_result`) and returns the response. `system_info` is PRIMARY: if it fails ŌåÆ `ok=False, errors=errors`; otherwise `ok=True` with per-fetch errors accumulated (fan/reboot failure degrades but does not fail the collector). Matches observe-only.
- **Uptime:** local `_uptime_to_seconds(v)` ŌĆö non-strŌåÆNone; split ":"; len!=3ŌåÆNone; non-numericŌåÆNone; else `float(h*3600+m*60+s)`. Strict 3-segment (corruption is skipped, not guessed).
- **Temp warning:** local `_temp_warn(info)` ŌĆö reads all three variants via `_bool_to_gauge`, emits 1.0 if ANY present-bool is True, 0.0 if Ōēź1 present bool but none True, None if none present (robust against a future DSM populating only one variant).
- **Identity `homelab_synology_info{model,serial,firmware,cpu_series}=1.0`:** anchor on `model` (skip the whole metric if model absent/non-str); omit any individual label whose value is absent/non-str (no empty-string labels); firmware label = `firmware_ver` value as-is ("DSM 7.3.2-86009", not stripped).
- **Fan status:** state-set `homelab_synology_fan_status{state}` via a local `_yesno_to_gauge(v)` ("yes"->1.0/"no"->0.0/else None): emit `{state="cool_fan"}`=yesno(cool_fan), `{state="all_disk_temp_fail"}`=yesno(all_disk_temp_fail) (both value-carrying so they're alertable: cool_fan==0 or all_disk_temp_fail==1), and `{state="<dual_fan_speed mode>"}=1.0` (mode member, e.g. "quietfan").
- **Reboot:** `homelab_synology_need_reboot{}` = `_bool_to_gauge(need_reboot)`.
- **Temp:** `homelab_synology_sys_temp_celsius{}` = `as_float(sys_temp)`. **Uptime:** `homelab_synology_system_uptime_seconds{}` = `_uptime_to_seconds(up_time)`.
- **Cap-routing:** keep the `capped_emitter`/`family(name,obs)` pattern (consistency with 005/006; fan_status/info state-sets can be multi-series).

HELPER EXTRACTION (rule-of-three now MET ŌĆö 005/006/007 all need the navigators): EXTRACT the byte-identical, structurally-stable navigators `_as_dict`, `_as_list_of_dicts`, `_nested`, `_bool_to_gauge` from storage.py + pool.py into `_shared.py`; update STAGE-008-005 `storage.py` AND STAGE-008-006 `pool.py` to import them from `_shared.py` (removing their local copies); `system.py` imports them too. The DIVERGING status/value mappers stay LOCAL per collector (`_status_str_to_gauge` in storage, `_raid_status_to_gauge` in pool, `_uptime_to_seconds`/`_yesno_to_gauge`/`_temp_warn` in system). This is a real refactor touching the committed 005/006 files ŌåÆ it rides 007's impl commit and gets explicit code-review attention; full `make verify` re-run validates no regression.

Files: NEW `apps/monitor/homelab_monitor/plugins/collectors/integrations/synology/system.py`; EDIT `_shared.py` (add 4 navigators); EDIT `storage.py` + `pool.py` (import navigators from `_shared.py`, delete local copies); EDIT `synology/__init__.py` (import + add `SynologySystemCollector` to `_SYNOLOGY_COLLECTORS`); NEW `apps/monitor/tests/test_synology_system_collector.py` (mirror 006's test, 100% BRANCH coverage ŌĆö real-payload fixtures for all 3 APIs + synthetic edge rows: each fetch erroring independently, primary-failsŌåÆok=False, malformed/absent fields, uptime malformed both ways, yesno none-branch, temp-warn none/all-false/any-true, info model-absent + individual-label-absent); EDIT `test_synology_integration_bundle.py` (add `test_register_all_registers_system`). Cadence 60s (`timedelta(seconds=60)`), timeout ~30s, `concurrency_group="synology"`. BACKEND stage ŌåÆ no user prompt in Refinement. NO card deliverables deferred.
