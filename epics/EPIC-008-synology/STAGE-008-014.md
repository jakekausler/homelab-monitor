## Design Phase

**Status**: [x] Design (Complete)

**Design Notes** (2026-06-26):

**Selected approach: Option B вЂ” curated alertable SMART subset + raw mdstat/CPU temp, bounded cardinality.** Brainstormer (Opus) presented 3 options; Option B selected (user-confirmed both judgment calls). Rationale: the SshProbe has NO cap/drop-gauge machinery (unlike DSM-API collectors with `capped_emitter`), so the parser itself must bound cardinality вЂ” a whitelist is the cap. Option A (full per-attribute, field-as-label) = ~690 UNBOUNDED series + would COLLIDE with existing DSM-API `homelab_synology_ups_*` names. Option C (failure-rollup only) discards cheap worst/threshold trend visibility.

**Probe shape:** new `apps/monitor/homelab_monitor/plugins/collectors/ssh/synology.py` вЂ” a `SynologyProbe(SshProbe)` (pure `parse(result: SshCommandResult) -> ProbeOutcome`, no ctx/vm access; framework auto-emits the `homelab_ssh_*` health surface). Register in `plugins/collectors/ssh/__init__.py`. The combined forced-command script (already live on the NAS as `/usr/local/bin/hm-probe-synology.sh`) emits marker-delimited sections (`===HM_UPTIME/DF/SYNODISK_ENUM/SMART .../MDSTAT/UPSC/HWMON/END===`). Parser splits on markers; each section parsed in its own try/except (one malformed section never aborts the others). A benign `Could not chdir to home directory ... Permission denied` banner + a `Init SSL without certificate database` upsc banner are tolerated (parse keys off markers/field-prefixes, not line position).

**Metrics emitted (Option B):**
- SMART whitelist keyed on STABLE Id (survives vendor name-spelling variance вЂ” `Star/Stop_Count` etc.): Ids 5 (reallocated_sectors), 196 (reallocated_event), 197 (current_pending_sector), 198 (offline_uncorrectable), 199 (udma_crc_error), 9 (power_on_hours), 194 (+190 fallback, temperature). Per whitelisted attr: `homelab_synology_smart_attr_raw{disk,attr_id,attr_name}`, `_worst{disk,attr_id}`, `_threshold{disk,attr_id}` (emit-on-presence). Duration-string Raw (e.g. POH `49180h+18m+27.502s`) parsed to float seconds; unparseable в†’ skip that one metric.
- `homelab_synology_smart_attr_failing{disk}` вЂ” count of ANY attr (not just whitelist) with `Status: FAIL` OR value<=threshold; SEEDED-ALERTABLE (emit 0.0 for every disk enumerated from `synodisk --enum`).
- `homelab_synology_smart_disk_present{disk,model}` вЂ” SEEDED 1.0 per enumerated disk (detects a disk dropping out of enumeration = death).
- `homelab_synology_mdstat_array_degraded{array}` вЂ” 0/1 SEEDED-ALERTABLE. Degraded iff a `_` appears among the FIRST n slots of the `[U_...]` block (n = active-count numerator from `[n/m]`) OR a fail marker. CRITICAL: `[12/8] [UUUUUUUU____]` = HEALTHY (12-slot Synology layout, 8 active, trailing `_` are spare slots 9-12, NOT degraded); `[8/8] [UUUUUUUU]` = healthy.
- `homelab_synology_mdstat_resync_progress_percent{array}` + `_resync_speed_kb_per_sec{array}` + `_disks_present{array}` + `_disks_active{array}` вЂ” emit-on-presence (TREND; only during rebuild).
- `homelab_synology_ssh_cpu_temp_celsius` (no label, the package `Physical id 0` Г·1000) вЂ” SEEDED-ALERTABLE single scalar (clean alert target). `homelab_synology_ssh_cpu_core_temp_celsius{core}` (cores 0-5) вЂ” emit-on-presence detail.
- `homelab_synology_ssh_uptime_seconds`, `_load1`, `_load5`, `_load15` вЂ” emit-on-presence (slice off Synology's `[IO:/CPU:]` suffix before parsing load).
- Liveness/up: `up=True` iff connected+ran (exit 0) AND the uptime section parsed a load average. Degraded NAS state (failing SMART / degraded array) is DATA, not probe failure вЂ” bad-state gauges emit with up still True.

**Card-vs-reality deltas (live capture 2026-06-26, /tmp/synology_probe_capture.txt вЂ” Lesson #3):**
- `upsc` does NOT expose input voltage / output voltage / load% / battery date on this APC Smart-UPS 1500-over-USB (the card's premise for `homelab_synology_ups_raw_*` was WRONG). Only battery.charge/runtime/voltage/status/mfr.date present вЂ” and DSM-API `ups.py` ALREADY emits all of those.
- SMART is BLOCK format (`Name:/Id:/Current:/Worst:/Threshold:/Raw:/Status:`, `---`-separated), NOT a table; "Current" not "Value"; Raw can be a duration string.
- `synodisk --enum` field is `>> Tempeture:` (Synology typo); blocks delimited by `************ Disk Info ***************`.
- mdstat md0/md1 use a 12-slot layout `[12/8] [UUUUUUUU____]` that must NOT false-flag as degraded.

**SCOPE-OUT (reality-forced descope, NOT silent): `homelab_synology_ups_raw_*` is DROPPED from the probe.** Rationale: the NUT vars the card assumed (input/output voltage, load%, battery date) do NOT exist on this UPS, and the vars that DO exist (charge/runtime/status) are already emitted by the DSM-API `ups.py` collector вЂ” re-emitting would COLLIDE on identical series names. The `upsc ups` capture STAYS in the forced-command script (future-proofing for UPS models that expose more), but the probe parser emits NOTHING for the UPSC section. No future stage owns UPS-over-SSH (DSM-API ups.py is the sole, sufficient UPS source) вЂ” this is a justified non-build, not a deferral.

**Consolidation decision (user-approved):** REMOVE the existing `uptime-synology` SSH probe (the old admin-account liveness probe). The combined 014 probe covers uptime/df liveness via the low-priv `homelab-probe` user, making the separate admin uptime probe redundant. Build phase removes/retires it.

**Live provisioning DONE in Design (the 004-deferred work):** the low-priv `homelab-probe` DSM user (uid 1030, group users, NOT admin) was created; key `ssh_probe_key_synology` installed; forced-command + restriction VERIFIED live (`hm ssh-probe test synology` в†’ PASS). A DSM-specific blocker was resolved: the inherited home-dir deny-ACL blocks `~/.ssh/authorized_keys`, so authorized_keys (`/etc/ssh/authorized_keys.d/homelab-probe`) AND the script (`/usr/local/bin/hm-probe-synology.sh`) live OUTSIDE the home (root-owned, world-traversable dir 0755 / file 0644), and a global `AuthorizedKeysFile ... /etc/ssh/authorized_keys.d/%u` was added to DSM sshd_config BEFORE the Match blocks. The overrides synology SSH target was corrected to `user: homelab-probe` (was drifted to admin `homelab-monitor`). See memory `reference_synology_ssh_probe_provisioning.md`.
