## Overview

Land the Synology SSH probe BODY on the EPIC-017 framework (the target config + key + host-key pin were set
up in STAGE-008-004). ONE forced-command read-only script, ONE `SshProbe` subclass, ONE 5-min run, emitting:
(1) liveness (`uptime`/`df`) вЂ” the independent liveness anchor (different failure domain than the DSM API),
(2) full per-attribute SMART via `/usr/syno/bin/synodisk --smart_info_get /dev/sdX` for all 8 disks (the raw
~21-attribute table the WebAPI does NOT expose), (3) raw `/proc/mdstat` rebuild/resync progress, and (4) raw
`upsc ups` NUT variables the UPS API omits (input voltage, load %, battery date). All UNPRIVILEGED (no sudo).

## Concrete deliverables

### 1. Combined forced-command read script

A single read-only script (installed under `homelab-probe` via the STAGE-008-004 install-instructions вЂ”
update that install body here to the real combined script) that emits, in one invocation, a
parse-friendly combined output of: `uptime` + `df`, `/usr/syno/bin/synodisk --enum`,
`/usr/syno/bin/synodisk --smart_info_get /dev/sd{a..h}`, `cat /proc/mdstat`, `/usr/bin/upsc ups`, and CPU
temp from hwmon sysfs. All unprivileged.

### 2. `SshProbe` subclass + parsers

`ssh_probe.py` (Synology bundle): an `SshProbe` subclass that runs the forced command via
`ctx.ssh.open("synology")`, parses the combined output, and emits:
- `homelab_ssh_up{target=synology}` + `homelab_ssh_probe_duration_seconds` + `_last_success_age_seconds` +
  `_host_key_mismatch` (EPIC-017 surface).
- `homelab_synology_smart_attr{disk,attribute}` (raw per-attribute SMART: value/worst/threshold/raw/status
  вЂ” the ~21 attributes Г— 8 disks) + a `_smart_attr_failing{disk}` rollup.
- `homelab_synology_mdstat_*` (array state, rebuild/resync `progress_percent`, speed) вЂ” raw RAID from the
  kernel.
- `homelab_synology_ups_raw_*` (input voltage, load %, battery date вЂ” the NUT vars the API omits).
- `homelab_synology_ssh_cpu_temp_celsius` (hwmon).

### 3. Tests

3a parser unit tests against CAPTURED fixtures of each tool's real output (the recon formats: `synodisk
--smart_info_get` attribute blocks, `synodisk --enum`, `/proc/mdstat` `[8/8] [UUUUUUUU]`, `upsc` key:value).
3b real end-to-end probe run.
