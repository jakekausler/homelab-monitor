- Real lines (verbatim): `<14>Jun 27 09:13:46 NAS System Test message from Synology Syslog Client from (70.229.192.120)` and `<14>Jun 27 09:19:40 NAS Connection: User [jakekausler] from [2600:387:15:700e::8] signed in to [DSM] successfully via [amfa-email].`
- **Format = RFC 3164 (BSD).** Every line `<PRI>Mmm dd HH:MM:SS HOSTNAME CATEGORY[: ]message`. All lines have `<PRI>`; no doubled hostname; single `NAS` host.
- **CORRECTION 1 ‚Äî facility is useless for bucketing.** ALL lines are PRI=14 (facility=1/user, severity=6/info) regardless of event type. The brainstormer's C3 "facility floor" is INVALID. Bucket by the CATEGORY WORD instead.
- **CORRECTION 2 ‚Äî the TAG is a human-readable CATEGORY word, not a Unix program name.** Seen: `System` (test heartbeat), `Connection` (DSM login). DSM Log Center also emits (per docs, to be confirmed by richer capture in Build/Refinement): `Connection`/`File`/`Package Center`/`Storage Manager`/`Application`/`Backup`/`Network`/`Surveillance Station`/etc.
- **CORRECTION 3 ‚Äî inconsistent category delimiter.** `Connection:` has a COLON after the category word; `System Test message‚Ä¶` has NO colon (space-delimited). The parser MUST handle BOTH: if a `:` follows the post-hostname token, category = token before `:`; else category = first whitespace token after the hostname.
- **CORRECTION 4 ‚Äî source IP is NOT .4 and is NOT stable.** DSM forwarded from 192.168.2.5; the NAS is multi-NIC (.4/.5/.6/.7). DECISION (user-confirmed): the source is NOT filtered by IP at all ‚Äî the dedicated PORT 5515 is the sole discriminator (only DSM is configured to send to 5515; UDM uses 5514). This is robust to NIC/re-IP changes. Trust model = identical to the existing UDM 5514 0.0.0.0 exposure (trusted homelab LAN).

**DECISION A ‚Äî Port (A1, dedicated): prod `0.0.0.0:5515/udp`, dev `127.0.0.1:15515/udp`.** New `[sources.synology_syslog]` socket source, mirroring the UDM block (same `profiles: ["host-collectors"]` gate so instance B omits it). Rejected A2 (share 5514 + branch by source IP) ‚Äî couples two devices into one mega-transform, and the multi-NIC reality (Correction 4) would have broken IP-dispatch.

**DECISION B ‚Äî Source type + parse (B1, lossless): `type="socket"` UDP + VRL `parse_syslog()` happy path + manual `parse_failed=1` fallback to `service="synology-other"`.** Honors the locked "nothing silently dropped / absence is hostile" philosophy (vector's `type="syslog"` source silently drops unparseable datagrams ‚Äî rejected). Because DSM emits standard RFC 3164, `parse_syslog()` covers the happy path (simpler than UDM's hand-rolled CEF regex); only the fallback bucket is hand-written. A `synology_parse_failed_metric` (mirroring `udm_parse_failed_metric`) is tapped BEFORE the throttle and wired into the redaction-metrics sink so dropped/unrecognized lines are visibly counted (satisfies the coverage-map requirement).

**DECISION C ‚Äî Bucketing (CATEGORY-WORD mapping, corrected from C3‚ÜíC2 by Correction 1+2).** After `parse_syslog()`, take the message and split the category word per Correction 3 (colon-or-space). Map a small, bounded, CLOSED set of category words ‚Üí `service="synology-<bucket>"` (case-insensitive, ordered):
  - `Connection` ‚Üí `synology-auth` (login/auth ‚Üí 023 failed-login-burst + abnormal-login)
  - `Storage Manager` / SMART-bearing categories ‚Üí `synology-smart` (‚Üí 023 SMART-event)

  - parse-failure / no-category fallback ‚Üí `synology-other` (`parse_failed=1`)
  - (`synology-surveillance` ‚Äî DEFERRED unless a richer capture shows `Surveillance Station` lines worth isolating; otherwise folds into `synology-system`.
