# STAGE-008-020: DSM syslog source + service labels + redaction + DSM-side enable + coverage map

## Status: Not Started

## Overview

Build the DSM-syslog → vector → VictoriaLogs events pipeline, mirroring EPIC-007's UDM syslog stage (the
METHOD mirrors Unifi; DSM emits standard RFC syslog, NOT Unifi's CEF — verify the live format at Design, the
parser differs). This is the SOLE log path for the Synology (Amendment 5: no API log polling). Forward all
logs DSM can emit, label them `service="synology-<facility>"` so the per-`service` Drain models partition and
the `<LogViewer>` filter works, add Synology-specific redaction patterns, document the DSM-side remote-syslog
enable, and record the syslog-vs-metric coverage map (Amendment 5/Q10b — what forwards vs. is covered by
metrics; nothing silently dropped).

**Vertical slice:** after this stage DSM forwards syslog → vector ingests it → VL has `service="synology-*"`
streams with redacted content; a coverage map documents which categories arrive via syslog vs. metrics.

**Spec references:** §4.3/§6.3 (logs pipeline; syslog forwarding). EPIC-004 (vector→VL, per-`service` Drain,
redaction pipeline, `<LogViewer>` contract). EPIC-007 STAGE-007-016 (UDM syslog source — the structural
precedent; DSM differs in RFC format). EPIC-008 Amendment 5 + Q10b coverage map.

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- **No API log polling** — this is the only Synology log path; single source per signal.
- `service="synology-<facility>"` labels so Drain models partition + the LogViewer facility filter works.
- Synology-specific redaction (DSM API tokens, session cookies, photo tokens) applied at ingest BEFORE VL.
- Syslog source on a non-privileged port (avoid root-bind), like the UDM source.
- Coverage map recorded: every category DSM forwards, and every signal NOT forwardable (covered by metrics
  instead) — no silent gaps.

## Concrete deliverables

### 1. vector Synology syslog source + VRL transform

Add an RFC-syslog `source` for the Synology (verify UDP/TCP + RFC 3164/5424 format live at Design) + a VRL
transform extracting fields and writing to VL with `service="synology-<facility>"` (e.g. `synology-auth` /
`synology-smart` / `synology-package` / `synology-system`). Mirror the UDM source structure
(`deploy/vector/...`), adapted to DSM's format.

### 2. Synology redaction patterns

Add DSM-specific patterns to `logs.redact:` (API tokens, session cookies, photo tokens) so DSM syslog is
redacted at ingest before reaching VL.

### 3. DSM-side enable + coverage map

Document the DSM Control Panel → Log Center → Log Sending steps (target `192.168.2.148`, all categories) in
`docs/integrations/synology.md`. Record the **syslog-vs-metric coverage map**: which log categories DSM
forwards (→ VL signals, STAGE-008-023) vs. which are not forwardable and are covered by the metric collectors
(Waves B/C/E) instead.

### 4. Validation

vector config golden/validation test; live ingest confirmed (`service="synology-*"` returns hits in VL after
the DSM-side enable) — in instance A if needed.
