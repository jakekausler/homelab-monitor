# STAGE-008-024: Backend panel data endpoints (Synology + Surveillance)

## Status: Not Started

## Overview

Build the backend data endpoints that feed the two UI pages, BEFORE the frontend stages (the backend-contract-
verification discipline: the data endpoint exists + returns real data before the widget is designed). Typed
response models (mirroring EPIC-005/007 `*Row[]` patterns) for: Synology summary/detail (storage, disks/SMART,
pool/RAID, system/temps, UPS, backup/replication, updates/packages, security, connections, mount-health,
SSH-probe state) and Surveillance summary/detail (cameras, events/recordings, license, HomeMode). Read from
VM per-series gauges (+ live DSM re-query where a field isn't in VM, e.g. the active-connection who-list).

**Vertical slice:** after this stage `GET /api/integrations/synology/summary` (+ detail) and
`GET /api/integrations/surveillance/summary` (+ detail) return typed rows from real data; OpenAPI +
`schema.ts` regenerated.

**Spec references:** §9.2 (Integrations panel = plugin-provided panel; data endpoint). EPIC-005 STAGE-005-021
+ EPIC-007 STAGE-007-019 (backend panel data endpoints — the precedent).

## Stage Flags

- Has Input Forms: [ ] No
- Touches secrets / production data: [ ] No (reads VM + live DSM re-query)

  unit-tested against fake VM/client) + 3b (real responses against the live stack). NO user viewport prompt
  (it's the backend feeding the panels).

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- Typed response models (no bare `Any`); OpenAPI + `schema.ts` regenerated (the openapi-export +
  generate-ui-types hooks fire — regenerate during Build so commit-time hooks are a no-op).
- Active-connection who-list = live DSM re-query at view-time (not stored series; privacy).
- Observe-only — read endpoints only.
- Mount-health + SSH-probe-state fields included so the widgets (026/027) can render them.

## Concrete deliverables

### 1. Synology data endpoints

`GET /api/integrations/synology/summary` (header-strip + section rollups) + detail endpoints (per-disk/SMART
rows, per-volume, pool/RAID, backup jobs, replication, packages, security findings, active connections,
mount rows, SSH-probe state). Typed rows from VM (+ live re-query for connection who-list).

### 2. Surveillance data endpoints

`GET /api/integrations/surveillance/summary` + detail (per-camera status/events/recording rows, license,
HomeMode).

### 3. OpenAPI + tests

Regenerate `openapi.json` + `schema.ts`. 3a endpoint tests against fake VM/client; 3b real responses.
