# STAGE-008-015: Surveillance camera collector

## Status: Not Started

## Overview

The first Surveillance Station collector: pull `SYNO.SurveillanceStation.Camera List` (+
`SYNO.SurveillanceStation.Info GetInfo`) and emit per-camera status/identity/recording-config. Recon: 3
cameras (id1 Driveway `192.168.2.103`, id2 Backyard `.99`, id3 Doorbell `.215`), all added as generic ONVIF;
`status=1` = ok. The `Camera.Status` v3 read method was NOT found, so camera online/offline is derived from
`Camera.List.status` (here) + `SS.Log` connection-lost/restored events (via syslog→VL, STAGE-008-023).

**Vertical slice:** after this stage `homelab_synology_ss_camera_*` (status, identity labels, recording
config) + `homelab_synology_ss_camera_count` + license-vs-camera are emitted for the 3 live cameras at 60s.

**Spec references:** §6.2, §9.2. EPIC-008 "Verified deployment reality" (SS Camera List shape; status=1=ok;
Camera.Status method absent).

## Concrete deliverables

### 1. Camera collector

`ss_camera.py`: calls `client.ss_camera_list()` (+ `ss_info()`), emits:
- `homelab_synology_ss_camera_status{camera,model,ip}` (1 = ok, from `status`),
  `homelab_synology_ss_camera_enabled{camera}`, `_recording_keep_days{camera}`, `_fps{camera}`,
  `_resolution{camera,resolution}` (label).
- `homelab_synology_ss_camera_count`, `homelab_synology_ss_license_used`/`_total`/`_max` (from `ss_info` /
  cross-checked in STAGE-008-017), `homelab_synology_ss_version{version}` (label).

### 2. Tests

3a fixture (the recon 3-camera shape; one synthetic offline camera `status!=1`); 3b live emission.
