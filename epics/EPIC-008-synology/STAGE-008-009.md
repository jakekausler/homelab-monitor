# STAGE-008-009: UPS collector + SystemHealth rule status

## Status: Complete

## Overview

Collect the UPS telemetry (`SYNO.Core.ExternalDevice.UPS` `get` — APC Smart-UPS 1500 over USB) and the DSM
overall-health `rule` (`SYNO.Core.System.SystemHealth` `get` — the active health condition, e.g.
`storage_is_attention`). UPS on-battery / low-battery is a fast-react critical signal; the SystemHealth rule
is a ready-made overall-status signal DSM computes for us.

**Vertical slice:** after this stage `homelab_synology_ups_*` (charge%, runtime, status enum, model labels)
+ `homelab_synology_health_rule{rule}` are emitted for the live NAS at 60s.

**Spec references:** §6.2. EPIC-008 "Verified deployment reality" (UPS + SystemHealth shapes).

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- `synology` concurrency group; self-metrics + API latency; 60s (catch on-battery fast).
- Graceful degrade if no UPS attached (emit a "no UPS" signal rather than failing).
- The UPS API omits load%/voltage — those richer raw NUT vars come from the SSH `upsc` probe (STAGE-008-014);
  this collector emits what the API gives.

## Concrete deliverables

### 1. UPS + health collector

`ups_health.py`: calls `client.ups_get()` + `client.system_health()`, emits:
- UPS: `homelab_synology_ups_charge_percent`, `_runtime_seconds`, `_status{state}` (from
  `usb_ups_status_*` enum → on-battery/online/low-battery booleans), `_connected` (1/0),
  `homelab_synology_ups_info{model,manufacture}`. Graceful "no UPS" when absent.
- Health: `homelab_synology_health_rule{rule}` (1 = the active rule id, e.g. `storage_is_attention`;
  absent/none = healthy), `_health_priority` if present.

### 2. Tests

3a fixtures: UPS online, UPS on-battery, UPS low-battery, no-UPS-attached; SystemHealth with-rule + healthy.
3b live emission.

## Design Phase

**Status**: [x] Design (Complete)

**Design Notes** (selected 2026-06-23; brainstormer used for the multi-fetch + status-representation + rule-of-three choices, recommendations adopted). LAST stage of Wave B.

Grounded against live-captured payloads (`/tmp/synology_ups_raw.json`, `/tmp/synology_system_health_raw.json`). Card field-name corrections vs ground truth: UPS `charge`(int %)/`runtime`(int seconds)/`status`(str "usb_ups_status_online")/`usb_ups_connect`(bool)/`model`+`manufacture`(str, BOTH with a TRAILING `\n` → must `.strip()`); DSM API does NOT surface load%/voltage (those come from the SSH `upsc` probe in STAGE-008-014 — out of scope here, named on that stage). SystemHealth `rule` is a dict-OR-NULL: ABSENT/null = the NAS is HEALTHY; PRESENT = a concern with `rule.id`(str "storage_is_attention")/`rule.priority`(float)/`rule.type`(int). `uptime` is NOT re-emitted (007's system collector already emits `homelab_synology_system_uptime_seconds`); `interfaces` skipped (network info, not health).

TWO-API multi-fetch (mirrors 007's `_fetch` wrapper) — but CO-EQUAL combine, NOT 007's primary-gated:
- **Combine semantic (Q5):** `ups_get()` and `system_health()` are co-equal independent concerns (power vs health). `ok=False` ONLY if BOTH fetches fail; `ok=True` if at least one succeeds (`ok = ups_resp is not None or health_resp is not None`). Documented as the co-equal variant (007 was primary-gated because system_info was load-bearing identity).

Decisions (theme: STATE-SET for fidelity + always-present BOOLEAN/GAUGE for alertability):
- **UPS status:** state-set `homelab_synology_ups_status{state}=1.0` (active state only, stale-series-stops like 005's volume_status) via a local `_ups_status_state(v)` that `.strip()`s + removeprefix's `usb_ups_status_` → bare state ("online"/"on_battery"/"low_battery"/"charging"), None for non-str. PLUS derived always-present alertable booleans `homelab_synology_ups_on_battery` and `homelab_synology_ups_low_battery` (1.0 when the normalized state matches, else 0.0; emitted whenever the state is parseable).
- **UPS scalars:** `homelab_synology_ups_charge_percent` = `as_float(charge)`, `homelab_synology_ups_runtime_seconds` = `as_float(runtime)` (emit-if-present).
- **UPS connected (intentional asymmetry):** `homelab_synology_ups_connected` ALWAYS emitted — `bool_to_gauge(usb_ups_connect)` when present, else `0.0`. This is the clean "is there a UPS / UPS lost" signal (==0 distinguishable from collector-down); charge/runtime/status stay emit-if-present. Documented as a deliberate divergence from the pure emit-if-present convention.
- **UPS identity:** `homelab_synology_ups_info{model,manufacture}=1.0` — anchor on a `.strip()`'d non-empty `model` (skip whole metric if absent); add `manufacture` label only if its stripped value is non-empty; via a local `_strip_label(v)`.
- **SystemHealth (null-means-healthy inversion):** detect via `as_dict(payload.get("rule"))` — None → HEALTHY → `homelab_synology_health_ok=1.0` (always-present, the alertable backbone), emit nothing else. PRESENT → `health_ok=0.0` + `homelab_synology_health_rule{rule=<rule.id>}=1.0` state-set (the specific concern's id, str non-empty) + `homelab_synology_health_priority = as_float(rule.priority)` (rule-scoped — emit only when a rule is present).
- **Cap-routing:** unchanged `capped_emitter`/`cap_for_synology`/`family(name,obs)` closure (all families single-or-low-series; far under 500).
- **Helpers:** import `as_dict`/`as_float`/`bool_to_gauge` + `client_unconfigured_result`/`fetch_or_result`/`cap_for_synology`/`capped_emitter` from `_shared` (NOT `nested`/`as_list_of_dicts` — flat payloads, one-level `rule` dict). New locals: `_ups_status_state`, `_strip_label`, and a LOCAL COPY of 007's `_fetch` multi-fetch wrapper (doc-comment noting 007 as origin).

RULE-OF-THREE RESOLUTION (the handoff predicted 009 would trigger a shared status-mapper extraction — it does NOT): the three existing per-collector mappers (`_status_str_to_gauge` storage / `_raid_status_to_gauge` pool / `_yesno_to_gauge` system) each have ONE consumer and operate on DIFFERENT value-domains; 009's transforms diverge AGAIN (`_ups_status_state` returns a LABEL not a gauge; the health-rule handling is inline presence-checking). No convergent shape to extract → mappers stay LOCAL (consistent with the 003/007 "diverging mappers stay local" principle). The `_fetch` wrapper is a legitimate 2nd-consumer candidate but 009 composes it with a DIFFERENT combine semantic (co-equal vs primary-gated) → kept a documented LOCAL COPY; the extraction trigger is named: a 3rd multi-fetch collector (potentially the 015-017 surveillance stages).

SCOPE (no silent deferral): UPS load%/input-voltage/output-voltage are NOT in the DSM API → owned by STAGE-008-014 (SSH `upsc` probe), named there. UPS `enable`/`delay_time`/`shutdown_device`/`snmp_*`/`ACL_*` are NAS-side config, not live metrics → intentionally not emitted (YAGNI, no downstream consumer). `interfaces`/`uptime`/`hostname` from SystemHealth not emitted (uptime owned by 007; interfaces/hostname are not health signals). All card metrics ARE emitted (`ups_charge_percent`/`_runtime_seconds`/`_status{state}`/`_connected`/`_info{model,manufacture}`, `health_rule{rule}`, `health_priority`); `health_ok`, `ups_on_battery`, `ups_low_battery` are additive alertability improvements over the card.

Files: NEW `apps/monitor/homelab_monitor/plugins/collectors/integrations/synology/ups.py`; EDIT `synology/__init__.py` (import + add `SynologyUPSCollector` to `_SYNOLOGY_COLLECTORS`); NEW `apps/monitor/tests/test_synology_ups_collector.py` (mirror 007's multi-fetch test, 100% BRANCH coverage — fixtures: UPS online / on-battery / low-battery / charging / no-UPS-or-empty + SystemHealth with-rule / healthy-null; multi-fetch edge: ups-fails-health-ok / health-fails-ups-ok / BOTH-fail→ok=False / both-succeed; non-str/absent field skips; `_strip_label` None branch; `_ups_status_state` non-str + each-state branches; client-unconfigured ok=False; non-dict payload).
