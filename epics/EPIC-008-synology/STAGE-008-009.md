# STAGE-008-009: UPS collector + SystemHealth rule status

## Status: Not Started

## Overview

Collect the UPS telemetry (`SYNO.Core.ExternalDevice.UPS` `get` — APC Smart-UPS 1500 over USB) and the DSM
overall-health `rule` (`SYNO.Core.System.SystemHealth` `get` — the active health condition, e.g.
`storage_is_attention`). UPS on-battery / low-battery is a fast-react critical signal; the SystemHealth rule
is a ready-made overall-status signal DSM computes for us.

**Vertical slice:** after this stage `homelab_synology_ups_*` (charge%, runtime, status enum, model labels)
+ `homelab_synology_health_rule{rule}` are emitted for the live NAS at 60s.

**Spec references:** §6.2. EPIC-008 "Verified deployment reality" (UPS + SystemHealth shapes).

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- `synology` concurrency group; self-metrics + API latency; 60s (catch on-battery fast).
- Graceful degrade if no UPS attached (emit a "no UPS" signal rather than failing).
- The UPS API omits load%/voltage — those richer raw NUT vars come from the SSH `upsc` probe (STAGE-008-014);
  this collector emits what the API gives.

## Concrete deliverables

### 1. UPS + health collector

`ups_health.py`: calls `client.ups_get()` + `client.system_health()`, emits:
- UPS: `homelab_synology_ups_charge_percent`, `_runtime_seconds`, `_status{state}` (from
  `usb_ups_status_*` enum → on-battery/online/low-battery booleans), `_connected` (1/0),
  `homelab_synology_ups_info{model,manufacture}`. Graceful "no UPS" when absent.
- Health: `homelab_synology_health_rule{rule}` (1 = the active rule id, e.g. `storage_is_attention`;
  absent/none = healthy), `_health_priority` if present.

### 2. Tests

3a fixtures: UPS online, UPS on-battery, UPS low-battery, no-UPS-attached; SystemHealth with-rule + healthy.
3b live emission.
