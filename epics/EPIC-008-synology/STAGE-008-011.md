# STAGE-008-011: Snapshot Replication collector (graceful-empty)

## Status: Complete

## Overview

Collect Btrfs snapshot + Snapshot Replication state: per-share snapshot count/age (`SYNO.Core.Share.Snapshot
list` — param is `name=<share>`, NOT `share_name`) and replication-task lag (`SYNO.Btrfs.Replica.Core`).
Recon found **no snapshots and no replication configured**, and `Btrfs.Replica.Core` had no working read
method among the guessed names — so this stage's Design must nail down the correct read method live (or
document that replication is absent and the collector ships graceful-empty until configured).

**Vertical slice:** after this stage `homelab_synology_snapshot_*` (per-share count/age) +
`homelab_synology_replication_*` (lag/status, when present) are emitted, degrading cleanly to empty for the
current no-snapshot/no-replication state.

**Spec references:** §6.2. EPIC-008 "Verified deployment reality" (Share.Snapshot empty, `name=` param;
Btrfs.Replica.Core read method TBD).

### 1. Snapshot + replication collector

`replication.py`:
- Snapshots: iterate configured shares, call `client.share_snapshot_list(name=<share>)`, emit

- Replication: call the confirmed `Btrfs.Replica.Core` read method, emit
  `homelab_synology_replication_status{task,state}`, `_last_sync_timestamp{task}`,
  `_lag_seconds{task}`, direction/target labels. Graceful-empty when no tasks.

## Design Phase

**Status**: [x] Design (Complete)

**Design Notes**:

Co-equal graceful-empty multi-fetch collector (mirrors `backup.py`/STAGE-008-010). New file
`plugins/collectors/integrations/synology/replication.py`. Cadence 5m (interval=300s), timeout 30s,
`concurrency_group="synology"`. Local `_fetch` wrapper + co-equal combine
(`ok = <any fetch succeeded>`); always calls `_emit`.

**Live recon (ground truth — RESOLVED the open replication-method question):**
- **Snapshots WORK.** `client.share_snapshot_list(name=<share>)` helper EXISTS:
  `SYNO.Core.Share.Snapshot` v2 `list`, param `name=` (REQUIRED — calling without it → DSM error 403).
  14 shares discovered live via `SYNO.Core.Share list` (Audiobooks, Books, Calibre, Comics, Documents,
  Games, Hidden, homes, HomeVideos, Movies, Pictures, Surveillance, TV, YouTube). Per-share snapshot
  payload `data` keys: `["snapshots", "total"]`; currently `total=0` for all shares (no snapshots
  configured → graceful-empty). Per-snapshot field names UNKNOWN (no snapshots live). Saved:
  `/tmp/synology_snapshot_raw.json`, `/tmp/synology_shares_raw.json`.
- **Btrfs replication has NO working read method (provably).** `SYNO.Btrfs.Replica.Core` exists in the API
  registry (maxVersion=1) but BOTH `list` and `get` return ERR:103 (unknown method) — the Snapshot
  Replication PACKAGE IS NOT INSTALLED, so the methods are stubs. `SYNO.SnapshotReplication.*` is entirely
  absent (ERR:102). The only replication API returning data is `SYNO.Core.ISCSI.Replication v1 list` →
  `{"tasks":[]}` — iSCSI LUN replication (a DIFFERENT feature, also empty). Probe table + API.Info
  enumeration saved: `/tmp/synology_api_info_replication.json`, `/tmp/synology_replication_raw.json`.

**Selected design (brainstormer options; coordinator selected per pre-authorization):**
1. **Share enumeration:** add a new `share_list()` client helper (`SYNO.Core.Share list`); enumerate shares
   each tick, loop `share_snapshot_list(name=<share>)` per share — all `_fetch`-wrapped co-equally
   (`ok = share_list succeeded OR any per-share snapshot fetch succeeded`). EMIT
   `homelab_synology_snapshot_count{share}=0` for EVERY enumerated share (do NOT skip empties — Wave-B
   "absence is hostile to alerting": a rule must be able to fire on "share X lost its snapshots"). API load:
   1 + 14 = 15 calls/5m tick (~0.05 req/s avg — trivial, well under the 30s timeout). Cardinality: 14 shares
   × 2 families = 28 series steady-state (far under the 500 cap).
2. **Btrfs replication SCOPED OUT** (package not installed → no read method → nothing to collect). Emit a
   single SELF-CORRECTING sentinel `homelab_synology_replication_available` (no labels): live-probe
   `Btrfs.Replica.Core list` (v1) inside a `_fetch` wrapper, set =1.0 ONLY if it returns a non-error dict
   payload, else =0.0 (currently 0.0 — ERR:103). This makes the absence alertable AND auto-flips to 1 if the
   package is ever installed. NO per-task replication families until then. This is NOT a silent deferral: a
   REGRESSION item (added in Refinement) names the trigger — "install Snapshot Replication package +
   confirm `SYNO.Btrfs.Replica.Core list` returns data, then add per-task replication families
   (status/last_sync/lag/direction/target)". Adding a new client helper `replica_core_list()`
   (`SYNO.Btrfs.Replica.Core list` v1, tagged scaffolding) for the probe.
3. **Snapshot field-name strategy:** mirror `backup.py` — a `_SNAP_FIELDS` constant (logical→inferred DSM
   key) + a local epoch-or-`"%Y-%m-%d %H:%M:%S"` `_parse_dsm_time` (copy backup.py's). Inferred per-snapshot
   fields (UNVERIFIED — 0 snapshots live): `time` (creation epoch/string), `create_time` (alt key),
   `snapshot` (the @GMT-… id), `desc`, `lock`. `latest_age_seconds{share} = max(0, now - max(parsed times))`;
   emit count even when no time field parses (skip only the age family). Field verification rides the same
   regression item (when a real snapshot exists).
4. **Always-emitted alertable signals:** per-share `homelab_synology_snapshot_count{share}` (0 included) +
   per-share `homelab_synology_snapshot_latest_age_seconds{share}` (only when a time parses) + a single
   no-label `homelab_synology_snapshots_configured` (1.0 if ANY share has ≥1 snapshot, else 0.0 — the
   top-level sentinel, mirrors backup.py's `no_backup_configured`; emit 0 even if the enumerate fetch fails).
   No redundant `snapshot_total_count` (derivable via `sum()` at query time — YAGNI).

All follow the locked Wave-B alerting philosophy + the "backup/replication built fully now, graceful-empty"
epic amendment (replication is graceful-empty via the documented-absent sentinel + tracked regression item,
which honors the amendment given the package is genuinely not installed).

**Known limitations (tracked, NOT silent deferrals):** (a) per-snapshot field names inferred (0 snapshots
live); (b) Btrfs replication per-task collection deferred (package not installed). BOTH owned by a single
regression item (added in Refinement) naming the triggers.

**Files:** `replication.py` (new) + new client helpers `share_list()` + `replica_core_list()` in
`kernel/synology/client.py` (tagged `# SCAFFOLDING: consumed in STAGE-008-011`) + add
`SynologyReplicationCollector` to `_SYNOLOGY_COLLECTORS` in `synology/__init__.py` +
`tests/test_synology_replication_collector.py` (new) + add `_EXPECTED_REPLICATION_INTERVAL = 300` +
`test_register_all_registers_replication` to `tests/test_synology_integration_bundle.py`. No migration /
openapi / frontend.

**Brainstormer:** used (real architectural choices — replication-method-absence + share enumeration).
Coordinator selected all four recommendations; deferral tracked via regression item (not a new sub-stage)
to preserve the clean 001–031 numbering.

## Build Phase

**Status**: [x] Build (Complete)

**Build Notes**:

Implemented `SynologyReplicationCollector` in `plugins/collectors/integrations/synology/replication.py`
(co-equal graceful-empty multi-fetch, mirrors `backup.py`). Three-part fetch (all `_fetch`-wrapped):
(1) `share_list()` enumerate shares, (2) per-share `share_snapshot_list(name=<share>)` loop (each isolated —
one share's fetch failing skips that share, NOT the run), (3) `replica_core_list()` self-correcting
replication probe. Co-equal combine: ok=False only if ALL fetch groups fail.

4 metric families: `homelab_synology_snapshot_count{share}` (always per share, 0 included),
`homelab_synology_snapshot_latest_age_seconds{share}` (emit-if-a-time-parses), `homelab_synology_snapshots_configured`
(no-label sentinel, 1.0 if any share has ≥1 snapshot), `homelab_synology_replication_available` (no-label
self-correcting sentinel: 1.0 only if the `Btrfs.Replica.Core list` probe returns a non-error dict payload,
else 0.0 — currently 0.0, ERR:103 package-not-installed). Centralized `_SNAP_FIELDS` (inferred DSM keys) +
local `_parse_dsm_time` (epoch-or-`"%Y-%m-%d %H:%M:%S"`); `_latest_snapshot_time` reads `time` then falls
back to `create_time`, MAX across the share's snapshots. **Deliberate choice:** a failed per-share snapshot
fetch SKIPS that share (does NOT emit `count=0`) — emitting 0 would be a false "no snapshots" when the fetch
actually failed.

Added client helpers `share_list()` (`SYNO.Core.Share list` v1) + `replica_core_list()`
(`SYNO.Btrfs.Replica.Core list` v1) to `kernel/synology/client.py` (tagged
