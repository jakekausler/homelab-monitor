# STAGE-008-004: SSH `homelab-probe` dedicated-user target config on the EPIC-017 framework

## Status: Complete

## Overview

Configure the Synology as an SSH probe target on the EPIC-017 framework, using a NEW dedicated low-privilege
DSM user `homelab-probe` (`dedicated-user` account-mode, forced-command, **no sudoers**). This stage lands
the target config + key + host-key pin + install instructions + restriction verification ‚Äî the framework
plumbing. The actual probe BODY (the combined read script + the Python `SshProbe` subclass) lands in Wave D
(STAGE-008-014). Recon proved everything the probe needs reads UNPRIVILEGED (`synodisk`, `/proc/mdstat`,
`upsc`, `df`, hwmon), so no sudo is granted.

**Vertical slice:** after this stage `ssh_targets:` declares the `synology` target (`dedicated-user` mode,
host `192.168.2.4`, port `53197`, key-secret `ssh_probe_key_synology`, pinned host key); `hm ssh-probe
keygen/capture-hostkey/install-instructions/test synology` work end-to-end; the install-instructions output
creates the `homelab-probe` low-priv user + installs the (placeholder/trivial) forced-command + the hardened
`authorized_keys` line, and `test` verifies the forced-command restriction holds (an arbitrary command is
refused).

**Spec references:** spec ¬ß2 Q29 / ¬ß3.1 (`SshClientFactory`, per-target scoped users). EPIC-017 (the
framework; `dedicated-user` mode; `hm ssh-probe` CLI; host-key pinning; forced-command-as-boundary) +
EPIC-017 STAGE-017-005/006 (install-instructions + the `uptime` exemplar) are the precedent. EPIC-008
Amendments 3 (no sudoers) + 4 (separate `homelab-probe` low-priv user).

## Concrete deliverables

### 1. `ssh_targets:` Synology target config

Declare the `synology` target in the overrides config (public default empty): `account_mode=dedicated-user`,
`host=192.168.2.4`, `port=53197`, `user=homelab-probe`, key-secret `ssh_probe_key_synology`, pinned host
key, the forced-command/script reference (placeholder here; real body in 008-014), and the per-target
concurrency group. Pydantic-validated via the EPIC-017 `ssh_targets` model.

### 2. Key + host-key provisioning

`hm ssh-probe keygen synology` ‚Üí ed25519 key in the secret store (`ssh_probe_key_synology`), public key
printed. `hm ssh-probe capture-hostkey synology` ‚Üí pin the Synology host key.

### 3. Install instructions + restriction verification

`hm ssh-probe install-instructions synology` (dedicated-user mode) ‚Üí create the `homelab-probe` low-priv DSM
user, install the forced-command script (placeholder/trivial body this stage), add the hardened
`authorized_keys` line (`command="..." + no-port-forwarding,no-pty,...`), **no sudoers**. `hm ssh-probe test
synology` ‚Üí connect, run the forced command, verify an arbitrary command is refused (the restriction holds).

### 4. Tests

3a unit tests for the CLI flows against the framework with a fake target; 3b live provisioning + `test`
verification on the real Synology.
