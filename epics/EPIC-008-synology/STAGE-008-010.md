# STAGE-008-010: Hyper Backup collector (graceful-empty + NoBackupConfigured)

## Status: Not Started

## Overview

The epic's single most important collector: pull Hyper Backup job status (`SYNO.Backup.Task list` +
`SYNO.Backup.Repository list`) and emit per-job last-run/result/size/next-run so the `BackupFailed` /
`BackupMissing` rules (STAGE-008-022) can fire. False negatives are unacceptable. Recon found HyperBackup
**installed + running but with 0 configured jobs**, so this collector must handle the empty case gracefully
AND emit a distinct `SynologyNoBackupConfigured` signal (running-but-zero-tasks is itself worth knowing).

**Vertical slice:** after this stage `homelab_synology_backup_*` is emitted per job (none live yet) and
`homelab_synology_backup_configured_count` + a `no_backup_configured` signal reflect the empty state; when
the user adds a Hyper Backup job, it lights up with no code change.

**Spec references:** §6.2. EPIC-008 cross-stage criterion ("backup-status is the single most important
rule; test the empty / failed / zero-size-but-ran cases"). EPIC-008 "Verified deployment reality" (Backup.Task
list empty; the per-job field set for when jobs exist).

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- `synology` concurrency group; self-metrics + API latency; 5m.
- **False negatives unacceptable** — test the empty case, the failed case, and the ran-but-zero-size case.
- Graceful-empty: 0 jobs → `homelab_synology_backup_configured_count=0` + `no_backup_configured=1`, NOT a
  collector failure.
- All timestamps UTC (last-run / next-run epochs → UTC).

## Concrete deliverables

### 1. Hyper Backup collector

`backup.py`: calls `client.backup_task_list()` (+ `backup_repository_list()`), emits per job:
- `homelab_synology_backup_last_result{job,result}` (success/fail), `_last_run_timestamp{job}` (UTC),
  `_last_run_age_seconds{job}`, `_next_run_timestamp{job}`, `_data_size_bytes{job}`, `_status{job,state}`
  (backuping/waiting/error/...), `_enabled{job}`.
- Aggregate: `homelab_synology_backup_configured_count`, `homelab_synology_no_backup_configured` (1 when
  HyperBackup running + 0 tasks), `homelab_synology_backup_repository_count`.
- **Zero-size guard:** expose `_data_size_bytes` so a "ran but produced 0 bytes" rule is possible
  (STAGE-008-022 owns the rule; this collector ensures the field is present even when 0).

### 2. Tests

3a fixtures: empty (the live state), one-job-success, one-job-failed, ran-but-zero-size, multi-job mixed.
3b live emission (empty/`NoBackupConfigured` path; if the user has added a job by then, validate the populated
path too).
