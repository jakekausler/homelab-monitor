# STAGE-008-005: Storage collector (volumes + disks + SMART status)

## Status: Not Started

## Overview

The first real collector: pull `SYNO.Storage.CGI.Storage` method `load_info` (ONE call = volumes + disks +
pools + caches) and emit per-volume and per-disk metrics. This is the workhorse storage signal — volume
usage/status and per-disk health/temperature. (The pool/RAID slice of the same response is its own collector
in STAGE-008-006 to keep each file single-purpose; both read the same one cached fetch via the shared
client.)

**Vertical slice:** after this stage `homelab_synology_volume_*` (usage/status/fs) and
`homelab_synology_disk_*` (model/temp/`smart_status`/`unc`/`remain_life`/`sb_days_left`) are emitted for the
live NAS (1 volume, 8 disks) at 5-min cadence, in the `synology` concurrency group.

**Spec references:** §6.2 (`homelab_synology_*`). EPIC-008 "Verified deployment reality" (the load_info
shape). EPIC-007 STAGE-007-005 (combined device collector parsing one fat response) is the structural
precedent.

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- `synology` concurrency group; emit `homelab_collector_run_*` self-metrics + `homelab_synology_api_took_seconds`.
- `ok=True` when the NAS reports degraded state (that's data, not a probe failure); `ok=False` only on client
  error.
- Graceful degrade if optional disk fields absent; cardinality cap as guardrail (8 disks is well under).
- All timestamps UTC.

## Concrete deliverables

### 1. Storage collector

`storage.py` in the bundle: calls `client.storage_load_info()`, parses `volumes[]` + `disks[]`, emits:
- **Per-volume:** `homelab_synology_volume_used_bytes{volume}`, `_total_bytes{volume}`,
  `_used_percent{volume}`, `_status{volume,status}` (1/0 per state incl. `has_unverified_disk`/
  `fs_almost_full`), `_fs_type{volume,fs}`, writable/encrypted/locked/inode-full flags.
- **Per-disk:** `homelab_synology_disk_temp_celsius{disk,model}`, `_smart_status{disk}` (1=normal),
  `_status{disk}`, `_unc_count{disk}` (uncorrectable), `_remain_life{disk}` (where trustable; -1 = N/A),
  `_sb_days_left{disk}` (bad-sector days-left), size, slot.

### 2. Tests

3a fixture-based unit tests (the recon `load_info` JSON shape: 1 volume, 8 disks, mixed sizes, normal SMART,
sdc 50 °C, `has_unverified_disk`) covering parse + emit + graceful-missing-field; 3b live emission check.
