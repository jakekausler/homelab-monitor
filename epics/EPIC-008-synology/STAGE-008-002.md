# STAGE-008-002: `integrations/synology/` bundle skeleton + registration

## Status: Complete

## Overview

Create the Synology integration-bundle package and its collector-registration seam, mirroring the
established pattern (HA `005-003`, Pi-hole `006-002`, Unifi `007-002`). This is pure scaffolding: an
`integrations/synology/` directory with an `__init__.py` exposing `register_all(loader)` that registers each
Synology collector in a try/except, a `_shared.py` placeholder for collector helpers, and the lifespan call
site that invokes `register_all` once. No collectors land here — they fill the skeleton in Waves B/C/D/E.

**Vertical slice:** after this stage the Synology bundle is importable and `register_all(loader)` is called
from lifespan (registering zero collectors so far, cleanly), establishing the registration point every later
collector plugs into.

**Spec references:** §5 (plugin/collector/integration_bundle framework). EPIC-005 STAGE-005-003 is the
canonical precedent (the bundle layout + registration pattern reused by 006/007/008); EPIC-006 STAGE-006-002
and EPIC-007 STAGE-007-002 are the most recent copies.
