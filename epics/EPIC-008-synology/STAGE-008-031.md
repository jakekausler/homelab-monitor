**Design Notes** (2026-06-29):

**Selected approach:** Option C (Hybrid 3-row) from the brainstormer — mirrors `synology.json` (STAGE-008-030) idioms exactly. Top non-collapsed "Alerts at a glance" stat strip + 3 collapsed `type:"row"` rows with nested children. Panel mix = stat (glance/detail) + table (multi-target `format:"table"` + `merge` + `organize` transform, joined on the `camera` label) + timeseries for time-varying scalars. NO barcharts (project rule). schemaVersion 40, uid `synology-surveillance`, datasource `{type:"prometheus", uid:"victoriametrics"}` on every panel+target, default range `now-6h`.

**Scope correction (locked, follows STAGE-008-030):** the dashboard embed lives on the GLOBAL Metrics page as a new "Surveillance" dashboard tab (placed after Synology), NOT a tab on a per-integration Surveillance page. The card prose's "Surveillance page" phrasing is superseded by EPIC-008 Amendment 7 + the 025/030 scope-correction. Same shape as 030.

**Glance strip (5 stats):** Cameras connected (`ss_cameras_connected_total`, green); Cameras disconnected (`ss_cameras_disconnected_total`, background-RED at ≥1 — the lone alarm; `0→"0"/green`); License used (`ss_license_used`, thresholds green / yellow@81 / red@90, noValue "—"); HomeMode (`ss_homemode_on`, `0→"Off"/grey-text`, `1→"On"/blue` — OFF is NORMAL, never red); Events today (`ss_events_today`, area sparkline, green).

**Row 1 — Cameras (operational, ≤6 cols, mobile-safe):** table joined on `camera` — Camera | Connected (`ss_camera_connected` color-bg 1→Connected/green 0→Disconnected/red) | Status (`ss_camera_status` color-text 1→OK/green 0→Fault/red) | FPS (`ss_camera_fps`) | Resolution (label from `ss_camera_resolution`) | IP (label from `ss_camera_info`). Plus stats: `ss_cameras_total` (Total), `ss_info_camera_number` (reported count cross-check).

**Row 2 — Recordings & identity:** table joined on `camera` — Camera | Recordings (`ss_recordings_count`) | Size (`ss_recordings_bytes`, unit bytes) | Keep days (`ss_camera_recording_keep_days`) | Keep size (`ss_camera_recording_keep_size_mb`, MB) | Retention (label `mode` from `ss_camera_recording_retention_mode`) | Model (label from `ss_camera_info`) | Vendor (label from `ss_camera_info`). Plus aggregate stats `ss_recordings_total` (All recordings), `ss_recordings_bytes_total` (bytes, Total size); timeseries `ss_recordings_bytes_total` (range, storage growth).

**Row 3 — License, HomeMode & events:** License stat group (`ss_license_used`, `ss_license_max`, `ss_license_total`, `ss_license_camera_count`, `ss_license_exhausted` `0→"OK"/green 1→"Exhausted"/red`; `ss_info_license_max` cross-check). HomeMode family stat group (5 stats, each `0→"Off"/grey 1→"On"/blue`, off=normal): `ss_homemode_on` (master), `ss_homemode_schedule_on`, `ss_homemode_streaming_on`, `ss_homemode_notify_on`, `ss_homemode_rec_schedule_on`. Events: `ss_events_today` stat + `ss_events_total_all` stat ("All events") + timeseries `ss_events_today` (range). Version: stat/1-row table surfacing `ss_info_version{version}` (version="9.2.4-11880").

**Card-vs-reality deltas (card is a strong prior; live prod VM 2026-06-29 is the contract):**
- HomeMode is a FAMILY (5 metrics), richer than the card's single flag — surfaced honestly as a stat group, off=normal.
- License surface richer than card (added `license_exhausted`/`license_total`/`license_camera_count` cross-checks).
- New-vs-card metrics folded in: `ss_camera_status`, `ss_camera_resolution` (label `resolution`), `ss_camera_recording_keep_days`/`keep_size_mb`/`retention_mode` (label `mode`), `ss_info_version`, `ss_info_camera_number`.
- All 3 cameras connected; HomeMode off; license 3/90; `cameras_disconnected_total`=0 → honest-normal greens, no false alarms.

**Join-safety (verified, prevents the 030 `disk`/`device` Critical):** every per-camera table column maps from a metric carrying the `camera` label — `camera_connected`, `camera_status`, `camera_fps`, `camera_resolution`, `camera_info`, `recordings_count`, `recordings_bytes`, `camera_recording_*` ALL carry `camera` with values Backyard/Doorbell/Driveway. Label-carrier columns (ip/model/vendor/resolution/mode) are KEPT via `organize.indexByName`/`renameByName` (not excluded).

**Documented near-omission (no silent drop):** `ss_events_total` and `ss_events_total_all` are identical series (both 2648). Surface ONE stat ("All events", using `ss_events_total_all`); note `ss_events_total` as its alias in that panel's `description` rather than rendering a duplicate panel. This is a YAGNI-collapse-with-rationale, not a drop.

**Bug found + fixed during Refinement (duplicate camera rows):** Both per-camera tables (Cameras id 11, Recordings id 21) duplicated each camera into TWO rows — one with all value columns but no IP/Model/Vendor, one with only those carrier-label columns. ROOT CAUSE: Grafana's `merge` transform joins on the FULL shared field set; the `homelab_synology_ss_camera_info` carrier metric has extra labels (id/ip/mac/model/vendor) the other per-camera metrics lack, so merge could not collapse them onto one camera row. FIX: replaced `merge` → `joinByField` `{byField:"camera", mode:"outer"}` in BOTH tables (organize blocks unchanged — they already reference the carried column names). JSON-only / refresh-only (no rebuild). Verified: `grep merge`=0, `grep joinByField`=2, JSON valid, dashboard re-provisioned to version 2, no errors, data joins to exactly 3 rows. (`synology.json`'s tables never hit this because every target there reduces via `max by (key)` to identical `{key,Value}` frames; the surveillance dashboard broke that invariant by feeding raw label-carrier metrics into merge — joinByField is the correct join-on-single-field transform.)
