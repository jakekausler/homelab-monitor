# STAGE-008-022: vmalert-metrics rules — protect/maintain + security + mount + surveillance

## Status: Not Started

## Overview

Author the second tranche of Synology metric alert rules: the protect/maintain signals (backup, replication,
updates, packages), Security Advisor status, mount-health (the root-cause critical that inhibits ContainerDown
via STAGE-008-019), and the surveillance-integrity signals (`synology-surveillance.yaml`: camera offline,
camera-online-but-not-recording, recording-storage-low, license-exhausted). Backup is the single most
important rule (false negatives unacceptable). Hybrid severity.

**Vertical slice:** after this stage these rules are deployed + validated; `SynologyMountDown` is the
inhibit source wired in 008-019; the backup rules cover missing/failed/zero-size/no-configured; the
surveillance-integrity rules cover the silent-failure cases.

**Spec references:** EPIC-008 "Alert catalog" (protect/security/mount/surveillance rows). EPIC-008 cross-stage
("backup is the most important rule; mount-health before container probes"). EPIC-006/007 golden-test
discipline.

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- **Backup false negatives unacceptable** — `SynologyBackupFailed` / `SynologyBackupMissing` (no success in N
  hours) / ran-but-zero-size; `SynologyNoBackupConfigured` (running + 0 tasks). Test all cases.
- `SynologyMountDown` (critical) is the inhibit SOURCE for dependent ContainerDown (008-019).
- Hybrid severity: backup-failed/missing, mount-down = critical; replication-lag, no-backup-configured,
  camera-not-recording, recording-storage-low, license-exhausted = warning; update-available, package-update,
  security-advisor-not-safe = info.
- Surveillance-integrity: `SurveillanceCameraOffline` (critical), `SurveillanceCameraNotRecording`
  (online-but-≈0-events anomaly, warning), `SurveillanceRecordingStorageLow` (warning),
  `SurveillanceLicenseExhausted` (warning).
- vmalert golden tests; validate against real data.

## Concrete deliverables

### 1. Protect/maintain/security/mount rules (`deploy/vmalert/metrics/synology.yaml` additions)

`SynologyBackupFailed`, `SynologyBackupMissing`, `SynologyMountDown` (critical); `SynologyReplicationLag`,
`SynologyNoBackupConfigured` (warning); `SynologyUpdateAvailable`, `SynologyPackageUpdateAvailable`,
`SynologySecurityAdvisorNotSafe` (info — per-category counts in annotation).

### 2. Surveillance-integrity rules (`deploy/vmalert/metrics/synology-surveillance.yaml`)

`SurveillanceCameraOffline` (critical), `SurveillanceCameraNotRecording` (warning),
`SurveillanceRecordingStorageLow` (warning), `SurveillanceLicenseExhausted` (warning).

### 3. Tests

zero-size/no-configured, the camera-silent anomaly); 3b live evaluation (mount/backup live-validation may be
instance-A).
