**Choice 2 — Backup:** no_backup_configured=true → warn banner "No backup configured" (advisory risk, NOT critical). Else: configured_count + last_result_ok (true→ok "Last run OK" / false→critical "Last run FAILED" / null→muted "No result yet"). Caption the delta (no per-job timeline/age/size).

**Choice 3 — Replication:** replication_available=false → muted "Replication not enabled" banner, but STILL render the 14-share table (snapshot_count is real btrfs data, independent of the flag) under "Btrfs snapshots per share". snapshot_count 0 → "0" (real count), null → "—". available=true → table without the banner.

**Choice 4 — Updates:** DSM badge (dsm_update_available true→warn "DSM update available" / false→ok "DSM up to date"). Packages: show ONLY packages with update_available=true (the actionable set; header "{packages_with_updates_count} package updates available"); 0 → muted "All packages up to date". Caption the delta (no per-package version/status string).

**Choice 5 — Security:** security_safe=false → warn "Security advisories present" / true→ok "Secure". Per-severity: chips for count>0 only + a muted "Other severities: 0" line (honest about suppressed zeros). severityVariant() map: danger→critical, risk→warn, warning→warn, outOfDate→warn, info→muted, unknown→muted. count null → "—".

**Choice 6 — Connections + Mounts:** Connections table (User|IP|Type) + a muted caption "Live read from DSM — not stored"; data_available=false → muted "Connection data unavailable"; data_available=true & empty → muted "No active connections"; type passed verbatim. Mounts table (Mount | Status [mount_up true→ok "Up"/false→critical "Down"] | Free [formatBytes, null→"—"]); mount_data_available=false → muted "Mount data unavailable"; true & empty → muted "No NFS mounts" (genuinely empty instance-A; graceful-empty instance-B). Caption the delta (no per-mount latency).

**Card-vs-contract deltas (ALL honest-empty-state captions in-UI, NONE dropped — the contract simply doesn't expose these fields, so they're surfaced as captions not silent omissions):** (1) backup per-job timeline/age/size; (2) per-package version/status string; (3) per-mount latency; (4) replication: card said "status", contract = per-share snapshot_count + an availability flag (rendered accurately + labeled to avoid misreading zero-snapshots as replication-on). No new tracked stage needed — these are collector-doesn't-expose-it facts, captioned honestly.

**Tone palette:** ok/warn/critical/muted (no "info"); advisories (updates/no-backup/security-advisory)→warn, live failures (backup-failed/mount-down)→critical, unavailable/disabled/zero-summary→muted.

**Two real issues found during Refinement (both user-found, both FIXED — backend code + prod config, folded in per user direction):**

1. **Stale-read bug (correctness-critical CODE fix).** The `/summary` + `/ops` endpoints read metrics from two HOURLY collectors (synology_updates=3600s, synology_security=3600s) via BARE vm_instant_query — VictoriaMetrics applies a 5-min staleness window, so for ~58min of each hour those metrics read EMPTY → endpoint defaulted to 0/false/[] → the UI FALSELY showed "DSM up to date / Secure / no package updates". Confirmed live: all 3 DSM update APIs succeed (DSM update IS available: "DSM 7.3.2-86009 Update 3", security update; 4 packages need updates) — the bug was purely the query-staleness mismatch, NOT DSM permissions/auth/parse. FIX: wrapped 5 query constants in `integrations_synology.py` in `last_over_time(<metric>[2h])` (_Q_DSM_UPDATE_AVAILABLE, _Q_SECURITY_SAFE, _Q_PACKAGES_WITH_UPDATES, _Q_PACKAGE_UPDATE_AVAILABLE, _Q_SECURITY_FINDINGS_TOTAL) so the last hourly value is always returned; updated the test mocks (keyed on query strings) + added a lock-in test asserting the 5 constants contain "last_over_time". Backend gate re-verified 6678 passed / 100% coverage. Live post-fix: dsm_update_available=1, security_safe=0, 4 packages — matches the DSM GUI.

2. **Mount-health no-op (PROD CONFIG fix).** The mount-health widget showed "No NFS mounts" but the host HAS 9 NFS mounts of the NAS (/rackstation/*). Root cause: (a) the compose binds `${HM_RACKSTATION_SRC:-/dev/null}:/rackstation` and HM_RACKSTATION_SRC was unset → container saw /dev/null not the real mounts; (b) the `synology_mounts` collector config defaults to empty (open-source-safe no-op) with no override file. FIX (host-specific, in the gitignored overrides repo + prod .env — NOT the public repo): set `HM_RACKSTATION_SRC=/rackstation` in `deploy/compose/.env` (gitignored); created `/var/lib/homelab-monitor/overrides/plugins/collectors/synology_mount_health.yaml` (in the gitignored overrides repo) listing the 9 mounts. Live post-fix: the container sees the 9 NFS /rackstation mounts; `homelab_synology_mount_up` emits 9 series all=1; the mounts widget shows all 9 mounts Up.
