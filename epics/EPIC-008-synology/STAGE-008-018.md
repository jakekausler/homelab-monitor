# STAGE-008-018: NFS/`/rackstation` mount-health collector

## Status: Not Started

## Overview

Build the mount-health collector that validates the monitor host's NFS/SMB mounts from the Synology
(`/rackstation/*`) are alive and responsive — emitting the root-cause signal that the Docker-alert
suppression (STAGE-008-019) consumes. The check must be NON-HANGING (a stale NFS mount blocks naive
`stat`/`statfs` forever), so it runs the probe with a hard timeout and treats a hang as "down." This is the
"mount-health before container probes" criterion: if `/rackstation` is gone, dependent containers are
*expected* to be sad, so this signal is the root cause.

**Vertical slice:** after this stage `homelab_synology_mount_up{mount}` + a mount-probe-latency metric are
emitted for each configured `/rackstation/*` mount at 60s, with a hung mount detected as down (not a wedged
collector).

**Spec references:** §6.2. EPIC-008 cross-stage criterion ("mount-health before container probes"). EPIC-008
Amendment 8 (built in Wave F, before rules/UI/dashboards). EPIC-003 (Docker — the suppression consumer, in
008-019).

## Cross-cutting standards (from EPIC-008 cross-stage acceptance criteria)

- Self-metrics; `ok=True` when a mount is down (data, not probe failure); `ok=False` only if the probe
  mechanism itself errors.
- **Non-hanging:** the responsiveness check runs with a hard timeout (e.g. a thread/subprocess `statfs` with
  cancellation); a hang → `mount_up=0`, never a wedged collector or blocked event loop.
- Configurable mount list (the `/rackstation/*` mounts this host depends on); public default empty.
- All timestamps UTC.

## Concrete deliverables

### 1. Mount-health collector

`mount_health.py`: for each configured mount, perform a timeout-bounded responsiveness probe (e.g.
`os.statvfs` in a cancellable thread, or a bounded subprocess) and emit:
- `homelab_synology_mount_up{mount}` (1 responsive / 0 down-or-hung),
- `homelab_synology_mount_probe_seconds{mount}` (probe latency),
- `homelab_synology_mount_free_bytes{mount}` / `_total_bytes{mount}` when responsive.

### 2. Config

A `synology_mounts:` (or reuse an existing mount-config) list of mounts to probe; public default empty,
user override declares `/rackstation/*`.

### 3. Tests

3a: responsive mount, down mount, HUNG mount (probe must time out and report down without hanging),
missing-mount. 3b (instance A): real `/rackstation/*` mounts probed on the host.
