Resolution: ISOLATED the 3 ScrubOverdue stanzas into a separate `synology.scruboverdue.tests.yaml` with `evaluation_interval: 6h` (valid because those stanzas are `for: 0m`/instant). Main file restored to ~2s; scrub file ~2s. Rule unchanged (production-correct).
