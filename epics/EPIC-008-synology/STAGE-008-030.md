# STAGE-008-030: Grafana `synology.json` dashboard + Metrics-tab embed + readability pass

## Status: Not Started

## Overview

Author the `deploy/grafana/dashboards/synology.json` dashboard (collapsible rows, glance-first) and embed it
as a new dashboard tab on the GLOBAL Metrics page (NOT a tab on the Synology integration page — that Metrics tab was removed in STAGE-008-025). Rows: health/storage, disks/SMART, system/UPS, backup/replication, security, mount-health, SSH-probe state. The `.json` is provisioned-as-code; the dashboard-tab embed is a `.tsx` change to the global Metrics page (FRONTEND — rebuild + sha256-prove + viewport sign-off). A readability pass ensures every metric is surfaced well against live data.

**Scope correction (from STAGE-008-025 Refinement, 2026-06-28):** the Synology dashboard embed lives on the GLOBAL Metrics page as a new dashboard tab, NOT as a tab on the Synology integration page (that tab was removed in 025).

**Vertical slice:** after this stage `synology.json` renders the full Synology metric surface in Grafana and
is reachable as a new dashboard tab on the global Metrics page in-app.

**Spec references:** §9.2 (Metrics = Grafana embed). EPIC-005 STAGE-005-026/030 + EPIC-007 STAGE-007-024
(dashboard + Metrics-tab embed + readability — the precedent). EPIC-008 Amendment 7.

## Concrete deliverables

### 1. `synology.json` dashboard

Collapsible rows: (1) health & storage (volume usage, pool/RAID, SystemHealth rule), (2) disks & SMART
(per-disk temp/status + SMART attributes), (3) system & UPS (CPU/mem/temps, UPS charge/runtime), (4) backup &
replication, (5) security (advisor status/findings, connections), (6) mount-health, (7) SSH-probe state.

### 2. Dashboard-tab embed on global Metrics page

Wire `synology.json` as a new dashboard tab on the existing global Metrics page (alongside Pi-hole, UDM, and other dashboards). NOT a tab on the Synology integration page.

### 3. Readability pass + sign-off

Iterate panels against live data; rebuild + sha256-prove; Desktop + Mobile viewport sign-off on the dashboard tab.
