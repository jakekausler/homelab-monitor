# STAGE-008-030: Grafana `synology.json` dashboard + Metrics-tab embed + readability pass

## Status: Not Started

## Overview

Author the `deploy/grafana/dashboards/synology.json` dashboard (collapsible rows, glance-first) and embed it
as a Metrics tab (the EPIC-005 Metrics-tabs pattern). Rows: health/storage, disks/SMART, system/UPS,
backup/replication, security, mount-health, SSH-probe state. The `.json` is provisioned-as-code; the
Metrics-tab embed is a `.tsx` change (FRONTEND — rebuild + sha256-prove + viewport sign-off). A readability
pass ensures every metric is surfaced well against live data.

**Vertical slice:** after this stage `synology.json` renders the full Synology metric surface in Grafana and
is reachable as a Metrics tab in-app.

**Spec references:** §9.2 (Metrics = Grafana embed). EPIC-005 STAGE-005-026/030 + EPIC-007 STAGE-007-024
(dashboard + Metrics-tab embed + readability — the precedent). EPIC-008 Amendment 7.

## Concrete deliverables

### 1. `synology.json` dashboard

Collapsible rows: (1) health & storage (volume usage, pool/RAID, SystemHealth rule), (2) disks & SMART
(per-disk temp/status + SMART attributes), (3) system & UPS (CPU/mem/temps, UPS charge/runtime), (4) backup &
replication, (5) security (advisor status/findings, connections), (6) mount-health, (7) SSH-probe state.

### 2. Metrics-tab embed

Wire `synology.json` as a Synology Metrics tab (`.tsx`).

### 3. Readability pass + sign-off

Iterate panels against live data; rebuild + sha256-prove; Desktop + Mobile viewport sign-off on the tab embed.
