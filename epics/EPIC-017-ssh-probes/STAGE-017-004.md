## Concrete deliverables

### 1. `hm ssh-probe keygen <target>`

Generate ed25519 keypair; store private key as `ssh_probe_key_<target>` in secrets; print the public key (for
the user to install per 005's instructions). Idempotency/rotation behavior decided at Design (e.g. refuse to
overwrite without `--rotate`).

### 2. `hm ssh-probe capture-hostkey <target>`

Connect to the target (host/port from 002 config) and pin its host key into the target config/secrets so 001
can verify it on every probe. Read-only.

## Design Phase

**Status**: [x] Design (Complete)

### Design Decisions (selected by coordinator, pre-authorized)

| Decision | Selection |
| --- | --- |
| A. keygen rotation/overwrite | Refuse if `ssh_probe_key_<target>` already exists UNLESS `--rotate` is passed. Logic: `existing = await repo.get(secret_name)`; if `existing is not None` and no `--rotate` â†’ stderr error (names that rotating BREAKS the probe until the new pubkey is reinstalled) + exit 1. Create path â†’ `repo.set(..., who="hm ssh-probe keygen")`; `--rotate` path â†’ `repo.rotate(..., who="hm ssh-probe keygen")` (rotate errors via `SecretNotFoundError` if absent â†’ surfaced as "omit --rotate to create it"). Default-refuse is safe + reversible. |
| B. host-key capture mechanism | asyncssh IN-PROCESS capture (NOT the factory â€” it raises `HostKeyNotPinned` before connect; NOT ssh-keyscan). `_CaptureClient(asyncssh.SSHClient)` overrides `validate_host_public_key(host, addr, port, key)` â†’ records `key`, returns `True`. `asyncssh.connect(host, port=, username=, client_factory=lambda: _CaptureClient(sink), client_keys=None, password=None, agent_path=None, preferred_auth="none", connect_timeout=, login_timeout=)`. Callback fires PRE-AUTH (no client key needed â†’ solves chicken-and-egg); the subsequent `asyncssh.PermissionDenied` is EXPECTED and swallowed (key already captured). Bare line = `key.export_public_key().decode().strip()` (passes `_validate_bare_host_key`, no hostname token). Fingerprint = `asyncssh.import_public_key(line).get_fingerprint()` â†’ `SHA256:...`. Connection refused/timeout/DNS â†’ mapped to clear stderr + exit 1 (mirror client.py taxonomy). |
| C. capture-hostkey storage / config write-back | **Print-only**: print the bare host-key line + `SHA256:` fingerprint + a TOFU WARNING (operator must verify out-of-band) + a paste instruction (set `ssh_targets[<id>].host_key`). Writes NOTHING (no secret, no YAML edit). NOT a descope: the 017-002 contract puts `host_key` in operator-authored YAML and NO code round-trip-edits that YAML; ruamel.yaml (comment-preserving) is NOT installed and PyYAML would destroy operator comments; the config may be absent in dev. The command captures+presents; the OPERATOR pins (paste) â€” identical to how host_key gets into config at all and to 005's operator-paste model, and mandatory because TOFU capture requires operator fingerprint verification. (Storing host_key in secrets is rejected â€” the transport reads `pinned_host_key` from the config projection, not secrets; that would need out-of-scope 001/002 changes.) |
| D. module layout + shared bootstrap | New `homelab_monitor/cli/ssh_probe.py` (two-level argparse dispatch mirroring `cli/secrets.py`), wired into `cli/main.py` `_build_parser()`. Sync `_handle(args)` â†’ `asyncio.run(_cmd_keygen/_cmd_capture_hostkey)`. EXTRACT a public `build_secrets_repo()` helper (so we don't import secrets.py's private `_build_repo` â€” avoids `reportPrivateUsage`); re-point `secrets.py._build_repo` to delegate to it (mechanical, tests unaffected). |
| E. keygen target validation | keygen does NOT require the target in `ssh_targets` config (it only forms the secret name `ssh_probe_key_<target>`; the slice runs `keygen udm` standalone). Validate id charset `[A-Za-z0-9._-]+`, non-empty (reject otherwise). Same charset guard on capture-hostkey's target arg. |
| F. capture host/port resolution | Resolve host/port from `load_ssh_targets()[target_id]`; if target not configured â†’ clear stderr error + exit 1. Accept optional `--host`/`--port` override (precedence: `--host` given â†’ use it without requiring config; else require config). Validate `--port` 1â€“65535. |

**Security invariants:** the ed25519 PRIVATE key (PEM) is NEVER printed/logged (only `ssh_probe_key_<target>` secret holds it; a test asserts stdout+stderr never contain `PRIVATE KEY`). capture-hostkey is TOFU (no prior trust) â†’ the command makes the operator responsible for out-of-band fingerprint verification (prints the warning).

**Command summary:**
- `hm ssh-probe keygen <target> [--rotate]` â€” writes secret `ssh_probe_key_<target>` (ed25519 PEM); prints the bare PUBLIC key line + install hint; never the private key. Exit 0 success; 1 on master-key error / refuse-without-rotate / rotate-but-absent; 2 usage.
- `hm ssh-probe capture-hostkey <target> [--host H] [--port P]` â€” resolves host/port (config or override), connects read-only via the capture callback, prints bare host-key line + fingerprint + TOFU warning + paste instruction; writes nothing. Exit 0 success; 1 on not-configured / connection failure / no-key / bad port; 2 usage.

**asyncssh capture API (empirically verified):** `validate_host_public_key` callback fires during kex, pre-auth, with `known_hosts` OMITTED (passing `None` disables verification + the callback); `export_public_key().decode()` yields a `_validate_bare_host_key`-conforming bare line; `get_fingerprint()` (method, not attribute) yields `SHA256:...`.
