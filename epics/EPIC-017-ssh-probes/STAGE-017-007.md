# STAGE-017-007: `ssh.yaml` framework-health alert rules

## Status: Not Started

## Overview

The framework-health alert rules (`deploy/vmalert/metrics/ssh.yaml`) over the `homelab_ssh_*` metrics. **This
stage comes AFTER the exemplar (006)** so the rules validate against REAL metrics the exemplar produces (the
"rules validate against real data" discipline from EPIC-006/007). These are the FRAMEWORK-health alerts;
target-specific alerts (Synology SMART, etc.) belong to consumer epics.

**Vertical slice:** the rules file provisions into vmalert-metrics; a representative rule fires end-to-end —
e.g. stopping the exemplar / a host-key mismatch produces `SshTargetUnreachable` / `SshHostKeyMismatch` →
Alertmanager.

**Spec references:** spec §3.1 (vmalert metrics), §4.1 (metric flow). EPIC-017 "Probe contract &
observability" (the metric surface) + the alert list. STAGE-017-006 (the exemplar producing the metrics these
rules key on). EPIC-006/007 metric-rules stages (promtool idioms; the §4.1 empty-join trap).

## Cross-cutting standards

- Severity vocab `info|warning|critical` ONLY.
- `SshHostKeyMismatch` is **critical** (potential MITM on a trusted target) — distinct from the warning-level
  "unreachable."
- Annotations name the target.
- Comes AFTER the exemplar (006) so rules validate against real metrics.

## Concrete deliverables

### 1. `deploy/vmalert/metrics/ssh.yaml`

- `SshTargetUnreachable` (warning) — `homelab_ssh_up == 0` sustained.
- `SshHostKeyMismatch` (**critical**) — `homelab_ssh_host_key_mismatch == 1` (MITM signal).
- `SshProbeStale` (warning) — `homelab_ssh_last_success_age_seconds` beyond N intervals.
