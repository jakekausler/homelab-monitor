# STAGE-017-003: `SshProbe` base collector + framework health metrics

## Status: Complete

## Overview

The `SshProbe` base тАФ a `BaseCollector` subtype that handles the probe boilerplate (open via
`ctx.ssh.open(target_id)` тЖТ run the pinned forced command тЖТ capture stdout/exit тЖТ parse тЖТ emit тЖТ close) so
concrete probes only declare their target + parser. Establishes the framework's consistent health-metric
surface that EVERY SSH probe emits regardless of payload, including the first-class host-key-mismatch signal.

**Vertical slice:** a trivial test `SshProbe` subclass (against the 001 mock server) emits the full health
surface тАФ `homelab_ssh_up`, duration, last-success-age тАФ and the `ok=True`-when-target-down convention is
demonstrated (target-down тЖТ `up=0` + `ok=True`; probe-error тЖТ `ok=False`); a host-key mismatch emits
`homelab_ssh_host_key_mismatch`.

**Spec references:** spec ┬з5.2 (Collector contract), ┬з5.7 (plugin self-observation тАФ `homelab_collector_run_*`
mandatory). EPIC-017 "Probe contract & observability". STAGE-017-001 (transport), the HA `ha_up.py` collector
(the `..._up`-gauge + `ok=True`-when-down precedent).

## Cross-cutting standards

- `SshProbe(BaseCollector)`: ClassVars (`name`/`interval`/`timeout`/`concurrency_group`/`run_kind=ASYNC`) +
  the openтЖТrunтЖТparseтЖТemitтЖТclose flow.
- **`ok=True`-even-when-target-down convention** (load-bearing): target reachable-but-sad тЖТ `homelab_ssh_up=0`,
  `CollectorResult(ok=True)`; `ok=False` ONLY when the probe itself errors (refused / host-key mismatch /
  timeout).
- Emits `homelab_collector_run_*` (kernel) + the SSH health surface.

## Concrete deliverables

### 1. `SshProbe` base

The base collector encapsulating the probe lifecycle; concrete probes override target + a `parse(stdout)` тЖТ
metrics hook. Maps transport errors (001) onto the `ok` semantics + the health metrics.

### 2. Framework health metrics

Every probe emits: `homelab_ssh_up{target}` (1/0), `homelab_ssh_probe_duration_seconds{target,probe}`,
`homelab_ssh_last_success_age_seconds{target,probe}`, and **`homelab_ssh_host_key_mismatch{target}`** (set on
the distinct `HostKeyMismatch` error from 001 тАФ separate from "target down"). Plus kernel
`homelab_collector_run_*`.

## Design Phase

**Status**: [x] Design (Complete)

### Design Decisions (selected by coordinator, pre-authorized)

| Decision | Selection |
| --- | --- |
| A. `parse` hook shape | `parse(self, result: SshCommandResult) -> ProbeOutcome` where `ProbeOutcome` is a frozen dataclass `(up: bool, metrics: list[ProbeMetric])` and `ProbeMetric = (name: str, value: float, labels: dict[str,str])`. The `SshProbe` base owns the ENTIRE emission path (health surface + payload + `metrics_emitted` counting + ok-semantics); the concrete probe provides only `target_id`/`command` + a PURE `parse`. Pure-function `parse` is unit-testable with no ctx/vm mock. |
| A-prereq. abstract-intermediate marker | Add `abstract: ClassVar[bool] = False` to `BaseCollector` (`kernel/plugins/base.py`); `__init_subclass__` returns early when `cls.__dict__.get("abstract") is True` (keep existing `cls.run is BaseCollector.run` check as a secondary guard). `SshProbe` sets `abstract = True`. REQUIRED because `SshProbe` overrides `run()`, which defeats the existing `cls.run is BaseCollector.run` abstract-detection тАФ without the marker, `SshProbe` itself trips the required-ClassVar (`name`/`interval`/`timeout`) enforcement. Build phase must empirically confirm this `__init_subclass__` behavior against the real `base.py`. |
| B. target + command declaration | Concrete probe declares `target_id: ClassVar[str]` and `command: ClassVar[str] = ""`. Base calls `await conn.run(self.command)`. Works against the 017-001 mock (echoes the command тЖТ deterministic parseable stdout) AND real forced-command targets (server ignores the client command). |
| C. `last_success_age_seconds` persistence | In-memory instance state (`self._last_success_wall`; scheduler reuses collector instances across ticks). Emit `homelab_ssh_last_success_age_seconds` ONLY after the first successful probe (OMIT the series before first success тАФ no sentinel). Restart caveat: after a process restart the age series is absent until the first success, by design. 017-007 `SshProbeStale` must combine `homelab_ssh_up==0` + the age metric with `absent()` handling. NOTE: DB-persistence for restart-survival is NOT in 017-003 scope and is NOT being descoped from it (017-003 never owned restart-survival); if operational need arises later it becomes its OWN tracked stage with a minimal kv repo method. |
| D. base location | New module `kernel/ssh/probe.py`, class `SshProbe`. Coverage gate is GLOBAL 100% branch (`source=["homelab_monitor"]`, `fail_under=100` in `apps/monitor/pyproject.toml`) so location is coverage-neutral; cohesion with the `kernel/ssh/` framework package decides. Concrete probes (017-006) live under `plugins/collectors/` and import `SshProbe`. |
| E. `host_key_mismatch` policy | Always emit `homelab_ssh_host_key_mismatch{target}` = 0 on every non-mismatch run, = 1 on the distinct `HostKeyMismatch` error (emitted even on the early error path where the mismatch raises from `async with` entry). Always-present 0/1 gauge тЖТ 017-007 critical rule is a clean `== 1`, no absence/stale-series handling. |

**Metric surface (every probe):** `homelab_ssh_up{target}` (1/0), `homelab_ssh_probe_duration_seconds{target,probe}`, `homelab_ssh_host_key_mismatch{target}` (always), `homelab_ssh_last_success_age_seconds{target,probe}` (only after first success). `probe` label = the concrete collector's `name`. Duration via `time.monotonic()`. `homelab_collector_run_*` is emitted by the scheduler, NOT the probe.

**Ok-semantics (pinned):** connected AND command ran тЖТ `ok=True`, `up` reflects payload health (1 up / 0 sad). Could-not-connect or command threw a transport error (`SshConnectionRefused`/`SshTimeout`/`HostKeyMismatch`/`HostKeyNotPinned`/`SshAuthError`/unknown `SshTransportError`) тЖТ `ok=False`, `up=0`. (Resolves the ha_up tension: ha_up can't distinguish transport-failure from app-down with one HTTP call; SshProbe CAN тАФ connect/command transport error = probe couldn't complete = ok=False; command-ran-but-sad = app-down = ok=True/up=0.)

## Build Phase

**Status**: [x] Build (Complete)

### Build Notes
- New `apps/monitor/homelab_monitor/kernel/ssh/probe.py`: `SshProbe(BaseCollector)` abstract base + `ProbeOutcome`/`ProbeMetric` dataclasses. Lifecycle: open via `ctx.ssh.open(self.target_id)` тЖТ `conn.run(self.command)` тЖТ `parse(result) -> ProbeOutcome` (pure abstract hook) тЖТ emit тЖТ close. Base owns the entire health-metric surface + `metrics_emitted` counting + ok-semantics.
- Metrics: `homelab_ssh_up{target}` (1/0), `homelab_ssh_probe_duration_seconds{target,probe}` (monotonic), `homelab_ssh_host_key_mismatch{target}` (always 0, =1 on `HostKeyMismatch`), `homelab_ssh_last_success_age_seconds{target,probe}`. Plus subclass payload metrics from `ProbeOutcome.metrics`.
- Ok-semantics: connected+command-ran тЖТ `ok=True`, `up` = parse result; any `SshTransportError` (refused/timeout/mismatch/not-pinned/auth/unknown) тЖТ `ok=False`, `up=0`.
- `last_success_age` semantic (corrected during Build): age = seconds since last HEALTHY (up==1) run; on EVERY up=1 run age=0.0 + timestamp updated; on up=0 with prior success age=now-ts; before first success the metric is omitted. In-memory (`_last_success_monotonic`); resets on process restart by design.
- `kernel/plugins/base.py`: added `abstract: ClassVar[bool] = False` marker; `__init_subclass__` returns early when `cls.__dict__.get("abstract") is True` (non-inheriting тЖТ concrete probes still enforced). `SshProbe` sets `abstract = True`.
- `kernel/ssh/__init__.py`: does NOT re-export the probe symbols (would create a circular import: `ssh/__init__ тЖТ probe тЖТ plugins.base`, while `plugins/io тЖТ ssh/result тЖТ ssh/__init__`). Consumers (017-006 + tests) import `SshProbe`/`ProbeOutcome`/`ProbeMetric` directly from `homelab_monitor.kernel.ssh.probe`. (This corrects the Design's "export from __init__" note тАФ cycle-correctness overrides it.)
- Tests: `apps/monitor/tests/ssh/test_ssh_probe.py` (63 tests incl. a real-loopback happy path + a configurable fake factory for every transport-error row + a deterministic fake clock for age). Build fix cycle: circular import (architectural fix тАФ drop probe re-export) тЖТ 2 test failures (1 test-assertion bug re abstract-marker MRO inheritance; 1 real IMPL bug in the age-on-re-success path, fixed). Verification GREEN: ruff check + ruff format --check + pyright --strict; probe tests 63/63 with 100% branch coverage on `probe.py`; fast full suite 4955/4955 (no regression from the core `base.py` change).
