# STAGE-017-003: `SshProbe` base collector + framework health metrics

## Status: Not Started

## Overview

The `SshProbe` base — a `BaseCollector` subtype that handles the probe boilerplate (open via
`ctx.ssh.open(target_id)` → run the pinned forced command → capture stdout/exit → parse → emit → close) so
concrete probes only declare their target + parser. Establishes the framework's consistent health-metric
surface that EVERY SSH probe emits regardless of payload, including the first-class host-key-mismatch signal.

**Vertical slice:** a trivial test `SshProbe` subclass (against the 001 mock server) emits the full health
surface — `homelab_ssh_up`, duration, last-success-age — and the `ok=True`-when-target-down convention is
demonstrated (target-down → `up=0` + `ok=True`; probe-error → `ok=False`); a host-key mismatch emits
`homelab_ssh_host_key_mismatch`.

**Spec references:** spec §5.2 (Collector contract), §5.7 (plugin self-observation — `homelab_collector_run_*`
mandatory). EPIC-017 "Probe contract & observability". STAGE-017-001 (transport), the HA `ha_up.py` collector
(the `..._up`-gauge + `ok=True`-when-down precedent).

## Cross-cutting standards

- `SshProbe(BaseCollector)`: ClassVars (`name`/`interval`/`timeout`/`concurrency_group`/`run_kind=ASYNC`) +
  the open→run→parse→emit→close flow.
- **`ok=True`-even-when-target-down convention** (load-bearing): target reachable-but-sad → `homelab_ssh_up=0`,
  `CollectorResult(ok=True)`; `ok=False` ONLY when the probe itself errors (refused / host-key mismatch /
  timeout).
- Emits `homelab_collector_run_*` (kernel) + the SSH health surface.

## Concrete deliverables

### 1. `SshProbe` base

The base collector encapsulating the probe lifecycle; concrete probes override target + a `parse(stdout)` →
metrics hook. Maps transport errors (001) onto the `ok` semantics + the health metrics.

### 2. Framework health metrics

Every probe emits: `homelab_ssh_up{target}` (1/0), `homelab_ssh_probe_duration_seconds{target,probe}`,
`homelab_ssh_last_success_age_seconds{target,probe}`, and **`homelab_ssh_host_key_mismatch{target}`** (set on
the distinct `HostKeyMismatch` error from 001 — separate from "target down"). Plus kernel
`homelab_collector_run_*`.
