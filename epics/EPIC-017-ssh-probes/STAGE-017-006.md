## Overview

The end-to-end framework proof — a trivial `uptime` `SshProbe` that exercises **BOTH account-modes against the
two real targets**: the UDM (`appliance`, fixed inlined command) and the Synology (`dedicated-user`, installed
script). This validates the entire framework (keygen → install → host-key pin → probe → verify-restriction)
before any consumer probe (Unifi lease / Synology DSM) is built — mirroring how EPIC-001 proved the collector
bones with a noop collector.

`homelab_ssh_up{target}` + `homelab_ssh_uptime_seconds{target}` for BOTH the UDM and the Synology; the full
setup-to-metric path works in both account-modes.

## Design Phase

**Status**: [x] Design (Complete)

### Selected Approach

**Decided 2026-06-17 (self-selected per pre-authorization; brainstormer Opus generated options).**

A generic, target-id-agnostic `uptime` exemplar probe + a `type()`-based per-target factory, registered by looping over the operator's configured `ssh_targets`. This keeps the PUBLIC repo free of the user's private target_ids (which live only in the gitignored overrides repo) while exercising BOTH account-modes (appliance + dedicated-user) against whatever targets the operator declares.

| Decision | Selection |
| --- | --- |
| Class structure | One generic `UptimeProbe(SshProbe)` carrying the pure `parse()` + `command` ClassVar, plus a `make_uptime_probe(target_id)` factory that synthesizes a per-target subclass via `type()`. NO hard-coded `udm`/`nas` in the public repo. |
| Source command | `command = "cat /proc/uptime"`; `parse()` reads `float(stdout.split()[0])` = seconds since boot. Keeps `parse()` PURE (no clock). `uptime -s` rejected (needs current time → impure). Real forced-command targets ignore `command`; it documents intent + drives the loopback fixture. |
| up/down + malformed | `up = (exit_status == 0 AND a float parsed)`. Emit `homelab_ssh_uptime_seconds{target}` ONLY on successful parse; on any failure (exit≠0, empty stdout → IndexError, garbage → ValueError) return `ProbeOutcome(up=False, metrics=[])` (no fake-zero series pollution). The base still emits `homelab_ssh_up{target}=0`. |
| Registration | In `kernel/api/lifespan.py`, loop over `load_ssh_target_configs()`, call `make_uptime_probe(target_id)`, register each via the existing per-collector `try/except` (log + `degraded.append` on failure). Each synthesized subclass sets `name=f"uptime-{target_id}"`, `target_id`, and `concurrency_group=f"ssh_{target_id}"` (per-target serialization). |

**Metric surface:** base-emitted `homelab_ssh_up{target}` + `homelab_ssh_probe_duration_seconds{target,probe}` + `homelab_ssh_host_key_mismatch{target}` + `homelab_ssh_last_success_age_seconds{target,probe}` (all from the SshProbe base), PLUS the payload `homelab_ssh_uptime_seconds{target}` from `parse()`.

**Test plan:** (1) pure-`parse()` unit tests for all 3 branches — ok (`"2150060.91 29871962.63\n"`,exit 0 → up, value 2150060.91), exit≠0, empty stdout (IndexError path), garbage / `"ran: ..."` echo (ValueError path); (2) factory tests — synthesized subclass has correct `target_id`/`name`/`concurrency_group` and inherits `parse`; (3) one loopback lifecycle test through `ssh_test_server_forced_command` asserting the base emit surface + graceful degradation (no crash; `homelab_ssh_uptime_seconds` absent because the forced echo doesn't parse to a float); (4) lifespan registration tests — registers one probe per stubbed target + covers the `except`/`degraded` branch when register raises. Global 100% branch coverage.

**Refinement (3b) plan:** real `keygen` + `capture-hostkey` (read-only) → pin host keys → `install-instructions` → USER performs the on-target install by hand (UDM authorized_keys line; Synology low-priv user + exemplar `hm-probe.sh` + authorized_keys line) → `hm ssh-probe test <target>` returns exit 0 PASS on BOTH → run the probe → confirm `homelab_ssh_up{target}=1` + `homelab_ssh_uptime_seconds{target}` flow in VictoriaMetrics. Read-only on targets except the user's one-time install.

**Build note (2026-06-17):** Implemented as a bundle: `plugins/collectors/ssh/uptime.py` (generic `UptimeProbe` + `make_uptime_probe(target_id)` `type()` factory, `/proc/uptime` parse) + `plugins/collectors/ssh/__init__.py` (`register_all(loader)` looping `load_ssh_target_configs()` with per-probe try/except isolation, mirroring the HA bundle) + a one-line `register_ssh_collectors(loader)` call in `kernel/api/lifespan.py`. Tests in `tests/ssh/test_uptime_probe.py` (10 tests: 4 parse branches, factory ClassVars, abstract-base guard, loopback lifecycle/graceful-degradation, 2 registration paths incl. the failure-isolation branch). ### Refinement Notes

**Validated 2026-06-17.** Both account modes proven end-to-end against the two real targets:
- Appliance target (inlined `cat /proc/uptime` forced command): `hm ssh-probe test` exit 0 PASS; probe `up=1`, real uptime emitted.
- Dedicated-user target (installed `hm-probe.sh` → `cat /proc/uptime`): `hm ssh-probe test` exit 0 PASS; probe `up=1`, real uptime emitted.
- Live metrics confirmed in VictoriaMetrics after a monitor rebuild: `homelab_ssh_up{target}=1` for both, `homelab_ssh_uptime_seconds` + `homelab_ssh_probe_duration_seconds` populated, collector success incrementing, zero failures, no quarantine.

**Code changes made during Refinement (both verified via full `make verify`, 100% branch coverage):**
1. `cli/ssh_probe.py` — the dedicated-user exemplar script body now renders `cat /proc/uptime` (was `uptime`) to match the probe's `/proc/uptime` parser (+ test_cli_ssh_probe assertion updated).
2. `kernel/ssh/client.py` — fixed an asyncssh `ValueError: Unknown local username` that crashed probes in the prod container (numeric UID, no passwd entry / USER env): seed `USER`/`LOGNAME` env defaults + map any unexpected connect-time exception to `SshTransportError` (clean `up=0` instead of scheduler crash/quarantine). New regression test `test_connect_unexpected_value_error_maps_to_transport_error`.

**Operational gotchas discovered (documented in `docs/ssh-probe-setup.md` troubleshooting):** paste line-wrap breaking authorized_keys; dedicated-user home path varies (use the real home in the forced-command path); a `nologin` login shell makes the forced command return empty stdout → `up=0` (set shell to `/bin/sh`); `usermod`/`chsh` may be absent on stripped-down hosts.

**Follow-ups recorded for later stages (not blocking 006):**
- STAGE-017-007 alert rules must carry remediation guidance in `annotations.description` (added to the 007 card) — incl. the appliance firmware-wipe failure mode.
- Possible future enhancement: `hm ssh-probe capture-hostkey` cannot force a host-key type (negotiates the server default, e.g. rsa); pinning a specific ed25519 host key currently requires `ssh-keyscan -t ed25519`. A `--type` flag would let `capture-hostkey` pin ed25519 directly. (Note this as a candidate; do NOT create a stage unless the workflow requires it — it is a nice-to-have, not a dependency.)
