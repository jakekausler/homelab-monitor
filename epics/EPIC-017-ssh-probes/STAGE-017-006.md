# STAGE-017-006: `uptime` exemplar probe — both account-modes against both real targets

## Status: Not Started

## Overview

The end-to-end framework proof — a trivial `uptime` `SshProbe` that exercises **BOTH account-modes against the
two real targets**: the UDM (`appliance`, fixed inlined command) and the Synology (`dedicated-user`, installed
script). This validates the entire framework (keygen → install → host-key pin → probe → verify-restriction)
before any consumer probe (Unifi lease / Synology DSM) is built — mirroring how EPIC-001 proved the collector
bones with a noop collector.

**Vertical slice:** with both targets set up via the 004/005 tooling, the exemplar probe emits
`homelab_ssh_up{target}` + `homelab_ssh_uptime_seconds{target}` for BOTH the UDM and the Synology; the full
setup-to-metric path works in both account-modes.

**Spec references:** EPIC-017 "Account model" (both modes) + "Stage decomposition" (the exemplar mandate from
the build-order banner). STAGE-017-001..005 (everything this exercises). The HA `ha_up.py` precedent.

- The exemplar exercises BOTH modes: UDM `appliance` (inlined `uptime` command in authorized_keys) AND
  Synology `dedicated-user` (a trivial installed `uptime` script run by the dedicated low-priv user).
- `ok=True`-when-target-down convention; emits the full `homelab_ssh_*` health surface + `uptime_seconds`.
- The restriction is verified for both targets (via 005 `test`) as part of 3b.
- The framework ships ONLY this trivial exemplar script body; real script bodies (Synology SMART/btrfs) are
  EPIC-008.

## Concrete deliverables

### 1. `uptime` exemplar probe

An `SshProbe` subclass emitting `homelab_ssh_up{target}` + `homelab_ssh_uptime_seconds{target}`, configured
for BOTH the UDM (appliance) and Synology (dedicated-user) targets. The trivial exemplar script (for the
dedicated-user mode) ships with the framework.

### 2. End-to-end setup on both targets (Refinement 3b)

Perform the real manual setup per 005's instructions on BOTH targets (UDM authorized_keys line; Synology
low-priv user + exemplar script + authorized_keys line), pin host keys, run the probe, verify the restriction,
and confirm metrics flow.
