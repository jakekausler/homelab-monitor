# STAGE-004-012: Stream picker sidebar (distinct service values with line counts; click-to-filter)

## Status: Not Started

## Overview

Add a left sidebar to the Logs Explorer that lists every distinct `service` value present in the current time window, with line counts. Clicking a service injects `service:"<name>"` (AND'd with existing filters) into the query. Provides discoverability for users who don't know LogsQL.

This is the biggest "wow" moment for first-time Explorer users: they see the menu of available log sources without having to remember any field names.

**Vertical slice:** `/logs` page now has a left sidebar showing "home-assistant (1,234), pi-hole (892), homelab-monitor (445), docker.service (102), sshd.service (12), ..." for the current time window. Click "home-assistant" → query updates to include `service:"home-assistant"`. Results refresh.

**Spec references:** master spec §9.2 (stream picker).

## Concrete deliverables

### 1. Backend endpoint: distinct services in window

`apps/monitor/homelab_monitor/kernel/api/routers/logs.py` — new endpoint:

```
GET /api/logs/services?start=<ISO>&end=<ISO>&limit=100
```

Response:

```python
class LogsServicesResponse(BaseModel):
    services: list[ServiceCount]  # sorted desc by count
    truncated: bool  # true if more services exist beyond limit

class ServiceCount(BaseModel):
    service: str
    count: int  # line count in window
```

Implementation: uses VL's `/select/logsql/stats` (or equivalent) with `stats by (service) count()` LogsQL. Caches result per (start, end, limit) for 30s to avoid hammering VL.

### 4. Recommended policy (Design confirms)

**Hybrid approach:**
- Stream-picker selection is a **separate filter state** (`selectedServices: string[]`) NOT directly part of the LogsQL string
- Backend combines `selectedServices` AND `expr` into the actual VL query: `(service:"a" OR service:"b") AND (<expr>)`
- UI shows selected services as chips above the search box: `[home-assistant ×] [pi-hole ×]`
- Click a service in sidebar → adds to `selectedServices`; click chip × → removes
- URL state: `&services=home-assistant,pi-hole`
- In LogsQL advanced mode, the user CAN write their own `service:` filter in the expr; the sidebar's `selectedServices` AND's with whatever they write

This is cleaner than mutating the LogsQL string and works in both plain-text and advanced modes uniformly.

## Out of scope for this stage

- Distinct values for OTHER fields (host, severity) — could be follow-up "Streams" tab; v1 sidebar is service-only

## Decisions in this stage

### Design phase will confirm

- **Filter composition policy** — separate state + AND vs string mutation. Recommendation: separate state (above)
- **Default limit** — recommendation: 100 services; truncated banner if exceeded
- **Cache TTL on backend** — recommendation: 30s
- **Mobile sidebar behavior** — top-of-page dropdown vs collapsible drawer

### Locked Design Decisions (inherited from brainstorming 2026-05-28)

- **D-SIDEBAR-LEFT** — left sidebar on the Explorer; lists distinct services with counts.
- **D-CLICK-INJECTS-FILTER** — clicking a service adds it to the query (via separate state per recommended policy).
- **D-SIDEBAR-FOR-DISCOVERABILITY** — biggest UX win for users without LogsQL knowledge.
