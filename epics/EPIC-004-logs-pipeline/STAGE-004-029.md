# STAGE-004-029: Signature annotations (timestamped notes)

## Status: Not Started

## Overview

Add free-form annotations to each signature: a chronological list of `(timestamp, author, note)` entries so the user can leave investigation notes like "2026-04-15 — investigated; root cause was HA config typo". Surface in the signature drill-in panel from STAGE-004-028.

**Vertical slice:** in the Signature drill-in panel, an "Annotations" section lists prior notes + an input to add a new one. Annotations persist in SQLite, ordered chronologically.

**Spec references:** master spec §9.2 (signature catalog detail).

## Concrete deliverables

### 1. SQLite schema

`apps/monitor/homelab_monitor/kernel/db/migrations/` — new alembic migration:

```sql
CREATE TABLE log_signature_annotations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    template_hash TEXT NOT NULL,
    service_key TEXT NOT NULL,
    note TEXT NOT NULL,
    author TEXT NOT NULL,  -- the username (single-user system → always 'admin' or similar)
    created_at INTEGER NOT NULL,  -- unix ms
    FOREIGN KEY (template_hash, service_key) REFERENCES log_signatures(template_hash, service_key) ON DELETE CASCADE
);
CREATE INDEX idx_signature_annotations_sig ON log_signature_annotations(template_hash, service_key);
CREATE INDEX idx_signature_annotations_created ON log_signature_annotations(created_at);
```

### 2. Repo layer

`apps/monitor/homelab_monitor/kernel/logs/signature_annotations_repo.py` (new):

- `create(template_hash, service_key, note, author) -> Annotation`
- `list(template_hash, service_key) -> list[Annotation]` (sorted by created_at desc)
- `delete(id) -> None`
- 100% test coverage

### 3. API endpoints

```
GET /api/logs/signatures/{template_hash}/{service_key}/annotations -> {annotations: [...]}
POST /api/logs/signatures/{template_hash}/{service_key}/annotations
  body: {note: string}  (author derived from session)
DELETE /api/logs/signatures/annotations/{id}
```

CSRF on POST/DELETE.

## Out of scope for this stage

- Markdown rendering of note bodies — plain text only in v1
- @mentions / linked references — YAGNI for a single-user system
- Annotation reactions — speculative
- Annotation export — YAGNI

## Decisions in this stage

### Design phase will confirm

- **Editable notes vs delete-only** — recommendation: delete-only for v1 (simpler; user can re-add a corrected note if needed)
- **Max note length** — 2,000 chars recommended
- **Author label format** — username from session

### Locked Design Decisions (inherited from brainstorming 2026-05-28)

- **D-ANNOTATIONS-TIMESTAMPED** — every annotation has author + creation timestamp.
- **D-ANNOTATIONS-PLAIN-TEXT** — no markdown; just plain text.
- **D-ANNOTATIONS-CASCADE-DELETE** — deleting a signature cascades-deletes its annotations.
