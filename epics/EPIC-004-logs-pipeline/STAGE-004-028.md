## Concrete deliverables

### 1. SQLite schema

`apps/monitor/homelab_monitor/kernel/db/migrations/` — new alembic migration:

```sql
CREATE TABLE log_signatures (
    template_hash TEXT NOT NULL,
    service_key TEXT NOT NULL,
    template_str TEXT NOT NULL,  -- the canonical Drain template
    label TEXT,  -- user-set free text
    status TEXT NOT NULL DEFAULT 'active',  -- 'active' | 'suppressed' | 'expected'
    first_seen_at INTEGER NOT NULL,  -- unix ms
    last_seen_at INTEGER NOT NULL,
    total_count INTEGER NOT NULL DEFAULT 0,  -- cumulative, populated from VM metric
    PRIMARY KEY (template_hash, service_key)
);
CREATE INDEX idx_signatures_service ON log_signatures(service_key);
CREATE INDEX idx_signatures_status ON log_signatures(status);
CREATE INDEX idx_signatures_last_seen ON log_signatures(last_seen_at);
```

### 2. Sync from Drain → signature catalog

`apps/monitor/homelab_monitor/kernel/logs/signature_sync.py` (new) — after each Drain cycle, upsert rows into `log_signatures`:

- New template: INSERT with `first_seen_at`, label=NULL, status='active'
- Existing template: UPDATE `last_seen_at` and `total_count`

Hooks into `DrainConsumer`'s cycle completion (don't add to the cycle hot path; runs after `engine.snapshot()`).

### 4. API endpoints


### 5. Sample-lines lookup

The samples endpoint translates the template back to a LogsQL filter (the template's literal tokens + wildcards) and queries VL for recent matches. For example, template `"User <*> connected from <*>"` becomes a LogsQL contains-pattern query.

Exact translation strategy locked in Design phase — drain3 provides the template; the filter is a best-effort match.

## Design Phase

### Design Decisions (resolved 2026-06-05)

Three architecturally-open decisions resolved via brainstormer (Opus). The four D-CATALOG-* decisions above remain locked (inherited). Code facts confirmed by reading source before deciding.

**DECISION A — Drain→catalog sync strategy: A1 (touched-only, injected collaborator)** *(CHOSEN)*
- New `SignatureCatalogSync` collaborator, injected into `DrainConsumer` (mirrors `metrics_writer`/`persistence` injection). Called inline at end of `_run_once_locked` AFTER `engine.snapshot()`.
- Reuses the per-cycle accumulators (`sig_state`, `cycle_counts`, `max_ts_seen`). Add `template_str` to the `sig_state` tuple value (already in scope).
- `total_count` = monotonic accumulation of per-cycle line deltas (`total_count + delta` in the UPSERT), NOT drain3's `cluster_size`. Rationale: `cluster_size` is re-feed-inflatable and resets on drain3 generalization (new hash); the summed per-cycle delta is the only re-feed-safe, generalization-correct line counter.
- `last_seen_at = max_ts_seen` (cycle newest line ts); `first_seen_at` set on INSERT only. `ON CONFLICT(template_hash, service_key) DO UPDATE`.
- Sync wrapped in try/except: a sync DB error is logged but does NOT fail the cycle or block watermark advance. On the VL-failure partial path, sync the partial `sig_state` (consistent with partial metrics emit).
- *Rejected A2 (add `all_templates()` engine accessor + full-sync every cycle):* O(all signatures) writes every cycle, new engine API surface, and `cluster_size`/per-model `last_processed_ts` give wrong per-signature counts and last-seen. Touched-only is correct AND cheap.

**DECISION B — template→LogsQL translation for samples endpoint: B1 (split-on-`<*>`, AND quoted phrases)** *(CHOSEN)*
- `expr = " AND ".join(logsql_quote_phrase(seg) for seg in template_str.split("<*>") if seg.strip())`. Finds lines containing every literal fragment; ignores wildcard position/order (acceptable for an illustrative 10-line sample view).
- Service scope: when `service_key` is a real service name → add a `service:` filter (exact VL field name to be confirmed in Build from how `/logs/query` scopes by service); when `cron:*` or `_unknown` → skip service scoping (best-effort, as D-CATALOG-SAMPLES-FROM-VL permits).
- Empty-literal fallback: if template is just `<*>` / has no non-whitespace literal segment → return `samples: [], reason: "template_too_generic"` (NOT a match-all `*` query).
- Time window: last 24h (server-side constant; configurable window deferred as a non-blocking TODO).
- Graceful degradation: `VictoriaLogsClientError` → `samples: [], reason: "vl_unavailable"`, never 500. Cap at 10 lines.
- *Rejected B2 (order-preserving regex):* second harder escaping problem (LogsQL regex syntax, no existing primitive), far costlier VL regex scans, brittle vs drain3 whitespace — disproportionate for an illustrative sample view.

**DECISION C — Frontend tab refactor: C1 (nested routes) + drill-in as side-panel aside/Sheet** *(CHOSEN)*
- Convert `logsRoute` (`/logs`) into a layout route with children `/logs/query` (existing `LogsExplorerPage` moved wholesale, search schema + `ln*` deep-link params intact) and `/logs/signatures` (new `SignaturesTab`). `/logs` index → `redirect` to `/logs/query` PRESERVING search params (`redirect({ to: '/logs/query', search: (prev) => prev })`) so existing `ln*` deep-links keep working with no `lnUrl` change. Fallback if redirect-with-search proves fiddly: one-line change of `lnUrl` base to `/logs/query`.
- A thin tab-bar shell lives in the layout-route component (one render test for the two tab links + active state, or coverage handling per project convention).
- Drill-in `SignatureDetailPanel`: reuse the EXISTING desktop right-side `<aside>` + mobile `<Sheet side="right">` inspector pattern from `LogsExplorerBody` (driven by a `selection` state). NOT a route, NOT a modal. Live VL sample lines load via a react-query hook keyed on `(template_hash, service_key)`; "Open in Explorer" reuses the existing `OpenInExplorerButton`/`lnUrl`.
- *Rejected C2 (single route, `?tab=` search param):* violates the locked path-segment URL requirement (`/logs/query` vs `/logs/signatures`) and entangles both tabs' params in one `validateSearch` schema.

**Route-ordering constraint (verified):** new `GET /signatures`, `GET/PATCH /signatures/{template_hash}/{service_key}`, `GET /signatures/{template_hash}/{service_key}/samples` routes MUST be declared AFTER the 027 literal `/signatures/refresh` + `/signatures/refresh/{cycle_id}` routes (currently at logs.py lines 668/700) so FastAPI does not let `{template_hash}` swallow `refresh`.
