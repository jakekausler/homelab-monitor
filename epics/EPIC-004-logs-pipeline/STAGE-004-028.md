# STAGE-004-028: Signature catalog backend + list/drill-in UI

## Status: Not Started

## Overview

Add the Signature Catalog as a tab on the `/logs` page (alongside the Query tab from STAGE-004-010). The catalog shows all distinct signatures Drain has identified, with operations: label, suppress, mark-as-expected, search/filter, and click-to-drill-in to a panel showing the full template + sample matching lines + a deep-link to the Explorer.

Annotations (timestamped notes) are added in the NEXT stage (STAGE-004-029).

**Vertical slice:** `/logs` page now has two tabs: Query, Signatures. The Signatures tab lists every active signature with template, service, count, first/last seen, label, status. Click a row → drill-in panel with full template, 10 sample lines from VL, label/suppress/mark-expected controls, "Open in Explorer" deep-link.

**Spec references:** master spec §6.3, §9.2.

## Concrete deliverables

### 1. SQLite schema

`apps/monitor/homelab_monitor/kernel/db/migrations/` — new alembic migration:

```sql
CREATE TABLE log_signatures (
    template_hash TEXT NOT NULL,
    service_key TEXT NOT NULL,
    template_str TEXT NOT NULL,  -- the canonical Drain template
    label TEXT,  -- user-set free text
    status TEXT NOT NULL DEFAULT 'active',  -- 'active' | 'suppressed' | 'expected'
    first_seen_at INTEGER NOT NULL,  -- unix ms
    last_seen_at INTEGER NOT NULL,
    total_count INTEGER NOT NULL DEFAULT 0,  -- cumulative, populated from VM metric
    PRIMARY KEY (template_hash, service_key)
);
CREATE INDEX idx_signatures_service ON log_signatures(service_key);
CREATE INDEX idx_signatures_status ON log_signatures(status);
CREATE INDEX idx_signatures_last_seen ON log_signatures(last_seen_at);
```

### 2. Sync from Drain → signature catalog

`apps/monitor/homelab_monitor/kernel/logs/signature_sync.py` (new) — after each Drain cycle, upsert rows into `log_signatures`:

- New template: INSERT with `first_seen_at`, label=NULL, status='active'
- Existing template: UPDATE `last_seen_at` and `total_count`

Hooks into `DrainConsumer`'s cycle completion (don't add to the cycle hot path; runs after `engine.snapshot()`).

### 3. Repo layer

`apps/monitor/homelab_monitor/kernel/logs/signatures_repo.py` (new):

- `list(filter: SignatureFilter) -> list[Signature]` (paginated, sortable; filter: service_key, status, label search, time range, etc.)
- `get(template_hash, service_key) -> Signature | None`
- `update_label(template_hash, service_key, label) -> Signature`
- `set_status(template_hash, service_key, status) -> Signature`
- 100% test coverage

### 4. API endpoints

```
GET /api/logs/signatures?service=<>&status=<>&label_q=<>&limit=100&offset=0
  -> {signatures: [...], total: int}
GET /api/logs/signatures/{template_hash}/{service_key}
  -> Signature
PATCH /api/logs/signatures/{template_hash}/{service_key}
  body: {label?: string, status?: 'active'|'suppressed'|'expected'}
GET /api/logs/signatures/{template_hash}/{service_key}/samples?limit=10
  -> {lines: LogLine[]}  (recent matching lines from VL)
```

CSRF on PATCH.

### 5. Sample-lines lookup

The samples endpoint translates the template back to a LogsQL filter (the template's literal tokens + wildcards) and queries VL for recent matches. For example, template `"User <*> connected from <*>"` becomes a LogsQL contains-pattern query.

Exact translation strategy locked in Design phase — drain3 provides the template; the filter is a best-effort match.

### 6. Frontend: Signatures tab

`apps/ui/src/routes/logs/SignaturesTab.tsx` (new):

- Table with columns: Service / Template (truncated) / Count (24h) / Last seen / Label / Status
- Filter controls: service dropdown, status dropdown, label search, time-range
- Sort by any column
- Click a row → drill-in panel

### 7. Frontend: drill-in panel

`apps/ui/src/routes/logs/SignatureDetailPanel.tsx` (new):

- Full template (monospace)
- Last 10 sample matching lines (via `<LogViewer>` in embed mode + samples endpoint)
- Label edit (inline input + Save)
- Status toggles: Active / Suppressed / Expected (radio or 3-button group)
- "Open in Explorer" deep-link (uses STAGE-004-021 helper)

### 8. Add tab to `/logs` page

`LogsExplorerPage.tsx` (or refactor to a parent with tabs) — Tabs: Query (existing) / Signatures (new). Default to Query. Tab state in URL: `/logs/query` vs `/logs/signatures`.
