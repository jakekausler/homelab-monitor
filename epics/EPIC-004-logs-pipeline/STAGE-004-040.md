# STAGE-004-040: Throttle/budget alerts (L2) — stream budget threshold warnings

## Status: Not Started

## Overview

The `log_stream_budget` collector (from STAGE-001-018) already emits per-stream byte/line metrics. This stage adds vmalert rules that alert when a stream is:

- Approaching its daily byte budget (e.g., 80% consumed)
- Being throttled by vector's rate limit (50 lines/sec/service today)
- Showing an unusually large jump in per-second line rate (operational anomaly)

Categorization: `anomaly_kind: log_budget_warning`.

**Vertical slice:** when a service's daily VL byte allocation hits 80%, a `LogStreamApproachingBudget` alert fires. When vector starts throttling a service, `LogStreamThrottling` fires. Both annotations link to the Explorer pre-filtered to that service so the user can investigate.

**Spec references:** master spec §6.3 (per-stream caps).

## Concrete deliverables

### 1. Vector throttled-line counter exposure

Verify vector exposes a `vector_events_throttled_total{component_id=...}` (or equivalent) metric. If yes, scrape it into VM via vmagent. If no, add a derived metric in the budget collector that detects "incoming rate > 50/s but stored rate < 50/s" as an inferred throttle signal.

### 2. vmalert rules

`deploy/vmalert/logs/budget_threshold.yml` (new):

```yaml
groups:
  - name: log_budget
    interval: 60s
    rules:
      - alert: LogStreamApproachingBudget
        expr: homelab_log_stream_bytes_today / homelab_log_stream_bytes_budget > 0.80
        for: 5m
        labels:
          severity: warning
          source_tool: homelab-monitor
          category: log-anomaly
          anomaly_kind: log_budget_warning
        annotations:
          summary: "Stream {{ $labels.service }} at {{ $value | humanizePercentage }} of daily budget"
          description: |
            Stream {{ $labels.service }} on {{ $labels.host }} has consumed {{ $value | humanizePercentage }} of its daily byte budget.
            URL: /logs?logsql=service:"{{ $labels.service }}"

      - alert: LogStreamThrottling
        expr: rate(vector_events_throttled_total[5m]) > 0
        for: 5m
        labels:
          severity: warning
          source_tool: homelab-monitor
          category: log-anomaly
          anomaly_kind: log_budget_warning
        annotations:
          summary: "Vector is throttling stream {{ $labels.component_id }}"
          description: |
            Vector is dropping lines from {{ $labels.component_id }} due to throttle rules.
            Recent rate of throttled events: {{ $value }}/sec.
            URL: /logs?logsql=service:"{{ $labels.component_id }}"

      - alert: LogStreamUnusualLineRate
        # Per-second rate is N× the service's 1-hour baseline
        expr: |
          rate(homelab_log_stream_lines_per_sec[5m])
          > 5 * avg_over_time(rate(homelab_log_stream_lines_per_sec[5m])[1h:5m])
        for: 5m
        labels:
          severity: info
          source_tool: homelab-monitor
          category: log-anomaly
          anomaly_kind: log_budget_warning
        annotations:
          summary: "Unusual line rate from {{ $labels.service }}"
          description: |
            Stream {{ $labels.service }} is at {{ $value }}/sec, 5× its 1-hour baseline.
            URL: /logs?logsql=service:"{{ $labels.service }}"
```

## Out of scope for this stage

- Per-stream auto-rate-adjust (vector dynamically tunes throttle) — speculative; manual yaml for v1
- Cross-stream budget reallocation — speculative
- Notification of the user when a stream is permanently exceeding (vs just temporarily) — included implicitly via the `for: 5m` clause

## Decisions in this stage

### Design phase will confirm

- **Warning threshold** — 80% recommended; second tier (95%) → critical?
- **Unusual rate baseline window** — 1 hour recommended (shorter than spike rules because budget anomalies are operational, not security)
- **Whether to alert on vector backpressure events** — yes if available

### Locked Design Decisions (inherited from brainstorming 2026-05-28)

- **D-BUDGET-WARN-AT-80** — first-tier warning at 80% of daily budget.
- **D-THROTTLE-VISIBILITY** — alert when vector is actively throttling.
- **D-RATE-SPIKE-AS-OPS-SIGNAL** — unusual rate vs 1-hour baseline = `anomaly_kind: log_budget_warning`, severity info.
