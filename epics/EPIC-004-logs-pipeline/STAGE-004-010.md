# STAGE-004-010: Logs Explorer skeleton at `/logs` (plain-text search, presets, custom range, paginated results)

## Status: Complete

## Overview

Add the top-level `/logs` route with the first usable iteration of the Logs Explorer. v1 surfaces:

- Plain-text **search box** at top: translates to `_msg:"..."` LogsQL under the hood (LogsQL advanced mode lands in STAGE-004-011)
- `<TimeRangeControl>` from STAGE-004-008 (since-presets + custom range)
- Run / Search button (or auto-run on edit-debounce ‚Äî Design decides)
- Results panel: paginated log lines via `<LogViewer>` (inheriting cursor pagination from STAGE-004-007, local-time rendering from STAGE-004-009)
- URL state: search query + range deep-linkable

No sidebar yet (STAGE-004-012), no saved queries (STAGE-004-013), no query history (STAGE-004-014), no state persistence beyond URL (STAGE-004-015), no field inspector (STAGE-004-016), no histogram (STAGE-004-019), no export (STAGE-004-020), no live tail (STAGE-004-023+).

Just a usable Explorer page.

**Vertical slice:** user navigates to `/logs`, types a search like "connection refused", picks a time range, clicks Search, sees matching lines from VL across all sources. Cursor pagination + custom range work. Deep-link `/logs?q=connection%20refused&since=24h` reproduces the view.

**Spec references:** master spec ¬ß9.2 (Logs screen / LogsQL explorer).

### 4. Plain-text ‚Üí `_msg:` translation

`apps/ui/src/lib/logsQlTranslate.ts` (new):

```typescript
function translateSearchToLogsQl(searchText: string): string {
  if (!searchText.trim()) return '*'  // empty search ‚Üí match all
  // Escape double quotes, embed in _msg:"..."
  const escaped = searchText.replace(/"/g, '\\"')
  return `_msg:"${escaped}"`
}
```

Tests for: empty input, basic phrase, quote escaping, leading/trailing whitespace.

## Decisions in this stage

### Design phase will confirm

- **Search trigger:** explicit button vs debounced auto-fire. Recommendation: explicit button for v1 (predictable, no surprising VL queries)
- **Default range** ‚Äî recommendation: 1h
- **Empty search behavior** ‚Äî `*` LogsQL (match all in range)? Or require a query? Recommendation: `*` so the user immediately sees the most recent lines and can refine
- **Search box affordances** ‚Äî clear button, prior-query autocomplete (defers to STAGE-004-014 query history)

### Locked Design Decisions (inherited from brainstorming 2026-05-28)

- **D-EXPLORER-TOP-LEVEL** ‚Äî `/logs` is a top-level route, peer to Dashboard / Integrations / Alerts.
- **D-EXPLORER-SKELETON-FIRST** ‚Äî v1 is intentionally minimal: search + range + pagination + viewer. Everything else is later stages.
- **D-PLAIN-TEXT-DEFAULT** ‚Äî search box defaults to plain text; LogsQL toggle is a later stage. Users who don't know LogsQL still find this useful immediately.
