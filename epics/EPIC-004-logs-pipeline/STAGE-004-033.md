# STAGE-004-033: Healthcheck-failure log enrichment (60s log window attached to healthcheck unhealthy alerts)

## Status: Not Started

## Overview

When a container's healthcheck transitions to `unhealthy`, fetch the surrounding 60s log window (before + after) from VL and attach the result as enrichment to the resulting alert. Parallel pattern to the crash correlation in STAGE-004-032, but triggered by healthcheck state changes rather than container exits.

Built on top of the shared `LogWindowFetcher` from STAGE-004-031.

**Vertical slice:** when a container healthcheck transitions unhealthy, the user sees the surrounding log window in both Karma's notification AND inline on the alert detail page.

**Spec references:** master spec §6.3; EPIC-004 plan #5.

## Concrete deliverables

### 1. Healthcheck transition detector

`apps/monitor/homelab_monitor/kernel/logs/healthcheck_correlator.py` (new):

- Subscribes to docker socket collector's healthcheck state-change events (already emitted per EPIC-003 work)
- For each transition `healthy → unhealthy`:
  - Fetch log window via `LogWindowFetcher` with filter `service:"<container_name>"`
  - Anchor timestamp: the transition time
  - Window: `window_before_s` (default 30s before), `window_after_s` (default 30s after)
  - Limit: 100 lines (configurable)
- Emit metric: `homelab_container_healthcheck_unhealthy{name, image, compose_project, compose_service}`
- Emit metric: `homelab_container_healthcheck_with_log_context{name, lines_count}` so vmalert can know enrichment is available
- Store enrichment in SQLite `container_healthcheck_enrichments` keyed by event ID

### 2. SQLite schema

```sql
CREATE TABLE container_healthcheck_enrichments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    container_name TEXT NOT NULL,
    transitioned_at INTEGER NOT NULL,  -- unix ms; healthy→unhealthy moment
    new_state TEXT NOT NULL DEFAULT 'unhealthy',  -- could later be other states; for now unhealthy
    image_name TEXT,
    compose_project TEXT,
    compose_service TEXT,
    lines_json TEXT NOT NULL,
    line_count INTEGER NOT NULL,
    truncated INTEGER NOT NULL DEFAULT 0,
    created_at INTEGER NOT NULL
);
CREATE INDEX idx_hc_enrich_container ON container_healthcheck_enrichments(container_name, transitioned_at);
```

Retention: 7 days, configurable via `HOMELAB_MONITOR_HEALTHCHECK_ENRICHMENT_RETENTION_DAYS=7`.

### 3. Window size env var

`HOMELAB_MONITOR_HEALTHCHECK_LOG_WINDOW_S=60` (total; split half-before / half-after; Design phase locks the asymmetry — 30/30 by default).

### 4. Enrichment endpoint

```
GET /api/integrations/docker/containers/{name}/healthcheck-incidents?limit=20
GET /api/integrations/docker/containers/{name}/healthcheck-incidents/{id}
```

### 5. vmalert rule

`deploy/vmalert/metrics/container_healthcheck.yml` (new):

```yaml
groups:
  - name: container_healthcheck
    interval: 30s
    rules:
      - alert: ContainerUnhealthy
        expr: increase(homelab_container_healthcheck_unhealthy[5m]) > 0
        for: 0m
        labels:
          severity: warning
          source_tool: homelab-monitor
          category: docker
        annotations:
          summary: "Container {{ $labels.name }} became unhealthy"
          description: |
            Container {{ $labels.name }} (image: {{ $labels.image_name }}) transitioned
            to unhealthy. Healthcheck context: see homelab-monitor UI.
            URL: /integrations/docker/containers/{{ $labels.name }}/overview
```

### 6. UI: render enrichment on ContainerOverviewTab + Alert detail

- `ContainerOverviewTab.tsx` (from EPIC-003) — when a recent healthcheck transition has enrichment, render a collapsible "Last healthcheck incident log context" section
- `AlertDetailPage.tsx` — for `ContainerUnhealthy` alerts, fetch + render enrichment inline

### 7. Tests

- Backend: detector unit tests, SQLite repo, endpoints
- Integration: synthetic healthcheck transition, assert metric + enrichment row + retrievable
- vmalert rule: planted metric fires
- Frontend: ContainerOverviewTab renders incident context; AlertDetailPage renders enrichment

## Out of scope for this stage

- Cross-container healthcheck correlation — speculative
- Auto-restart on unhealthy — EPIC-009 (auto-fix); just emit alert
- Healthy → unhealthy → healthy flap detection — could be a separate signal; YAGNI for v1

## Decisions in this stage

### Design phase will confirm

- **Window split** — 30s before / 30s after recommended (symmetric)
- **Line limit** — 100 recommended
- **Whether to also enrich `unhealthy → healthy` transitions** — recommendation no for v1 (just unhealthy → healthy is the actionable direction; the recovery transition doesn't need a log window)

### Locked Design Decisions (inherited from brainstorming 2026-05-28)

- **D-HEALTHCHECK-ENRICHMENT-60S-DEFAULT** — env-var configurable.
- **D-HEALTHCHECK-USES-SHARED-FETCHER** — `LogWindowFetcher` from STAGE-004-031.
- **D-HEALTHCHECK-PERSISTED-7D** — enrichment cached 7 days for UI lookup.
