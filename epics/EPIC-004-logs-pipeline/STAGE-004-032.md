# STAGE-004-032: Container crash log correlation + `homelab_container_crash` metric + alert annotation + UI render

## Status: Complete

## Overview

When a container exits with a non-zero code, fetch the last N seconds of its stderr from VL and attach the result as enrichment to a crash alert. The crash metric `homelab_container_crash{name, exit_code, image}` increments per crash; vmalert fires on the metric; Alertmanager renders the enrichment in notifications; the homelab-monitor UI's alert detail page also renders the enrichment inline.

Configurable window via env var `HOMELAB_MONITOR_CRASH_LOG_WINDOW_S=60` (default 60s).

This is one of the docker-specific log requirements from the EPIC-004 plan (item #3). Built on top of the shared `LogWindowFetcher` from STAGE-004-031.

**Vertical slice:** when a container exits with code != 0, the next time the user looks at the resulting alert (in Karma OR in homelab-monitor's UI), they see the container's last ~60 seconds of stderr inline alongside the alert metadata. No need to manually open the Docker log viewer.

**Spec references:** master spec §6.3; EPIC-004 plan #3.

## Design Decisions (selected 2026-06-07)

### Selected approach

- **D1 — Crash detection:** NEW `ContainerCrashReconciler` (a BaseCollector, 30s interval, mirroring CronRunReconciler) that each tick reads `targets_docker` rows in an exited/dead state with `exit_code != 0` and, for any crash not already in the enrichment table, fetches the log window, persists an enrichment row, and increments the crash counter. Dedup is structural: UNIQUE `(logical_key, finished_at)` — re-runs hit INSERT OR IGNORE. The existing `docker_socket_collector` (30s poll) is extended to also persist the container's `State.FinishedAt` (Docker's RFC3339 crash time) into a NEW `targets_docker.finished_at` column; the `ContainerState` TypedDict gains `FinishedAt: NotRequired[str]`. Crash anchor = FinishedAt, falling back to `now()` when missing/zero (`0001-01-01T00:00:00Z`).
  - Rejected (a): extend docker_socket_collector to do correlation inline — rejected: couples a VL query into the inventory tick's timeout budget, inflates that collector's coverage surface, single VL hang risks the inventory tick.
  - Rejected (c): add `die` handling to the docker /events stream — rejected: creates a two-source dedup problem, breaks the discoverer's single responsibility, and event streams drop on reconnect so polling fallback is needed anyway (would build both).
- **D2 — Window + streams:** `window_before_s = 60` (configurable via `HOMELAB_MONITOR_CRASH_LOG_WINDOW_S`, default 60), `window_after_s = 5` (fixed const), `limit = 200` (fixed const). LogsQL anchor = `container_name:"<name>" AND source_type:docker`. stdout+stderr are MERGED by vector's docker_logs source — stderr-only is NOT separable, so the window includes both (this differs from the card's "stderr" wording — corrected here).
- **D3 — Persistence:** single row per crash in NEW table `container_crash_enrichments`; the full log window stored as a JSON blob in `lines_json` (read as a unit, never queried by line content). Columns: crash_id (TEXT PK, uuid4 hex), logical_key, container_name, container_id, exit_code, finished_at, image_name, compose_project, compose_service, lines_json, line_count, truncated (0/1), degraded (0/1), window_start, window_end, created_at. UNIQUE index on `(logical_key, finished_at)` = the dedup mechanism. Retention: age-based prune (`HOMELAB_MONITOR_CRASH_ENRICHMENT_RETENTION_DAYS`, default 7) PLUS a per-logical-key count cap (keep newest 50), both in ONE transaction mirroring CronRunRepository.prune_runs. NEW alembic migration 0036 (down_revision "0035"); bump EXPECTED_TABLES 33 -> 34 in tests/test_db_migrations.py.
- **D4 — API:** two endpoints in the existing `/integrations/docker` router: `GET /integrations/docker/containers/{name}/crashes` returns a list of SUMMARIES (crash_id, exit_code, finished_at, line_count, truncated, degraded, image_name, compose_project, compose_service — NO lines); `GET /integrations/docker/containers/{name}/crashes/{crash_id}` returns DETAIL including the full parsed `lines`. `{name}` resolves to `logical_key` server-side (404 on unknown, matching get_container_logs). Session auth (`Depends(require_session())`).
- **D5 — UI:** render ONLY in `apps/ui/src/routes/integrations/ContainerOverviewTab.tsx` — a "Recent crashes" section listing recent crashes (from the list endpoint), each expandable to render the log window via the existing `<LogViewer>`/`LogLineList` (the sole log-line renderer), plus an `OpenInExplorerButton` deep-link (via `buildExplorerUrl`) anchored on the crash window. Frontend delta is container-tab-only → FE-half Refinement is 3a only.
- **D6 — vmalert + alert channel:** the reconciler emits a COUNTER `homelab_container_crash_total{container_name, exit_code, compose_project, compose_service}` (incremented once per newly-enriched crash). NEW rule file `deploy/vmalert/metrics/container_crash.yml` fires on `increase(homelab_container_crash_total[5m]) > 0`. Labels: `severity: warning`, `source_tool: vmalert-metrics`, `category: container` (EPIC invariant: alerts carry a `category` label). Annotations carry a deep-link URL to the container overview tab (where the enrichment renders) rather than embedding log lines. Tested via an integration test (plant metric -> assert rule fires). (File renamed `.yml`→`.yaml` in yaml-glob bugfix, 2026-06-08.)

## Concrete deliverables

### 1. Crash detector

`apps/monitor/homelab_monitor/kernel/logs/crash_correlator.py` (new):

- Subscribes to the docker socket collector's container-stopped events (already emitted per STAGE-003-003 / STAGE-003-004)
- For each stopped event with `exit_code != 0`:
  - Fetch log window via `LogWindowFetcher` with filter `service:"<container_name>"` (or `container_name:"<name>"`)
  - Anchor timestamp: container's stop_time
  - Window: `window_before_s` (default 60s before stop), `window_after_s` (default 5s — small grace for post-stop emissions)
  - Limit: 100 lines (configurable)
- Emit metric: `homelab_container_crash{name, exit_code, image_name, compose_project, compose_service}` (1 per crash event)
- Emit metric: `homelab_container_crash_with_log_context{name, lines_count}` so a vmalert rule can know the enrichment is available
- Store enrichment in a small SQLite cache `container_crash_enrichments(crash_id, fetched_at_ms, lines_json)` keyed by an event ID so the UI can look it up

### 2. SQLite schema

```sql
CREATE TABLE container_crash_enrichments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    container_name TEXT NOT NULL,
    exit_code INTEGER NOT NULL,
    stopped_at INTEGER NOT NULL,  -- unix ms
    image_name TEXT,
    compose_project TEXT,
    compose_service TEXT,
    lines_json TEXT NOT NULL,  -- JSON-serialized list[LogLine]
    line_count INTEGER NOT NULL,
    truncated INTEGER NOT NULL DEFAULT 0,
    created_at INTEGER NOT NULL
);
CREATE INDEX idx_crash_enrich_container ON container_crash_enrichments(container_name, stopped_at);
```

Retention: 7 days (rolling delete). Single env var `HOMELAB_MONITOR_CRASH_ENRICHMENT_RETENTION_DAYS=7`.

### 3. Crash detail endpoint

```
GET /api/integrations/docker/containers/{name}/crashes?limit=20 -> list[CrashSummary]
GET /api/integrations/docker/containers/{name}/crashes/{crash_id} -> CrashDetail (with full lines)
```

### 4. vmalert rule

`deploy/vmalert/metrics/container_crash.yaml` (renamed from `.yml` in yaml-glob bugfix, 2026-06-08):

```yaml
groups:
  - name: container_crash
    interval: 30s
    rules:
      - alert: ContainerCrashed
        expr: increase(homelab_container_crash[5m]) > 0
        for: 0m
        labels:
          severity: warning
          source_tool: homelab-monitor
          category: docker
        annotations:
          summary: "Container {{ $labels.name }} crashed (exit {{ $labels.exit_code }})"
          description: |
            Container {{ $labels.name }} (image: {{ $labels.image_name }}, compose: {{ $labels.compose_project }}/{{ $labels.compose_service }})
            exited with code {{ $labels.exit_code }}.

            Log context: see the homelab-monitor UI for the crash log window.
            URL: /integrations/docker/containers/{{ $labels.name }}/overview
```

(The 60s log window isn't embedded in the annotation directly — it's looked up from SQLite via the URL. Alertmanager/Karma notification template can choose to fetch + embed.)

### 5. UI: render enrichment on alert detail + container overview

- `apps/ui/src/routes/integrations/ContainerOverviewTab.tsx` (from EPIC-003) — when there's a recent crash with enrichment, render a collapsible "Last crash log context" section showing the lines
- `apps/ui/src/routes/alerts/AlertDetailPage.tsx` (or equivalent) — when the alert is a `ContainerCrashed` alert, fetch + render the enrichment inline
