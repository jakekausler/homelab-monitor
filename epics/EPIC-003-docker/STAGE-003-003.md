# STAGE-003-003: Docker drill-down UI skeleton — Integrations sub-page + routes + empty states

## Status: Complete

## Overview

Land the "Docker" tab under Integrations as a SKELETON: the new route, the page layout, the navigation entry, and empty-state components for every section that later stages will fill. After this stage, every subsequent EPIC-003 stage has a user-visible surface to render its new data into — Refinement always has something to sign off on.

The skeleton is intentionally inert: empty container grid (with placeholder columns), empty "Pending suggestions" panel, empty "Recent actions" panel. As stages land:

- STAGE-003-004 → container grid populates with status/restart/health columns + cadvisor resource columns.
- STAGE-003-005 → suggestions panel populates with discovered-but-unaccepted containers.
- STAGE-003-006 / 007 → probe-source badges appear on grid rows.
- STAGE-003-008 / 009 → image-update badge column appears.
- STAGE-003-010 → action buttons appear; "Recent actions" panel populates.
- STAGE-003-011 → "Logs" column gets a "View logs →" link; per-container log viewer route is filled in.
- STAGE-003-012 → final mop-up; suggestions panel becomes interactive.

**Vertical slice:** logged-in users navigate Integrations → Docker; the page renders with all panels in their empty states. No 404, no console errors, no missing components.

**Spec references:** §9.1 (top-level navigation — Integrations); §9.2 (Integrations sub-page per integration); §9.4 (frontend stack — TanStack Router, Tremor).
