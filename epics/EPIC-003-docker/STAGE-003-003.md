# STAGE-003-003: Docker drill-down UI skeleton — Integrations sub-page + routes + empty states

## Status: Not Started

## Overview

Land the "Docker" tab under Integrations as a SKELETON: the new route, the page layout, the navigation entry, and empty-state components for every section that later stages will fill. After this stage, every subsequent EPIC-003 stage has a user-visible surface to render its new data into — Refinement always has something to sign off on.

The skeleton is intentionally inert: empty container grid (with placeholder columns), empty "Pending suggestions" panel, empty "Recent actions" panel. As stages land:

- STAGE-003-004 → container grid populates with status/restart/health columns + cadvisor resource columns.
- STAGE-003-005 → suggestions panel populates with discovered-but-unaccepted containers.
- STAGE-003-006 / 007 → probe-source badges appear on grid rows.
- STAGE-003-008 / 009 → image-update badge column appears.
- STAGE-003-010 → action buttons appear; "Recent actions" panel populates.
- STAGE-003-011 → "Logs" column gets a "View logs →" link; per-container log viewer route is filled in.
- STAGE-003-012 → final mop-up; suggestions panel becomes interactive.

**Vertical slice:** logged-in users navigate Integrations → Docker; the page renders with all panels in their empty states. No 404, no console errors, no missing components.

**Spec references:** §9.1 (top-level navigation — Integrations); §9.2 (Integrations sub-page per integration); §9.4 (frontend stack — TanStack Router, Tremor).

### 1. New route `/integrations/docker`

`apps/ui/src/router.tsx` — register a new route `/integrations/docker` under the existing integrations layout route (or create the integrations layout route if it does not yet exist). The route renders `DockerIntegrationPage`.

If the Integrations sidebar / layout route does not yet exist, this stage creates the MINIMAL Integrations layout route + sidebar entry — the sidebar lists only Docker for now (other integrations will register their own entries when their epics start).

### 2. `DockerIntegrationPage` component

`apps/ui/src/routes/integrations/DockerIntegrationPage.tsx` — top-level page with three sections:

- **Container grid** — table on desktop (`hidden md:block`) + card list on mobile (`md:hidden`). Empty state shows "No containers discovered yet — cadvisor metrics will populate this grid once containers are seen." Column scaffold (placeholders for now): Name, Status, Image, CPU, RAM, Image Update, Healthcheck, Probes, Logs, Actions. (Each column is rendered as a header with "—" cells in the empty state.) The **Logs** column is a forward-scaffolded slot — STAGE-003-011 fills it with a "View logs →" link per row.
- **Pending suggestions** — panel below the grid. Empty state: "No pending suggestions." (STAGE-003-005 populates; STAGE-003-012 makes interactive.)
- **Recent actions** — panel below pending suggestions. Empty state: "No recent actions." (STAGE-003-010 populates.)

## Locked Design Decisions

D-DRILLDOWN-EARLY — the Docker drill-down skeleton lands as the SECOND stage of EPIC-003, immediately after cadvisor. Every subsequent backend stage fills its slots so Refinement always has UI to sign off on. (User decision during EPIC-003 brainstorm.)

D-SKELETON-COMMENTS — every scaffolded component carries an explicit comment referencing the stage(s) that will fill it. Code reviewers must check stage doc forward references before flagging "unused" scaffolding code for removal. (Per phase-build skill "Scaffolding and Stub Stages" guidance.)

D-SUGGESTIONS-STUB-XREF — the in-epic suggestions stub is TEMPORARY. Code carries a comment block referencing EPIC-011 explicitly: "// EPIC-011 will subsume this with the global Suggestions inbox; this stub may be removed or refactored to link to the global inbox at that time." STAGE-003-010 finalizes; EPIC-011 replaces.

D-CONTAINER-ROW-OPTIONAL — every forward-known field on `ContainerRow` is `T | undefined` (with `exactOptionalPropertyTypes` strictness from STAGE-002-015 learnings). Subsequent stages add data without breaking callers.
