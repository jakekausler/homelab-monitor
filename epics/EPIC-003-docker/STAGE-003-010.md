### 2. Suggestions panel — interactive buttons

`apps/ui/src/components/docker/PendingSuggestionsPanel.tsx`:

- **Accept** button: emits `POST /api/integrations/docker/suggestions/{id}/accept`. The endpoint creates a `probe_targets` row with `config_source="discovered_accepted"`, the same shape STAGE-003-005's label-based path would create. A default HTTP probe is created if the container has an exposed port + healthcheck; otherwise no probe is created (just inventory entry). The suggestion is marked `state="accepted"`.
- **Customize** button: opens a `CustomizeSuggestionModal` (new component) — a form to define one or more probes for this container (kind/name/target/interval). Submitting calls `POST /api/integrations/docker/suggestions/{id}/customize` with the user-defined probes.
- **Ignore** button: emits `POST /api/integrations/docker/suggestions/{id}/ignore`. The suggestion is marked `state="ignored"` and is NOT re-suggested on subsequent discoverer ticks.

`apps/ui/src/components/docker/CustomizeSuggestionModal.tsx` — new component. Form fields: probe kind (dropdown), probe name (text), target (text — context-aware: URL for http, port:host for tcp, command for exec), interval_seconds (number, default 30). "Add another probe" button to add multiple. Save button submits.

### 3. New API endpoints

`apps/monitor/homelab_monitor/kernel/api/routers/docker.py`:

- `POST /api/integrations/docker/suggestions/{id}/accept` — accept with default probe behavior. Returns the created `probe_targets` row(s).
- `POST /api/integrations/docker/suggestions/{id}/customize` — accept with user-specified probes. Body: `{"probes": [ProbeDescriptor[]]}`. Returns the created `probe_targets` row(s).
- `POST /api/integrations/docker/suggestions/{id}/ignore` — archive the suggestion.

All three are auth-required, audit-logged, and idempotent (re-accepting a suggestion is a no-op + 200; double-action doesn't crash).

## Locked Design Decisions

D-IN-EPIC-PANEL-TEMPORARY — the in-epic Pending Suggestions panel is TEMPORARY. EPIC-011 subsumes. Comment in code + entry in `regression.md` + entry in `epics/EPIC-011-*/EPIC-011.md` (when it begins).

D-ACCEPT-DEFAULT-CONSERVATIVE — accepting a suggestion without customizing creates an inventory entry but a probe ONLY if the container has an exposed port + a healthcheck (conservative). Otherwise no probe — just inventory.

D-IGNORE-NOT-DELETE — Ignore archives the suggestion (`state="ignored"`), it does NOT delete. The user (or a future stage) can un-ignore.

D-CUSTOMIZE-PROBE-LIST — the Customize modal collects a LIST of probes (multiple probes per container, mirroring labels). Same schema as STAGE-003-005 `ProbeDescriptor`.

D-CARRY-FORWARD-EXPLICIT — EPIC-011 carry-forward is documented in: (a) this stage file, (b) EPIC-003.md, (c) `epics/EPIC-003-docker/regression.md`, (d) `epics/EPIC-011-*/EPIC-011.md` (when EPIC-011 begins — added to its docs at that time).
