## Cross-cutting standards (extensions to STAGE-006-027/028's contract)

- **Depends on STAGE-006-027** (the `unifi_name` label) + **STAGE-006-028** (VL indexing, phrasematch-aware client enrichment).
- **Honest caveats are the deliverable's core, not an afterthought.** Build ONLY on explicit user request.
- **VM immutability boundary:** old unlabeled series coexist forever; backfill does not retroactively label them.
- **VL retention boundary:** ~30 days (byte-cap pruning); older records are gone. Historical IP→name mapping in `unifi_client_observations` pruned (~24h cutoff) — ALREADY LOST for most old IPs.
- **Forward note from STAGE-006-028 code review (loopback/host indexed-filter caveat):** STAGE-006-028 promoted `client_ip` to an INDEXED VictoriaLogs field, but the indexed value is the RAW client IP (e.g. `127.0.0.1` for loopback), NOT the loopback-attributed host LAN IP — the LAN IP lives in the separate `attributed_host` field. So a future forensic `client_ip:"<host_lan_ip>"` exact filter would MISS loopback-attributed records. If/when this stage (or a forensic-search stage) swaps the per-client query from phrase-match to the indexed `client_ip` filter, it must ALSO index/filter `attributed_host` to preserve the loopback→host attribution that STAGE-006-027/028 established (the Client-page enrichment currently keeps phrase-match specifically to avoid this).

## Concrete deliverables

### 1. Reconstruct historical per-client counts

Extract historical per-client query and blocked counts from the `pihole-queries` VL feed (~30d retention) + `unifi_client_observations` history (same IP→name mapping window), and write NEW `homelab_pihole_client_queries{client_ip,unifi_name,…}` / `_client_blocked{…}` series for past timestamps via VM `/api/v1/import/prometheus` (raw exposition POST, pattern in `cli/dev.py::_post_synthetic_series`).

### 2. Honest caveats — document, do not engineer away

- **VM samples are immutable:** old unlabeled series coexist forever in every Grafana panel; backfill creates new labeled series alongside them — queries must union/prefer.
- **Only ~30 days back** (VL retention boundary). Byte-cap days have gaps → undercount.
- **Historical IP→name mapping is ALREADY GONE:** `unifi_client_observations` is pruned (~24h cutoff). Most reconstructed series cannot be name-resolved. This is the load-bearing reason the user declined in 027 + why this is LOSSIER than first presented.
- **Incomplete coverage:** no way to reconstruct names for IPs that were never observed in the `unifi_client_observations` window.

### Success Criteria

- Backfill script completes without error.
- New labeled series appear in VM for the 30-day window.
- Grafana panel queries union old + new series (both appear).
- Caveats are documented in code + CHANGELOG.
- User explicitly confirms via approval.
