# STAGE-006-003: Unbound-control access layer (docker-exec stats parser + extended-stats graceful-degrade)

## Status: Complete

## Overview

Build the access layer the unbound collector (006-013) will use: run `unbound-control stats_noreset`
inside the `pihole-unbound` container via `docker exec` (the control socket is a unix socket inside the
container; no TCP), parse the key/value stat output into a typed structure, and **detect whether
extended-statistics is enabled** so the collector can gracefully degrade. Extended-statistics was enabled
on the live host during the 2026-06-16 brainstorm, but the layer must work either way (open-source-safe).

**Vertical slice:** a helper that returns a parsed unbound-stats object (default keys always; extended
keys â€” `num.answer.rcode.*`, `num.answer.secure`/`bogus`, `histogram.*`, `num.query.type.*` â€” when present)
plus a boolean `extended_enabled`; unit-tested against captured stat fixtures (both modes); 3b runs it live.

**Spec references:** spec Â§5.4 (subprocess/exec concurrency; collectors that shell out). EPIC-006
"Verified deployment reality" (unbound via docker exec, extended-stats enabled persistently) + "Credentials
& config" (unbound access = docker exec, reuses the existing Docker socket from EPIC-003).

## Cross-cutting standards

- Reuse the existing Docker socket access (mounted for EPIC-003) â€” do NOT add unbound TCP exposure or a new
  Pi-hole-side config.
- The exec must time out + fail gracefully (a wedged unbound-control never hangs the collector).
- Graceful degrade is mandatory: emit `homelab_pihole_unbound_extended_stats_enabled` (1/0) and only emit
  the extended-derived metrics when the keys are present.

## Concrete deliverables

### 1. Exec + parse helper

A helper that runs `unbound-control stats_noreset` (via the Docker exec path; Design confirms the exact
exec mechanism â€” likely the same Docker client EPIC-003 uses) and parses the `key=value`-per-line output
into a typed dict/dataclass. Handle: missing binary, exec error, timeout, empty output.

### 2. Extended-stats detection

Detect extended mode by key presence (e.g. `num.answer.rcode.NOERROR` / `histogram.000000.000000.to...`
present â†’ extended on). Expose `extended_enabled: bool`. The 006-013 collector uses this to choose its
emit set and to emit the `_extended_stats_enabled` flag.

### 3. Fixtures

Capture two real stat outputs (default-mode and extended-mode) as test fixtures (the extended one from the
live host now that it's enabled; anonymize nothing â€” these are resolver counters, not query data). Parser
unit-tested against both.

**Decisions selected:**
- **A â€” exec backend (A2):** add a NEW `exec_capture()` method to `DockerSocketClient` (`kernel/docker/socket_client.py`) returning `(exit_code, stdout, stderr)` â€” Docker socket API with `AttachStdout/Stderr: True`, reading the `/exec/{id}/start` body and de-multiplexing Docker's 8-byte stream-frame headers (Tty:false framing) via an isolated `_demux_stream` helper. (Rejected A1 = mutate the shared method, A3 = shell out via the `docker` CLI.)
- **B â€” module + shape (B1):** `kernel/pihole/unbound_control.py` (beside the 006-001 v6 client, where 006-013 looks). Public surface: frozen `UnboundStats(raw: dict[str, float], extended_enabled: bool, raw_line_count: int)` + `async def fetch_unbound_stats(...) -> UnboundStats | UnboundError`. Keep the FULL parsed keyâ†’value map â€” 006-013 picks the curated emit subset. `extended_enabled` computed here (any `histogram.*`/`num.query.type.*` key present).
- **C â€” wiring (C2):** config `HOMELAB_MONITOR_PIHOLE_UNBOUND_CONTAINER` default `pihole-unbound` via a `PiholeUnboundConfig`/loader mirroring `load_pihole_config`. The exec backend is INJECTED via a narrow `Protocol` (`async def exec_capture(...) -> tuple[int, str, str]`) â€” so the 3a parser/degrade tests use a FAKE backend (no docker required). The access layer does NOT construct the socket client itself.
- **D â€” degrade contract (D2):** `UnboundStats | UnboundError` union (typed outcome, NEVER raises into the collector â€” matches the SSH/Unifi precedent). `UnboundError(reason: Literal["container_unreachable","socket_error","control_error","empty_output","parse_error"], message: str)`. Reasons by layer: exec-layer (`container_unreachable` = container down/not found; `socket_error` = docker socket perm-denied/unreachable/CLI-absent â€” folded in since A2 is socket-only, no CLI), unbound-layer (`control_error` = exitâ‰ 0 with stderr `error ...`), parse-layer (`empty_output` = exit 0 but no key=value lines; `parse_error` = garbage). IMPORTANT: `extended_enabled=False` is NOT an error â€” when extended-stats is OFF the open-source-safe path returns a valid `UnboundStats` with `extended_enabled=False`.

**Tests:** **3a** â€” parser + degrade tested against TWO captured fixtures (real default-mode + real extended-mode `stats_noreset` output) with a MOCKED exec backend: full parse, `extended_enabled` true/false, and each `UnboundError` reason (container_unreachable, socket_error, control_error, empty_output, parse_error). **Consumed by:** STAGE-006-013 (Unbound stats collector) â€” it calls `fetch_unbound_stats`, reads `extended_enabled` to choose its emit set, and emits `homelab_pihole_unbound_extended_stats_enabled` (1/0). The `exec_capture` method is partly SCAFFOLDING for THIS stage (only `unbound-control stats_noreset` exercises it now) but is a general capability; comment any unused breadth.

**Scope:** NO migration (alembic HEAD `0044`). NO API/openapi change. NO frontend. NO secrets (reads resolver counters only). **Host-integration: YES** â†’ Refinement 3a (fixtures + mocked backend) + 3b (live `docker exec`). No user viewport prompt (backend stage).
