# STAGE-006-003: Unbound-control access layer (docker-exec stats parser + extended-stats graceful-degrade)

## Status: Not Started

## Overview

Build the access layer the unbound collector (006-013) will use: run `unbound-control stats_noreset`
inside the `pihole-unbound` container via `docker exec` (the control socket is a unix socket inside the
container; no TCP), parse the key/value stat output into a typed structure, and **detect whether
extended-statistics is enabled** so the collector can gracefully degrade. Extended-statistics was enabled
on the live host during the 2026-06-16 brainstorm, but the layer must work either way (open-source-safe).

**Vertical slice:** a helper that returns a parsed unbound-stats object (default keys always; extended
keys ‚Äî `num.answer.rcode.*`, `num.answer.secure`/`bogus`, `histogram.*`, `num.query.type.*` ‚Äî when present)
plus a boolean `extended_enabled`; unit-tested against captured stat fixtures (both modes); 3b runs it live.

**Spec references:** spec ¬ß5.4 (subprocess/exec concurrency; collectors that shell out). EPIC-006
"Verified deployment reality" (unbound via docker exec, extended-stats enabled persistently) + "Credentials
& config" (unbound access = docker exec, reuses the existing Docker socket from EPIC-003).

## Cross-cutting standards

- Reuse the existing Docker socket access (mounted for EPIC-003) ‚Äî do NOT add unbound TCP exposure or a new
  Pi-hole-side config.
- The exec must time out + fail gracefully (a wedged unbound-control never hangs the collector).
- Graceful degrade is mandatory: emit `homelab_pihole_unbound_extended_stats_enabled` (1/0) and only emit
  the extended-derived metrics when the keys are present.

## Concrete deliverables

### 1. Exec + parse helper

A helper that runs `unbound-control stats_noreset` (via the Docker exec path; Design confirms the exact
exec mechanism ‚Äî likely the same Docker client EPIC-003 uses) and parses the `key=value`-per-line output
into a typed dict/dataclass. Handle: missing binary, exec error, timeout, empty output.

### 2. Extended-stats detection

Detect extended mode by key presence (e.g. `num.answer.rcode.NOERROR` / `histogram.000000.000000.to...`
present ‚Üí extended on). Expose `extended_enabled: bool`. The 006-013 collector uses this to choose its
emit set and to emit the `_extended_stats_enabled` flag.

### 3. Fixtures

Capture two real stat outputs (default-mode and extended-mode) as test fixtures (the extended one from the
live host now that it's enabled; anonymize nothing ‚Äî these are resolver counters, not query data). Parser
unit-tested against both.
