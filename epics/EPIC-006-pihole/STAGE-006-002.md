# STAGE-006-002: integrations/pihole/ bundle skeleton + registration

## Status: Not Started

## Overview

Create the `integrations/pihole/` bundle directory + establish the collector-registration pattern the
rest of EPIC-006 plugs into, mirroring EPIC-005 STAGE-005-003 (which created
`plugins/collectors/integrations/homeassistant/` and the registration pattern reused by 006/007/008/021).
This is pure scaffolding — no collector logic yet — but it locks the layout so Wave-B/C collectors are
"drop a module + register" rather than ad-hoc.

**Vertical slice:** the `pihole` integration bundle directory exists with the bundle's
registration entry point wired into the plugin loader (same mechanism HA used); a trivial no-op or the
006-001 client smoke registers cleanly and appears in the collector/integration status surface.

**Spec references:** spec §5 (integration_bundle = collectors + discoverer(s) + default rules + UI panel +
runbook; "drop a Python module" modularity). EPIC-005 STAGE-005-003 (the exemplar registration pattern).
EPIC-006 Overview ("consumes foundation already built — integration-bundle skeleton/registration").

## Cross-cutting standards

- Follow the EXACT registration pattern EPIC-005 established (read `plugins/collectors/integrations/
  homeassistant/` + its registration in `kernel/api/lifespan.py` / the loader). Do not invent a parallel
  mechanism.
- All plugins observe themselves — `homelab_collector_run_*` self-metrics are emitted by the kernel for
  in-process collectors; the bundle inherits this automatically.

## Concrete deliverables

### 1. Bundle directory

`apps/monitor/homelab_monitor/plugins/collectors/integrations/pihole/` with the package init + the place
Wave-B/C collectors will live. Mirror the homeassistant bundle's file organization (a `_shared.py` for
helpers will accrete as collectors are added — same as HA's `_shared.py` extraction pattern in 005-007/009).

### 2. Registration

Wire the bundle's collectors into the plugin loader the same way HA's were (the `lifespan.ctx_factory` /
loader registration). At this stage there may be zero collectors yet (or just a placeholder) — the point
is the registration plumbing is in place and tested so each subsequent stage's collector registers by
adding one entry, not by touching the loader.

### 3. Confirm the bundle gets the Pi-hole client

Verify the 006-001 `PiholeClient` is reachable from collectors in this bundle via the context (the
injection done in 006-001), so Wave-B collectors can call `ctx.<pihole client>` without re-wiring.
