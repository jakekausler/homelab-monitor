### 1. Bundle directory

`apps/monitor/homelab_monitor/plugins/collectors/integrations/pihole/` with the package init + the place
Wave-B/C collectors will live. Mirror the homeassistant bundle's file organization (a `_shared.py` for
helpers will accrete as collectors are added — same as HA's `_shared.py` extraction pattern in 005-007/009).

### 2. Registration

Wire the bundle's collectors into the plugin loader the same way HA's were (the `lifespan.ctx_factory` /
loader registration). At this stage there may be zero collectors yet (or just a placeholder) — the point
is the registration plumbing is in place and tested so each subsequent stage's collector registers by
adding one entry, not by touching the loader.

Verify the 006-001 `PiholeClient` is reachable from collectors in this bundle via the context (the
injection done in 006-001), so Wave-B collectors can call `ctx.<pihole client>` without re-wiring.

## Design Phase

**Status**: [x] Design (Complete)

**Design Notes**

---
**Design complete.** Mirror the verified integration-bundle precedent (EPIC-005 homeassistant + EPIC-007 unifi): a `plugins/collectors/integrations/pihole/` package whose `__init__.py` exposes `register_all(loader)` iterating a module-level `_PIHOLE_COLLECTORS: list[type[BaseCollector]]` and registering each with per-collector `try/except` isolation (`loader.register(cls, config_from_classvars(cls))`, logging `pihole_integration.collector_register_failed`), wired into `lifespan.py` next to the HA (~599-603) and Unifi (~622-626) registration calls via a bare `register_all as register_pihole_collectors` import + call.

**Decision — placeholder collector (Unifi-style), NOT first-real-collector (HA-style). Brainstormer SKIPPED — the two options + trade-offs were fully enumerated by code/precedent recon and the card language resolves the choice (documented rationale below).**

- **Why placeholder over a real `pihole_up`:** (1) The card frames this as PURE scaffolding ("there may be zero collectors yet (or just a placeholder)... the point is the registration plumbing is in place"), matching Unifi's placeholder precedent (STAGE-007-002), not HA's ship-the-real-`ha_up` precedent (STAGE-005-003). (2) A real `pihole_up`/reachability metric is OWNED by STAGE-006-014 (DNS health probe → `homelab_pihole_up` composite). Shipping a real `pihole_up` now would pre-commit and COLLIDE with 006-014's metric contract. (3) The placeholder has a clean removal contract.
- **Why not an empty `_PIHOLE_COLLECTORS = []`:** an empty list leaves the `register_all` for-loop body + success path uncovered → fails the GLOBAL 100% branch-coverage gate. The bundle MUST register ≥1 collector. (Verified against HA/Unifi which each ship ≥1.)
- **`PiholePlaceholderCollector`** (`placeholder.py`): `name="pihole_placeholder"`, `interval=60s`, `timeout=5s`, `concurrency_group="default"` (NOT a real "pihole" group — must not consume the real concurrency budget), always `ok=True`, emits ONE static gauge `homelab_pihole_bundle_loaded = 1.0`, NEVER touches `ctx.pihole`. A comment at the top of `placeholder.py` AND on its `_PIHOLE_COLLECTORS` entry names **STAGE-006-005** (first real collector — core query-stats) as the REMOVER. This is throwaway scaffolding tracked for removal in 006-005 (no silent deferral — the removal is named on the card-equivalent comment + cited here).
- **No `_shared.py`/`_parsing.py` yet:** helpers accrete as real collectors land (HA grew `_shared.py` in 005-007/009; Unifi `_parsing.py` similarly). Creating an empty helper module now would be premature/uncoverable. Wave-B stages add it.

**Tests (3a, mirror `test_ha_integration_bundle.py` / `test_unifi_integration_bundle.py`):** (1) `register_all` registers the placeholder → assert it appears in `loader.load_all()` with correct name/interval/timeout/config; (2) isolation: monkeypatch `loader.register` to raise → assert `register_all` swallows it (does NOT propagate) and load_all is empty — covers the `except` branch; (3) the placeholder collector's `run(ctx)` emits `homelab_pihole_bundle_loaded=1.0` and returns `ok=True` (covers the collector body). 100% branch coverage required.

**Scope:** NO migration (alembic HEAD `0044`). NO API/openapi change. NO frontend. NO secrets (006-001 already wired `ctx.pihole`). **Host-integration: NO** → Refinement is **3a ONLY** (bundle registers cleanly, loader picks it up, status surface shows the bundle) — NO 3b live test, NO user prompt. Verify: `make verify` green at 100% branch.
