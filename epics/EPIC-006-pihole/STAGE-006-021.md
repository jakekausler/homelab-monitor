# STAGE-006-021: Panel shell + sidebar/router registration + header status strip

## Status: Not Started

## Overview

The Integrations → Pi-hole panel shell: register the route + sidebar nav entry (mirroring EPIC-005's
panel/router/sidebar pattern) and build the header status strip — the at-a-glance row: up/down badge,
blocking state + live auto-re-enable countdown, block %, queries/sec, FTL-messages indicator. Subsequent
stages (022/023/024) fill in the widgets.

**Vertical slice:** navigating to Integrations → Pi-hole renders the panel shell with a live header status
strip wired to the 006-020 overview endpoint (auto-refreshing); the rest of the panel is placeholder.

**Spec references:** spec §9.1 (Integrations nav), §9.2 (per-integration panel). EPIC-006 "Operator panel"
(header status strip). EPIC-005 STAGE-005-022 (panel shell + sidebar/router registration precedent).

## Concrete deliverables

### 1. Route + sidebar registration

Add the Pi-hole integration route + sidebar nav entry (under Integrations), mirroring the HA panel
registration.

### 2. Header status strip

A header component bound to `GET /api/integrations/pihole/overview` (auto-refresh): up/down badge,
blocking state + live countdown (from `blocking_timer_seconds`), block %, queries/sec, FTL-messages
indicator (red if count>0). Placeholder regions for the widgets (022/023/024).
