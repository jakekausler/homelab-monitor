## Design Phase

### Design Notes

**Selected approach (recon-verified against live source + brainstormer-assisted; coordinator selected).** Mirror the EPIC-005 HA + EPIC-007 Unifi panel frontends (`apps/ui/src/routes/integrations/`) — the binding pattern: openapi-fetch + TanStack Query, `unwrap<T>()`, Badge variants `ok`/`warn`/`critical`/`muted`, the verbatim 502 "temporarily unavailable" yellow banner, hand-rolled strip state machine (Unifi's `UnifiStatusStrip` precedent).

**UI route slug decision: `/integrations/pihole` (NO hyphen).** Matches the API path (`/api/integrations/pihole/`), the generated schema component names (`Pihole*`), and the backend router prefix. The project spells it "pihole" as one token throughout EPIC-006; consistency with API+schema+spec outweighs matching the `home-assistant` hyphen.

**Decision 1 — Status strip = EXTRACTED component (`PiholeStatusStrip.tsx`) + colocated test (Option B).** The live countdown is effect-bearing (`setInterval`) and deserves an isolated test surface; the repo already files effect-bearing widgets separately (e.g. `UnifiControllerHealthWidget.tsx`), so this is an existing convention, not a new pattern. Layout stays a pure shell (HA style).

**Decision 2 — Live auto-re-enable countdown = `setInterval` tick, re-synced on refetch (Option B).** Local `remaining` state seeded from `blocking_timer_seconds` on each successful fetch; a 1s interval decrements, floored at 0; effect keyed on `[blocking_enabled, blocking_timer_seconds]` re-seeds on the 30s refetch and clears the interval on any transition (with cleanup). Chosen over an absolute-deadline approach (C) because the source is a 30s snapshot anyway — C's drift-resistance buys nothing and costs dual Date.now()+timer mocking in tests.

**Decision 3 — Register TWO tabs now: Overview + Logs, with placeholder bodies (Option B).** The Overview/Logs split is the only division the card mandates (024 is explicitly an embedded LogViewer = a Logs tab; 022/023 are Overview-family widgets). Registering both now means 022/023/024 are pure body-fill stages that never re-touch `router.tsx` or the Layout `TABS` — minimal-but-extensible, no routing churn, no over-building. The Overview body gets clearly-marked placeholder `<section>`s (Blocking control / Gravity & adlists / Messages / Upstreams & Unbound / Clients / Recent blocked / Version & container) as insertion points for 022/023.

**Decision 4 — Header states = hand-rolled Unifi-style state machine in the strip (Option A).** Precedence: `isPending` ("Loading…") → `error?.status === 502` (verbatim yellow banner) → non-502 `isError` (`<ErrorDisplay>`) → `data` (badge row). The domain DOWN state (`up === false`) is a DATA state (200 response), NOT an error — rendered as a leading `critical` "Pi-hole down" badge with the other indicators showing honest `—`. Nullable fields render as `—` (honest empty states).

**Status-strip indicator layout** (`flex flex-wrap items-center gap-2 text-sm`, left→right):
1. `up` → Badge `ok` "Pi-hole up" / `critical` "Pi-hole down" (always present).
2. blocking → Badge `ok` "Blocking on" (enabled) / `warn` "Blocking off · re-enables in M:SS" + live countdown (disabled, timer>0) / `critical` "Blocking off" (disabled, timer null) / `muted` "Blocking —" (null or down).
3. `percent_blocked` → text "X% blocked" / "—% blocked".
4. `query_frequency` → text "N q/s" / "— q/s".
5. `messages_count` → Badge `critical` "N messages" if >0; else muted "No messages".

**Files to CREATE:** `apps/ui/src/api/pihole.ts` (`piholeQueryKeys` + `usePiholeOverview()` → `/api/integrations/pihole/overview`, `refetchInterval` 30_000, mirror `unifi.ts`); `apps/ui/src/routes/integrations/PiholeLayout.tsx` (shell + strip + Overview/Logs tab nav + `<Outlet/>`); `PiholeStatusStrip.tsx`; `PiholeOverviewTab.tsx` (placeholder sections); `PiholeLogsTab.tsx` (placeholder for 024); `PiholeStatusStrip.test.tsx` (primary surface — countdown + all states); `PiholeLayout.test.tsx` (shell trivial); optionally `PiholeOverviewTab.test.tsx` if the coverage gate needs it.

**Live countdown — 3 attempts to satisfy the React-Compiler-style eslint purity rules (root cause documented in lessons):** the project's `react-hooks` recommended-latest ruleset enforces `set-state-in-effect`, no-refs-during-render, and no-`Date.now()`-during-render as ERRORS. Attempt 1 (setState-in-effect seed) and attempt 2 (refs+Date.now() during render) both failed lint. Attempt 3 (the working one): extracted `useReenableCountdown(timerSeconds)` hook that confines ALL impure/stateful work to the interval callback + a lazy `useState` initializer (mirroring the blessed `useNowTick`/`useMediaQuery` convention) — the deadline lives in a ref read/written ONLY inside the interval callback (the exempt context), `setRemaining` is called only inside `tick()` (callback context), and the render body is pure. Re-anchors on each refetch when `timerSeconds` changes.

**Verification (first full cycle, after pre-flight resolved the countdown lint):** `make verify` GREEN — python `6029 passed, 1 skipped` (0 failed; the skip is the always-present `vector binary not on PATH`), **100.00% branch coverage**; frontend **vitest 1698 passed (164 files)** (+25 new tests), eslint **0 errors** (97 pre-existing warnings), tsc clean, vite build ✓.
