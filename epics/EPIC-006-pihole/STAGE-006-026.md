# STAGE-006-026: Grafana pihole.json dashboard + Metrics-tab embed + readability review pass

## Status: Not Started

## Overview

The Grafana deliverable: a single `deploy/grafana/dashboards/pihole.json` dashboard-as-code (seven
collapsible rows over the `homelab_pihole_*` / `homelab_unbound_*` metrics), embedded as a Pi-hole tab in
the Metrics screen, with a final readability/review pass against live data (mirroring EPIC-005's
dashboard-review stage). Grafana stays on AGGREGATE metrics — no query-feed panels (those are LogViewer /
EPIC-020 territory).

**Vertical slice:** the Pi-hole Grafana dashboard provisions at startup, is embedded in the Metrics screen
as a tab, and renders all seven rows against live Pi-hole/unbound metrics; the readability pass confirms it
actually reads well (legends, units, thresholds, row order).

**Spec references:** spec §9.2 (Metrics = Grafana embed; dashboards-as-code under `deploy/grafana/
dashboards/`). EPIC-006 "Grafana dashboard" (the 7 rows). EPIC-005 STAGE-005-026 (dashboard) + 005-041
(readability review pass) precedent. Datasource `{type:prometheus, uid:victoriametrics}`.

## Concrete deliverables

### 1. Dashboard

`deploy/grafana/dashboards/pihole.json`, 7 collapsible rows:
1. Health & blocking (up/down, blocking+timer, block%, q/s, active clients, FTL-messages).
2. Query volume over time (total/blocked/cached/forwarded + block% over time).
3. Query composition (by type / status / reply / upstream).
4. Gravity / blocklists (domain count, last-update age, per-adlist table).
5. Clients (Tier 2: top talkers, top blocked-per-client, top blocked/permitted domains; host shown).
6. Unbound (cache-hit ratio, recursion p50/95/99, queries/hits/misses, requestlist; DNSSEC secure/bogus +
   RCODE NXDOMAIN/SERVFAIL).
7. FTL/process & API (uptime, CPU%, mem%, DB size + on-disk count, privacy level, query-logging, API took).

### 2. Metrics-tab embed

Add the Pi-hole dashboard as a tab in the Metrics screen (the EPIC-005 Metrics-tabs pattern — System + HA;
add Pi-hole).

### 3. Readability review pass

Interactive pass against live data: legends, units (bytes/percent/seconds), threshold colors, row order,
collapse defaults. Adjust until it reads well.
