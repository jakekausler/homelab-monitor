# STAGE-006-023: Upstreams/unbound + clients (Tier 2 drill-down) + recent-blocked + version widgets + container-control buttons

## Status: Not Started

## Overview

The second set of panel widgets: upstreams table + Unbound resolver sub-card (cache-hit ratio, recursion
p50/p95, DNSSEC secure/bogus, SERVFAIL rate, extended-stats-enabled indicator), the clients widget (top
talkers with per-client drill-down, live-queried), the recent-blocked live feed, the version/updates
widget, and the container-control buttons (restart/start/stop `pihole-unbound`, calling the 006-019
generic action).

**Vertical slice:** the panel shows the upstreams/unbound card with live DNSSEC + latency, a top-talkers
table (host `192.168.2.148` visible) with a working per-client drill-down (recent queries / top domains,
live-queried on the app's refresh interval), the recent-blocked feed, version/update flags, and
confirm-gated restart/start/stop buttons for the Pi-hole container.

**Spec references:** EPIC-006 "Operator panel" (widgets 4-7 + 9 container control). Consumes 006-006/013
(upstreams/unbound), 006-012 (clients, Tier 2 live drill-down), 006-011 (versions), 006-019 (container
actions), 006-020 (panel data). Q9 = live-query drill-down on the existing refresh.

## Cross-cutting standards

- Per-client drill-down LIVE-queries at view-time (no stored history) using the app's refetch-interval
  convention — reuse the existing refresh, don't add a new timer.
- Host `192.168.2.148` shown as a top device (loopback-attributed) — never hidden.
- Container-control buttons use confirm-on-destructive + reflect the action outcome (audited in 006-019).
- Unbound card shows the extended-stats-enabled indicator (so the user knows if DNSSEC/latency metrics are
  available).
- FRONTEND stage: Desktop + Mobile testing + explicit approval; any code change resets both.

## Concrete deliverables

### 1. Upstreams + Unbound card

Upstreams table (share each) + Unbound sub-card: cache-hit ratio, recursion p50/p95, DNSSEC secure/bogus,
SERVFAIL rate, `extended_stats_enabled` indicator.

### 2. Clients widget + drill-down

Top-talkers table (queries/blocked/block%); click a client → drill-down panel showing its recent queries /
top domains (live-queried). Host shown.

### 3. Recent-blocked feed + version widget

Live recent-blocked stream; version/updates widget (per-component version + update-available flags).

### 4. Container-control buttons

Restart / start / stop buttons for `pihole-unbound` → the 006-019 generic action (confirm + audit).
