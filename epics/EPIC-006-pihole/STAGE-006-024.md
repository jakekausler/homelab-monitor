# STAGE-006-024: Embedded LogViewer (docker-stdout scoped) + Tier-3 query-feed view toggle

## Status: Not Started

## Overview

Embed the EPIC-004 `<LogViewer>` in the Pi-hole panel, scoped to the Pi-hole container's docker-stdout
`service` label (the FTL log lines: gravity runs, rate-limits, errors). And — when Tier 3 is enabled —
add a toggle/tab to view the per-query feed stream (the forensic line-level view). Uses the documented
LogViewer embedding contract.

**Vertical slice:** the Pi-hole panel shows an embedded LogViewer scoped to the Pi-hole container logs
(FTL lines visible); when `pihole_stream_query_feed` is on, a toggle reveals the per-query feed stream in
the same viewer.

**Spec references:** EPIC-006 "Operator panel" (widget 10) + "Privacy tiers" (Tier 3 view). `apps/ui/src/
components/logs/README.md` (LogViewer embedding contract). EPIC-005 STAGE-005-025 (embedded LogViewer
scoped to `service:"homeassistant"` precedent). Uses the vector service label pinned in 006-001 and the
Tier-3 VL stream from 006-025.

## Stage Flags

- Has Input Forms: [ ] No (a view toggle, not a form).
- Touches secrets / production data: [x] Yes — the Tier-3 view shows per-query log data (PII; the user's
  own DNS history). Only visible when Tier 3 is enabled.
- Host-integration concerns: [ ] No new backend; consumes VL via the existing logs API. FRONTEND stage:
  Desktop + Mobile viewport approval.

## Cross-cutting standards

- Reuse `<LogViewer>` via the documented embedding contract (caller-provided `useLogs`/`useLogsQuery`
  scoped to the Pi-hole service label) — no parallel log UI (EPIC-005/004 convention).
- The Tier-3 query-feed toggle is HIDDEN when `pihole_stream_query_feed` is off (default public) — only the
  docker-stdout FTL view shows. When on (user), both available.
- Privacy: the Tier-3 view is the most sensitive surface — gated behind the toggle + the feed being enabled.
- FRONTEND stage: Desktop + Mobile testing + explicit approval; any code change resets both.

## Concrete deliverables

### 1. Embedded docker-stdout LogViewer

`<LogViewer>` scoped to the Pi-hole container's docker-stdout `service` label (from 006-001). Shows FTL
lines (gravity/rate-limit/error). Reuses the existing logs API + component.

### 2. Tier-3 query-feed view toggle

When `pihole_stream_query_feed` is enabled, a toggle/tab switches the viewer to the Tier-3 per-query feed
VL stream (from 006-025) — scoped to that stream's service label, with the LogViewer's filter/search.
Hidden entirely when the feed is off.
