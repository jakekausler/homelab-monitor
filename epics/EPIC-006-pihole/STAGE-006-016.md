# STAGE-006-016: vmalert-METRICS rules (the 22 metric alerts)

## Status: Not Started

## Overview

Author the 22 metrics-based vmalert rules over the Wave-B/C metrics, in `deploy/vmalert/metrics/pihole.yaml`
(+ test mirror) and the unbound rules. Severities per the EPIC-006 catalog (info|warning|critical).
Absolute-threshold rules carry the load immediately; anomaly rules use the rolling-baseline idiom with
warm-up.

**Vertical slice:** all 22 rules load `health=ok` on the live vmalert-metrics; the absolute-threshold ones
(PiholeDown, BlockingDisabled, GravityStale, AdlistFailing, MessagesPresent, the SERVFAIL/bogus absolute
forms) fire correctly on planted conditions and are silent at the real baseline; anomaly rules promtool-
validate where possible (live-replay for any subquery forms).

**Spec references:** EPIC-006 "Alert catalog" (metrics rules table). Inherits the user-authored-MetricsQL-
rule machinery (005-005) so the user can add/tune Pi-hole thresholds without code. Pattern precedent:
EPIC-005A vmalert-metrics stages + the rolling-baseline `clamp_min(K*avg_over_time(...))` idiom.

## Cross-cutting standards

- Severity vocab info|warning|critical ONLY (no `error`). Labels: `severity` + `source_tool: vmalert-metrics`
  + `target_kind` (likely `pihole` / `dns`; match the nearest sibling convention).
- A noisy rule is a FAILED rule — confirm no-false-fire at the real baseline (3b).
- Anomaly rules: rolling-baseline + floor; warm-up acceptable (absolute rules cover immediately).
- Watch the §4.1 empty-join trap on any cross-metric comparison; prefer single-metric/threshold forms.

## Concrete deliverables

### 1. The 22 rules

`deploy/vmalert/metrics/pihole.yaml` + unbound rules (Design decides one file vs split pihole/unbound):
**critical** — PiholeDown (`homelab_pihole_up==0`), PiholeDnsProbeFailing, PiholeContainerDown (over
EPIC-003's container metric), PiholeDnsSplitDivergence (pihole-path-fail AND direct-ok), PiholeBlockRateCollapsed,
PiholeUpstreamAllDown. **warning** — PiholeBlockingDisabled (`for:5m`), PiholeBlockingDisabledIndefinitely
(`==0` AND no timer `for:15m`), PiholeUpstreamDown, PiholeGravityStale (TZ-guarded), PiholeGravityDomainsDropped,
PiholeAdlistFailing, PiholeMessagesPresent (text in annotation), PiholeDbTooLarge, PiholeClientFlooding,
UnboundDnssecBogusSpike, UnboundServfailSpike, UnboundRecursionSlow. **info** — PiholeUpdateAvailable,
PiholeBlockRateSpike, PiholeApiSlow, UnboundCacheHitLow.

### 2. Test mirror

`deploy/vmalert/metrics/__tests__/pihole.tests.yaml` — FIRE + NO-FIRE per promtool-validatable rule; for
anomaly/subquery rules that promtool can't evaluate, document the live-replay recipe (EPIC-005A pattern).
