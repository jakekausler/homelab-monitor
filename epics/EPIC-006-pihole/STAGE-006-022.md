# STAGE-006-022: Blocking control + gravity/adlist + FTL-messages widgets (+ write buttons + privacy banner)

## Status: Not Started

## Overview

The first set of panel widgets, including the first WRITE controls in the UI. Blocking control (state +
timer + enable/disable buttons calling the 006-018 write endpoint with confirm), gravity/blocklist widget
(count, last-update age, per-adlist health table + "Update gravity now" button), FTL diagnostic-messages
widget (live message list), and the conditional privacy banner (shown only when privacy level is elevated).

**Vertical slice:** the panel shows live blocking state with working enable/disable buttons (confirm-gated,
audited), the per-adlist table (the 2 live failing adlists shown by name) with an "Update gravity now"
button, the live FTL-messages list, and — only if privacy is up — the privacy banner.

**Spec references:** EPIC-006 "Operator panel" (widgets 1-3 + privacy banner). Consumes 006-018 (write
endpoints), 006-020 (panel data), 006-007 (adlists), 006-010 (messages). Q9 = write-capable + privacy
banner only when elevated. Confirm-on-destructive per spec §9.3.

## Cross-cutting standards

- Write buttons use confirm-on-destructive (typed phrase / session PIN per spec §9.3) and surface the audit
  outcome; reflect the new state live after the action.
- Privacy banner renders ONLY when privacy level elevated (conditional; hidden otherwise) — decided.
- FRONTEND stage: Desktop + Mobile testing + explicit approval; any code change resets both viewports.
- Per-adlist table reuses the app's table conventions; failing adlists visually distinct.

## Concrete deliverables

### 1. Blocking control widget

State + live countdown + enable/disable buttons → `POST /api/integrations/pihole/blocking` (confirm +
audit). Optimistic/refetch update so the new state shows immediately.

### 2. Gravity / blocklists widget

Domain count + last-update age + per-adlist health table (name/domains/enabled/status; the live 2 failing
adlists shown by name) + "Update gravity now" button → `POST /api/integrations/pihole/gravity/update`
(confirm + audit).

### 3. FTL diagnostic-messages widget

Live list from `.../messages` (text + type) — the "Pi-hole is unhappy" panel.

### 4. Privacy banner

Conditional: if privacy level elevated, show "query logging restricted — data may be incomplete"; else
render nothing.
