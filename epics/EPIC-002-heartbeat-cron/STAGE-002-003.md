# STAGE-002-003: Schema redesign — fingerprint identity, drop integration_mode, rename archived_at→hidden_at, add source_path + wrapper_installed_at

## Status: Not Started

## Overview

Replace the UUID-keyed cron schema with a fingerprint-keyed schema that matches the derived-state model from the 2026-05-11 cron-derived-state-redesign spec. After this stage, `crons.fingerprint` is the PK (SHA256 of host/source_path/schedule/command); `integration_mode` is gone; `archived_at` is renamed `hidden_at`; new columns `source_path` and `wrapper_installed_at` are present. Heartbeat receiver and CronRepo are updated to use fingerprint as the cron identifier.

This is a **destructive migration**: existing dev seed data is dropped and re-seeded by the migration's `upgrade()` itself (no production data exists per session note). Test fixtures from STAGE-002-001 and STAGE-002-002 are rewritten to use fingerprint identity.

**Spec references:** `docs/superpowers/specs/2026-05-11-cron-derived-state-redesign.md` §Identity, §Schema changes.

## Decisions in this stage

These need brainstormer-Opus input during Design phase. Pre-recommended position in **bold**.

1. **Fingerprint format: full 64-char SHA256 hex vs truncated 16-char?**
   - **Recommended: full 64-char SHA256 hex.** Collision resistance, no ambiguity in URLs, fits modern systems easily. URL length is acceptable.
   - Alternative: truncated 16-char (current discovery suggestion fingerprint). Shorter URLs, but introduces a non-zero collision probability across the registry's lifetime.

2. **Schema field for `source_path` of remote-only crons: empty string vs NULL?**
   - **Recommended: NULL.** Clear semantic distinction: NULL = "no known disk source" vs `""` = "disk source is the empty string" (which would be a bug).
   - Alternative: empty string + NOT NULL. Avoids three-state SQL but conflates "no source" with "discovery failure to set."

3. **`name` default derivation: command basename vs first 40 chars of command vs custom rule?**
   - **Recommended: command basename (os.path.basename equivalent on the schedule string).** Simple, deterministic, user can edit. Most cron commands invoke a script by path.
   - Alternative: trim to 40 chars + ellipsis. Loses information for short commands; useful for very long inline commands; less common in homelab.
