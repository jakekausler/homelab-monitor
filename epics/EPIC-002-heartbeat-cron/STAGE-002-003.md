# STAGE-002-003: Schema redesign — fingerprint identity, drop integration_mode, rename archived_at→hidden_at, add source_path + wrapper_installed_at

## Status: Complete

## Overview

Replace the UUID-keyed cron schema with a fingerprint-keyed schema that matches the derived-state model from the 2026-05-11 cron-derived-state-redesign spec. After this stage, `crons.fingerprint` is the PK (SHA256 of host/source_path/schedule/command); `integration_mode` is gone; `archived_at` is renamed `hidden_at`; new columns `source_path` and `wrapper_installed_at` are present. Heartbeat receiver and CronRepo are updated to use fingerprint as the cron identifier.

This is a **destructive migration**: existing dev seed data is dropped and re-seeded by the migration's `upgrade()` itself (no production data exists per session note). Test fixtures from STAGE-002-001 and STAGE-002-002 are rewritten to use fingerprint identity.

**Spec references:** `docs/superpowers/specs/2026-05-11-cron-derived-state-redesign.md` §Identity, §Schema changes.

## Decisions in this stage

These need brainstormer-Opus input during Design phase. Pre-recommended position in **bold**.

1. **Fingerprint format: full 64-char SHA256 hex vs truncated 16-char?**
   - **Recommended: full 64-char SHA256 hex.** Collision resistance, no ambiguity in URLs, fits modern systems easily. URL length is acceptable.
   - Alternative: truncated 16-char (current discovery suggestion fingerprint). Shorter URLs, but introduces a non-zero collision probability across the registry's lifetime.

2. **Schema field for `source_path` of remote-only crons: empty string vs NULL?**
   - **Recommended: NULL.** Clear semantic distinction: NULL = "no known disk source" vs `""` = "disk source is the empty string" (which would be a bug).
   - Alternative: empty string + NOT NULL. Avoids three-state SQL but conflates "no source" with "discovery failure to set."

3. **`name` default derivation: command basename vs first 40 chars of command vs custom rule?**
   - **Recommended: command basename (os.path.basename equivalent on the schedule string).** Simple, deterministic, user can edit. Most cron commands invoke a script by path.
   - Alternative: trim to 40 chars + ellipsis. Loses information for short commands; useful for very long inline commands; less common in homelab.

### Design decisions locked (2026-05-12)

| # | Decision | Choice |
|---|---|---|
| D1 | Fingerprint format | **Full 64-char SHA256 hex.** Matches existing `alerts.fingerprinting.quarantine_fingerprint` precedent (same project, same hash function, same hex format). URL length 84 chars is acceptable; TanStack Router doesn't care. |
| D2 | `source_path` for remote-only crons | **NULL (column nullable).** Matches every other "absent" column in the kernel (`archived_at`, `quarantined_at`, `expected_next_at`). Zero precedent for `= ''` as a sentinel in the kernel. |
| D3 | `name` default derivation | **`basename(command.split()[0])`** with fallback `"cron"`. Sets a poor default for interpreter-prefixed commands (`python3 /opt/sync.py` → `python3`) but the field is user-editable; one-click fix beats interpreter-list maintenance burden. |
| D4 | Fingerprint computation | **JSON-canonical dict**: `json.dumps({"host":..., "source_path":..., "schedule":..., "command":...}, sort_keys=True, separators=(",", ":"), ensure_ascii=False).encode("utf-8")`. Matches `quarantine_fingerprint` F19 precedent for delimiter-collision safety. NULL `source_path` serializes as JSON `null`, distinct from empty string. |
| D5 | Audit verb for migration seeds | **Skip audit entirely.** Migrations are infrastructure, not user actions; no meaningful actor or "before" state. STAGE-002-007's discovery emits `crons.discover` for real scan events; not pre-empted here. |
| D6 | Downgrade fidelity | **Restore old shape, drop all data.** Symmetric with destructive upgrade; matches existing project pattern (0006, 0007). Docstring warns both directions are destructive; prod rollback uses backup, not `alembic downgrade`. |
| D7 | UI breakage scope | **Minimal field-rename + integration_mode-removal patches in this stage.** Throwaway-by-design; STAGE-002-006 rewrites these components anyway. Stage-spec's claim "UI doesn't change" is technically incorrect because `make verify` enforces tsc/vitest/UI-build GREEN. |

### Cross-decision interactions

- **D1 (full hex) × D4 (JSON-canonical) — paired alignment with `alerts.fingerprinting` precedent.** Both decisions reinforce consistency with the existing kernel pattern for content-addressable identity.
- **D2 (NULL `source_path`) × D4 (JSON-canonical):** in the hashed JSON, `null` ≠ `""`. Wrapper installers (STAGE-002-009) MUST send `null` (not `""`) for remote-only crons; otherwise the fingerprint computed by the wrapper installer will differ from one computed by local discovery on the same machine. Document in STAGE-002-005's `/register` API contract.
- **D3 (name) × D4 (fingerprint):** `name` is NOT part of the fingerprint tuple. Name edits (or re-derivations) don't change identity. Test invariant: changing `name` alone leaves `fingerprint` unchanged.
- **D5 (skip audit) × D6 (downgrade drops all):** cleanest pair — no orphan audit rows pointing to non-existent fingerprints after downgrade.
- **D7 (minimal UI patches) × Refinement gate:** Refinement smoke test must confirm: (a) `/inventory/crons` renders against new types, (b) mode-badge column absent (no console errors), (c) mode filter dropdown absent from toolbar, (d) detail-page archive button still works (still says "Archive", under-the-hood writes `hidden_at`).

### Algorithm specs locked

**`_compute_fingerprint`** (per D4):
\`\`\`python
import hashlib
import json

def _compute_fingerprint(host: str, source_path: str | None, schedule: str, command: str) -> str:
    payload = json.dumps(
        {"host": host, "source_path": source_path, "schedule": schedule, "command": command},
        sort_keys=True,
        separators=(",", ":"),
        ensure_ascii=False,
    ).encode("utf-8")
    return hashlib.sha256(payload).hexdigest()
\`\`\`

**`derive_name`** (per D3):
\`\`\`python
import os

def derive_name(command: str) -> str:
    stripped = command.strip()
    if not stripped:
        return "cron"
    first_token = stripped.split(maxsplit=1)[0]
    return os.path.basename(first_token) or "cron"
\`\`\`
