# STAGE-002-004: API removal — drop POST /api/crons + AddCronModal + CronCreate + create-mode of CronForm

## Status: Not Started

## Overview

Strip the manual-create path from the cron registry. After this stage there is NO way to create a cron row except via auto-discovery (STAGE-002-007) or `/register` (STAGE-002-005). The `POST /api/crons` endpoint is removed; the `CronCreate` Pydantic schema is removed; the `AddCronModal` UI component is removed; the "+ Add cron" button is removed; `CronForm` collapses to edit-only mode.

This is the "no add" enforcement stage. After it ships, manual-add cannot return without a deliberate code change reverting the deletion.

**Spec references:** `docs/superpowers/specs/2026-05-11-cron-derived-state-redesign.md` §API surface changes.

## Decisions in this stage

These need brainstormer-Opus input during Design phase. Pre-recommended position in **bold**.

1. **Removed POST /api/crons response: 405 Method Not Allowed vs 404 Not Found vs leave-route-but-return-410-Gone?**
   - **Recommended: just delete the route. FastAPI returns 405 Method Not Allowed automatically because GET /api/crons still exists.** Cleanest.
   - Alternative: explicitly route POST to a 410 Gone handler with deprecation message. Helpful for clients still calling the old endpoint, but adds a kept-for-deprecation handler that pollutes the codebase.

2. **CronForm trim level: drop create-mode branch entirely vs keep skeleton with dead `if mode === 'create'` branches?**
   - **Recommended: drop the create branch entirely.** The mode prop is removed; the form is a single-branch edit form. Simpler, less dead code.
   - Alternative: keep the skeleton with `// future: re-enable create if discovery model changes`. Defends against a hypothetical reversal but adds clutter.

3. **Test cleanup approach: delete tests for removed code vs keep skipped with reason?**
   - **Recommended: delete the tests outright.** The removed code IS the contract being tested; keeping skipped tests pretends a removed feature is "temporarily disabled." If the feature comes back later, write fresh tests.
   - Alternative: pytest.skip with `reason="POST /api/crons removed in STAGE-002-004"`. Preserves test history but is process noise.
