## Design Phase

**Status**: [x] Design (Complete) — INCLUDES a live audit of the current Inventory page before removal

### Design Notes

**Selected approach** (coordinator selection per stage-resume pre-authorization; brainstormer consulted; no user prompt). Reuses the EPIC-005 Home Assistant registration pattern (static `NAV_ITEMS` entry + TanStack `createRoute`); no new abstraction.

**Live Inventory audit (Design deliverable #1):** `apps/ui/src/routes/inventory/Inventory.tsx` renders only a header + a single "Crons" tab + `<Outlet/>` (the redirect `/inventory` → `/inventory/crons`). No "upcoming" tabs are wired to real functionality. Safe to remove; only the Crons routes need relocating.

**Decision 1 — Crons path: full rename to `/integrations/crons*` (no redirect).** TanStack Router is typed, so the ~60 `/inventory` references (`<Link to>` + `useSearch({from})` across the 5 Crons routes + `components/crons/CronsTable.tsx` + `RecentRunsPanel.tsx` + ~6 test files) are compile-checked by `make ui-verify`'s tsc step — a missed reference fails the build, not silently breaks. Single-user localhost → redirects from `/inventory/crons` are pure overhead (rejected option 1c). Crons becomes a FLAT set of routes parented directly to `protectedLayoutRoute` (mirroring Docker); the `InventoryLayout` wrapper (only existed to host a single-tab strip) is deleted. Move the directory `routes/inventory/` → `routes/integrations/crons/` (imports are self-contained). Crons functionality preserved EXACTLY — relocation, not rewrite.

**Decision 2 — route shape: Unifi = layout-with-child-tabs NOW; Network = flat stub NOW.** Unifi's multi-tab future is CERTAIN per look-ahead (020 panel w/ device table+drill-down+threats/DPI/Teleport/controller, 022 Clients tab + client detail route, 023 embedded LogViewer + threat-forensics) → create `unifiLayoutRoute` at `/integrations/unifi` (index redirect + ONE honest placeholder tab), mirroring HA, so 020-023 just add child `createRoute`s with zero route-shape refactor. Network's tab-need is speculative (021) → a flat stub page at `/integrations/network` is YAGNI-correct; promote to a layout in 021 if tabs are needed (trivial, self-contained). Placeholders are HONEST empty states ("coming in a later stage" / "not yet configured") — NO fake device/network data (real data arrives in 019/020/021).

**Decision 3 — NO Integrations landing page (explicit non-goal, not a deferral).** The card scope is "add Integrations entries", not a `/integrations` hub. "Integrations" stays a nav-only section label (status quo). A landing/hub page would be its own future stage if ever wanted (pointless now while integrations are placeholders) — not a dropped card deliverable.

**Decision 4 — sidebar:** remove the top-level "Inventory" entry; under the existing "Integrations" section label, indented in ALPHABETICAL order: Crons → Docker → Home Assistant → Network → Unifi. Pick distinct `lucide-react` icons for Network/Unifi (e.g. `Network`/`Wifi`, `Router`/`ShieldCheck`-adjacent; `Cable` is already the section-label icon). Do NOT touch the "Coming soon" disabled placeholders (Runbooks, Auto-fix, etc.) — out of scope.

**Files that will change:** `apps/ui/src/router.tsx` (rename cron paths, delete inventory layout/index routes + import, add unifi layout+index+placeholder-tab + network stub, fix cron import paths, wire routeTree); `apps/ui/src/components/SidebarNav.tsx` (NAV_ITEMS + icons); the 5 Crons route files + `components/crons/CronsTable.tsx` + `components/crons/RecentRunsPanel.tsx` (path strings); DELETE `routes/inventory/Inventory.tsx` + its test; NEW `UnifiLayout.tsx` (+ placeholder tab) + `NetworkPage.tsx`; directory move `routes/inventory/` → `routes/integrations/crons/`. Tests: rewrite `/inventory/crons*` paths in the cron test files + `SidebarNav.test.tsx` (drop Inventory assertion), delete `Inventory.test.tsx`, add minimal render tests for the two new placeholder routes (assert honest empty-state text, no fake data).

**Cross-epic provenance:** Crons is an EPIC-002 cron-heartbeat UI surface; relocated here because closed epics aren't reopened (noted per the card + to go in the commit message).

**Frontend gate:** `make ui-verify` (tsc strict + vitest + prettier + eslint + build). Refinement = Desktop + Mobile user sign-off against `make dev` (host UI :5180).
