# STAGE-007-022: Clients tab (registry table) + Client page (per-client drill-down + DNS-enrichment extension point)

## Status: Not Started

## Overview

The **Clients tab** (under Network) + the per-client **Client page** — the surfaces EPIC-007 builds heavily
and EPIC-006 enhances. The Clients tab renders the `unifi_clients` registry as a sortable table; clicking a
row opens the rich Client page. The Client page exposes a documented **DNS-enrichment extension point** (a
stable contract slot) that EPIC-006 plugs Pi-hole DNS-behavior into — EPIC-006 does NOT build its own client
view.

**Vertical slice:** the Clients tab lists the live roster (host `192.168.2.148` first-class); clicking a
client opens its page showing identity + connection (AP+signal / switch+port) + uptime + bandwidth +
per-client DPI + DHCP-lease field + online/offline history; the DNS-enrichment slot renders as an empty/
"provided by Pi-hole integration" placeholder until EPIC-006 fills it.

**Spec references:** spec §9.2 (inventory drill-downs). EPIC-007 "UI structure" (surface 3 + the enrichment
extension point) + "Build order + client-identity ownership" (007 owns the view, 006 enhances). EPIC-006
STAGE-006-027 (the enhancement that plugs in here). STAGE-007-019 (endpoints + extension slot),
STAGE-007-012 (lease field).

  `<LogViewer>` embed contract) — EPIC-006 plugs into it without modifying this stage's components. Document
  it in a short README/contract note so 006 has a clean seam.

- FRONTEND stage: Desktop + Mobile testing + explicit approval; any code change resets both.

## Concrete deliverables

### 1. Clients tab (registry table)

Sortable table of the registry: name/hostname, IP, MAC, connection (AP+signal / switch+port), uptime,
bandwidth, device type, online/offline, optional lease-expiry column. Row click → Client page.

### 2. Client page (per-client drill-down)

Identity + connection + uptime + bandwidth + per-client DPI + DHCP-lease field (expiry / static-vs-dynamic)
+ online/offline history. Host is a first-class client page.

### 3. DNS-enrichment extension point

A documented slot/contract on the Client page (and a column hook on the Clients tab) that EPIC-006 populates
with Pi-hole DNS behavior (query volume / block rate / top domains / recent blocks / DNSSEC). Renders an
empty/"provided by Pi-hole integration" state until 006 lands.
