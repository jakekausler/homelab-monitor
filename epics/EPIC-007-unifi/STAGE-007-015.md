## Cross-cutting standards

- Severity vocab `info|warning|critical` ONLY.
- Anomaly rules use the rolling-baseline `clamp_min(K*avg_over_time(metric[window]), floor)` idiom with
  warm-up; absolute-threshold rules (down/probe) carry the load immediately.
- Avoid the §4.1 cross-metric empty-join trap (a binary op between two different metrics returning empty =
  dead rule). promtool can't eval `[w:s]` subqueries / `label_value()` — those validate via live-replay.
- Annotations carry the device/client name so an alert says WHICH device.

### Deferred Implementation Details

**1. UnifiDhcpPoolExhaustion & UnifiDnsSteeringDrift rule mechanics (Deferred from STAGE-007-011 Design — Decision 2)**

The `UnifiDhcpPoolExhaustion` rule computes DHCP pool occupancy as a PromQL join: `homelab_unifi_client_count_by_network{network}` (per-network active count, emitted by STAGE-007-007 active_client.py) divided by `homelab_unifi_dhcp_pool_size{network}` (emitted by STAGE-007-011 networkconf collector). STAGE-007-011 deliberately emits pool_size ONLY; the occupancy ratio/recording-rule + the exhaustion alert live HERE. Reference `homelab_unifi_client_count_by_network`, NOT a non-existent `dhcp_pool_used` series. When the opt-in SSH lease collector (STAGE-007-012) is enabled, `homelab_unifi_dhcp_lease_count` is the AUTHORITATIVE occupancy numerator that sharpens this (active-client count is the API approximation when leases are off). Also: the `UnifiDnsSteeringDrift` rule compares the `dns` label of `homelab_unifi_dhcp_dns_primary{network,dns}` against the configured expected Pi-hole IP (192.168.2.148) and needs an `absent()` companion clause to catch a vanished metric.

**2. DPI per-counter spike clamp (Deferred from STAGE-007-009 Design — Decision 3)**

The DPI per-counter spike clamp: the `homelab_unifi_client_dpi_bytes{client,app,cat}` collector (STAGE-007-009) emits RAW cumulative bytes; the clamp for Ubiquiti's known per-counter spike bug is a PromQL safeguard that lives HERE (a `clamp_max(...)` / rate-sanity guard in the DPI-consuming rules/recording rules), NOT in the collector. Apply it in any 015 rule that computes rate()/increase() over the DPI byte series so a spurious counter jump doesn't produce a false anomaly.

**Key design decisions (brainstormer-consulted, coordinator-selected):**
- D1 GatewayDown vs ControllerDown: KEEP BOTH (distinct blast radius). ControllerDown = API unreachable (`homelab_unifi_up==0`); GatewayDown = controller reachable but reports the UDM device down (`homelab_unifi_device_up{gateway}==0`). NOT off wan_up (WAN-down is its own alert).
- D2 Flapping: derived via `changes(device_up[15m]) > 3` (no flap metric exists); for: 0m (window encodes the time).
- D3 Uplink saturation: switch PORT tx rate vs port_speed_bps (NOT WAN xput); denominator guarded `> 0`.
- D4 ClientPoorExperience: the only available series is the `clients_poor_satisfaction{threshold}` COUNT rollup -> alert on count > 3; annotation names the count, not per-client (a rollup can't name clients) — acceptable.
- D5 Anomaly rules use clamp_min idiom; warm-up means they don't fire until ~1d history fills (acceptable/desired).
- D6 DHCP exhaustion: per-network `client_count_by_network / on(network) pool_size > 0.9`. `homelab_unifi_dhcp_lease_count` is a SITE total `{}` with NO network label -> CANNOT join per-network -> it is NOT the numerator. 015 ships the per-network ratio ONLY; no site-level lease alert. Empty-join trap is understood-and-safe here (both sides come from the same per-network collector set).
- D7 DNS steering: `dhcp_dns_primary{dns!="192.168.2.148"}` (label-NEQ; the IP is the `dns` label) + `absent(homelab_unifi_dhcp_dns_primary)` companion as a distinct critical alert. Pi-hole IP hardcoded (versioned config) per design.
- D9 Threat: `homelab_unifi_threat{type} > 0` (NOT threat_count scalar); annotation names `{type}`.
- D10 Single `unifi` group; 23 promtool-unit-testable + 2 live-replay-only (#24/#25 subqueries).

- **DHCP pool exhaustion (from STAGE-007-011)** -> BUILT here as rule #18 (per-network ratio).

STAGE-007-024 (`unifi.json` "DPI top-apps") is the first `rate()`-over-DPI consumer and now carries the clamp as a concrete deliverable. NOT a silent drop; NOT a fabricated rule (YAGNI).
