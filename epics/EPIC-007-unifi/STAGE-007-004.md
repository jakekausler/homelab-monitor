# STAGE-007-004: Cardinality caps + identity-upsert helper (`stat/sta` + `stat/alluser` → registry)

## Status: Not Started

## Overview

The shared helper layer the Wave-B collectors build on: reuse the EPIC-005-004 cardinality-cap mechanism for
per-client/per-device/per-DPI series, and build the **identity-upsert helper** that maps `stat/sta` (active)
and `stat/alluser` (known) records into the `unifi_clients` registry (upsert by MAC, append a time-stamped
IP↔MAC observation, attribute the host). This isolates the identity-normalization logic so the active-client
collector (007) and any later consumer reuse it.

**Vertical slice:** given fixture `stat/sta` + `stat/alluser` payloads, the helper upserts the registry,
appends observations, attributes the host correctly, and respects the cardinality cap; unit-tested against
fixtures.

**Spec references:** spec §6.1 (registry). EPIC-005 STAGE-005-004 (the cardinality cap — REUSE, do not
reinvent). EPIC-007 "Persistent client registry" + "Client-join contract". STAGE-007-003 (the table this
writes to).

## Cross-cutting standards

- REUSE the EPIC-005-004 cardinality cap (default top-N) — do not invent a new cap.
- Host attribution: the configured `unifi_host_lan_ip`=`192.168.2.148` maps to the first-class host row.
- All internal timestamps UTC; observation appends are time-stamped.

## Concrete deliverables

### 1. Cardinality-cap wiring

Apply the 005-004 cap to per-client series (identity rollups, per-client stats, DPI). Document the
per-family caps in the bundle config.

### 2. Identity-upsert helper

A pure function/helper: `stat/sta` + `stat/alluser` records → registry upserts (by MAC) + time-stamped
IP↔MAC observation appends. Handles: wired vs wireless field shapes (`sw_mac`/`sw_port` vs `ap_mac`/`essid`/
`signal`), `fixed_ip`/`use_fixedip`, online/offline, host attribution, and graceful handling of records
missing fields. Returns a structured result (counts upserted/observed) for the collector to emit as
self-metrics.
