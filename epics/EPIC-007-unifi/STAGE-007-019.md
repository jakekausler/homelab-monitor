## Design Phase

**Status**: [x] Design (Complete)

### Design Notes

**Selected approach** (coordinator selection per stage-resume pre-authorization; brainstormer consulted; no user prompt). Mirrors the EPIC-005 Home Assistant integration-router pattern exactly (`APIRouter(prefix="/integrations/unifi")`, per-section GET endpoints with Pydantic `response_model`, `Depends(require_session())` â€” CSRF not enforced on GET, VM reads via the shared `vm_instant_query()`/`vm_count()` helpers, `HttpProblem(502)` on VM failure). No new abstraction.

**Upstream-contract verification (this stage CREATES the contract 020-023 consume; verified its SOURCES are REAL):** Unifi `homelab_unifi_*` metric series are emitted by live collectors (device/wan/alarms/active_client/client_stats/client_dpi/networkconf/controller_up/vpn_teleport) â€” REAL, not stubs. Registry `UnifiClientRepo` (`list_clients`/`get_client`/`find_mac_by_ip_at`) returns `UnifiClientRow` carrying use_fixedip/network/lease_expiry/online/is_host â€” REAL. VL log stream via existing `/api/logs/query` (`service:X AND source_type:Y`). `use_fixedip` already extracted in `kernel/unifi/identity.py` + stored in the registry.

**Decision 1 â€” reservation_count = NEW collector metric (1a), NOT endpoint-computed.** Emit `homelab_unifi_dhcp_reservation_count{network}` from a collector (count live `stat/sta` clients with `use_fixedip==true` grouped by network). Rationale: the card NAMES it as a metric ("Expose `homelab_unifi_dhcp_reservation_count{network}`") AND the STAGE-025 Grafana network dashboard must be able to query it â€” a registry-only count is invisible to Grafana. No migration (a metric series is not a table). Honors the STAGE-011 deferral. (Endpoint-computed from the registry was rejected: not Grafana-queryable, contradicts the card's metric naming, registry network may lag live stat/sta.)

**Decision 2 â€” granular endpoints (2a): one endpoint per panel section, mirroring HA, + a `/summary` aggregate.** Gives 020-023 clean independent subsets, per-widget error isolation, clean per-endpoint Pydanticâ†’OpenAPI codegen. (Fat per-page aggregate endpoints rejected: diverge from HA pattern, couple unrelated widgets, churn the shared contract.) Endpoint list:
- Unifi panel: `GET /integrations/unifi/summary` (status strip: controller_up, devices_up/total, threat_count, teleport_up, wan_up), `/devices`, `/devices/{mac}` (404 if no series), `/threats`, `/dpi`, `/teleport`, `/controller-health`.
- Network page: `/network/wan` (CURRENT WAN + latest speedtest values + lastrun + failover), `/network/dhcp` (pools + occupancy + reservation_count), `/network/wifi` (experience + SSID distributions), `/network/dns-posture`.
- Clients: `/clients` (paginated/sortable roster from registry), `/clients/{mac}` (registry identity/lease/reservation/online JOINED with VM per-client signal/bandwidth/DPI + DNS slot; 404 if get_client None).

**Decision 3 â€” speedtest/WAN HISTORY via the EXISTING `GET /api/metrics/range` proxy (3b), NOT a new 019 endpoint.** `vm_query.py` has only instant helpers; a generic typed `/api/metrics/range` proxy already exists (`routers/metrics.py`). 019 serves CURRENT WAN/speedtest values (instant); the Network page draws trend sparklines via `/api/metrics/range` with a Unifi PromQL expr. Avoids duplicating tested range code; respects the STAGE-024/025 Grafana-history boundary. Documented in the `/network/wan` docstring.

**Decision 4 â€” DNS-enrichment extension point = in-payload nullable slot (4a); registry is source-of-truth for `online`.** Client-detail response carries `dns: DnsEnrichment | None = None` (docstring: "populated by EPIC-006 Pi-hole join via UnifiClientRepo.find_mac_by_ip_at"; ships None until EPIC-006 â€” non-breaking to fill later). `DnsEnrichment` reserves `top_domains/blocked_count/last_query_at`. The roster/detail `online` flag comes from the REGISTRY (authoritative MAC-keyed record), NOT a per-client VM presence query; VM presence feeds only the aggregate `/summary` counts.

**Empty/error states IN the payload** (mirror HA's `ha_up=false`): "no threats" â†’ `[]`; "controller down" â†’ `controller_up=false` + last-known values; only genuine VM transport/query failure â†’ `HttpProblem(502)`. `/clients/{mac}` not-found â†’ 404.

**Gate:** Python â†’ `make verify` (GLOBAL 100% BRANCH coverage, pyright --strict, no bare Any).
