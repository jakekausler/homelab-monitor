# STAGE-007-019: Backend panel/page data endpoints (Unifi panel + Network page + Client page)

## Status: Not Started

## Overview

The backend data endpoints the three UI surfaces consume — typed FastAPI routes that aggregate the collected
metrics + registry into panel/page-shaped payloads. Built before the frontend stages (per the backend-
contract-first principle) so the UI is never built against a non-existent contract.

**Vertical slice:** `GET` endpoints return real, typed data for: the Unifi panel (device list + drill-down +
threats + DPI + Teleport + controller health), the Network page (WAN/DHCP/WiFi-experience/SSID/DNS-posture),
and the Client page + Clients tab (registry rows + per-client detail). Verified against live data.

**Spec references:** spec §9.2 (per-integration panel = plugin-provided panel; typed FastAPI routes →
OpenAPI → frontend). EPIC-007 "UI structure" (the three surfaces). EPIC-005/006 panel-endpoint precedent
(006-020).

## Cross-cutting standards

- Typed routes → OpenAPI → `openapi-typescript-codegen` (the frontend consumes generated types).
- Read-only (observe-only epic) — no mutating endpoints.
- The Client-page endpoint exposes the **DNS-enrichment extension point** shape (a documented place EPIC-006
  plugs DNS-behavior data into) so the frontend Client page renders a slot EPIC-006 later fills.
- Empty/error states represented in the payloads.

## Concrete deliverables

### 1. Unifi panel endpoints

Device list + per-device drill-down (ports/PoE/radios/outlets), threats list, DPI top-apps, Teleport state,
controller/API health, overview/status-strip.

### 2. Network page endpoints

WAN/ISP + speedtest history, DHCP pool/lease occupancy, WiFi-experience distributions, SSID client
distribution, DNS-posture (handout-vs-expected).

### 3. Clients tab + Client page endpoints

Registry roster (paginated/sortable) for the Clients tab; per-client detail (identity + connection + uptime
+ bandwidth + per-client DPI + lease field + online/offline history) for the Client page, **with the
documented DNS-enrichment extension slot** for EPIC-006.
