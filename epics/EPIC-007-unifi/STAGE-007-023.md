# STAGE-007-023: Embedded `<LogViewer>` (UDM syslog) + threat-forensics view

## Status: Not Started

## Overview

Wire the embedded `<LogViewer>` into the Unifi panel, scoped to the UDM syslog stream (`service:"udm-*"`,
with device/category filtering), plus a **threat-forensics** view (the raw IDS/IPS CEF lines — forensics
only, the alert fires off the structured alarm path). Uses the documented EPIC-004 `<LogViewer>` embedding
contract.

**Vertical slice:** the Unifi panel shows an embedded LogViewer over the UDM syslog stream; the device
drill-down pre-filters to that device's lines; a threat-forensics view surfaces IDS/IPS detection lines for
incident review.

**Spec references:** spec §9.2 (Logs embed). EPIC-004 STAGE-004-003 (`<LogViewer>` embedding contract;
`apps/ui/src/components/logs/README.md`). EPIC-007 "Events pipeline" (CEF stream, `service="udm-*"`; threat
forensics-only). STAGE-007-016 (the stream this views).

## Cross-cutting standards

- Consume `<LogViewer>` via the documented EPIC-004 embedding contract (caller-provided `useLogs` hook scoped
  to `service:"udm-*"` [+ device/category]). Do NOT build custom log infrastructure.
- Threat lines are forensics ONLY (no alert from this surface).
- FRONTEND stage: Desktop + Mobile testing + explicit approval; any code change resets both.

## Concrete deliverables

### 1. Panel LogViewer

Embed `<LogViewer>` (panel-level, all UDM categories) using the contract; the device drill-down embeds a
device/category-filtered instance.

### 2. Threat-forensics view

A view/filter over the `service="udm-*"` security-category lines for IDS/IPS incident review (the
`UnifiThreatLog` saved filter — forensics, no alert).
