# STAGE-007-002: `integrations/unifi/` bundle skeleton + registration

## Status: Not Started

## Overview

Stand up the Unifi integration-bundle skeleton and register it, mirroring EPIC-005's STAGE-005-003 (the
integration-bundle skeleton + registration pattern). This is the structural home for all Wave-B/C
collectors, the alert-rule files, the panel, and the Grafana dashboards — created once so subsequent stages
drop into a known layout.

**Vertical slice:** the `unifi` integration bundle is discovered + registered at startup (visible in the
plugin/collector status API) with a no-op/placeholder collector that proves the bundle loads; the bundle's
directory structure exists for collectors/rules/panel to land in.

**Spec references:** spec §5 (plugin/collector/integration_bundle framework), §5.5 (plugin discovery +
registration). EPIC-005 STAGE-005-003 (the exemplar skeleton/registration). EPIC-006 STAGE-006-002 (sibling
Pi-hole skeleton).

## Concrete deliverables

### 1. Bundle skeleton

The `integrations/unifi/` package: `__init__.py` (bundle registration), a place for collectors, and the
config-schema (pydantic) for the bundle's settings (`unifi_url`, cardinality caps, intervals, the opt-in
SSH-lease flag default-false, the `unifi_host_lan_ip`=`192.168.2.148` for host attribution). Mirror the
EPIC-005 layout.

### 2. Registration + placeholder

Register the bundle so the plugin host loads it; a placeholder/no-op collector proves the bundle wires into
the scheduler and appears in the collector status API. Real collectors replace it in Wave B/C.
