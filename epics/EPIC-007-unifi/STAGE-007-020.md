## Design Phase

### Design Notes

**Selected approach** (brainstormer-backed; recommendation selected per pre-authorized Design rule — no user prompt):

1. **Panel structure** — Lift the header status strip into `UnifiLayout` (persistent across tabs, future-proofs 022/023 tabs). Keep a single `Overview` content tab (`UnifiOverviewTab`) holding the device table + the four widgets (threats / DPI / Teleport / controller-health) in a responsive `grid-cols-1 sm:grid-cols-2` PanelSection grid, mirroring `HomeAssistantHealthTab`. Avoids manufacturing an empty "Security" tab — threats and DPI are legitimately empty on this UDM.
2. **Device drill-down** — Dedicated route-param page `/integrations/unifi/devices/$device` with a back-link, mirroring the repo's locked `ContainerPage` / `CronDetailPage` convention. Deep-linkable, refresh-safe, natural home for the `/devices/{device}` 404 case, best mobile ergonomics. Build gotcha: the `$device` param is a MAC containing colons — URL-encode/decode it.
3. **Kind-conditional drill-down rendering** — Data-driven: render a `PanelSection` for each non-empty array (`ports[]`, `radios[]`, `outlets[]`) plus an always-on System section (cpu/mem/load/temps). No `switch (kind)` branching (which can drift from actual data).

**Honored regression item:** radio `satisfaction == -1.0` (UniFi "not computed" sentinel) is rendered as "n/a"/blank inside the radio renderer (owned by this stage per EPIC-007 regression.md).

**Backend contract verification (DONE):** the seven consumed endpoints (`/summary`, `/devices`, `/devices/{device}`, `/threats`, `/dpi`, `/teleport`, `/controller-health`) in `kernel/api/routers/integrations_unifi.py` were confirmed to return REAL VM-derived data (not stubs), with typed contracts present in `apps/ui/src/api/schema.ts`. `UnifiDeviceRow` has NO `satisfaction` field; `satisfaction` lives only on ports/radios. 200-empty vs 502 vs 404 semantics confirmed.

**Patterns to mirror:** new `apps/ui/src/api/unifi.ts` per-endpoint react-query hooks (clone `home_assistant.ts`); router child routes under `unifiLayoutRoute` + a `$device` detail route mirroring `containerPageRoute`; loading/502/empty/data states per `HomeAssistantHealthTab`.

**Scope guard:** flat device table, NO topology graph (deferred per card). Embedded `<LogViewer>` is NOT in this stage (owned by STAGE-007-023).
