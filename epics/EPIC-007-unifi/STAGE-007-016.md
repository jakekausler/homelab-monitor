# STAGE-007-016: vector CEF syslog source + VL `service="udm-*"` labels + UDM-side enable + validate

## Status: Not Started

## Overview

The events pipeline — add a CEF-aware `syslog` source to vector, enable UDM remote syslog forwarding (a
UDM-side write, done HERE at the stage, not earlier), and land the raw event stream into VictoriaLogs with
`service="udm-<category>"` labels so the per-`service` Drain models (EPIC-004) partition automatically and
the `<LogViewer>` device/category filter works. **This MUST precede the log-alert rules (017) so the rules
validate against real lines.**

**Vertical slice:** UDM syslog (CEF) flows into VL with correct `service` labels; the parse extracts CEF
fields (`UNIFIcategory`/`UNIFIsubCategory`/`src`/`msg`/signature-id); lines are searchable in the Logs
explorer.

**Spec references:** spec §6.3 (VL streams; UDM syslog forwarded → VL ingest), §4.3 (logs flow). EPIC-004
(the vector→VL pipeline + per-`service` Drain models). EPIC-007 "Events pipeline" + "Verified deployment
reality" (CEF-over-syslog, NOT plain RFC5424). EPIC-007 follow-up notes mirror EPIC-008's syslog-stage
redaction/`service`-label requirements.

## Cross-cutting standards

- **CEF-aware parse** (NOT strict RFC5424 — UniFi OS messages fail strict RFC5424 "version must be 1"
  parsers). VRL transform extracts CEF fields; `service="udm-<category>"` per facility/category.
- Redaction at ingest: add UDM-specific bearer/API/session-token patterns to the redaction pipeline
  (`homelab-monitor.yaml` under `logs.redact:`) so syslog content is redacted before reaching VL (mirrors
  the EPIC-004/EPIC-008 follow-up requirement).
- Per-stream caps (lines/sec, bytes/day) on the new UDM stream.
- The UDM-side enable step is documented (Settings → Control Plane → Integrations → Activity Logging/SIEM →
  `192.168.2.148:<port>`, all categories) and performed at THIS stage.

## Concrete deliverables

### 1. vector CEF syslog source

A new `[sources.udm_syslog]` (`type = "syslog"`, UDP/TCP, non-privileged port) + a VRL transform that parses
CEF and emits `service="udm-<category>"` labels into VL. Add the per-stream cap + the redaction patterns.

### 2. UDM-side enablement + validation

Enable UDM remote syslog → the vector listener; confirm CEF lines land in VL with correct labels and are
searchable in the Logs explorer. Document the enable steps.
