## Overview

The events pipeline — add a CEF-aware `syslog` source to vector, enable UDM remote syslog forwarding (a
UDM-side write, done HERE at the stage, not earlier), and land the raw event stream into VictoriaLogs with
`service="udm-<category>"` labels so the per-`service` Drain models (EPIC-004) partition automatically and

fields (`UNIFIcategory`/`UNIFIsubCategory`/`src`/`msg`/signature-id); lines are searchable in the Logs
explorer.

(the vector→VL pipeline + per-`service` Drain models). EPIC-007 "Events pipeline" + "Verified deployment

### Design Notes

**Selected approach (coordinator-selected from brainstormer options; user not prompted per stage pre-authorization):**

**Protocol / port / LAN reachability.** vector gains a UDP `syslog` source listening on `0.0.0.0:5514` (non-privileged; avoids 514). Because the UDM (192.168.2.1) is a LAN device and CANNOT reach a container-internal sidecar, the PROD compose publishes this ONE host port — a **deliberate, documented exception** to the "sidecars are container-internal / 127.0.0.1-only" invariant. Prod: `ports: ["${HM_UDM_SYSLOG_BIND_HOST:-0.0.0.0}:${HM_UDM_SYSLOG_PORT:-5514}:5514/udp"]` (defaults to 0.0.0.0 so the LAN UDM reaches 192.168.2.148:5514; operator may pin to the LAN IP via env for a tighter bind). Dev: `ports: ["127.0.0.1:${HM_DEV_UDM_SYSLOG_PORT:-15514}:5514/udp"]` (dev stays 127.0.0.1-only, 1xxxx port). The exception is recorded in the CLAUDE.md port-map table + spec. UDP chosen over TCP: matches UniFi's native remote-syslog, no TCP backpressure into the pipeline.

**CEF parse (VRL, vector 0.41.1 has NO parse_cef, NO lookaround support).** New `udm_parse` remap transform: anchor on the `CEF:` marker via `find`/`slice!` (envelope-agnostic — tolerates UniFi's loose, non-RFC5424 syslog prefix), `split("|", limit: 8)` the guaranteed 7-field CEF header (extract signature-id, name, severity), then `parse_key_value()` the extension tail (field_delimiter=" ", key_value_delimiter="=") to read UNIFIcategory / UNIFIsubCategory / src / msg WITHOUT regex lookarounds. Error-tolerant `??` fallbacks so a malformed line resolves to `udm-other` and never poisons the pipeline. `.message` is set to the human-readable `msg` (feeds redaction + VL `_msg`).

**Service mapping (low-cardinality stream-field hygiene).** `service` is a VL STREAM field, so it must stay low-cardinality. UNIFIcategory is freeform/unbounded → map it through a bounded allow-list VRL if-chain to: `udm-threat` / `udm-firewall` / `udm-vpn` / `udm-client` / `udm-system` / `udm-wireless` / `udm-other` (fallback). The RAW category is preserved in a non-stream field `.udm_category` for LogViewer secondary filtering + operator drill-down. (The allow-list buckets are a best-guess vocabulary — see regression item; real category coverage is validated at live Refinement 3b.)

**source_type.** Set `.source_type = "udm"` directly in `udm_parse` (do NOT modify the existing asserted `source_type_classify` chain). Both `service` and `source_type` are load-bearing: the backend log-query endpoint + STAGE-023 LogViewer filter on `service:X AND source_type:Y`.

### Refinement Findings — Design Invalidated (returning to Design)

Refinement validated the pipeline INFRASTRUCTURE end-to-end (synthetic CEF packets: source → VRL → VL with `service`/`source_type` labels, redaction, throttle, LogViewer-compatible `service:X AND source_type:Y` filtering all confirmed). TWO real bugs were found+fixed during Refinement (kept in the redesign): (a) `to_string(x) ?? ""` is illegal VRL (infallible → E651) — fixed 5 lines; (b) Vector's `syslog` source parses `CEF` as the syslog appname tag and strips the `CEF:` prefix — VRL now reconstructs `raw = "CEF:" + message` when `appname == "CEF"`.

**BUT live UDM data (operator enabled remote syslog) invalidated the core format assumption:**

1. **The real UDM stream is NOT CEF.** A 200-record live sample (real UDM at `source_ip=192.168.2.1`, `appname=UDM-Pro`) showed: **iptables firewall logs 70%** (`DESCR="..." IN=br0 SRC=... DST=... PROTO=... SPT=... DPT=... MARK=...`), **systemd process logs 26%** (`systemd[pid]: ...`), **sshd auth logs 2%** (`sshd[pid]: ...`), **misc daemon 1%**. **ZERO real CEF lines.** 100% fell to `udm-other` because the CEF field-extraction got no CEF data.

2. **Why:** Per Ubiquiti SIEM docs (https://help.ui.com/hc/en-us/articles/33349041044119-UniFi-System-Logs-SIEM-Integration), UniFi 9.x SPLIT logging into TWO paths with DIFFERENT formats: **Activity Logging (Control Plane → Integrations) = CEF** (admin/device/client events, `UNIFIcategory=...`) and **Traffic Logging (CyberSecure/Policy Engine) = iptables** (firewall/IPS). Firewall syslog is also PER-RULE (each rule → Advanced → Enable Syslog Logging). The single-format CEF assumption in the original EPIC-007 design (labelled "Verified deployment reality") was wrong — that recon predated enabling the real stream.

3. **CRITICAL — silent packet loss.** Vector's `type="syslog"` source strictly parses RFC3164/5424 and SILENTLY DROPS unparseable datagrams BEFORE the VRL runs (~24-30+ packets/hour observed dropped in vector logs: "unable to parse input as valid syslog message"). This is data loss with no visibility — exactly the failure mode the parser must NOT have.

**Redesign direction (locked at re-Design): parse ALL formats, never drop, alert on parse failure (operator-requested):**
- **Source: `syslog` → `socket` (type="socket", mode="udp", 0.0.0.0:5514).** Raw UDP — every datagram becomes an event, ZERO source-level drops. The VRL does its OWN envelope parsing.
- **VRL: error-tolerant `parse_syslog()` first** (fall back to raw `.message` if the envelope is malformed — never lose the line), then **multi-format content branching**: CEF (when present → existing extraction), iptables (`DESCR=`/`IN=`/`SRC=` → field extraction, with quoted-`DESCR="..."` handled via regex-extract-then-strip-then-parse_key_value, since DESCR contains spaces that break naive space-split), systemd/sshd/plain (passthrough with sensible labels).
- **Service buckets reworked for the REAL taxonomy:** `udm-firewall` (iptables), `udm-system` (systemd), `udm-auth` (sshd), `udm-threat`/`udm-client`/etc. (CEF when it arrives), `udm-other` (genuinely unclassifiable — should be near-zero).
- **Parse-failure metric + alert:** emit `homelab_udm_parse_failed_total` (incremented in VRL when a line can't be classified) via the existing vector internal-metrics → prometheus_exporter:9598 → vmagent scrape → VictoriaMetrics path; alertable by vmalert (pairs with STAGE-017).
- **Keep** the redaction patterns, throttle, the appname/CEF reconstruction fix, and the Python reference-test approach (extended for the new formats).

**Operator note (logging enablement):** the CEF (`UNIFIcategory`) events require the Activity Logging (Syslog) → SIEM Server toggle specifically under Control Plane → Integrations (currently producing nothing — re-verify it's on + pointed at 192.168.2.148:5514); firewall coverage requires per-rule syslog enablement; multi-site requires per-Site config. The redesigned parser does NOT depend on any of these being complete — it parses whatever arrives.

### Redesign — Locked Design (multi-format, binary-verified)

The redesign was verified by running ALL 20 real captured fixture lines through the actual `timberio/vector:0.41.1-debian` binary (20/20 → correct events, zero VRL errors). Real samples captured via tcpdump from the live UDM Pro (UniFi Network 10.4.57) are in `apps/monitor/tests/fixtures/udm_syslog_samples.txt` (anonymized: admin name → "Admin User"; public IPs → doc ranges; MAC host-octets masked; a real mcad authkey was REDACTED).

**Real stream taxonomy (from the capture):** iptables firewall (UDP/TCP/ICMP/IPv6, ~80%), systemd/sshd/daemon (~18%), **CEF audit events (login "Network Accessed" + config-change "Config Modified", `UNIFIcategory=Audit`)** (~rare but the highest-value). Two envelope shapes share the stream:
- **Standard lines** (iptables/systemd/etc.): `<PRI>` + **DOUBLED hostname** `UDM-Pro UDM-Pro` + body.
- **CEF audit lines:** `<PRI>` (`<30>`) + **SINGLE hostname** `UDM-Pro` + `CEF:0|Ubiquiti|UniFi Network|...`.

**Root cause the CEF audit events were "missing" (DEFINITIVELY: mis-parse, NOT drop):** the syslog source ACCEPTS them, but the old `parse_key_value(ext, field_delimiter:" ")` SILENTLY TRUNCATES every space-containing CEF extension value (`UNIFIadmin=Admin User`→"Admin", `msg=<full sentence>`→"Admin"), so audit events land with a near-empty message and look absent when searched by content. (If audit events are ENTIRELY absent from VL, that's an upstream operator/routing check for Refinement — the vector config accepts them.)

**Locked redesign (each point binary-verified against the real fixtures):**

1. **Source: `type="syslog"` → `type="socket"` (mode="udp", raw, LOSSLESS).** Socket never drops and hands VRL the INTACT wire line (full `<PRI>` + envelope). The syslog source pre-mangled `.appname` (took the doubled hostname / took "CEF" as the tag) — socket avoids all that. `parse_syslog()` does NOT handle the doubled hostname (it consumes the 2nd `UDM-Pro` as appname and leaves the real process tag in `.message`), so we do a CUSTOM envelope strip, not `parse_syslog`.

2. **Custom two-shape envelope strip (handles BOTH):** `parse_regex(raw, r'^<(?P<pri>\d+)>(?P<ts>[A-Z][a-z]{2}\s+\d{1,2}\s+\d{2}:\d{2}:\d{2})\s+(?P<host>\S+)\s+(?P<body>.*)$')`; then strip the DUPLICATED leading hostname only when present (`dup = host + " "; if starts_with(body, dup) { body = slice past it }`) — single-hostname CEF lines simply don't match and pass through. Severity from `mod(to_int(pri), 8)`.

3. **Three-way body branch:** `starts_with(body, "CEF:")` → **CEF/audit** (`service="udm-audit"`); `match(body, r'^\[[A-Z]')` (the `[CHAIN]` bracket) → **iptables/firewall** (`service="udm-firewall"`); else → **systemd/sshd/daemon** (`service="udm-system"`). `source_type="udm"` always.

4. **CEF extension parse — FIX the space-truncation bug** via a boundary pre-split: `norm = replace(ext, r' (UNIFI[A-Za-z]+|src|dst|spt|dpt|msg|cs\d|cn\d)=', "\n$$1=")` then `parse_key_value(norm, field_delimiter: "\n", key_value_delimiter: "=")`. Recovers FULL `UNIFIadmin='Admin User'`, `UNIFIsettingsSection='Firewall Group'`, `UNIFIcategory='Audit'`, `src`, and full `msg`. Extract CEF header sig-id/name/severity from the `|`-split. (CRITICAL: `$$1` NOT `$1` — `$1` collides with vector's env-var interpolation and breaks config load with "Missing environment variable name=1".)

5. **iptables parse:** extract `[CHAIN]` via regex; extract quoted `DESCR="..."` via regex (values contain spaces + brackets, e.g. `PortForward DNAT [Nginx SSL]`, truncated rule names) then `replace()` it out; `parse_key_value` the space-free remainder for IN/OUT/MAC/SRC/DST/LEN/PROTO/SPT/DPT/MARK + TCP(SEQ/ACK/WINDOW/SYN)/ICMP(TYPE/CODE)/IPv6(FLOWLBL/HOPLIMIT/TC) extras.

6. **systemd/sshd/daemon parse:** `parse_regex(body, r'^(?P<proc>[a-zA-Z0-9._-]+)(\[(?P<pid>\d+)\])?:\s+(?P<msg>.*)$')` → `.process`, clean `.message`.

7. **Parse-failure metric + alert (operator-requested "alert when a log fails to parse"):** when the envelope regex fails OR the body matches no branch cleanly, set a marker and emit `homelab_udm_parse_failed_total` via the existing vector internal-metrics → prometheus_exporter:9598 → vmagent → VictoriaMetrics path (mirror the `redaction_metric`/`vector_redactions_total` pattern). vmalert alerts on `homelab_udm_parse_failed_total > 0` (pairs with STAGE-017). This is the safety net for formats we haven't seen.

8. **Service buckets reworked for the REAL taxonomy:** `udm-audit` (CEF), `udm-firewall` (iptables), `udm-system` (systemd/sshd/daemon), `udm-other`/parse-failed (fallback — should be near-zero). Keep the per-stream throttle + redaction branch. **Redaction MUST cover the mcad-authkey-style token** seen in cleartext on the wire — add a UDM authkey pattern to DEFAULT_REDACT_PATTERNS.

9. **VRL fallibility discipline (binary-verified):** `to_string()` is INFALLIBLE on `split()` elements + `parse_regex` named captures (NO `?? ""`), but FALLIBLE on `.field` paths and `parse_key_value` members (REQUIRE `?? ""`). No regex lookarounds anywhere (Rust-regex constraint).

**Tests (operator asked for MANY regression types):** replace the synthetic CEF fixture with the REAL captured samples (`apps/monitor/tests/fixtures/udm_syslog_samples.txt`). Build a comprehensive pure-Python reference + per-format regression tests covering EVERY captured format: CEF-login, CEF-config-change (with space-containing UNIFIadmin/settings/msg — the truncation regression), iptables UDP/TCP/ICMP/IPv6/named-rule/quoted-DESCR, systemd, sshd, other-daemon, doubled-hostname envelope, single-hostname CEF envelope, a parse-failure/unknown line (asserts the parse-failed marker fires), and a redaction case (authkey redacted). Keep the `@pytest.mark.slow` `vector validate` test as the authoritative VRL-execution gate; live Refinement re-captures real events to confirm.

### Refinement Notes

Validated live end-to-end on the prod rig (synthetic + real traffic). THREE parse bugs were found by live validation and fixed (none catchable by `make verify` — all needed the real binary + real UDM traffic):
1. **Trailing newline** — every real UDP datagram ends with `\n`; Rust-regex `.*$` (no `(?m)`) rejected ~100% of real lines → all fell to `udm-other`. Fix: `raw = strip_whitespace(...)`. Caught by the parse-failed metric (it read ~100% failure on real traffic while synthetic `nc` packets — no trailing `\n` — passed). My synthetic test packets hid the bug.
2. **No-<PRI> CEF** — some real CEF audit lines arrive as `Mon DD HH:MM:SS host CEF:...` with no `<PRI>`. Added a no-PRI envelope fallback (severity unknowable → `syslog_severity=""`).
3. **iptables empty-value field** — `OUT= SRC=...` made `parse_key_value` swallow the next key. Replaced with per-field `parse_regex(r'\b<KEY>=(?P<v>\S*)')`. (Found at Build render+validate.)

**Live results:** real traffic parses at 0% failure (315 lines / 0 failed in a 40s window). Real CEF audit events (operator login + Pi-Hole-DNS Firewall-Group rename revert) → `udm-audit` with `udm_admin="Jake Kausler"`, `udm_settings_section="Firewall Group"`, `udm_settings_entry="Pi-Hole DNS"` ALL fully captured (the $$1 boundary pre-split on real multi-word values). Real iptables → `udm-firewall` (src/dst/proto/chain/descr); real systemd/mcad → `udm-system` (process set).

**Operator-requested change:** the `UnifiUdmLogParseFailed` vmalert alert `for:` was changed 10m→0m so ANY unparseable UDM line surfaces immediately (no 10m silent window where "some get through"). The alert is LOADED + ACTIVE in prod vmalert-metrics; the metric `udm_lines_total{parse_failed="1"}` reaches VM via vector:9598→vmagent→VM.
