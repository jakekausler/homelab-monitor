# STAGE-007-010: Alarms/threats collector (`rest/alarm`)

## Status: Complete

## Overview

The structured threat collector — polls `rest/alarm?archived=false` for IDS/IPS / threat alarms and emits a
count + by-type breakdown. This is the **structured** threat path that drives the `UnifiThreatDetected`
alert (the syslog CEF stream provides the raw forensic lines separately, but only THIS path pages — no
double-page).

**Vertical slice:** the collector emits `homelab_unifi_threat_*` metrics against the live UDM; visible in VM
and the threats widget; feeds the alert.

**Spec references:** spec §6.2. EPIC-007 "Metric families" (alarms row) + "Events pipeline" (the
two-paths-one-alert design). EPIC-007 alert catalog (`UnifiThreatDetected`).

## Design Phase

**Status**: [x] Design (Complete)

### Design Notes (locked)

**Live recon (READ-ONLY, today):** `rest/alarm?archived=false` returns `{"meta":{"rc":"ok"},"data":[]}` — EMPTY (quiet network). Even `archived=true` is empty; `stat/alarm`/`stat/event` 404. ZERO alarms to observe live. Refinement validates the empty path live; the POPULATED path is validated via a synthetic fixture against the known UniFi IPS/IDS alarm shape: `{ "_id", "key" (e.g. EVT_IPS_IpsAlert / EVT_IDS_*), "subsystem" ("ids"/"wlan"/"lan"/"vpn"), "catname" (human category), "msg", "time" (epoch), "datetime" (iso), "archived", ... }`.

**Client helper:** `rest_alarm()` ALREADY exists on the UnifiClient Protocol + UnifiRestClient (hits `rest/alarm?archived=false`). No new helper needed.

**Dedup semantics (Decision 1 = A, stateless):** "Dedup by `_id` so a still-open alarm doesn't re-fire each poll" is satisfied by GAUGE IDEMPOTENCY at the collector + change/threshold detection at the alert (STAGE-007-015). The collector is STATELESS (like every sibling unifi collector). "Dedup by `_id`" concretely = count DISTINCT active alarm `_id`s within the current response (defensive against duplicate records). Cross-poll "don't re-page" is OWNED by STAGE-007-015's PromQL (`> 0` for active-threat, or `increase()`/`changes()` for new-alarm semantics) on the idempotent gauge — NOT by collector state. (Rejected: cross-poll state in the collector — breaks the stateless-collector invariant + smuggles alert logic into the collector + restart-fragile.)

**`{type}` label (Decision 2 = `key` + fallback):** `homelab_unifi_threat{type}` label value = the alarm's `key` (machine-readable, low/stable cardinality, e.g. `EVT_IPS_IpsAlert`), with a fallback chain `key` -> `subsystem` -> `"unknown"` so the label is NEVER empty. Human-readable `catname` is NOT the metric label (free-text -> cardinality drift); it is surfaced in 015's alert annotation / the UI display mapping (020/024) instead.

**Metric set + always-emit (Decision 3 = A):**
- ALWAYS emit `homelab_unifi_threat_count` = number of DISTINCT active alarm `_id`s (0 on the quiet/common path). Always-present count=0 avoids the `absent()` trap STAGE-007-015's UnifiThreatDetected alert would otherwise hit.
- ALWAYS emit `homelab_unifi_api_took_seconds{endpoint=<result.endpoint>}` (use the response's actual endpoint string, which is `rest/alarm?archived=false`, mirroring wan.py's use of `result.endpoint`).
- Emit `homelab_unifi_threat{type=...}` ONLY for types present in the current response (no zero-series for never-seen types — types are open-ended). Each is the distinct-`_id` count for that type.

**Cardinality cap (Decision 4 = none):** No cap. `key`/`subsystem` are a bounded IDS/IPS enum; `CappedEmitter`'s LEXICAL top-N drop would arbitrarily HIDE threat types (bad for a security signal). **Conditional deferral:** if the `{type}` label source ever becomes free-text `catname` (e.g. a future UI/dashboard request at STAGE-007-020/024), add a `unifi_threat` cap family in `kernel/config.py _DEFAULT_CARDINALITY_FAMILIES` at that time. Tracked as a conditional note here; the current `key`-based design needs no cap.

**Error/empty/graceful:** `ctx.unifi is None` -> ok=False. `rest_alarm()` UnifiError -> ok=False. Malformed payload (non-dict / non-list data) or empty data -> records=[] -> still emit latency + threat_count=0, ok=True. `_parse_records(payload)` local helper (copy from client_dpi.py): non-dict payload / non-list data -> []; skip non-dict entries; a record with no usable `_id` (missing/empty/non-str) is skipped from the distinct-id set (document the skip — coverage branch).

**ClassVars:** `name="unifi_threat"`, `interval=timedelta(seconds=60)`, `timeout=timedelta(seconds=15)`, `concurrency_group="unifi"`. Register in the unifi bundle `_UNIFI_COLLECTORS`. (`concurrency_group` is a ClassVar, NOT a CollectorConfig field — the test `_ctx`/`CollectorConfig` must NOT pass it.)

**Brainstormer:** ran (Opus); coordinator selected 1A / 2(key+fallback) / 3A / 4(none) per pre-authorization.
