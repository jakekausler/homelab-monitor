# STAGE-007-011: DHCP config + DNS-steering + pool-usage collector (`rest/networkconf`)

## Status: Complete

## Overview

The DHCP/DNS-posture collector — reads `rest/networkconf` for the per-network DHCP pool range, the
DNS-server handout (`dhcpd_dns_1..3`), and reservations. This drives the **DNS-steering drift** detector
(the sole DNS-protection signal after the firewall DNS-force rules were deleted) and DHCP pool-exhaustion.

**Vertical slice:** the collector emits `homelab_unifi_dhcp_*` + `dhcp_dns_primary` metrics against the live
UDM; visible in VM and the Network DHCP / DNS-posture widgets; feeds the alerts.

**Spec references:** spec §6.2. EPIC-007 "Metric families" (DHCP row) + "Verified deployment reality" (DNS
steering = DHCP handout, firewall rules deleted — Amendment 3) + alert catalog (`UnifiDnsSteeringDrift`,
`UnifiDhcpPoolExhaustion`).

- `unifi` group; 5m cadence (config changes slowly).
- The DNS-steering drift check compares `dhcp_dns_primary` against the configured expected
  (`192.168.2.148`) — the expected value is config, not hardcoded.
- Pool occupancy here is the API approximation (active-client/lease count vs pool size); the opt-in lease
  collector (012) sharpens it to authoritative lease count when enabled.
- Emits `homelab_collector_run_*`.

## Concrete deliverables

### 1. DHCP config / DNS-steering collector

Reads `rest/networkconf`; emits `dhcp_pool_size`, `dhcp_pool_used` (approx from active count),
`dhcp_dns_primary` (label/value for the drift check), reservation count, per-network. Feeds
`UnifiDnsSteeringDrift` (DNS handout no longer → Pi-hole) and `UnifiDhcpPoolExhaustion` (occupancy nearing
pool size). Surfaces in the Network page DNS-posture indicator + DHCP-pool widget + `network.json`.

## Design Phase

**Status**: [x] Design (Complete)

### Design Notes (locked)

**Live recon (READ-ONLY, today) — REAL data:** `rest/networkconf` returns 3 networks: 2 WAN (`purpose="wan"`, NO dhcp fields — filter out) + 1 "Default" (`purpose="corporate"`) with full DHCP. The "Default" net: `name="Default"`, `ip_subnet="192.168.2.1/24"`, `dhcpd_enabled=True`, `dhcpd_start="192.168.2.6"` (STRING IP), `dhcpd_stop="192.168.2.254"` (STRING IP), `dhcpd_dns_enabled=True`, `dhcpd_dns_1="192.168.2.148"` (Pi-hole — DNS steering ACTIVE), `dhcpd_dns_2="1.1.1.1"`, `dhcpd_dns_3="8.8.8.8"`. Pool size = `int(ip(stop)) - int(ip(start)) + 1` = 249. `rest/networkconf` has NO reservation array and NO active/used count.

**Client helper:** `rest_networkconf()` ALREADY exists (hits `rest/networkconf`). No new helper. Fakes already have a `rest_networkconf` stub.

**`dhcp_dns_primary` encoding (Decision 1 = A, info-gauge):** Emit `homelab_unifi_dhcp_dns_primary{network=<name>, dns=<dhcpd_dns_1>} = 1.0` (info-gauge; the IP is a LABEL since a gauge value must be numeric). The EXPECTED DNS ("must be 192.168.2.148") lives in STAGE-007-015's vmalert rule file (which IS versioned config) — the collector is a pure observer and bakes in NO policy. 015's UnifiDnsSteeringDrift alert compares the `dns` label (e.g. `homelab_unifi_dhcp_dns_primary{dns!="192.168.2.148"} == 1` plus an `absent()` companion for a vanished metric). Do NOT overload the existing `UnifiConfig.host_lan_ip` as the expected-DNS source. The Network page (021) reads the raw IP from the `dns` label directly.

**`dhcp_pool_used` (Decision 2 = A, downstream join):** This collector emits `dhcp_pool_size` ONLY. Occupancy ("approx from active count") is a PromQL join in STAGE-007-015: `homelab_unifi_client_count_by_network{network}` (VERIFIED to exist, emitted by STAGE-007-007 active_client.py, same `network` label) / `homelab_unifi_dhcp_pool_size{network}`. **Tracked deferrals:** the occupancy ratio / pool-exhaustion expr -> STAGE-007-015 (PromQL/recording rule, referencing `homelab_unifi_client_count_by_network`, NOT a non-existent `dhcp_pool_used` series); the AUTHORITATIVE lease-based "used" -> STAGE-007-012 (opt-in lease collector). No second API fetch here (single responsibility).

**`reservation_count` (Decision 3 = A, dropped + tracked):** `rest/networkconf` has NO reservation data — per-client fixed IPs (`use_fixedip`/`fixed_ip`) live on `stat/sta`. So `reservation_count` is NOT emitted by this collector. **Tracked owner: STAGE-007-019** (backend panel/page data endpoints) will emit/expose `homelab_unifi_dhcp_reservation_count{network}` = count of `stat/sta` clients with `use_fixedip==true` per network (the data is already parsed in active_client.py). Added as an explicit deliverable on the STAGE-007-019 card (see that card).

**Final metric set (Decision 4):** Per DHCP-enabled network (filter records where `purpose=="wan"` OR `as_bool(dhcpd_enabled)` is not True):
- `homelab_unifi_dhcp_pool_size{network}` = address count (e.g. 249), via `int(ipaddress.ip_address(stop)) - int(ipaddress.ip_address(start)) + 1`. If start/stop missing/unparseable -> skip pool_size for that network (coverage branch).
- `homelab_unifi_dhcp_pool_start{network}` / `homelab_unifi_dhcp_pool_end{network}` = `float(int(ipaddress.ip_address(<ip>)))` (numeric, graphable; epic §6.2 lists pool start/end).
- `homelab_unifi_dhcp_dns_primary{network, dns=<dhcpd_dns_1>}` = 1.0 (only when dhcpd_dns_1 is a non-empty str; if absent, skip the dns info-gauge for that network — coverage branch).
ALWAYS emit:
- `homelab_unifi_dhcp_enabled_network_count` (no labels) = number of DHCP-enabled networks found (0 if none — avoids the absent() trap in 015 + gives 021 a deterministic signal).
- `homelab_unifi_api_took_seconds{endpoint=<resp.endpoint>}` (resp.endpoint = "rest/networkconf").

**Error/empty/graceful:** `ctx.unifi is None` -> ok=False. `rest_networkconf()` UnifiError -> ok=False. Malformed payload (non-dict / non-list data) or empty data -> records=[] -> still emit latency + `dhcp_enabled_network_count=0`, ok=True. `_parse_records(payload)` local helper (copy from alarms.py). Skip non-dict entries. A WAN/non-dhcp record contributes nothing (the filter false-side is a coverage branch).

**ClassVars:** `name="unifi_networkconf"`, `interval=timedelta(seconds=300)` (5m), `timeout=timedelta(seconds=15)`, `concurrency_group="unifi"`. Register in the unifi bundle `_UNIFI_COLLECTORS`. (`concurrency_group` is a ClassVar, NOT a CollectorConfig field — the test `_ctx`/`CollectorConfig` must NOT pass it.) No cardinality cap (per-network = bounded by VLAN count).

**`ipaddress` stdlib:** use Python's `ipaddress` module for parsing string IPs to ints (top-level import). Guard parse failures (invalid IP string) with try/except ValueError -> skip that field (coverage branch).

**Brainstormer:** ran (Opus); coordinator selected 1A / 2A / 3A / 4 per pre-authorization.

**015 design assumption (note for STAGE-007-015):** UnifiDhcpPoolExhaustion must reference `homelab_unifi_client_count_by_network` (the real per-network active count), NOT a non-existent `dhcp_pool_used`. UnifiDnsSteeringDrift compares the `dns` label of `homelab_unifi_dhcp_dns_primary` against the configured expected (192.168.2.148) + an absent() companion.
