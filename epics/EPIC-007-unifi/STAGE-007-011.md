# STAGE-007-011: DHCP config + DNS-steering + pool-usage collector (`rest/networkconf`)

## Status: Not Started

## Overview

The DHCP/DNS-posture collector ‚Äî reads `rest/networkconf` for the per-network DHCP pool range, the
DNS-server handout (`dhcpd_dns_1..3`), and reservations. This drives the **DNS-steering drift** detector
(the sole DNS-protection signal after the firewall DNS-force rules were deleted) and DHCP pool-exhaustion.

**Vertical slice:** the collector emits `homelab_unifi_dhcp_*` + `dhcp_dns_primary` metrics against the live
UDM; visible in VM and the Network DHCP / DNS-posture widgets; feeds the alerts.

**Spec references:** spec ¬ß6.2. EPIC-007 "Metric families" (DHCP row) + "Verified deployment reality" (DNS
steering = DHCP handout, firewall rules deleted ‚Äî Amendment 3) + alert catalog (`UnifiDnsSteeringDrift`,
`UnifiDhcpPoolExhaustion`).

## Cross-cutting standards

- `unifi` group; 5m cadence (config changes slowly).
- The DNS-steering drift check compares `dhcp_dns_primary` against the configured expected
  (`192.168.2.148`) ‚Äî the expected value is config, not hardcoded.
- Pool occupancy here is the API approximation (active-client/lease count vs pool size); the opt-in lease
  collector (012) sharpens it to authoritative lease count when enabled.
- Emits `homelab_collector_run_*`.

## Concrete deliverables

### 1. DHCP config / DNS-steering collector

Reads `rest/networkconf`; emits `dhcp_pool_size`, `dhcp_pool_used` (approx from active count),
`dhcp_dns_primary` (label/value for the drift check), reservation count, per-network. Feeds
`UnifiDnsSteeringDrift` (DNS handout no longer ‚Üí Pi-hole) and `UnifiDhcpPoolExhaustion` (occupancy nearing
pool size). Surfaces in the Network page DNS-posture indicator + DHCP-pool widget + `network.json`.
