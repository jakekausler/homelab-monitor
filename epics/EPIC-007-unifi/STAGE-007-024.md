# STAGE-007-024: Grafana `unifi.json` (gear-centric) + Metrics-tab embed + readability pass

## Status: Not Started

## Overview

The gear-centric Grafana dashboard — `deploy/grafana/dashboards/unifi.json`, dashboards-as-code, embedded as
a tab in the Metrics screen (the EPIC-005 Metrics-tabs pattern), with a readability-review pass against live
data. Covers Unifi-as-product metrics: device health, switch ports/PoE, AP airtime/satisfaction,
threats/DPI, controller/API.

**Vertical slice:** the dashboard provisions at startup, is embedded as a Unifi tab in the Metrics screen,
and renders all rows against live `homelab_unifi_*` metrics; the readability pass confirms it reads well.

**Spec references:** spec §9.2 (Metrics = Grafana embed; dashboards-as-code). EPIC-007 "Grafana" (unifi.json).
EPIC-005 dashboard + readability-review precedent; EPIC-006 STAGE-006-026 (sibling Grafana stage). Datasource
`{type:prometheus, uid:victoriametrics}`.

## Concrete deliverables

### 1. Dashboard

`deploy/grafana/dashboards/unifi.json`, collapsible rows: (1) Device health (up/state/firmware/cpu/mem/temp),
(2) Switch ports & PoE (per-port link/speed/PoE-power/errors), (3) AP radios/airtime (cu/satisfaction/num_sta/
tx-power), (4) Threats/IDS-IPS, (5) DPI top-apps, (6) Controller & API (up/api_took). Gear-centric — network
concept lives in `network.json` (025).

### 2. Metrics-tab embed

Add the Unifi dashboard as a tab in the Metrics screen (EPIC-005 Metrics-tabs pattern).

### 3. Readability review pass

Interactive pass against live data: legends, units (watts/percent/seconds/dBm), thresholds, row order,
collapse defaults.
