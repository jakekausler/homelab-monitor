# STAGE-007-005: Combined device collector (health/firmware/ports/PoE/radios/PDU-relay/temp)

## Status: Complete

## Overview

The single combined device collector — ONE fetch of `stat/device` per tick, parsed into device health,
per-port PoE/link, AP radio/airtime, PDU outlet relays, and temperatures. One fetch (not N) minimizes
controller load and gives an atomic snapshot.

**Vertical slice:** the collector runs on its interval, fetches `stat/device` once, and emits
`homelab_unifi_device_*` / `_port_*` / `_radio_*` / `_outlet_*` metrics for all 8 devices; visible in the
collector status API and queryable in VM.

**Spec references:** spec §6.2 (`homelab_unifi_*`). EPIC-007 "Metric families" (combined-device row) +
"Verified deployment reality" (the `stat/device` FAT fields). EPIC-005 collector stages are the structural
precedent.

## Cross-cutting standards

- Joins the `unifi` concurrency group; 60s cadence.
- Graceful degrade: PDU outlet POWER is NOT emitted (HA owns USP PDU Pro wattage) — only relay state/name;
  fields absent on a device class are skipped, not errored.
- Cardinality: per-port / per-radio / per-outlet series bounded by real hardware (8 devices) — no cap needed
  beyond device count, but verify no unbounded label (e.g., transient port indices).
- Emits `homelab_collector_run_*`.

## Concrete deliverables

### 1. Combined device collector

Fetches `stat/device` once; parses and emits:
- **Device:** `device_up{device,model,kind}`, `state`, `firmware_info` (info), `update_available`
  (drives `UnifiFirmwareUpdateAvailable`), uptime, `cpu_percent`, `mem_percent`, load, `temperature`.
- **Switch ports** (`port_table`): per-port link/`port_speed_bps`, `poe_power/current/voltage`, `poe_good`,
  rx/tx bytes, error/drop counts, `mac_table_count`, `link_down_count`.
- **AP radios** (`radio_table_stats`): `cu_total`/`cu_self_rx`/`cu_self_tx` (airtime), `num_sta`, `tx_power`,
  `tx_retries_pct`, `satisfaction`, channel, bandwidth.
- **PDU** (`outlet_table`): `outlet_relay_state{outlet,name}` only (NO power).
- These metrics feed the Wave-D alerts (DeviceDown, PoePortFault, SwitchPortErrors, ApHighChannelUtil,
  ApLowSatisfaction, DeviceHighTemp, DeviceHighCpuMem, FirmwareUpdateAvailable, SwitchUplinkSaturation).

## Design Phase

**Status**: [x] Design (Complete)

### Design Notes & Selected Approach

**Selected approach:** Single `UnifiDeviceCollector(BaseCollector)` mirroring the HA collector pattern (`plugins/collectors/integrations/homeassistant/ha_up.py` + the bundle `register_all` in that package's `__init__.py`). One `await ctx.unifi.stat_device()` per 60s tick; parse `payload["data"]` (the classic `{"meta":{"rc":"ok"},"data":[...]}` wrapper); emit `homelab_unifi_device_*` / `_port_*` / `_radio_*` / `_outlet_*` gauges via `ctx.vm.write_gauge`. Brainstormer skipped: single obvious implementation, no architectural choice, HA collectors are the locked structural precedent.

**STAGE-007-005 also REMOVES the placeholder scaffolding:** delete `plugins/collectors/integrations/unifi/placeholder.py` and drop `UnifiPlaceholderCollector` from `_UNIFI_COLLECTORS` (and its import) in `plugins/collectors/integrations/unifi/__init__.py`.

**Collector ClassVars:** `name="unifi_device"`, `interval=timedelta(seconds=60)`, `timeout=timedelta(seconds=15)`, `concurrency_group="unifi"`.

**Error handling:** on `UnifiError` from `stat_device()`, return `CollectorResult(ok=False, metrics_emitted=0, errors=[result.message], ...)` (return-not-raise) AND emit `homelab_unifi_device_up=0` is NOT done (no devices known) — instead emit nothing device-level and still record the run as failed. If `ctx.unifi is None`, treat as a failed run (errors=["unifi client not configured"]). Always emit `homelab_unifi_api_took_seconds{endpoint="stat/device"}` from `result.took_seconds` on a SUCCESSFUL response (the classic endpoint's `meta` has NO `took` field — latency comes from the client-measured `took_seconds`).

**LIVE-RECON FIELD MAP (from the real UDM, 8 devices — authoritative; the card's field names were partly aspirational):**

- **Wrapper:** `{"meta":{"rc":"ok"},"data":[...]}`. No `meta.took`. 8 device records.
- **Device kind detection:** `type` ∈ {`udm`(gateway), `usw`(switch), `uap`(ap)}. The USP PDU Pro reports `type="usw"`, `model="USPPDUP"` — detect PDU by `model=="USPPDUP"` OR presence of a non-empty `outlet_table`. Assign `kind` label: gateway/switch/ap/pdu.
- **Device labels:** `device` = the device `name` (string), `model` (string), `kind` (derived). (Anonymize never needed in code; labels carry real names — that's expected in metrics.)
- **`device_up`** ← `1.0 if state==1 else 0.0` (`state` is a number; 1=connected). Field `update_available`/`disabled` do NOT exist.
- **`homelab_unifi_device_firmware_info`** info gauge ← carries `version` (full build str) and/or `displayable_version` (short) as labels; value 1.0.
- **`update_available`** ← `1.0 if upgradable else 0.0` (`upgradable` is a BOOLEAN; the card's `update_available` field name does NOT exist on the device).
- **`uptime`** ← `uptime` (number, seconds elapsed — NOT a timestamp; emit as-is).
- **cpu/mem:** under `system-stats` object, STRING-encoded: `cpu`="21.9", `mem`="44.4". Parse str→float defensively (may be absent/empty on USWED35 EdgeSwitch). `system-stats.uptime` is a STRING (skip; use top-level numeric `uptime`).
- **load:** under `sys_stats` object: `loadavg_1`/`loadavg_5`/`loadavg_15` (STRINGS). USWED35 has `sys_stats: {}` (empty) — graceful skip. (`mem_total`/`mem_used` are byte ints if you want a derived mem% but `system-stats.mem` already gives %.)
- **temperature:** `temperatures` array — UDM ONLY. Entries `{name:str, type:str("cpu"/"board"), value:float Celsius}`. Emit `homelab_unifi_device_temperature_celsius{device,name,type}` per entry. No `general_temperature`. Skip devices without the array (graceful degrade).
- **`port_table`** (switches + gateway; APs have `[]`; PDU has 1 uplink entry). Per-port fields: `port_idx`(num,1-based), `name`(str), `up`(bool), `enable`(bool), `full_duplex`(bool), `speed`(num Mbps int → emit `port_speed_bps`=speed*1_000_000), `poe_enable`(bool), `poe_good`(bool), `poe_power`(**STRING** watts→float), `poe_current`(**STRING** mA→float), `poe_voltage`(**STRING** V→float), `rx_bytes`/`tx_bytes`(num), `rx_errors`/`tx_errors`(num), `rx_dropped`/`tx_dropped`(num), `mac_table_count`(num), `link_down_count`(num), `satisfaction`(num 0-100). Non-PoE ports carry `poe_power="0.00"`, `poe_enable=false`. Label per-port series with `{device, port=str(port_idx)}`.
- **`radio_table_stats`** (APs; U7 Pro In-Wall has 3 radios wifi0/1/2). Fields: `name`(str e.g. "wifi0"), `radio`(str "ng"/"na"/"6e"), `channel`(num), `bw`(num MHz — use for bandwidth; no `channel_width`/`ht`), `cu_total`/`cu_self_rx`/`cu_self_tx`(num airtime %), `num_sta`(num), `user-num_sta`(num), `tx_power`(num dBm), `tx_retries_pct`(num float %), `satisfaction`(num; -1 = not computed → skip or emit -1). Label `{device, radio=name}`.
- **`outlet_table`** (PDU only — 20 entries; USB caps=1, AC caps=3). Emit ONLY `outlet_relay_state{device, outlet=str(index), name}` ← `1.0 if relay_state else 0.0` (`relay_state` is a BOOLEAN). Field for outlet number is `index` (NOT `outlet_idx`). Do NOT emit `outlet_power/current/voltage` (strings) — HA owns USP PDU Pro wattage per spec.
- **Timestamps:** all epoch-second INTEGERS (uptime/startup_timestamp/last_seen/connected_at). This collector emits `uptime` as elapsed seconds (already relative) — no ISO conversion needed here (no registry writes in this stage).

**Graceful-degrade rule:** every nested field is isinstance-guarded; absent arrays (no `port_table`/`radio_table_stats`/`outlet_table`/`temperatures`) are skipped, never errored. STRING numeric fields (`poe_*`, `cpu`/`mem`, `loadavg_*`) are float-parsed inside a guard (skip the series on ValueError, not crash). NOTE: exclude `bool` BEFORE `int`/`float` isinstance checks where a numeric field could be a bool (`isinstance(True,int)` is True).

**Cardinality:** bounded by real hardware (8 devices × ~48 ports + 3 radios + 20 outlets) — no CappedEmitter needed this stage; per-port/radio/outlet labels are stable hardware indices (not transient). Verify no unbounded label leaks.
