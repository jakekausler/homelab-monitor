# STAGE-007-005: Combined device collector (health/firmware/ports/PoE/radios/PDU-relay/temp)

## Status: Not Started

## Overview

The single combined device collector — ONE fetch of `stat/device` per tick, parsed into device health,
per-port PoE/link, AP radio/airtime, PDU outlet relays, and temperatures. One fetch (not N) minimizes
controller load and gives an atomic snapshot.

**Vertical slice:** the collector runs on its interval, fetches `stat/device` once, and emits
`homelab_unifi_device_*` / `_port_*` / `_radio_*` / `_outlet_*` metrics for all 8 devices; visible in the
collector status API and queryable in VM.

**Spec references:** spec §6.2 (`homelab_unifi_*`). EPIC-007 "Metric families" (combined-device row) +
"Verified deployment reality" (the `stat/device` FAT fields). EPIC-005 collector stages are the structural
precedent.

## Cross-cutting standards

- Joins the `unifi` concurrency group; 60s cadence.
- Graceful degrade: PDU outlet POWER is NOT emitted (HA owns USP PDU Pro wattage) — only relay state/name;
  fields absent on a device class are skipped, not errored.
- Cardinality: per-port / per-radio / per-outlet series bounded by real hardware (8 devices) — no cap needed
  beyond device count, but verify no unbounded label (e.g., transient port indices).
- Emits `homelab_collector_run_*`.

## Concrete deliverables

### 1. Combined device collector

Fetches `stat/device` once; parses and emits:
- **Device:** `device_up{device,model,kind}`, `state`, `firmware_version` (info), `update_available`
  (drives `UnifiFirmwareUpdateAvailable`), uptime, `cpu_percent`, `mem_percent`, load, `temperature`.
- **Switch ports** (`port_table`): per-port link/`port_speed_bps`, `poe_power/current/voltage`, `poe_good`,
  rx/tx bytes, error/drop counts, `mac_table_count`, `link_down_count`.
- **AP radios** (`radio_table_stats`): `cu_total`/`cu_self_rx`/`cu_self_tx` (airtime), `num_sta`, `tx_power`,
  `tx_retries_pct`, `satisfaction`, channel, bandwidth.
- **PDU** (`outlet_table`): `outlet_relay_state{outlet,name}` only (NO power).
- These metrics feed the Wave-D alerts (DeviceDown, PoePortFault, SwitchPortErrors, ApHighChannelUtil,
  ApLowSatisfaction, DeviceHighTemp, DeviceHighCpuMem, FirmwareUpdateAvailable, SwitchUplinkSaturation).
