# STAGE-007-025: Grafana `network.json` (network-centric) + Metrics-tab embed + readability pass

## Status: Not Started

## Overview

The network-centric Grafana dashboard — `deploy/grafana/dashboards/network.json`, dashboards-as-code,
embedded as a tab in the Metrics screen, with a readability-review pass. Covers network-as-concept metrics:
WAN/ISP + speedtest, DHCP pool/lease occupancy, client counts/distribution, WiFi experience. The final
stage of the epic.

**Vertical slice:** the dashboard provisions at startup, is embedded as a Network tab in the Metrics screen,
and renders all rows against live metrics; the readability pass confirms it reads well.

**Spec references:** spec §9.2. EPIC-007 "Grafana" (network.json). EPIC-005 dashboard + readability-review
precedent. STAGE-007-006/011/012 (WAN/DHCP/lease data). Datasource `{type:prometheus, uid:victoriametrics}`.

## Cross-cutting standards

- Dashboard-as-code; datasource `victoriametrics`; `jq -e .` valid; no gridPos overlap.
- The DHCP-pool/occupancy panel uses the authoritative lease count (from 012) when on, the active-client
  approximation when off — the panel reads sensibly either way.
- Readability pass is a real sub-step.

## Concrete deliverables

### 1. Dashboard

`deploy/grafana/dashboards/network.json`, collapsible rows: (1) WAN/ISP (up/latency/drops/throughput) +
speedtest history, (2) DHCP pool/lease occupancy, (3) Clients (active count, by-network/SSID/band
distribution, count-anomaly), (4) WiFi experience (signal/satisfaction distributions, per-AP airtime). DNS-
posture (handout-vs-expected) panel optional here (also a UI indicator).

### 2. Metrics-tab embed

Add the Network dashboard as a tab in the Metrics screen.

### 3. Readability review pass

Interactive pass against live data: legends, units (Mbps/percent/seconds), thresholds, row order, collapse
defaults.
