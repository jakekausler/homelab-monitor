# STAGE-007-012: SSH DHCP-lease collector (opt-in, default-OFF, on the EPIC-017 framework)

## Status: Complete

## Overview

The optional DHCP-lease-table collector — reads the UDM's authoritative dnsmasq lease file (the one piece of
DHCP truth the API cannot supply: lease expiry + lease-only devices) over the **EPIC-017 SSH probe
framework**. It is **opt-in, default-OFF**, and **pure enrichment**: it adds `lease_expiry` to existing
registry rows (and can add rows for lease-only devices) but is NEVER the identity source — the epic is fully
functional API-only without it.

**This stage is built AFTER EPIC-017** (the whole-epic build order is 017 → 007 → 006 → 008, so the SSH
framework already exists when this stage runs). It consumes 017's scoped per-target forced-command SSH
probe; it does NOT create its own SSH access path.

**Vertical slice:** with the collector enabled (and an SSH probe configured per 017), it reads the lease
table, parses the dnsmasq format, and enriches the registry with `lease_expiry` + `dhcp_lease_count`;
disabled (the default), it is inert and the registry/DHCP widgets degrade gracefully.

**Spec references:** spec §2 Q29 (SSH probes — per-target low-priv users, forced commands), §3.1
(`SshClientFactory`). EPIC-017 (the framework this builds on — built first; carries the mandate to replace
any unscoped SSH access with a scoped user). EPIC-007 "Verified deployment reality" (the lease file) +
"Credential / transport model" (opt-in, default-off, neutrally worded — no committed doc reveals the
existing passwordless-root trust).

## Concrete deliverables

### 1. Lease-file SSH collector

When enabled: runs the 017 scoped probe whose forced command emits the lease file
(`/data/udapi-config/dnsmasq.lease`); parses the standard dnsmasq 5-field format
(`<expiry-epoch> <mac> <ip> <hostname|*> <clientid>`); enriches `unifi_clients` rows with `lease_expiry`,
adds rows for lease-only devices, and emits `dhcp_lease_count` (the authoritative count that sharpens the
DHCP-pool occupancy metric/widget).

### 2. Consumer surfaces (call out — must not be a write-only enrichment)

The lease data surfaces in, all with graceful-degrade when the collector is OFF:
- **Client page** — a "DHCP lease" field (expiry / time-remaining; static-vs-dynamic indicator).
- **Network → Clients tab** — an optional lease-expiry column.
- **Network DHCP widget** — authoritative lease count vs pool size (replaces the active-client approximation).
- **`network.json` Grafana** — the DHCP-pool/occupancy panel uses the authoritative lease count when on.

(The Client-page / Clients-tab / Network-widget / Grafana stages each reference this; this stage provides the
data + the graceful-degrade contract.)

### 3. SSH-probe STATE rendering (EPIC-017 is headless/Option-B)

EPIC-017 ships no UI — consumers render probe state from its `homelab_ssh_*` metrics. This stage also
surfaces the **SSH lease probe's own state** (last success / host-key status / probe duration / up-down) on
the Unifi or Network page (alongside the lease data) and in `network.json` — so a failing/ stale/ host-key-
mismatched lease probe is visible. When the collector is OFF, no probe-state is shown.

## Design Phase

**Status**: [x] Design (Complete)

### Design Notes (locked)

**Verified framework facts (EPIC-017):** `SshProbe.parse(result: SshCommandResult) -> ProbeOutcome` is PURE (no `ctx`); the base `SshProbe.run()` owns ctx and has NO DB-write hook. The SSH bundle registers a probe per ssh_target UNCONDITIONALLY (no gate). Therefore a registry-WRITING, opt-in-GATED, unifi-identity collector CANNOT cleanly subclass SshProbe.

**Base class (Decision 1 = A, plain BaseCollector; EPIC-017 untouched):** Implement as a plain `BaseCollector` (NOT an SshProbe subclass), appended to the unifi bundle `_UNIFI_COLLECTORS` (static list). Its own `run(ctx)` opens `ctx.ssh.open(...)`, runs the command, parses (pure helper), writes the registry via `ctx.db.transaction()`, and emits metrics. EPIC-017 framework is NOT modified (it's a completed epic; modifying it = unnecessary blast radius). The ~15-line health-metric emission is hand-written here, not extracted from SshProbe.

**Metric-collision avoidance (Decision 1, load-bearing):** The SSH-bundle uptime probe already emits target-level `homelab_ssh_up{target}` / `homelab_ssh_host_key_mismatch{target}` (labeled by `{target}` ONLY) for every ssh_target — INCLUDING `udm` if a udm uptime probe is configured. If this collector also emitted those exact series, two emitters would flap the same series. AVOIDED by emitting DISTINCT `_probe_`-infixed, probe-scoped names: `homelab_ssh_probe_up{target,probe="unifi_dhcp_lease"}`, `homelab_ssh_probe_host_key_mismatch{target,probe="unifi_dhcp_lease"}`, plus `homelab_ssh_probe_duration_seconds{target,probe}` and `homelab_ssh_last_success_age_seconds{target,probe}` (these two already carry `probe`, so they wouldn't collide, but use them probe-scoped). NEVER emit the bare `homelab_ssh_up{target}` / `homelab_ssh_host_key_mismatch{target}` from this collector. This satisfies card deliverable #3 (the lease probe's OWN state visible) without colliding with the framework.

**Registration + gate (Decision 2 = A):** Register in `_UNIFI_COLLECTORS` (static, one-line append). Gate at RUNTIME: `run()` first checks `load_unifi_config().ssh_lease_enabled`; if False -> return `CollectorResult(ok=True, metrics_emitted=0, errors=[], duration_seconds=...)` with NO SSH attempt (inert). Always-register + runtime-gate matches every other unifi collector and allows hot-toggle (next 5m tick) without restart.

**SSH target (Decision 2):** Add a new `UnifiConfig.ssh_lease_target_id: str = "udm"` field (env `HOMELAB_MONITOR_UNIFI_SSH_LEASE_TARGET_ID`, read in `load_unifi_config()` alongside `ssh_lease_enabled`). The collector opens `ctx.ssh.open(cfg.ssh_lease_target_id)`. The public repo hardcodes nothing host-specific; the operator overrides the target id if theirs differs. (NOTE for Build: this is a NEW UnifiConfig field — add it + its env parse + a test, mirroring the existing `ssh_lease_enabled` field/parse.)

**Registry writes (Decision 3 = A; enrich + lease-only rows):** Add a NEW UPDATE-only static `set_lease_expiry_conn(conn, *, mac, lease_expiry)` to `UnifiClientRepo` (mirrors the static-in-transaction pattern of `upsert_client_conn`/`promote_to_host_conn`). **MAC MATCHING (load-bearing):** the registry stores MACs VERBATIM from the UniFi API (NO normalization — confirmed); the dnsmasq.lease file MAC case may differ. So `set_lease_expiry_conn` matches CASE-INSENSITIVELY: `UPDATE unifi_clients SET lease_expiry = :expiry WHERE LOWER(mac) = LOWER(:mac)`. The collector: snapshot prior MACs via `list_clients()` (lowercased set), then in ONE `ctx.db.transaction()`: for a lease MAC whose lowercase is already in the registry -> `set_lease_expiry_conn` (case-insensitive UPDATE); for a lease-only MAC NOT present -> `upsert_client_conn(minimal)` then `set_lease_expiry_conn`. Minimal lease-only row: `mac=<dnsmasq mac as-is (lowercase)>`, `ip=<lease ip>`, `hostname=<dnsmasq hostname or None if "*">`, `name=None`, `oui=None`, `network=None`, `ap_mac=None`, `sw_mac=None`, `sw_port=None`, `use_fixedip=False`, `fixed_ip=None`, `online=False`, `first_seen=last_seen=now_iso`. Enrichment-only: NEVER the identity source.

**Parse (Decision 4):** pure helper over `result.stdout`: split lines; per line `split()`; require `>= 2` fields (epoch + mac); `int(fields[0])` -> `datetime.fromtimestamp(epoch, tz=UTC).isoformat()` (matches the registry's epoch->ISO format `YYYY-MM-DDTHH:MM:SS+00:00`); mac = `fields[1]` (dnsmasq lowercases it; used as-is for inserts, matched case-insensitively for updates); ip = `fields[2]` if present; hostname = `fields[3]` if present and != "*" else None. Skip malformed lines (non-int epoch, `< 2` fields) gracefully. Use `datetime.now(tz=UTC).isoformat()` for now_iso (matches `utc_now_iso()`).

**ok-semantics + metrics per path (Decision 4 + 5):**
- Gate OFF -> ok=True, 0 metrics, NO ssh attempt, NO db write.
- Success (incl. empty/garbage file -> 0 valid leases): ok=True; emit `homelab_unifi_dhcp_lease_count` = N (0 if empty); `homelab_ssh_probe_up{target,probe}` = 1; `homelab_ssh_probe_duration_seconds{target,probe}` = elapsed; `homelab_ssh_last_success_age_seconds{target,probe}` = 0; DB enrich/add (0 rows if empty).
- `HostKeyMismatch` (kernel.ssh.errors) -> ok=False; `homelab_ssh_probe_up`=0; `homelab_ssh_probe_host_key_mismatch`=1; NO lease_count, NO db write.
- Other `SshTransportError` (unknown target / HostKeyNotPinned / SshAuthError / SshConnectionRefused / SshTimeout) -> ok=False; `homelab_ssh_probe_up`=0; `homelab_ssh_probe_host_key_mismatch`=0; NO lease_count, NO db write, errors=[exc message].
- Non-zero `exit_status` (lease file missing / cmd error) -> ok=False; `homelab_ssh_probe_up`=0; NO lease_count, NO db write.
- CRITICAL: a transport/exit failure must NOT be read as "0 leases" — absence of `homelab_unifi_dhcp_lease_count` on failure (vs =0 on a real empty file) prevents STAGE-007-011's occupancy from misreading a failure as an empty pool.
- Latency metric (Decision 5): emit `homelab_ssh_probe_duration_seconds{target,probe}`, NOT `homelab_unifi_api_took_seconds` (this is SSH, not an API call).

**ClassVars:** `name="unifi_ssh_lease"` (or "unifi_dhcp_lease" — pick one and use it as the `probe` label value consistently; LOCK: `probe`-label value = "unifi_dhcp_lease", collector `name` = "unifi_ssh_lease"), `interval=timedelta(seconds=300)` (5m), `timeout=timedelta(seconds=15)`, `concurrency_group="unifi"`. (`concurrency_group` is a ClassVar, NOT a CollectorConfig field — the test `_ctx`/`CollectorConfig` must NOT pass it.)

**last_success_age handling:** emit `homelab_ssh_last_success_age_seconds{target,probe}` = 0.0 on success, OMIT on failure. A true persisted cross-run last-success age would need collector-side state this plugin doesn't have; freshness is computed downstream in PromQL from the always-emitted-on-success series (`time() - timestamp(...)`). NOT deferred to a new stage — PromQL handles freshness; no separate state store is warranted.

**Live validation caveat (Refinement):** the live UDM SSH target + forced-command key are OPERATOR-PROVIDED (not in the public repo). If the user has NOT installed the key / configured the `udm` ssh_target, the live read (3b) cannot run — that's an acceptable "pending operator key" outcome (document it). The parser + enrichment + gate + collision-avoidance are fully validated via fixture/synthetic SSH (3a) regardless. Build will GUIDE the operator on the one-time key install (the USER installs; we never write their auth config).

**Brainstormer:** ran (Opus); coordinator selected 1A+C1 / 2A+target-id-config / 3A / 4 / 5 per pre-authorization.
