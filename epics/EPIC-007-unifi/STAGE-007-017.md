# STAGE-007-017: vmalert-LOGS rules (the ~5 CEF-stream alerts)

## Status: Not Started

## Overview

Author the Unifi log alert rules (`deploy/vmalert/logs/unifi.yaml`) over the CEF syslog→VL stream that
STAGE-007-016 established. **This stage comes AFTER 016** so the stream exists and the rules validate against
real lines.

**Vertical slice:** the log rules provision into vmalert-logs; a planted/real CEF line (e.g. a port flap)
fires the matching rule → Alertmanager.

**Spec references:** spec §3.1 (vmalert logs — separate instance over VL), §4.3 (logs flow). EPIC-007 "Alert
catalog" (logs table) + "Events pipeline" (two-paths-one-alert: `UnifiThreatLog` is forensics-only, NO
alert). EPIC-006 STAGE-006-017 (sibling log-rules stage).

## Concrete deliverables

### 1. `deploy/vmalert/logs/unifi.yaml`

The ~5 log rules:
- UnifiPortFlapLog (warning) — port up/down flap lines.
- UnifiDeviceDisconnectLog (warning) — AP/gateway disconnect lines.
- UnifiFirmwareEventLog (info) — firmware update/apply lines.
- UnifiAdminLoginLog (info) — admin login lines.
- (UnifiThreatLog — a saved query / LogViewer filter for forensics; NOT an alert.)
