# STAGE-005A-002: Disk-fill rules (host filesystems + per-slot self-disk budgets)

## Status: Complete

## Overview

Today there is **NO host-filesystem disk-fill alert at all** ‚Äî the only disk alerting is the monitor's OWN aggregate self-disk (`SelfDiskWarn`/`SelfDiskError`/`SelfDiskCritical` on `homelab_self_disk_used_pct`). This stage closes two real gaps the user explicitly asked for ("disk space running low in any of the volumes"):

1. **Host filesystem fill** ‚Äî the actual physical filesystems `/` and `/storage` (the 2026-06-15 snapshot saw ~246G and ~3.4T respectively; **re-verify sizes live in Design** before tuning thresholds), which are completely unalerted.
2. **Per-slot self-disk budget fill** ‚Äî the named docker volumes the monitor manages (`vl`, `vm`, `sqlite`, `runbook_transcripts`), each with its own budget. The *aggregate* `homelab_self_disk_used_pct` is alerted, but a SINGLE slot blowing its budget (e.g. `vl` filling) can go undetected because the aggregate doesn't trip. This adds per-slot fill alerts.

This stage is threshold-based only ‚Äî **no predict_linear projection rule** (user decision: not needed).

**Verified live (scoping session):** `homelab_host_disk_bytes{mountpoint,type}` ‚Äî mountpoints `/` and `/storage` (relabeled in STAGE-005-041 via `disk_mountpoint_relabel`), `type` ‚àà {used, total}. `homelab_self_disk_used_bytes{slot}` and `homelab_self_disk_budget_bytes{slot}` ‚Äî slots {vm, vl, sqlite, runbook_transcripts} (runbook shares the sqlite budget tier).

## Concrete deliverables

A new rule file `deploy/vmalert/metrics/disk_fill.yaml` (+ `__tests__/disk_fill.tests.yaml`) containing:

1. **Host filesystem fill ‚Äî warning + critical, per mountpoint.** Expr (used % per mountpoint, the same form as the STAGE-005-041 Grafana disk panel):
   - warning: `100 * homelab_host_disk_bytes{type="used"} / ignoring(type) homelab_host_disk_bytes{type="total"} > 85` for `10m` ‚Üí warning, label `target_kind: host`, `{{mountpoint}}` in the annotation.
   - critical: same expr `> 95` for `5m` ‚Üí critical.
   This fires per-mountpoint (so `/` and `/storage` alert independently, and any future allowlisted mountpoint like `/rackstation` inherits the rule automatically).
2. **Per-slot self-disk budget fill ‚Äî warning + critical, per slot.** Expr:
   - warning: `100 * homelab_self_disk_used_bytes / ignoring() homelab_self_disk_budget_bytes > 85` (joined on the `slot` label) for `10m` ‚Üí warning, `{{slot}}` in annotation.
   - critical: `> 95` for `5m` ‚Üí critical.
   Confirm the join works (both metrics carry `slot`); guard against `budget == 0` (divide-by-zero ‚Üí use `> 0` filter on the denominator, mirroring the existing `homelab:vl_disk_used_pct` recording rule in `vl_health.yaml` which already does `(homelab_self_disk_budget_bytes{slot="vl"} > 0)`).

**Overlap note (avoid double-fire):** `vl_health.yaml` ALREADY has `VictoriaLogsDiskWarning`/`VictoriaLogsDiskCritical` on the `vl` slot specifically (via the `homelab:vl_disk_used_pct` recording rule). The new per-slot rule (item 2) would ALSO fire on the `vl` slot ‚Üí DUPLICATE. **Design must resolve this:** either (a) exclude `slot="vl"` from the new per-slot rule (vl keeps its dedicated alert), or (b) supersede the vl-specific rules with the generic per-slot one. Pick one; do not double-fire on vl.

## Design Phase

**Status**: [x] Design (Complete)

### Design Notes

**Live-rig verification (this Design session):**
- Host fs `homelab_host_disk_bytes{mountpoint,type}`: mountpoints `/` (46.4% = 122G/263G) and `/storage` (9.4% = 346G/3.67T, ~3.3T free); `type` ‚àà {used, total}. **Bare division `{type="used"} / {type="total"}` returns EMPTY ‚Äî MUST use `/ ignoring(type)`** (confirmed live; same gotcha as STAGE-005A-001's memory fix).
- Per-slot `homelab_self_disk_used_bytes{slot}` / `homelab_self_disk_budget_bytes{slot}`: 4 slots, each a DISTINCT series in BOTH metrics ‚Äî `vl` (2.4%), `vm` (0.75%), `sqlite` (0%), `runbook_transcripts` (0%, distinct 5.37G budget series ‚Äî NOT shared with sqlite). All budgets >0. Both metrics carry the identical `{slot}` label set ‚Üí a native `used / budget` division matches per-slot WITHOUT `ignoring()`.
- Existing coverage: host `/`+`/storage` have NO alert (genuine gap). Aggregate `SelfDiskWarn/Error/Critical` (70/85/95 on `homelab_self_disk_used_pct`, which is aggregate-only, no slot label). `vl` slot has dedicated `VictoriaLogsDiskWarning/Critical` (70/85) via the `homelab:vl_disk_used_pct` recording rule (which uses the `/ (budget > 0)` divide-by-zero guard).

**Decisions (brainstormer ‚Üí main-agent picks; no user prompt per standing instruction):**

- **D-005A-002-VL-OVERLAP ‚Üí (a) exclude `slot="vl"` from the generic per-slot rule.** Leave `VictoriaLogsDiskWarning/Critical` + the `homelab:vl_disk_used_pct` recording rule in `vl_health.yaml` UNTOUCHED. vl has hand-tuned 70/85 thresholds (logs fill fast); letting both fire = duplicate noise. Rejected (b) replace-vl (would silently relax vl 70‚Üí85 warn = a regression on the fastest filler).

- **D-005A-002-PER-SLOT-FORM ‚Üí (b) single generic inline-join rule, NO per-slot recording rules.** `100 * homelab_self_disk_used_bytes{slot!="vl"} / (homelab_self_disk_budget_bytes > 0)` ‚Äî both metrics share the `{slot}` label so the division matches per-slot natively (NO `ignoring()` here ‚Äî distinct from the host-fs case). The `( > 0)` guard drops zero-budget slots. One warn + one crit rule auto-covers vm/sqlite/runbook_transcripts AND any future non-vl slot; `{{ $labels.slot }}` in the annotation. Rejected (a) recording-rule-per-slot (verbose; new slot = new rules; the vl precedent is justified only because vl needed DIFFERENT thresholds, which the generic slots don't).

- **D-005A-002-THRESHOLDS ‚Üí uniform 85/95% (warn/crit) for BOTH host-fs and per-slot; NO free-bytes floor.** % is the standard "disk running low" signal; `/storage` at 95% (~180G free) on a 3.67T volume is runaway growth worth knowing; `/` at 95% (~13G free) is a real emergency. Per-slot budgets are small so % is the right unit. Rejected an absolute free-bytes floor (complexity + magic constant to suppress one page that's itself worth seeing; raise `/storage` crit later if it ever proves noisy).

- **D-005A-002-FOR-DURATION ‚Üí 15m warn / 10m crit** (host-fs AND per-slot). Disks fill over hours/days; 15m/10m is the calmest sensible window (one notch beyond the card's 10m/5m and vl's 10m/5m) ‚Äî these are slow capacity signals, not latency-sensitive. Crit (10m) shorter than warn (15m) so a sprint to 95% pages faster. Rejected 5m/0s (0s crit invites flap on a scrape blip).

- **D-005A-002-RUNBOOK-SLOT ‚Üí no special handling.** Confirmed live: `homelab_self_disk_budget_bytes` has a DISTINCT `{slot="runbook_transcripts"}` 5.37G series (separate from sqlite). The `{slot!="vl"}` generic rule covers `{vm, sqlite, runbook_transcripts}` natively ‚Äî nothing silently dropped.

- **Label convention (main-agent lock):** host-fs rules carry `target_kind: host` (consistent with STAGE-005A-001 + the card); per-slot rules carry `target_kind: self_disk`. All rules carry `source_tool: vmalert-metrics` + `severity` (vocab info|warning|error|critical). Annotations `summary` + `description` with `{{ $labels.mountpoint }}` / `{{ $labels.slot }}` / `{{ $value }}`.

**Final rule set for `deploy/vmalert/metrics/disk_fill.yaml` (4 rules; + `__tests__/disk_fill.tests.yaml` mirror). NO recording rules; NO edits to existing files:**
1. `HostDiskFillWarning` (warning, target_kind: host, for 15m) ‚Äî `100 * homelab_host_disk_bytes{type="used"} / ignoring(type) homelab_host_disk_bytes{type="total"} > 85` (per `{mountpoint}`).
2. `HostDiskFillCritical` (critical, target_kind: host, for 10m) ‚Äî same with `> 95`.
3. `SelfDiskSlotFillWarning` (warning, target_kind: self_disk, for 15m) ‚Äî `100 * homelab_self_disk_used_bytes{slot!="vl"} / (homelab_self_disk_budget_bytes > 0) > 85` (per `{slot}`; native join; no ignoring()).
4. `SelfDiskSlotFillCritical` (critical, target_kind: self_disk, for 10m) ‚Äî same with `> 95`.

Test mirror validates fire/no-fire per rule (plant high-usage to fire, baseline to stay silent; per-mountpoint + per-slot planted series). Validated per STAGE-005A-001's method: promtool for these threshold/division rules (all 4 are promtool-validatable ‚Äî plain division + `ignoring(type)`, no count()-broadcast or rate-subquery) + live-rig `health=ok` + no-false-fire-at-baseline.
