# STAGE-005A-002: Disk-fill rules (host filesystems + per-slot self-disk budgets)

## Status: Not Started

## Overview

Today there is **NO host-filesystem disk-fill alert at all** ŌĆö the only disk alerting is the monitor's OWN aggregate self-disk (`SelfDiskWarn`/`SelfDiskError`/`SelfDiskCritical` on `homelab_self_disk_used_pct`). This stage closes two real gaps the user explicitly asked for ("disk space running low in any of the volumes"):

1. **Host filesystem fill** ŌĆö the actual physical filesystems `/` and `/storage` (the 2026-06-15 snapshot saw ~246G and ~3.4T respectively; **re-verify sizes live in Design** before tuning thresholds), which are completely unalerted.
2. **Per-slot self-disk budget fill** ŌĆö the named docker volumes the monitor manages (`vl`, `vm`, `sqlite`, `runbook_transcripts`), each with its own budget. The *aggregate* `homelab_self_disk_used_pct` is alerted, but a SINGLE slot blowing its budget (e.g. `vl` filling) can go undetected because the aggregate doesn't trip. This adds per-slot fill alerts.

This stage is threshold-based only ŌĆö **no predict_linear projection rule** (user decision: not needed).

**Verified live (scoping session):** `homelab_host_disk_bytes{mountpoint,type}` ŌĆö mountpoints `/` and `/storage` (relabeled in STAGE-005-041 via `disk_mountpoint_relabel`), `type` Ōłł {used, total}. `homelab_self_disk_used_bytes{slot}` and `homelab_self_disk_budget_bytes{slot}` ŌĆö slots {vm, vl, sqlite, runbook_transcripts} (runbook shares the sqlite budget tier).

## Concrete deliverables

A new rule file `deploy/vmalert/metrics/disk_fill.yaml` (+ `__tests__/disk_fill.tests.yaml`) containing:

1. **Host filesystem fill ŌĆö warning + critical, per mountpoint.** Expr (used % per mountpoint, the same form as the STAGE-005-041 Grafana disk panel):
   - warning: `100 * homelab_host_disk_bytes{type="used"} / ignoring(type) homelab_host_disk_bytes{type="total"} > 85` for `10m` ŌåÆ warning, label `target_kind: host`, `{{mountpoint}}` in the annotation.
   - critical: same expr `> 95` for `5m` ŌåÆ critical.
   This fires per-mountpoint (so `/` and `/storage` alert independently, and any future allowlisted mountpoint like `/rackstation` inherits the rule automatically).
2. **Per-slot self-disk budget fill ŌĆö warning + critical, per slot.** Expr:
   - warning: `100 * homelab_self_disk_used_bytes / ignoring() homelab_self_disk_budget_bytes > 85` (joined on the `slot` label) for `10m` ŌåÆ warning, `{{slot}}` in annotation.
   - critical: `> 95` for `5m` ŌåÆ critical.
   Confirm the join works (both metrics carry `slot`); guard against `budget == 0` (divide-by-zero ŌåÆ use `> 0` filter on the denominator, mirroring the existing `homelab:vl_disk_used_pct` recording rule in `vl_health.yaml` which already does `(homelab_self_disk_budget_bytes{slot="vl"} > 0)`).

**Overlap note (avoid double-fire):** `vl_health.yaml` ALREADY has `VictoriaLogsDiskWarning`/`VictoriaLogsDiskCritical` on the `vl` slot specifically (via the `homelab:vl_disk_used_pct` recording rule). The new per-slot rule (item 2) would ALSO fire on the `vl` slot ŌåÆ DUPLICATE. **Design must resolve this:** either (a) exclude `slot="vl"` from the new per-slot rule (vl keeps its dedicated alert), or (b) supersede the vl-specific rules with the generic per-slot one. Pick one; do not double-fire on vl.

## Decisions to confirm in Design

- **D-005A-002-VL-OVERLAP** ŌĆö exclude `slot="vl"` from the generic per-slot rule (keep `VictoriaLogsDisk*`) vs. replace the vl-specific rules. (Re-verify the live rule set first.)
- **D-005A-002-THRESHOLDS** ŌĆö 85/95 warn/crit as starting points (matching the Grafana disk panel + the existing SelfDisk tiers); tune for `/storage` which is large (~3.4T per the snapshot, so 95% still leaves ~170G free ŌĆö maybe a tighter % or an absolute-free-bytes floor). Decide whether `/storage` warrants a different threshold than `/`. (Verify the live sizes first.)
- **D-005A-002-FOR-DURATION** ŌĆö `for:` windows (10m warn / 5m crit proposed; disk fills slowly so longer windows are fine and reduce flapping).

## Acceptance criteria

- Shortened-window test mirror under `__tests__/` + integration test (fires on planted high-usage, silent at baseline).
- NO double-fire on the `vl` slot (D-005A-002-VL-OVERLAP resolved).
- Divide-by-zero guarded (budget/total `> 0`).
- Per-mountpoint and per-slot labels surface in annotations (`{{mountpoint}}`, `{{slot}}`).
- Severity vocab + `source_tool: vmalert-metrics` + `target_kind` labels.
- Thresholds tuned against the real `/`, `/storage`, and per-slot usage on the prod rig.
