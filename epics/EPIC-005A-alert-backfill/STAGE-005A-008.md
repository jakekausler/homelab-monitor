# STAGE-005A-008: Watched-directory size collector + config-driven mounts + alerts + Grafana panel

## Status: Not Started

## Overview

The ONLY non-alerting-only stage in EPIC-005A. The existing disk metrics report per-FILESYSTEM usage (`homelab_host_disk_bytes` per mountpoint) but cannot size an arbitrary DIRECTORY (e.g. `/var`, which lives on the `/` filesystem). The user wants alerts when a configurable list of directories exceeds absolute size thresholds â€” e.g. a runaway `/tmp` or a bloated `/var`. This requires a new collector that recursively measures directory sizes, a config-driven `:ro` bind-mount mechanism so the collector can SEE host directories, absolute-threshold alerts, and a Grafana panel parallel to the existing disk-mountpoint bargauge.

**This is a backend + host-integration + dashboard stage** â€” collector code, a compose-mount generator change, and Grafana JSON. It is meatier than the alerting-only stages and **Design may split it into two sub-stages** (008a collector+mounts+config, 008b alerts+panel) if it balloons â€” decide in Design.

**Verified feasibility (scoping session):**
- **No existing directory-size measurement** in the codebase (only `psutil.disk_usage`/`shutil.disk_usage` for filesystem-level, and `Path.stat().st_size` for single files). The collector must add `os.walk`-sum-st_size OR a `du -x -s` shell-out, WITH A TIMEOUT (the host collector's timeout is 5s; `/var` can be tens of GB / millions of inodes â€” walking it can exceed 5s). Mitigation options: per-directory timeout + a `truncated` flag, raise the collector timeout, or cache + only re-walk on mtime change.
- **Host namespace problem:** the monitor runs in a container; its `/tmp` and `/var` are the CONTAINER's, not the host's. Host `/var`/`/tmp` are NOT currently mounted in. They must be bind-mounted read-only.
- **The project ALREADY has the config-driven-mount machinery:** `scripts/generate-compose-override.sh` reads a config YAML (`build-sources.yaml`) and GENERATES `deploy/compose/docker-compose.override.yml` with `:ro` bind-mount entries, applied via `make generate-build-mounts` + container recreate. `host-setup.sh` provides the idempotent host-prep idiom. The watched-directories feature REUSES this exact pattern (config â†’ generated `:ro` mounts â†’ restart).
- **The existing generator has ZERO collision/dedup logic** (just `sort -u` on prefixes). The watched-dirs generator MUST add collision/overlap/containment validation (a `:ro` watched mount must not shadow or be shadowed by an existing mount; occupied container targets include `/host/etc`, `/host/proc`, `/host-crontab-snapshot`, `/host-ipc`, `/config`, `/host-compose`, `/host-build-contexts`, `/var/run/docker.sock`, and the named volumes). A fresh `/host-watch/<name>:ro` target namespace is confirmed disjoint from all existing mounts.

## Stage Flags

- Has Input Forms: No (config is a YAML file + `HostCollectorConfig` defaults, not a UI form).
- Touches secrets / production data: No new secrets; reads host directory SIZES only (not contents â€” `:ro` and only `du`/walk for byte totals). Note the monitor already has broad host read visibility (`/config`=host-`/`, docker socket), so a read-only host-dir mount is not a new exposure class.
- Host-integration concerns: YES, strongly â€” new compose bind-mounts, a host-path generator, real `du` against host directories. Refinement REQUIRES 3a (synthetic dirs / fixture) AND 3b (REAL `/tmp`, `/var` on the prod rig, with the generated mounts applied + container recreated + host-setup script run).

## Concrete deliverables

### Collector + config
1. **New metric `homelab_host_directory_bytes{path}`** â€” emitted by a new collector (or an extension to `HostCollector`) that, for each configured watched directory, measures its recursive size and emits the byte total. Include a `truncated` label or a companion `homelab_host_directory_walk_timed_out` signal if the walk hits its per-directory timeout (so a too-big-to-walk dir is visible, not silently zero).
2. **`watched_directories` config** â€” a config list of `{path, warn_bytes, crit_bytes}` (or path + thresholds), following the established baked-default pattern in `HostCollectorConfig` (like `disk_mountpoint_relabel`/`extra_mountpoints`; the live config mechanism is baked defaults until the dangling "STAGE-014" YAML loader lands). **Default + test config: `/tmp` (warn 1 GB / crit 4 GB) and `/var` (warn 10 GB / crit 25 GB)** â€” realistic starting points, tuned in Refinement.
3. **du-with-timeout measurement** â€” `os.walk` summing `st_size` OR `du -x -s` shell-out, with a per-directory timeout (so `/var` can't hang the collector past its budget). Decide the approach + timeout + the truncated-flag behavior in Design. `-x` (don't cross filesystem boundaries) is important so sizing `/` doesn't recurse into mounted volumes.

### Config-driven mounts
4. **Watched-directory `:ro` mount generation** â€” extend `scripts/generate-compose-override.sh` (or a sibling `generate-watched-dirs-mounts.sh`) to read the watched-directories config and emit `<host_path>:/host-watch/<name>:ro` mount entries into the override file, applied via `make` + container recreate. The collector then `du`s `/host-watch/<name>` (the container-visible path) and emits the metric labeled with the FRIENDLY host path (`/tmp`, `/var`).
5. **Collision / overlap / containment validation (NEW logic â€” the existing generator lacks it).** Before emitting a mount, the generator MUST reject/dedupe: a watched path that equals, contains, or is contained-in an existing mount target; a duplicate watched path; a target-name collision under `/host-watch/`. This is a real, tested deliverable â€” not free. A `make test-generate-watched-dirs` unit test (mirroring `test-generate-build-mounts`) covers the collision cases.
6. **`host-setup.sh` / Makefile wiring** â€” a `make generate-watched-dirs-mounts` target + any host-setup step (the host paths are read by root-readable `du`; confirm the container user can read them via the `:ro` mount, or whether host-setup needs an ACL like the cron path does).

### Alerts
7. **Absolute-threshold alerts** â€” `deploy/vmalert/metrics/watched_directories.yaml` (+ test mirror): per watched directory, `homelab_host_directory_bytes{path="/var"} > <crit_bytes>` â†’ critical, `> <warn_bytes>` â†’ warning, with `{{path}}` in the annotation. Thresholds read from / consistent with the config defaults (`/tmp` 1G/4G, `/var` 10G/25G). The rule should be generic over the `path` label where possible, with per-path thresholds expressed via the metric value vs a companion threshold metric OR per-path rule entries (Design decides â€” a companion `homelab_host_directory_warn_bytes{path}`/`_crit_bytes{path}` emitted from config would let ONE generic rule cover all dirs; otherwise one rule per dir).

### Grafana
8. **Grafana panel parallel to the disk-mountpoint bargauge ("the mount component").** A new bargauge panel on `deploy/grafana/dashboards/host-overview.json` showing `homelab_host_directory_bytes{path}` per watched directory, unit bytes, with warn/crit thresholds colored (mirroring the existing "Disk used % per mountpoint" bargauge â€” same type/options, next free grid slot, jq-validated, no gridPos overlap). **Re-dump the dashboard's CURRENT panels/ids/gridPos in Design** (`jq '.panels[] | {id,title,gridPos}'`) before placing the new panel â€” the 2026-06-15 snapshot saw ~9 panels with the disk panel at id=4, but the layout shifts; do not hardcode the panel id or next-y from this card, derive them live.

## Decisions to confirm in Design

- **D-005A-008-SPLIT** â€” keep as one stage vs split into 008a (collector+mounts+config) and 008b (alerts+panel). If split, create the second card + bump the epic denominator.
- **D-005A-008-MEASURE** â€” `os.walk` vs `du -x -s` shell-out; per-directory timeout value; truncated-flag/metric behavior; the `-x` no-cross-filesystem decision.
- **D-005A-008-MOUNT-NAMESPACE** â€” confirm `/host-watch/<name>:ro` target namespace (verified disjoint) and the friendly-path relabel (container `/host-watch/var` â†’ emitted `path="/var"`).
- **D-005A-008-COLLISION-RULES** â€” the exact collision/overlap/containment rejection rules + whether to error-and-abort or skip-and-warn on a colliding entry.
- **D-005A-008-THRESHOLD-DELIVERY** â€” per-path thresholds: companion threshold metrics (one generic alert rule) vs per-path rule entries. Companion-metric approach is cleaner for a config-driven list but adds emitted metrics.
- **D-005A-008-CONFIG-MECHANISM** â€” baked `HostCollectorConfig` default (the only live mechanism today) vs reading a `watched-directories.yaml` from `/config` (the same surface the dangling STAGE-014 YAML loader would use â€” confirm whether `/config` YAML reading is wired for ANY collector yet, or strictly baked defaults).
- **D-005A-008-PERMISSIONS** â€” confirm the container user can `du` the `:ro`-mounted host dirs, or whether host-setup needs to grant read (like the cron ACL).

## Acceptance criteria

- `homelab_host_directory_bytes{path}` emits real sizes for `/tmp` and `/var` on the prod rig (3b), with the mounts generated + applied via the script flow.
- The mount generator's collision/overlap/containment validation has a unit test (`make test-generate-watched-dirs`) covering: duplicate path, path contains existing mount, path contained-in existing mount, target-name collision.
- Walking `/var` does NOT hang the collector past its timeout (the timeout + truncated behavior works; a too-big dir is visible, not silently zero).
- Absolute-threshold alerts fire on a planted oversized dir (test mirror) and are silent at baseline.
- The Grafana panel renders the watched-directory sizes against thresholds (jq-valid, no gridPos overlap) â€” visual sign-off if the dashboard is reviewed.
- Config-driven: changing the watched-directories list + thresholds + regenerating mounts + recreating the container picks up the new dirs WITHOUT hardcoded per-directory edits (the host-root/host-watch mount + config list is the only surface).
- Severity vocab + `source_tool: vmalert-metrics` + `target_kind: host` labels. Open-source-safe defaults (`/tmp`, `/var` are generic; no host-specific paths baked unless flagged).
