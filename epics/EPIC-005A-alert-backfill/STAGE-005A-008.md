# STAGE-005A-008: Watched-directory size collector + config-driven mounts + alerts + Grafana panel

## Status: Complete

## Overview

The ONLY non-alerting-only stage in EPIC-005A. The existing disk metrics report per-FILESYSTEM usage (`homelab_host_disk_bytes` per mountpoint) but cannot size an arbitrary DIRECTORY (e.g. `/var`, which lives on the `/` filesystem). The user wants alerts when a configurable list of directories exceeds absolute size thresholds — e.g. a runaway `/tmp` or a bloated `/var`. This requires a new collector that recursively measures directory sizes, a config-driven `:ro` bind-mount mechanism so the collector can SEE host directories, absolute-threshold alerts, and a Grafana panel parallel to the existing disk-mountpoint bargauge.

**This is a backend + host-integration + dashboard stage** — collector code, a compose-mount generator change, and Grafana JSON. It is meatier than the alerting-only stages and **Design may split it into two sub-stages** (008a collector+mounts+config, 008b alerts+panel) if it balloons — decide in Design.

**Verified feasibility (scoping session):**
- **No existing directory-size measurement** in the codebase (only `psutil.disk_usage`/`shutil.disk_usage` for filesystem-level, and `Path.stat().st_size` for single files). The collector must add `os.walk`-sum-st_size OR a `du -x -s` shell-out, WITH A TIMEOUT (the host collector's timeout is 5s; `/var` can be tens of GB / millions of inodes — walking it can exceed 5s). Mitigation options: per-directory timeout + a `truncated` flag, raise the collector timeout, or cache + only re-walk on mtime change.
- **Host namespace problem:** the monitor runs in a container; its `/tmp` and `/var` are the CONTAINER's, not the host's. Host `/var`/`/tmp` are NOT currently mounted in. They must be bind-mounted read-only.
- **The project ALREADY has the config-driven-mount machinery:** `scripts/generate-compose-override.sh` reads a config YAML (`build-sources.yaml`) and GENERATES `deploy/compose/docker-compose.override.yml` with `:ro` bind-mount entries, applied via `make generate-build-mounts` + container recreate. `host-setup.sh` provides the idempotent host-prep idiom. The watched-directories feature REUSES this exact pattern (config → generated `:ro` mounts → restart).
- **The existing generator has ZERO collision/dedup logic** (just `sort -u` on prefixes). The watched-dirs generator MUST add collision/overlap/containment validation (a `:ro` watched mount must not shadow or be shadowed by an existing mount; occupied container targets include `/host/etc`, `/host/proc`, `/host-crontab-snapshot`, `/host-ipc`, `/config`, `/host-compose`, `/host-build-contexts`, `/var/run/docker.sock`, and the named volumes). A fresh `/host-watch/<name>:ro` target namespace is confirmed disjoint from all existing mounts.

## Stage Flags

- Has Input Forms: No (config is a YAML file + `HostCollectorConfig` defaults, not a UI form).
- Touches secrets / production data: No new secrets; reads host directory SIZES only (not contents — `:ro` and only `du`/walk for byte totals). Note the monitor already has broad host read visibility (`/config`=host-`/`, docker socket), so a read-only host-dir mount is not a new exposure class.
- Host-integration concerns: YES, strongly — new compose bind-mounts, a host-path generator, real `du` against host directories. Refinement REQUIRES 3a (synthetic dirs / fixture) AND 3b (REAL `/tmp`, `/var` on the prod rig, with the generated mounts applied + container recreated + host-setup script run).

- EPIC-001 (`HostCollector` + `HostCollectorConfig` config pattern; the `self_disk`/`host` collector idioms).
- The existing mount-generator machinery (`scripts/generate-compose-override.sh`, `make generate-build-mounts`, `docker-compose.override.yml`).
- vmalert-metrics + Alertmanager; Grafana host-overview dashboard (STAGE-005-041 — the disk bargauge panel this one parallels).

## Concrete deliverables

### Collector + config
1. **New metric `homelab_host_directory_bytes{path}`** — emitted by a new collector (or an extension to `HostCollector`) that, for each configured watched directory, measures its recursive size and emits the byte total. Include a `truncated` label or a companion `homelab_host_directory_walk_timed_out` signal if the walk hits its per-directory timeout (so a too-big-to-walk dir is visible, not silently zero).
2. **`watched_directories` config** — a config list of `{path, warn_bytes, crit_bytes}` (or path + thresholds), following the established baked-default pattern in `HostCollectorConfig` (like `disk_mountpoint_relabel`/`extra_mountpoints`; the live config mechanism is baked defaults until the dangling "STAGE-014" YAML loader lands). **Default + test config: `/tmp` (warn 1 GB / crit 4 GB) and `/var` (warn 10 GB / crit 25 GB)** — realistic starting points, tuned in Refinement.
3. **du-with-timeout measurement** — `os.walk` summing `st_size` OR `du -x -s` shell-out, with a per-directory timeout (so `/var` can't hang the collector past its budget). Decide the approach + timeout + the truncated-flag behavior in Design. `-x` (don't cross filesystem boundaries) is important so sizing `/` doesn't recurse into mounted volumes.

### Config-driven mounts
4. **Watched-directory `:ro` mount generation** — extend `scripts/generate-compose-override.sh` (or a sibling `generate-watched-dirs-mounts.sh`) to read the watched-directories config and emit `<host_path>:/host-watch/<name>:ro` mount entries into the override file, applied via `make` + container recreate. The collector then `du`s `/host-watch/<name>` (the container-visible path) and emits the metric labeled with the FRIENDLY host path (`/tmp`, `/var`).
5. **Collision / overlap / containment validation (NEW logic — the existing generator lacks it).** Before emitting a mount, the generator MUST reject/dedupe: a watched path that equals, contains, or is contained-in an existing mount target; a duplicate watched path; a target-name collision under `/host-watch/`. This is a real, tested deliverable — not free. A `make test-generate-watched-dirs` unit test (mirroring `test-generate-build-mounts`) covers the collision cases.
6. **`host-setup.sh` / Makefile wiring** — a `make generate-watched-dirs-mounts` target + any host-setup step (the host paths are read by root-readable `du`; confirm the container user can read them via the `:ro` mount, or whether host-setup needs an ACL like the cron path does).

### Alerts
7. **Absolute-threshold alerts** — `deploy/vmalert/metrics/watched_directories.yaml` (+ test mirror): per watched directory, `homelab_host_directory_bytes{path="/var"} > <crit_bytes>` → critical, `> <warn_bytes>` → warning, with `{{path}}` in the annotation. Thresholds read from / consistent with the config defaults (`/tmp` 1G/4G, `/var` 10G/25G). The rule should be generic over the `path` label where possible, with per-path thresholds expressed via the metric value vs a companion threshold metric OR per-path rule entries (Design decides — a companion `homelab_host_directory_warn_bytes{path}`/`_crit_bytes{path}` emitted from config would let ONE generic rule cover all dirs; otherwise one rule per dir).

### Grafana
8. **Grafana panel parallel to the disk-mountpoint bargauge ("the mount component").** A new bargauge panel on `deploy/grafana/dashboards/host-overview.json` showing `homelab_host_directory_bytes{path}` per watched directory, unit bytes, with warn/crit thresholds colored (mirroring the existing "Disk used % per mountpoint" bargauge — same type/options, next free grid slot, jq-validated, no gridPos overlap). **Re-dump the dashboard's CURRENT panels/ids/gridPos in Design** (`jq '.panels[] | {id,title,gridPos}'`) before placing the new panel — the 2026-06-15 snapshot saw ~9 panels with the disk panel at id=4, but the layout shifts; do not hardcode the panel id or next-y from this card, derive them live.

## Decisions to confirm in Design

- **D-005A-008-SPLIT** — keep as one stage vs split into 008a (collector+mounts+config) and 008b (alerts+panel). If split, create the second card + bump the epic denominator.
- **D-005A-008-MEASURE** — `os.walk` vs `du -x -s` shell-out; per-directory timeout value; truncated-flag/metric behavior; the `-x` no-cross-filesystem decision.
- **D-005A-008-MOUNT-NAMESPACE** — confirm `/host-watch/<name>:ro` target namespace (verified disjoint) and the friendly-path relabel (container `/host-watch/var` → emitted `path="/var"`).
- **D-005A-008-COLLISION-RULES** — the exact collision/overlap/containment rejection rules + whether to error-and-abort or skip-and-warn on a colliding entry.
- **D-005A-008-THRESHOLD-DELIVERY** — per-path thresholds: companion threshold metrics (one generic alert rule) vs per-path rule entries. Companion-metric approach is cleaner for a config-driven list but adds emitted metrics.
- **D-005A-008-CONFIG-MECHANISM** — baked `HostCollectorConfig` default (the only live mechanism today) vs reading a `watched-directories.yaml` from `/config` (the same surface the dangling STAGE-014 YAML loader would use — confirm whether `/config` YAML reading is wired for ANY collector yet, or strictly baked defaults).
- **D-005A-008-PERMISSIONS** — confirm the container user can `du` the `:ro`-mounted host dirs, or whether host-setup needs to grant read (like the cron ACL).

## Acceptance criteria

- `homelab_host_directory_bytes{path}` emits real sizes for `/tmp` and `/var` on the prod rig (3b), with the mounts generated + applied via the script flow.
- The mount generator's collision/overlap/containment validation has a unit test (`make test-generate-watched-dirs`) covering: duplicate path, path contains existing mount, path contained-in existing mount, target-name collision.
- Walking `/var` does NOT hang the collector past its timeout (the timeout + truncated behavior works; a too-big dir is visible, not silently zero).
- Absolute-threshold alerts fire on a planted oversized dir (test mirror) and are silent at baseline.
- The Grafana panel renders the watched-directory sizes against thresholds (jq-valid, no gridPos overlap) — visual sign-off if the dashboard is reviewed.
- Config-driven: changing the watched-directories list + thresholds + regenerating mounts + recreating the container picks up the new dirs WITHOUT hardcoded per-directory edits (the host-root/host-watch mount + config list is the only surface).
- Severity vocab + `source_tool: vmalert-metrics` + `target_kind: host` labels. Open-source-safe defaults (`/tmp`, `/var` are generic; no host-specific paths baked unless flagged).

## Design Phase

**Status**: [x] Design (Complete)

### Design Notes

**Verified (source + live rig):**
- HostCollector `HostCollectorConfig` (host.py:55-77) uses Pydantic baked defaults via `Field(default_factory=...)`, read at runtime via `getattr(ctx.config, field, default)`. PluginLoader constructs a BASE CollectorConfig — STAGE-014 YAML loading is NOT live. → baked defaults are the ONLY working config mechanism (resolves D-005A-008-CONFIG-MECHANISM).
- Existing recursive dir-size: `dir_size_bytes()` self_disk.py:30-42 (`os.walk(followlinks=False)` + `st_size`). NO subprocess/`du` collector exists anywhere.
- HostCollector `timeout = 5s` (ClassVar), scheduler-enforced via `asyncio.wait_for`; a slow walk would cancel the WHOLE host collector run.
- Mount generator `scripts/generate-compose-override.sh` reads build-sources.yaml, `sort -u` only, ZERO collision logic. `/host-watch/` target namespace is FREE. Test target embedded in Makefile (`test-generate-build-mounts`).
- Monitor runs as uid 1000; `:ro` mounts preserve host perms → `/var` has root-only subdirs (e.g. /var/lib/private) that uid 1000 cannot read.
- Dashboard host-overview.json bind-mounted (30s poll). Disk bargauge = panel id 4 (type bargauge, datasource {prometheus, victoriametrics}, unit percent, horizontal gradient, absolute thresholds). MAX panel id = 9; bottom-most panel y=40 (next free y=40).

**Decisions:**
- **D-005A-008-SPLIT** → ONE stage (NOT split). Pieces are thin increments on existing patterns sharing one acceptance test; honors the user directive ("008 will be normal"). No denominator change.
- **NEW-vs-EXTEND** → NEW `WatchedDirSizeCollector` class (own config `WatchedDirSizeCollectorConfig`), own longer timeout (~30s) + slower cadence. Isolates the slow `/var` walk from HostCollector's shared 5s core-metric budget. The locked epic's "HostCollectorConfig / generated override" is an EXAMPLE of the config-driven mechanism, not a mandate; intent (config-driven, not hardcoded) preserved.
- **D-005A-008-MEASURE** → `os.walk` + `st_size` in `run_in_executor` (so the sync walk doesn't block the async loop), with a monotonic per-directory DEADLINE check (break early → `truncated` flag), `st_dev` pruning for `-x` (don't cross filesystem), and `os.walk(onerror=...)` to skip unreadable subdirs. Reuses the existing dir_size_bytes idiom; NO new subprocess/`du` dependency. REJECTED: `du -x -s` subprocess (no pattern to mirror, new kill-on-timeout branch + binary dependency + heavier test surface).
- **D-005A-008-MOUNT-NAMESPACE** → container target `/host-watch/<name>` where `<name>` = strip leading `/` and replace remaining `/`→`-` (`/var`→`var`, `/tmp`→`tmp`, `/var/log`→`var-log`); reject `/`. The collector reads `/host-watch/<name>` but emits the FRIENDLY host path (`path="/var"`) via an in-collector config-built `{container→friendly}` map (mirrors `disk_mountpoint_relabel`).
- **D-005A-008-COLLISION-RULES** → a Python validator (pytest-covered for 100% coverage) that the bash generator shells out to; 4 rules ALL error-and-abort with descriptive messages: (1) duplicate path, (2) watched path CONTAINS an existing mount target, (3) watched path CONTAINED-IN an existing mount, (4) target-name collision under `/host-watch/`. Normalize paths (normpath, require absolute) before checking. REJECTED: bash-only validation (awkward to unit-test); skip-and-warn (a silently-unmounted configured dir = no metric; build-time config errors must fail fast).
- **D-005A-008-THRESHOLD-DELIVERY** → **Per-path rule entries** in `deploy/vmalert/metrics/watched_directories.yaml` (one warn + one crit rule per path, thresholds matching the baked config: /tmp 1G/4G, /var 10G/25G), `{{ $labels.path }}` in annotations. **OVERRODE the brainstormer's companion-metric + generic `> on(path) group_left()` join** because that join is the epic's #1 recurring bug (§4.1 cross-metric label-match silently returns empty = dead rule). Per-path rules are promtool-trivial and still config-driven in spirit (the rule file's thresholds derive from the same baked source-of-truth config; the file changes only when the baked config changes, which is a code change anyway). A generic-join / generated-rules approach can be revisited when STAGE-014's dynamic YAML config lands (future enhancement under STAGE-014's scope, not a deferral of 008 scope). REJECTED: companion `_warn_bytes`/`_crit_bytes{path}` + generic join (fragile empty-join trap).
- **D-005A-008-CONFIG-MECHANISM** → baked `watched_directories: list[WatchedDirectory]` default on `WatchedDirSizeCollectorConfig`, read via `getattr`. `WatchedDirectory{path: str, warn_bytes: int, crit_bytes: int}`. Default: /tmp (1G/4G), /var (10G/25G). REJECTED: reading a `/config` YAML (not wired for any collector today).
- **D-005A-008-PERMISSIONS** → graceful skip via `os.walk(onerror=handler)`; size what's readable, count unreadable subdirs into `homelab_host_directory_unreadable_subdirs{path}`; NEVER fail the whole walk on a PermissionError. NO host setfacl requirement. REJECTED: host-setup.sh setfacl on /var (heavy/invasive; root-only `private` dirs exist by design).

**Metric names (consistent with the `homelab_host_disk_bytes` family):**
- `homelab_host_directory_bytes{path}` — recursive byte total per watched dir.
- `homelab_host_directory_walk_truncated{path}` — 1 if the deadline was hit (size is a lower bound), else 0.
- `homelab_host_directory_unreadable_subdirs{path}` — count of subdirs skipped due to PermissionError.

**Deliverable surfaces (ONE stage):**
1. NEW `WatchedDirSizeCollector` + `WatchedDirSizeCollectorConfig` (Python, baked default config, run_in_executor walk with deadline + onerror, friendly-path relabel) + plugin registration.
2. NEW Python collision validator + `scripts/generate-watched-dirs-mounts.sh` (or extend the existing generator) emitting `/host-watch/<name>:ro` mounts into `docker-compose.override.yml`, + `make generate-watched-dirs-mounts` + `make test-generate-watched-dirs` (pytest over the validator: dup / contains / contained-in / name-collision).
3. NEW `deploy/vmalert/metrics/watched_directories.yaml` (+ test mirror) — per-path warn+crit rules, `severity` + `source_tool: vmalert-metrics` + `target_kind: host`, `{{ $labels.path }}`.
4. NEW Grafana bargauge panel on host-overview.json (id 10, gridPos x=0 y=40 w=12 h=8, unit bytes/decbytes, `homelab_host_directory_bytes` per path, legendFormat `{{path}}`, thresholds), jq-validated, derive id/y LIVE in Build (re-dump first).

**Validation:** 100% kernel coverage on the new collector (every branch: normal walk, deadline-truncated, unreadable-subdir, empty/missing dir). Rule file promtool-validatable (per-path threshold comparisons, no subqueries/joins). 3a synthetic dirs + collision unit test; 3b REAL /tmp//var on prod rig with mounts applied + monitor RECREATE + panel review.
