# STAGE-009-003: `fixer-runner` container — Dockerfile + static CLAUDE.md + compose wiring

## Status: Complete

## Overview

Build the dedicated `fixer-runner` container (LOCKED Decision 1): a minimal image with the `claude` CLI installed, running as `homelab-fixer`, that the orchestrator `docker exec`s into to launch a runbook non-interactively. Ships a **static, container-level `CLAUDE.md`** baked into / mounted in the container that tells the fixer it is running **inside a Docker container** (not the host) and gives it an explicit allow/deny list plus the hard rule that it is **fully non-interactive** (never wait for or request user input). Wired into the prod compose as an **OPTIONAL container** (compose `profiles`, OFF by default per the conservative public posture).

**Vertical slice:** after this stage, the `fixer-runner` container builds and can be started (under its profile); the orchestrator can `docker exec` a non-interactive `claude -p <folder>` into it (proven with the FAKE claude binary, not the real CLI, in CI); the static container CLAUDE.md is present and correct; mid-run `docker kill` terminates it.

**Spec references:** §7.4 (two execution flavors — container chosen; egress allowed for Anthropic API, inbound denied; killable via `docker kill`), §10.1 (fixer-runner OPTIONAL container in the compose list), §10.3 (resource budget: fixer-runner 0 idle, bursty during runs). EPIC-009 LOCKED Decision 1.

## Cross-cutting standards (from EPIC-009 cross-stage acceptance criteria)

- Non-negotiable #2 (scope/intent): the static container CLAUDE.md is the always-applied scope manifest (distinct from each runbook's own CLAUDE.md); it constrains the fixer regardless of runbook.
- Non-negotiable #3 (identity): the container runs as `homelab-fixer`, never root.
- Non-negotiable #7 (kill switch): the container is killable mid-run via `docker kill` — proven here at the container level (the orchestrator-side kill gate is STAGE-009-007).
- **FAKE claude in CI:** the container's invocation contract is exercised with a fake `claude` shell script (records args, emits recorded transcript). The real CLI is never invoked in CI.
- Open-source-safe: the container is OFF by default (profile-gated); a fresh install has no running fixer.

## Concrete deliverables

### 1. `fixer-runner` Dockerfile

Minimal image, `claude` CLI installed, runs as `homelab-fixer` (from STAGE-009-002). Anthropic API credential resolved at runtime via the secrets store (egress to the Anthropic API allowed; inbound denied — egress allow-list detail is STAGE-009-008).

### 2. Static container-level CLAUDE.md

Baked in / mounted: "you are inside a Docker container (not the host); you are fully non-interactive — never wait for or request input; here is your hard allow/deny list." This applies to ALL runbooks.

### 3. Non-interactive invocation contract

Define how the orchestrator invokes `claude` in the container: headless `-p <runbook-folder>`, stdin `/dev/null`, `--dangerously-skip-permissions`, no prompts. Document the exact `docker exec` command shape the orchestrator (STAGE-009-005) will use.

### 4. Compose wiring (OPTIONAL container)

Add `fixer-runner` to `deploy/compose/docker-compose.yml` under a `profiles:` gate (off by default). Document how the user opts it in.

### 5. Inherited from STAGE-009-002 (provisioning hand-off): OS user identity

The `homelab-fixer` in-container OS user MUST adopt the `HM_FIXER_UID` / `HM_FIXER_GID` numeric identity that STAGE-009-002's host ACLs grant (defaults 1002:1002; real values in the overrides repo). This is the seam that makes the cross-container transcript ACLs work.

### 6. Inherited from STAGE-009-002: transcript bind-dir mount

Mount the host transcript bind-dir (`${HM_FIXER_TRANSCRIPTS_SRC:-/var/lib/homelab-monitor/runbook-transcripts}`) into the `fixer-runner` container at `/data/runbook-transcripts` **read-write** (the fixer-runner is the WRITER; the monitor mounts the same dir `:ro`). Contrast with the monitor's `:ro` mount added in STAGE-009-002.

### 7. Inherited from STAGE-009-002: Real-host transcript-dir provisioning

Instance-A Refinement must apply the real-host transcript-dir provisioning by running `sudo bash scripts/host-setup.sh` (section 3.9 — creates `/var/lib/homelab-monitor/runbook-transcripts` + applies the monitor-`rX` / fixer-`rwX` POSIX default ACLs). STAGE-009-002 validated this logic in a sandbox but deliberately did NOT mutate the real host (no writer existed yet); 003 is where the prod path is actually created and an end-to-end fixer-writes / monitor-reads transcript round-trip is validated.

### 8. Tests / validation

CI: the FAKE-claude exec path works non-interactively (stdin closed; records args; emits transcript). Instance-A: the container builds and starts under its profile; `docker kill` terminates it.

### Design Notes

**Selected design (Design phase, brainstormer-Opus options, coordinator-selected):**

A. **claude CLI install** — Native binary, version-pinned via Dockerfile `ARG CLAUDE_VERSION`, checksum-verified, auto-update disabled, NO Node runtime (base is python:3.12-slim / apt-get). ONE Dockerfile with `ARG CLAUDE_BINARY_SOURCE=real|fake`: `real` fetches+verifies the pinned binary; `fake` COPYs the in-repo fake-claude script to the same PATH location. CI builds with `--build-arg CLAUDE_BINARY_SOURCE=fake` so it tests the real container topology (identical user/entrypoint/mount surface) with only the binary differing; the real binary never lands in the CI image and the fake never lands in the prod image. (The exact current install URL/mechanism is verified against official Claude Code docs during Build, not trusted from a memorized version number.)

B. **Static container CLAUDE.md** — Hybrid: a PUBLIC-SAFE floor is BAKED into the image (universal invariants only: "you are inside a Docker container, not the host; you are fully non-interactive — never wait for or request input; deny-by-default; if uncertain, exit non-zero rather than improvise"). The HOST-SPECIFIC allow/deny target list lives in the overrides repo and is mounted READ-ONLY (e.g. /data/policy/CLAUDE.host.md); the baked floor references/appends it. Rationale: the container is dangerous WITHOUT a CLAUDE.md, so the safety invariants must be baked (can't be unmounted away), while host targets stay out of the public repo per §11. 003 ships the baked public file + the mount point + compose plumbing; the real host allow/deny TARGET content is authored in the overrides repo and is STAGE-009-008's scoped-capability job.

C. **Anthropic API credential** — In scope for 003: `ANTHROPIC_API_KEY` wired as a compose `.env`/host-env passthrough (`environment: ANTHROPIC_API_KEY: ${ANTHROPIC_API_KEY:-}`), documented as overrides/host-only, NEVER committed to the public repo. CI uses the fake binary so no real key is needed — the test asserts the var is absent/empty under fake. DEFERRED to STAGE-009-005: move to exec-time injection through the existing HOMELAB_MONITOR_MASTER_KEY secrets vault (key present only during an in-flight remediation, not for the idle container's lifetime — least-privilege per non-negotiable #3). A TODO comment in the compose file + this note name STAGE-009-005 as the owner.

D. **Egress network posture** — Dedicated user-defined bridge network for fixer-runner, NO published ports (no inbound), internet-reachable for the Anthropic API. Delivers the spec's egress-only INBOUND posture and isolates fixer-runner laterally from the monitoring sidecars. DEFERRED to STAGE-009-008: per-runbook egress DESTINATION allow-listing (api.anthropic.com-only filtering) — 008 explicitly owns "fixer-runner egress control". A compose comment names 008 as the owner.

E. **Non-interactive guarantee + entrypoint** — Idle-keepalive entrypoint: PID 1 is `tail -f /dev/null` (does not swallow signals), container starts and stays alive idle as the homelab-fixer user. claude is NEVER run at boot; it is launched ONLY via `docker exec`. The orchestrator (STAGE-009-005) exec shape that 003 validates manually + in CI:
  `docker exec -i -u homelab-fixer -e ANTHROPIC_API_KEY=<005-injected> <container> claude -p <runbook-folder> --dangerously-skip-permissions < /dev/null`
Four layered non-interactive guarantees (defense in depth): (1) stdin `< /dev/null` → reads return EOF immediately, can never block on a prompt; (2) `-p` headless/print mode → no TUI/REPL; (3) `--dangerously-skip-permissions` → no permission prompts that wait for yes/no; (4) baked CLAUDE.md instructs the model to never request input and exit non-zero rather than wait. Kill switch (#7): because PID 1 is `tail -f /dev/null` and claude runs as an exec'd child, `docker kill <container>` tears down the container AND any in-flight exec regardless of run state.

F. **FAKE claude CI harness** — A small fake `claude` script (shell or python) lives in-repo in the fixer-runner build context (e.g. deploy/compose/fixer-runner/test/fake-claude). It: reads stdin to EOF (proves stdin=/dev/null doesn't hang), appends argv to a record file under the mounted transcript dir, writes a transcript file there (proves the RW mount + HM_FIXER_UID:GID ownership), honors a `FAKE_CLAUDE_SLEEP` env (sleeps before exit so the kill test has a long-running in-flight exec), exits 0. Validation = pytest integration test under apps/monitor/tests/integration/, gated by `require_rig_components(...)` (auto-skips fast when Docker/rig absent, runs under `make integration`, never blocks make verify / test-fast). Asserts: (a) non-interactive exec records expected argv + emits a transcript owned by HM_FIXER_UID:GID; (b) `docker kill` terminates a `FAKE_CLAUDE_SLEEP`-stalled in-flight exec.

**Explicit scope boundary for 003:** builds + compose-wires + validates the fixer-runner container with a fake claude and manual/CI `docker exec`. It does NOT launch claude from the app (STAGE-009-005 orchestrator), does NOT implement egress DESTINATION filtering (STAGE-009-008), does NOT author the host allow/deny TARGET content (overrides repo + STAGE-009-008), does NOT do vault-based exec-time key injection (STAGE-009-005).

**Files this design will touch (anticipated):** deploy/compose/fixer-runner/Dockerfile, deploy/compose/fixer-runner/CLAUDE.md (baked public floor), deploy/compose/fixer-runner/test/fake-claude, deploy/compose/docker-compose.yml (new `fixer-runner` service under `profiles: ["fixer"]`, OFF by default), deploy/compose/.env.example (ANTHROPIC_API_KEY passthrough + any new vars), apps/monitor/tests/integration/ (new test + a docker/fixer-image rig-health component), docs/deploy/autofix-provisioning.md (extend with credential + profile-enable + claude-CLI sections).

**Non-negotiables touched (Design-level assertion):** #3 identity (homelab-fixer created in-image with HM_FIXER_UID/GID 1002:1002, ends USER homelab-fixer, NEVER root, NO docker group, NO sudoers) and #7 kill switch (docker kill terminates mid-run; the four-layer non-interactive guarantee makes blocking-on-input impossible). Code review at Finalize must PASS/FAIL all seven.
