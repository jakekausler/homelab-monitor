# STAGE-009-010: Auto-fix UI — Runbooks screen + manual "Run fix" trigger surface

## Status: Not Started

## Overview

Build the Runbooks screen (replacing the existing 'Coming soon' nav placeholder) and the manual "Run fix" trigger surface. The Runbooks catalog shows one card per registered runbook: name, alert match patterns, risk tag, enabled/auto-trigger state, last run, success rate. The manual trigger lets the user run a runbook on demand — with a dry-vs-real toggle that is **forced to dry for risky runbooks** and a **confirm-on-destructive** step for real runs. NOTE: the original stage table said this "extends STAGE-001-019's alert-detail drawer" — that drawer DOES NOT EXIST (alerts currently route through the Karma iframe). This stage CREATES the trigger surface (on the Runbooks screen, and/or a new alert-detail panel using the existing `GET /api/alerts/{id}`).

**Vertical slice:** after this stage, the Runbooks screen lists runbooks with their state; the user can trigger a runbook manually; risky runbooks force dry mode; real runs require confirm-on-destructive. Desktop + Mobile viewport sign-off.

**Spec references:** §9.1 (Runbooks nav entry), §9.2 (Runbooks screen — catalog cards: name, match patterns, risk tag, last run, success rate, "Run now / Dry run / Edit CLAUDE.md"; Alerts screen "Run fix" button), §7.2 (confirm-on-destructive on a real runbook run). ## Concrete deliverables

### 1. Runbooks catalog screen

Cards per registered runbook: name, match patterns, risk tag, enabled + auto-trigger state, last run, success rate. Enable/disable + auto-trigger toggles (calling the STAGE-009-004 API). "Edit config" link.

### 2. Manual "Run fix" trigger surface

A trigger control (on the Runbooks screen and/or an alert-detail panel using `GET /api/alerts/{id}`): pick a runbook (or it's pre-selected from an alert match), dry-vs-real toggle FORCED to dry for risky, confirm-on-destructive for real. Calls the orchestrator (STAGE-009-005) / dry-run flow (STAGE-009-006).

### 3. Tests + sign-off

Vitest: catalog renders runbook state; risky → real toggle disabled; real → confirm required; trigger calls the authed endpoint. Desktop + Mobile viewport sign-off.
