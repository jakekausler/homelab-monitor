# STAGE-009-002: `homelab-fixer` provisioning + transcript ACLs

## Status: Not Started

## Overview

Stand up the `homelab-fixer` low-privilege identity and the filesystem permissions the auto-fix subsystem needs, per non-negotiable #3 (Claude runs as `homelab-fixer`, never `jakekausler` or root). Because LOCKED Decision 1 puts the fixer in a dedicated **container**, the `homelab-fixer` user lives *inside* the `fixer-runner` container — this stage clarifies what host-side provisioning is still required: the `/data/runbook-transcripts/` ACLs so the fixer user can write transcripts, and confirming the orchestrator (monitor container) has the Docker-socket access it needs to `docker exec` into `fixer-runner` (it already has the socket from EPIC-003 — verify and document).

**Vertical slice:** after this stage, the `homelab-fixer` identity + transcript-dir ACLs are documented and provisioned (setup script in the host-overrides repo, instructions in the public release), and the orchestrator's docker-socket access path is verified. No orchestration yet.

**Spec references:** §7.4 (#3 identity, curated ACLs, narrow sudoers), §10.2 (volume mounts — `data/runbook-transcripts/` + the override repo's `runbooks/` bind mount), §11 (public vs `homelab-monitor-overrides` split). EPIC-009 LOCKED Decision 1.

## Concrete deliverables

### 1. `homelab-fixer` identity + transcript ACLs

The `homelab-fixer` user inside the `fixer-runner` container (UID/GID strategy documented so transcript files written in-container are readable by the monitor for the history UI). Host-side: `/data/runbook-transcripts/` ACLs allowing the fixer to write, the monitor to read. Setup script in `homelab-monitor-overrides`; public-release docs describe the entire OS-side setup.

### 2. Orchestrator docker-socket access verification

Confirm + document that the monitor container can `docker exec` into a sibling container (it already mounts the docker socket for EPIC-003 discovery). Note any additional permission needed for `docker exec` specifically (vs read-only discovery).

### 3. Tests / validation

Instance-A validation: the transcript dir is writable by the fixer identity and readable by the monitor; the documented setup is reproducible. (No Python unit gate unless this stage adds code; if it adds config/ACL-management code, the 100%-branch gate applies.)
