# STAGE-009-011: Auto-fix history UI — filterable table + transcript viewer + feedback display

## Status: Not Started

## Overview

Build the Auto-fix history screen: a filterable table of `runbook_runs` (by runbook, mode, exit code, date), a transcript viewer that renders the full captured Claude session for a selected run, and the run metadata (exit code, durations, dry-vs-real, runbook hash). Also displays the improvement-feedback items (from STAGE-009-009) attached to each run. This is the read-only audit/observability surface for everything the orchestrator records.

**Vertical slice:** after this stage, the user can browse all auto-fix runs, filter them, open any run to read its full transcript + metadata, and see any improvement feedback the fixer emitted. Desktop + Mobile viewport sign-off.

**Spec references:** §9.1 (Auto-fix history nav entry), §9.2 (Auto-fix history screen — filterable table; row click opens transcript viewer with full Claude session, exit code, durations, before/after diff if generated), §7.4 (#4 audit — this is where the audit becomes visible). EPIC-009 LOCKED Decision 3 (feedback display).

## Cross-cutting standards (from EPIC-009 cross-stage acceptance criteria)

- Frontend stage → Desktop + Mobile viewport sign-off against the prod stack (rebuild + sha256-prove the `.tsx`).
- TypeScript strict, eslint 0 errors, vitest tests (semantic queries).
- Non-negotiable #4: the history view is READ-ONLY — there is no delete/edit affordance for runs/transcripts (immutability enforced backend-side in STAGE-009-012; the UI must not offer deletion).
- Surfaces both `dry` and `real` runs (the digest section consumes the same data; the UI is the human view).

## Concrete deliverables

### 1. Filterable runs table

`runbook_runs` table with filters (runbook, mode dry/real, exit code, date range). Columns: runbook, mode, exit code, started/ended, duration, alert.

### 2. Transcript viewer

Row click → full captured Claude session (transcript/stdout/stderr), exit code, durations, runbook hash. Read-only.

### 3. Feedback display

Show the `runbook_run_feedback` items (STAGE-009-009) for the selected run.

### 4. Tests + sign-off

Vitest: table renders + filters; row opens transcript; feedback items display; no delete affordance present. Desktop + Mobile viewport sign-off.
