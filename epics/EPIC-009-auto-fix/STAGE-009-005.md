# STAGE-009-005: Orchestrator core — match → checks → docker-exec claude → capture → persist

## Status: Not Started

## Overview

Build the heart of the auto-fix subsystem: the runbook orchestrator. Given an ingested alert, it matches the alert to a registered runbook, runs the gate sequence (kill switch → allow-list/enabled → rate-limit → cooldown), claims the runbook, grants its declared scoped capabilities to the `fixer-runner` container, `docker exec`s `claude` non-interactively, captures the full transcript + stdout/stderr + exit code, and writes `runbook_runs` + an `alert_outcomes('auto_fixed')` row + an `audit_log` row. This is the keystone stage — it cannot be built before the schema (001), the fixer-runner (003), and the registry (004) exist. **Built and tested entirely against the FAKE `claude` binary** (a shell script that records args + emits a recorded transcript); the real Claude API is NEVER called in CI.

**Vertical slice:** after this stage, an alert that matches an enabled+auto-trigger runbook flows through the full gate sequence and (in real mode) execs the fake claude in the fixer-runner, capturing a complete audited run record. Rate-limit + cooldown are enforced. The dry-run gate (STAGE-009-006) and kill switch (STAGE-009-007) layer on top.

## Concrete deliverables

### 1. Alert → runbook match

Match an ingested alert against registered runbooks' `match_patterns` (regex vs alertname/labels). Resolve to at most one runbook (or none → no auto-fix; manual button only).

### 2. Gate sequence

In order: kill switch (STAGE-009-007 provides the check; wire the call), allow-list/enabled + auto-trigger, rate-limit (max N/hour globally), cooldown (per-runbook). A failed gate denies the run with a recorded reason.

### 3. Claim + exec

Claim the runbook (prevent concurrent duplicate runs), grant its declared scoped capabilities to the fixer-runner (STAGE-009-008 provides the granting mechanism; wire it), `docker exec` the non-interactive `claude` (STAGE-009-003 contract). Capture transcript/stdout/stderr/exit/timestamps.

### 4. Persist

Write `runbook_runs` (mode, prompt, transcript_path, exit_code, timestamps, runbook_hash, fixer_user, host), `alert_outcomes('auto_fixed')`, `audit_log`.

### 5. Tests (fake claude)

End-to-end with the fake claude: match → gates → exec → capture → persist; rate-limit denial; cooldown denial; full audit record written; `auto_fixed` outcome written; no-match → no run.
