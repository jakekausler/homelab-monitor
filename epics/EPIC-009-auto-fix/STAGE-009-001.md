# STAGE-009-001: Runbook schema & config-file contract

## Status: Complete

## Overview

Lay the data foundation for the auto-fix subsystem: an additive alembic migration that fills in the behavioral columns on the stub `runbooks` and `runbook_runs` tables (which EPIC-001 created as `id`/`path`/`created_at` and `id`/`runbook_id`/`created_at` placeholders), plus a pydantic model for the **runbook-folder config file** (the machine-read manifest co-located with the runbook's markdown). Per LOCKED Decision 2, a runbook is a folder of markdown + a config file, and the **config file is authoritative** — the DB row is a registration/enablement record plus a content hash (for audit non-negotiable #4).

**Vertical slice:** after this stage, the schema and the config-file contract exist and validate; a runbook folder's config can be parsed into a typed model; the migration applies up/down cleanly. No orchestration yet — this is the contract everything else reads/writes.

**Spec references:** §6.1 (`runbooks`, `runbook_runs` table shapes), §7.4 (the seven non-negotiables the schema must support). EPIC-009 LOCKED Decisions 1–4. The existing stub tables in alembic migration 0001 (with SCAFFOLDING comments naming the columns this stage adds).

## Concrete deliverables

### 1. Additive migration

Add to `runbooks`: `alert_match_patterns` (JSON), `risk_tag` (`safe`/`risky`, default `risky`), `dry_run_required` (bool, default true), `rate_limit_per_hour` (int), `cooldown_seconds` (int), `enabled` (bool, default false), `auto_trigger` (bool, default false), `content_hash` (text). Add to `runbook_runs`: `alert_id`, `mode` (`dry`/`real`), `prompt`, `transcript_path`, `exit_code`, `started_at`, `ended_at`, `fixer_user`, `host`, `runbook_hash`. Migration applies up/down; tests cover both directions.

### 2. Runbook-folder config pydantic model

A pydantic model for the config file (YAML/TOML) declaring: `match_patterns` (regex vs alertname/labels), `risk_tag`, `dry_run_required`, `rate_limit_per_hour`, `cooldown_seconds`, and the **`scoped_capabilities`** block — `docker` (container + allowed actions), an optional `ssh` (`target_id` reference to an existing EPIC-017 target, READ-only), and an `egress` allow-list. Validation rejects malformed configs with a clear error. Decide + document the file-authoritative registry model (config = truth, DB = registration + hash).

### 3. Tests

Migration up/down; config-model validation (valid config parses; risky/disabled defaults applied; malformed config rejected; scoped-capabilities block parses docker + ssh-read + egress shapes).

## Design Notes (Design phase — selected approach)

Card verified against REAL code before adopting (charter requirement). Key reality corrections vs the charter hints:
- Alembic HEAD is **0044** (charter said 0032, now stale). New migration = **0045**, down_revision **0044**.
- Current stub schema confirmed: `runbooks`(id TEXT PK, path TEXT NN, created_at TEXT NN); `runbook_runs`(id TEXT PK, runbook_id TEXT FK→runbooks.id NN, created_at TEXT NN). No later migration alters them.
- The `alerts` table EXISTS (id TEXT PK, UUIDv7) — `runbook_runs.alert_id` can FK to it.
- Repo conventions: SQLAlchemy Core (no ORM); ALL timestamps ISO-8601 UTC TEXT; JSON stored as TEXT + `json.dumps`; enums stored as TEXT (validated in pydantic, no DB CHECK); pydantic v2 (`ConfigDict(extra="forbid")`, `StrEnum`, `@field_validator`/`@model_validator`, `load_from_path` classmethod idiom per `SubprocessManifest`); YAML is the dominant config format; content-hash idiom is `hashlib.sha256(json.dumps(...,sort_keys=True,separators=(",",":"),ensure_ascii=False).encode()).hexdigest()`.

### Decision A — Migration ALTER strategy
Use `with op.batch_alter_table("t") as batch_op: batch_op.add_column(sa.Column(...))` (the `0005_alerts_columns.py` SQLite-safe idiom). Supports the `alert_id` FK via `batch_op.create_foreign_key()` and yields a clean reversible downgrade (drop columns in reverse) for the both-directions tests.
- Booleans → `sa.Boolean()` with `server_default=sa.text("0")`/`("1")`: `dry_run_required` default 1; `enabled` default 0; `auto_trigger` default 0.
- `alert_match_patterns` → `sa.Text()` nullable (canonical JSON via `json.dumps(...,sort_keys=True,separators=(",",":"))`).
- `risk_tag` → `sa.Text()` `server_default=sa.text("'risky'")`.
- `rate_limit_per_hour`, `cooldown_seconds` → `sa.Integer()` nullable (file-authoritative; no server_default).
- `content_hash` → `sa.Text()` nullable.

### Decision B — Config pydantic model shape
Structured label matchers (NO regex on the privileged path) + nested BaseModels, all `extra="forbid"`, following `SshTargetConfig`/`SubprocessManifest`:
- `RiskTag(StrEnum)`: SAFE="safe", RISKY="risky".
- `AlertMatcher`: `alertname: str | None`, `labels: dict[str,str]`; `@model_validator` requires at least one of alertname/labels non-empty.
- `DockerCapability`: `container: str`, `allowed_actions: list[str]`.
- `SshCapability`: `target_id: str` (READ-only reference to an SshTargetConfig.id; details NOT inlined).
- `ScopedCapabilities`: `docker: DockerCapability | None`, `ssh: SshCapability | None`, `egress: list[str]`.
- `RunbookConfig`: `runbook: Literal[1]=1` (schema version), `name` (pattern), `match_patterns: list[AlertMatcher]` (min_length=1), `risk_tag: RiskTag=RISKY`, `dry_run_required: bool=True`, `rate_limit_per_hour: int (ge=0)`, `cooldown_seconds: int (ge=0)`, `scoped_capabilities: ScopedCapabilities`. Has a `load_from_path` classmethod (yaml.safe_load → isinstance guard → model_validate; wrap ValidationError→ValueError with file path).

### Decision C — Content-hash definition
`content_hash` = sha256 of the CANONICAL PARSED config (semantic, stable across YAML reformatting/comments), per the repo structured-hash idiom:
`hashlib.sha256(json.dumps(config.model_dump(mode="json"), sort_keys=True, separators=(",",":"), ensure_ascii=False).encode("utf-8")).hexdigest()`.
Covers the safety contract (capabilities, risk_tag, rate limits). Does NOT cover markdown-intent prose.
**Deferred to STAGE-009-012:** decide whether per-run drift detection must also cover markdown-intent changes (whole-folder hash incl. `*.md`) rather than config-only. Tracked as a regression item under STAGE-009-012.

### Decision D — File-authoritative registry contract (resolves card's apparent column/config duplication)
File is the source of truth; the DB row is an enablement record + a cached, queryable snapshot of scalar file values keyed by `content_hash` (drift = live-file-hash ≠ stored content_hash).
- DB-ONLY (operator gates, user-toggled via API, conservative default 0, NOT in `RunbookConfig`): `enabled`, `auto_trigger`.
- FILE-AUTHORITATIVE, cached in DB for query/audit: `risk_tag`, `rate_limit_per_hour`, `cooldown_seconds`, `alert_match_patterns` (as canonical JSON TEXT).
- FILE-ONLY (read live by the loader in STAGE-009-004; NOT a DB column): `scoped_capabilities`.
- DERIVED (DB): `content_hash`, `path`. Identity: `id`, `created_at`.
- Three orthogonal gates: registered (row exists) ≠ enabled (`enabled=1`) ≠ auto_trigger (`auto_trigger=1`).
RESOLUTION of card duplication: `enabled`/`auto_trigger` are DB-only gates and are intentionally ABSENT from `RunbookConfig`. The registry LOADER is STAGE-009-004 — this stage builds only the schema + config model so 004 can implement cleanly.

### Decision E — `runbook_runs` audit column types
TEXT-for-enums + ISO-TEXT timestamps + Integer-nullable exit_code (repo convention; enforcement in pydantic):
- `alert_id` → `sa.Text()` nullable, FK to `alerts.id` via `batch_op.create_foreign_key()`. Nullable so a manual/dry-run-test invocation without a triggering alert is allowed (audit still records it when present).
- `mode` → `sa.Text()` nullable ('dry'/'real', StrEnum-enforced).
- `prompt`, `transcript_path`, `fixer_user`, `host`, `runbook_hash` → `sa.Text()` nullable.
- `exit_code` → `sa.Integer()` nullable (null while running).
- `started_at`, `ended_at` → `sa.Text()` nullable (ISO-8601 UTC). `runbook_hash` = per-run snapshot of the runbook's content_hash at execution time (009-012 compares it against the runbook's current content_hash for drift).

### Seven non-negotiables — inputs this stage encodes
#1 trigger allow-list → `match_patterns`; #2 scope → `scoped_capabilities` (docker/ssh-read/egress); #5 dry-run/risky → `risk_tag` default `risky` + `dry_run_required` default true; #6 rate-limit+cooldown → `rate_limit_per_hour`+`cooldown_seconds`. #3 identity (`fixer_user`), #4 audit (alert_id/prompt/transcript_path/exit_code/started_at/ended_at/host/runbook_hash) captured in `runbook_runs`. (#7 kill-switch is STAGE-009-007.)
