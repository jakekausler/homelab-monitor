# STAGE-009-012: Audit immutability + runbook_hash enrichment + transcript rotation

## Status: Not Started

## Overview

Harden the audit surface to satisfy the full breadth of non-negotiable #4. Enforce **audit immutability**: `runbook_runs` and `audit_log` rows cannot be deleted via the API (no delete endpoint; any attempt is rejected). Ensure every run carries the **`runbook_hash`** of the runbook's authoritative content at run time (so the system can detect whether a runbook was modified between runs — and invalidate a stale dry-run approval). Implement **transcript rotation** per §10.2: keep the last N=100 transcripts per runbook and a max age of 365 days, both configurable; prune oldest — but never silently delete (the `runbook_runs` audit row is retained even after the transcript file is pruned).

**Vertical slice:** after this stage, runs/audit rows are non-deletable via the API; every run records its runbook hash; transcripts rotate per policy without losing the audit row. This hardens the data the orchestrator (STAGE-009-005) writes.

**Spec references:** §7.4 (#4 audit — full detail + immutability), §10.2 (transcript rotation: keep last N=100, max age 365d, configurable, prune-not-silently-delete, audit row retained after transcript gone), §6.1 (`runbook_runs.runbook_hash`, `audit_log`). EPIC-009 LOCKED Decision 2 (file-authoritative → hash detects drift).

## Concrete deliverables

### 1. Audit immutability

No delete endpoint for `runbook_runs` / `audit_log`. Any delete attempt (direct or via a generic path) is rejected. Test: an authenticated delete attempt → rejected.

### 2. runbook_hash enrichment

Every run records the hash of the runbook's authoritative content at run time. A dry-run approval references the hash; if the runbook changes, the approval is invalidated (ties to STAGE-009-006).

### 3. Transcript rotation

A rotation job: keep last N=100 per runbook + max age 365d (configurable); prune oldest transcript FILES; retain the `runbook_runs` row (with a marker that the transcript file is gone). Never silently delete an audit row.

### 4. Tests

Delete attempt rejected; hash recorded + drift detected; rotation prunes files beyond N/age but keeps audit rows; the "transcript pruned" marker is set.
