# STAGE-005-023: Entity-health + battery widgets

## Status: Design (Not Started)

## Overview

Fill the first two sections of the HA panel shell (005-022) with live widgets: an entity-health grid
(total / available / unavailable / stale counts + a drill list) and a battery summary (devices low /
critical + a list). Consumes the `/api/integrations/home-assistant/summary` (+ detail) endpoint from
005-021.

**Vertical slice:** after this stage, the HA panel shows real entity-health counts and a battery
summary that updates from live data, letting the user see "12 entities unavailable, 2 batteries below
10%" at a glance.

**Spec references:** master spec §9.2 (HA panel shows entity health), EPIC-005 §"Stage decomposition"
Wave F. Data: 005-021 endpoint; UI pattern: the Docker integration list/widgets.

## Concrete deliverables

### 1. Data hook

A `useHomeAssistantSummary()` (and entity-detail hook if 021 split it out) in
`apps/ui/src/api/home_assistant.ts`, wrapping `GET /api/integrations/home-assistant/summary` via the
generated client (mirrors `apps/ui/src/api/docker.ts` hooks).

### 2. Entity-health grid widget

Counts (total / available / unavailable / stale) as stat tiles + a drill list of unavailable/stale
entities (entity_id, domain, last-changed age). Reuses the project's stat-tile / table primitives.

### 3. Battery summary widget

A summary (count low <20% / critical <10%) + a list of low/critical batteries (device, level). Reuses
severity tinting where sensible.

### 4. Tests

Render with mocked summary data → assert counts + lists render; empty state (no unavailable / no low
batteries) renders calmly; loading + error states. Desktop + Mobile layout.

## Out of scope for this stage

- Embedded LogViewer (005-025).
- The backend endpoint (021) and Grafana (026).
