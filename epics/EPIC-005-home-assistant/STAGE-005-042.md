# STAGE-005-042: HA cadence detail API + Status-tab idle/disabled cadence widget

## Status: Not Started

## Overview

Split off from STAGE-005-038 (D-CADENCE-SURFACE-SPLIT, locked in 038 Design). STAGE-005-038 shipped the script-cadence **Grafana** panel (parity with the automation panels). This stage adds the **in-app** cadence surface: a cadence detail API endpoint plus a Status-tab widget/drill that lists idle (>24h) + disabled automations and idle scripts ‚Äî mirroring the existing `/entities` `/batteries` detail-endpoint + `HaEntitiesDrill`/`HaUpdatesDrill` drill pattern.

**Ordering boundary note (why 042 runs AFTER 041):** STAGE-005-041 is the interactive **Grafana** dashboard review (System + HA dashboards). 041's deliverables are Grafana dashboard JSON edits + metric‚Üípanel coverage reports. The cadence artifact 041 reviews ‚Äî the script-cadence **Grafana** panel ‚Äî is built in 038, BEFORE 041, so 041 reviews a complete Grafana surface. This stage's deliverable is the **in-app** Status-tab widget, which is NOT a Grafana panel and is therefore OUTSIDE 041's Grafana-dashboard review scope. Consequently 042 legitimately runs after 041 with no forward-dependency violation: 041 does not depend on the in-app widget.

**Depends on:** STAGE-005-038 (cadence surface decision + script Grafana panel), STAGE-005-034 (clean idle data). Soft-depends STAGE-005-037 (registry-filtered availability ‚Äî keeps the cadence row data clean).

## Stage Flags

- Has Input Forms: No.
- Touches secrets / production data: No (automation/script entity_ids + ages + enabled flag; same posture as the existing cadence metrics and detail endpoints).
- Host-integration concerns: Yes for the backend portion ‚Äî cadence API validated against real automation/script metrics in VM; Refinement 3a + 3b. The widget portion ‚Üí 3a Desktop + Mobile (frontend dual sign-off).

## Concrete deliverables

1. **Cadence detail endpoint** `GET /api/integrations/home-assistant/cadence` returning typed rows:
   - automation rows: `entity_id`, `enabled` (bool), `last_triggered_age_seconds` (float | None)
   - script rows: `entity_id`, `last_triggered_age_seconds` (float | None) ‚Äî NO enabled column (scripts have no enabled flag; D-CADENCE-SCRIPT-DISABLED)
   - Sourced from VM per-series gauges (`homelab_ha_automation_last_triggered_seconds`, `homelab_ha_automation_enabled`, `homelab_ha_script_last_triggered_seconds`) via the `_sample_float` / `vm_instant_query` pattern in `kernel/api/routers/integrations_home_assistant.py` (D-CADENCE-API-SOURCE: VM, consistent with locked D-DETAIL-CONSUMES-VM).
   - New Pydantic response models ‚Üí OpenAPI regen ‚Üí `schema.ts` ‚Üí drill test-mock cascade (the 031 schema-cascade gotcha applies; re-added `T | None` fields become `required` for TS consumers ‚Äî add new fields to every sibling drill mock).
   - Privacy posture: `entity_id` shown to the authenticated owner; same D-ENRICH-PRIVACY posture as the other detail endpoints (free-text/identifying fields stay out of metrics/logs, may be shown to the owner).
2. **Frontend Status-tab widget + drill** mirroring `HaEntitiesDrill`/`HaUpdatesDrill`: list idle (>24h, threshold 86400s ‚Äî D-CADENCE-IDLE-THRESHOLD reuses the automation Grafana panel's threshold) + disabled automations and idle scripts. New `useHomeAssistantCadence` hook in `apps/ui/src/api/home_assistant.ts`; new `types.ts` exports; widget + drill component + `.test.tsx`.

## Decisions inherited from STAGE-005-038 Design (locked)

- D-CADENCE-API-SOURCE ‚Äî VM per-series (consistent with locked D-DETAIL-CONSUMES-VM). LOCKED.
- D-CADENCE-IDLE-THRESHOLD ‚Äî 24h / 86400s, matching the automation Grafana panels. LOCKED.
- D-CADENCE-SCRIPT-DISABLED ‚Äî scripts have no enabled flag; script rows carry idle-age only, no enabled column. LOCKED.
