## Design Phase

**Status**: [x] Design (Complete)

### Design Notes

Backend contract verified against the REAL prod rig (VM) before design. Key data finding: `homelab_ha_automation_enabled` has 80 series (the authoritative automation set; 35 disabled), but `homelab_ha_automation_last_triggered_seconds` has only 43 — so **37 automations have NEVER triggered**. The automation join must be a LEFT JOIN on `enabled` (80) ← `last_triggered` (43); the 37 missing get `last_triggered_age_seconds = None` (matching the card's `float | None`). Scripts: 19 `last_triggered` series (16 idle >24h); no `script_enabled` metric exists (confirms D-CADENCE-SCRIPT-DISABLED). VM carries only `entity_id` (no friendly name) — enrich via live HA `get_states()` like the other detail endpoints.

**Chosen approach — Option A: one endpoint, two named collections, server-side filtered** (brainstormer presented A/B/C; coordinator selected A per the standing "go with your recommendation" instruction).

- **Endpoint:** `GET /api/integrations/home-assistant/cadence` (mirrors `/entities`, in `kernel/api/routers/integrations_home_assistant.py`; `require_session()` auth, no CSRF on GET).
- **Response model `HaCadenceResponse`:** `automations: list[HaCadenceAutomationRow]`, `scripts: list[HaCadenceScriptRow]`, plus `automations_total`, `automations_returned`, `scripts_total`, `scripts_returned`, `filtered_to`.
- **`HaCadenceAutomationRow`:** `entity_id: str`, `enabled: bool`, `last_triggered_age_seconds: float | None`, `friendly_name: str | None`.
- **`HaCadenceScriptRow`:** `entity_id: str`, `last_triggered_age_seconds: float | None`, `friendly_name: str | None` (NO `enabled` field — D-CADENCE-SCRIPT-DISABLED, scripts have no enable concept).
- **Filter (server-side, to the widget's exact need):** automation included iff `enabled == False` OR `last_triggered_age_seconds is None` (never triggered = idle) OR `age > 86400` (idle >24h, D-CADENCE-IDLE-THRESHOLD). Script included iff `age > 86400`.
- **Never-triggered automations (37) are treated as IDLE** (`age = None`); `None` sorts FIRST (most idle). LEFT JOIN: `enabled` is the authoritative universe.
- **Sort** each collection by age DESC (None first); reuse the existing top-N cap (=100; counts are tiny so it never bites, kept for contract parity). `*_total` = full filtered count pre-cap, `*_returned` = after cap. `filtered_to` echoes the predicate string (e.g. `"idle_24h_or_disabled"`).
- **Script universe = the 19 `last_triggered` series** (no `script_enabled` to LEFT JOIN from); never-run scripts are invisible — documented as an acceptable limitation in the handler docstring.

**Rejected:** Option B (two endpoints) — worse single-widget ergonomics (two fetches + stitch), duplicated handler logic. Option C (one flat `rows` list + `kind` discriminator + `enabled: bool | None`) — abandons the named-collection convention and structurally re-muddies D-CADENCE-SCRIPT-DISABLED (nullable enabled on scripts).

**Frontend (mirror pattern):** new `useHomeAssistantCadence` hook in `apps/ui/src/api/home_assistant.ts`; new type re-exports in `apps/ui/src/routes/integrations/types.ts` (`HaCadenceResponse`, `HaCadenceAutomationRow`, `HaCadenceScriptRow` from generated schema); drill component(s) mirroring `HaEntitiesDrill` + `DrillList`, slotted into `HomeAssistantStatusTab.tsx` (PanelSection card(s) in the 2-col grid); `.test.tsx` using `vi.mock('@/api/home_assistant')` + `makeResult` builder. NEW Pydantic models → `make openapi-export` → `packages/shared-types/openapi.json` → `pnpm run generate-types` → schema.ts → update sibling drill test mocks if needed (the schema-cascade gotcha).
