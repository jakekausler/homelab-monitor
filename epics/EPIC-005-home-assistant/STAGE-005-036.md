# STAGE-005-036: UI: render repair description + learn-more link in Repairs drill

## Status: Not Started

## Overview

`HaRepairsDrill.tsx` `renderRow` (lines 17-29) shows only `issue_id`, `domain`, and a severity Badge; the `description` (human-readable prose) and `learn_more_url` (documentation URL) that STAGE-005-031 added to `HaRepairRow` (and `/repairs` already returns) are fetched but never displayed. This stage renders the `description` text and links `learn_more_url`, degrading gracefully when null. Frontend-only — model + hook (`useHomeAssistantRepairs`) already carry the data. Note the 031 privacy posture: these are HA-derived display fields shown to the single authenticated owner — render them, but they must remain out of any client-side logging (consistent with 031's never-logged enrichment).

**Depends on:** none

## Stage Flags

- Has Input Forms: No.
- Touches secrets / production data: Possibly — `description` is HA-derived free text shown in-UI to the authenticated owner (same posture 031 locked: display-only, never logged). No metric/log emission; render-only.
- Host-integration concerns: No — frontend-only; Refinement 3a Desktop + Mobile only.

## Concrete deliverables

1. Extend `renderRow` in `HaRepairsDrill.tsx` to render `description` when present (a secondary line under the issue_id/domain row; safe text render — `description` is plain text, not HTML; do NOT `dangerouslySetInnerHTML`).
2. Link `learn_more_url` when present (external link, `rel="noreferrer"` + `target="_blank"`), defensively hidden when null.
3. Keep the existing issue_id/domain + severity Badge layout; description + link is a secondary line.
4. Update / add to `HaRepairsDrill.test.tsx`: a row with description + learn_more_url renders the text + link href; a row with both null renders the existing top line only (no crash, no empty link).

## Decisions in this stage

### Design phase will confirm

- D-REPAIRS-UI-DESCRIPTION-RENDER — safe-render approach for `description` (plain text node; confirm no HTML in stock HA repair descriptions; truncate/clamp long descriptions?).
- D-REPAIRS-UI-LAYOUT — placement of description + learn-more link; Mobile readability.
- D-REPAIRS-UI-PRIVACY — confirm the render does not introduce client logging of description (mirror 031 never-logged posture; lean: trivially satisfied — render only).
