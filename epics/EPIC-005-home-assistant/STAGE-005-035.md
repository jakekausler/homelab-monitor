# STAGE-005-035: UI: render update versions + release link in Updates drill

## Status: Not Started

## Overview

`HaUpdatesDrill.tsx` `renderRow` (lines 6-14) shows only the title/entity_id; the `installed_version`, `latest_version`, and `release_url` fields that STAGE-005-031 added to `HaUpdateRow` (and the `/updates` endpoint already returns) are fetched but never displayed. This stage renders the version transition (e.g. `1.2.3 → 1.3.0`) and links `release_url`, falling back gracefully when any field is null (HA-down / not-enriched). Frontend-only — the api hook (`useHomeAssistantUpdates`) and the model already carry the data; no backend or OpenAPI change. Directly parallels STAGE-005-033 (which rendered the already-present `friendly_name`).

**Depends on:** none

## Stage Flags

- Has Input Forms: No.
- Touches secrets / production data: No (renders already-returned, already-authenticated display fields).
- Host-integration concerns: No — frontend-only; Refinement 3a Desktop + Mobile only (no 3b host-integration delta).

## Concrete deliverables

1. Extend `renderRow` in `HaUpdatesDrill.tsx` to render the version transition when `installed_version` and/or `latest_version` are present: render `{installed_version} → {latest_version}` (use an arrow/`→`); if only one side present, render what exists; if both null, render nothing (no empty arrow).
2. Link `release_url` when present (e.g. a small "release notes" external link / icon), `rel="noreferrer"` + `target="_blank"`, defensively hidden when null.
3. Keep the existing title|entity_id layout; the version + link is a secondary line/segment (match the drill's existing visual density — confirm placement in Design with the Mobile prompt).
4. Update / add to `HaUpdatesDrill.test.tsx`: a row with all three fields renders the transition + a link with the right href; a row with all three null renders neither the arrow nor the link (no crash).

## Decisions in this stage

### Design phase will confirm

- D-UPDATES-UI-LAYOUT — where the version transition + release link sit (second line vs inline trailing) and how they read on Mobile.
- D-UPDATES-UI-PARTIAL — exact render when only one of installed/latest is present (lean: show the present one, no arrow).
- D-UPDATES-UI-LINK-AFFORDANCE — link text/icon for `release_url` (e.g. external-link icon vs "Release notes" text).
