# STAGE-005-038: Script-cadence Grafana panel + automation/script cadence API + Status-tab widget

## Status: Not Started

## Overview

`homelab_ha_script_last_triggered_seconds` has NO Grafana panel, NO alert, NO API, NO UI (the Grafana dashboard has automation idle/disabled/oldest panels at lines 682/741/800 but nothing for scripts). Automation last_triggered/enabled exist only on Grafana — there is no integration-panel API or UI for cadence. This stage surfaces cadence beyond Grafana. At MINIMUM it adds the missing script-cadence Grafana panel (parity with the automation panels). OPTIONALLY it adds a cadence detail endpoint (mirroring `/entities` / `/batteries`) and a Status-tab widget/drill listing idle/disabled automations + idle scripts. Because the alert noise from disabled automations is fixed by STAGE-005-034, this stage MUST be numbered after 034 so the idle data it surfaces is clean.

**RECOMMENDED SPLIT.** The Grafana-panel work is small (config-only); the cadence API + Status-tab widget is substantial (new endpoint, response models, OpenAPI regen, new widget + drill + tests). If Design confirms the API+UI scope, SPLIT this into two stages:
- **STAGE-005-038a** (this number) — Grafana script-cadence panel only (config; 3a+3b). Small, ships parity immediately.
- A FOLLOW-ON stage (next free number at split time) — cadence detail endpoint + Status-tab widget/drill (backend 3a+3b + frontend 3a Desktop+Mobile).

The split decision is made in this stage's Design (D-CADENCE-SURFACE-SPLIT). The card below covers the full scope; if split, the API+UI deliverables move to the follow-on card. (NOTE: splitting bumps N from 7 to 8 and the epic total from 40 to 41 — the later session updates the counter math accordingly and inserts the follow-on row.)
