# STAGE-005-033: Entities drill — render friendly_name

## Status: Not Started

## Overview

STAGE-005-031 adds `friendly_name: str | None` to the `HaEntityRow` backend model (live-HA enrichment),
but does NOT wire the 028 entities drill widget to display it (031 is backend-only). This stage updates
the frontend entities drill-list (`HaEntitiesDrill` / its row render) to show the human-readable
`friendly_name` alongside or instead of the raw `entity_id`, so the "Unavailable entities" list reads
"Master Bedroom Motion" rather than `binary_sensor.master_bedroom_motion`.

**Deferred from:** STAGE-005-031 Design (2026-06-14), D-ENRICH-ENTITY-FRIENDLY-NAME — the backend field
was added in 031; this stage is the frontend follow-up that renders it.
