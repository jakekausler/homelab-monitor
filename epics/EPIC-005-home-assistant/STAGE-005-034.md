# STAGE-005-034: Cadence — skip disabled automations from last_triggered

## Status: Not Started

## Overview

`ha_cadence.py` emits `homelab_ha_automation_last_triggered_seconds{entity_id}` for EVERY automation, including those in `state=="off"` (disabled — they can never trigger). A disabled automation's last-triggered age grows unbounded, inflating the Grafana "Idle Automations (>24h)" panel and (once GAP E/F land) the idle widget + idle alert with noise that is not actionable (the automation is intentionally off). This stage skips the `last_triggered_seconds` family for disabled automations while KEEPING the `homelab_ha_automation_enabled==0` series (disablement is still detectable — it is just decoupled from idle-age). The script path is verified: scripts have no on/off enable flag (a script's HA state is its run status, not an enabled toggle), so confirm in Design whether any script filter is meaningful or whether scripts are left unchanged.

**Depends on:** none

## Stage Flags

- Has Input Forms: No.
- Touches secrets / production data: No (reads HA states already polled; emits fewer series).
- Host-integration concerns: Yes — collector behavior validated against real HA automation/script entities; Refinement 3a + 3b.

## Concrete deliverables

1. In `ha_cadence.py` automation branch (currently lines 127-143): only append to `automation_triggered_obs` when `state.state == _STATE_ON`. The `automation_enabled_obs` append for both on (1.0) and off (0.0) is UNCHANGED. Net effect: `homelab_ha_automation_last_triggered_seconds` carries only enabled automations; `homelab_ha_automation_enabled` still carries both.
2. Script branch (145-154): confirm + document that scripts have no disabled state on `/api/states`, so the script path is left as-is (or apply the Design-confirmed filter if Design finds a meaningful one). Update the module docstring (lines 1-36) which currently says scripts are "same" — clarify the disabled-skip applies only to automations.
3. Update `M_AUTOMATION_LAST_TRIGGERED` family docstring (lines 6-7) to state disabled automations are excluded.
4. Tests: cover (a) enabled automation with `last_triggered` → triggered series emitted; (b) disabled (`state=="off"`) automation with `last_triggered` → NO triggered series but enabled=0 series IS emitted; (c) automation in `unavailable`/`unknown` state → neither series (existing behavior, regression-guard); (d) script with `last_triggered` → triggered series emitted (unchanged); (e) `metrics_emitted` count reflects the reduced survivor count.
5. (Optional, confirm in Design) update the Grafana "Idle Automations" panel comment/query if it relied on the old behavior — likely no query change needed since the metric simply has fewer series.

## Decisions in this stage

### Design phase will confirm

- D-CADENCE-DISABLED-SCRIPTS — do scripts have any analogous "disabled" notion to filter, or is the script path strictly unchanged? (Lean: unchanged; a script has no enable flag.)
- D-CADENCE-OFF-VS-UNKNOWN — confirm only `state=="off"` is excluded from triggered (and `unavailable`/`unknown`/`""` were already skipped for the enabled metric but NOT for triggered — verify current triggered behavior for those states and whether they should also be excluded, since a transiently-unavailable automation's stale last_triggered may also be noise).
- D-CADENCE-METRIC-COUNT — confirm the `metrics_emitted` accounting still holds (survivors + drop gauges) with fewer triggered observations.
