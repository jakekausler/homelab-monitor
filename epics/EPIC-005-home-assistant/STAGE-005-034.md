# STAGE-005-034: Cadence — skip disabled automations from last_triggered

## Status: Complete

## Overview

`ha_cadence.py` emits `homelab_ha_automation_last_triggered_seconds{entity_id}` for EVERY automation, including those in `state=="off"` (disabled — they can never trigger). A disabled automation's last-triggered age grows unbounded, inflating the Grafana "Idle Automations (>24h)" panel and (once GAP E/F land) the idle widget + idle alert with noise that is not actionable (the automation is intentionally off). This stage skips the `last_triggered_seconds` family for disabled automations while KEEPING the `homelab_ha_automation_enabled==0` series (disablement is still detectable — it is just decoupled from idle-age). The script path is verified: scripts have no on/off enable flag (a script's HA state is its run status, not an enabled toggle), so confirm in Design whether any script filter is meaningful or whether scripts are left unchanged.

**Depends on:** none

Net effect: `homelab_ha_automation_last_triggered_seconds` carries only enabled automations; `homelab_ha_automation_enabled` still carries both.

4. ## Decisions in this stage

### Design decisions (LOCKED 2026-06-14)

- **D-CADENCE-DISABLED-SCRIPTS [LOCKED]** — Scripts UNCHANGED. A `script.*` entity has no persistent enable/disable toggle; its state is transient run-status (`on` only while executing, `off` the rest of the time — the common steady state). `off` means "not currently running," not "disabled," so there is no script analog to a disabled automation. Filtering `off` scripts would drop essentially every script every tick and erase the metric. Leave the script branch (loop lines ~145-154) as-is; the module docstring already states scripts have no enabled metric.

- **D-CADENCE-OFF-VS-UNKNOWN [LOCKED]** — Emit `homelab_ha_automation_last_triggered_seconds` ONLY when `state.state == _STATE_ON` (the `== _STATE_ON` form), which excludes `off` (disabled) AND `unavailable`/`unknown`/`""` in one check — NOT the weaker `!= _STATE_OFF` form. Rationale: the downstream consumers (Grafana "Idle Automations (>24h)" panel `count(... > 86400)` and the planned STAGE-005-039 `HaAutomationIdle` alert) query the triggered metric DIRECTLY without joining `homelab_ha_automation_enabled` — so series MEMBERSHIP is the filter. The only correct invariant is "a triggered series exists iff the automation is live-and-enabled (could be idle when it shouldn't be)," which is exactly `state == "on"`. An `unavailable` automation can't trigger right now either, so its unbounded idle-age is the same noise `off` produces. The single-tick flapping from an HA-restart blip is immaterial at the 24h threshold (a one-scrape gap can't cross a 24h boundary; vmalert `for:` + Grafana `count()` are insensitive to it). The exact edit: at the triggered-append line (~143), change `elif seconds is not None:` to `elif seconds is not None and state.state == _STATE_ON:`. **The `parse_errors` increment (the `if err: parse_errors += 1` branch, ~lines 140-141) MUST stay BEFORE/OUTSIDE the state gate** — a present-but-unparseable `last_triggered` on a disabled/unavailable automation is still a genuine data-quality signal for the parse-error health gauge, not an idle signal.
  - Rejected `!= _STATE_OFF` (exclude only off): would leak `unavailable`/`unknown` automations into the idle count → false idle alerts during/after the HA restarts that make them unavailable.

- **D-CADENCE-METRIC-COUNT [LOCKED]** — No accounting change. `metrics_emitted = s1 + s2 + s3 + 3` (+1 on parse_errors) is a DERIVED survivor count from `emit_family`; fewer `automation_triggered_obs` simply yields a smaller `s1` and a correct total. The `+3` drop gauges (one per `emit_family` call) and the parse-error `+1` are unaffected. No hardcoded triggered-count expectation exists in `run()`. Follow-on: existing unit tests that asserted a disabled/unavailable automation emits a triggered series, or that asserted specific `metrics_emitted` values, must be updated in Build to match the new gating.

**Backend-only collector change:** No API/OpenAPI/UI change; emits FEWER series on the existing `homelab_ha_automation_last_triggered_seconds` family. Real-HA 3a + 3b (validate disabled automations drop out of the metric on the real rig — prior 009 Refinement measured 45 on / 35 off automations).
