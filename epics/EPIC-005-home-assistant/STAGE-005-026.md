# STAGE-005-026: Grafana dashboard home-assistant.json

## Status: Design (Not Started)

## Overview

The final stage: ship a default Grafana dashboard for Home Assistant, provisioned as code, that
visualizes the HA collector metrics (entity availability, batteries, updates, automation cadence,
anomaly z-scores, HA up). Completes the exemplar integration's "default dashboard + rules" deliverable
(rules are 005-014/015/016; this is the dashboard).

**Vertical slice:** after this stage, a "Home Assistant" dashboard appears in Grafana (provisioned at
startup) and embeds in the monitor's Metrics screen, showing HA health over time.

**Spec references:** master spec §9.2 (Metrics: project-owned dashboards as code under
`deploy/grafana/dashboards/`), §17 (Grafana dashboards per epic). EPIC-005 §"Stage decomposition"
Wave F. Pattern: `deploy/grafana/dashboards/host-overview.json` +
`deploy/grafana/provisioning/dashboards/default.yaml`.

## Concrete deliverables

### 1. `deploy/grafana/dashboards/home-assistant.json`

A Grafana dashboard JSON (matching `host-overview.json` structure: `description`, `panels`,
`datasource: {type: "prometheus", uid: "victoriametrics"}`, `editable: false`) with panels for:

- Entity availability over time (count available / unavailable / stale).
- Battery levels (low/critical counts; per-device table or low-battery list).
- Updates available (count over time).
- Automation last-triggered ages / idle automations.
- Anomaly z-scores (top |z| sensors).
- HA up (`homelab_ha_up`) status + collector run health for the HA collectors.

Panel PromQL/MetricsQL targets reference the metric names emitted in Waves B (and `homelab_collector_run_*`
for the HA collectors).

### 2. Provisioning

Confirm the dashboard is picked up by the existing provisioning (`deploy/grafana/provisioning/
dashboards/default.yaml` scans `/var/lib/grafana/dashboards`); ensure the deploy/compose mount includes
the new file. It should appear in the "Homelab" folder and embed in the Metrics screen.

### 3. Tests / validation

A validation that the JSON is well-formed + references real metric names (a lightweight check, since
Grafana JSON isn't unit-tested). 3b: confirm panels populate with real HA data on the prod rig.

## Out of scope for this stage

- vmalert rules (005-014/015/016).
- The integration UI panel (021-025) — distinct from the Grafana dashboard (this embeds in the Metrics
  screen, the panel lives under Integrations).
- Per-entity bespoke dashboards (the default is an overview).
