## Concrete deliverables

### 1. `deploy/grafana/dashboards/home-assistant.json`

A Grafana dashboard JSON (matching `host-overview.json` structure: `description`, `panels`,
`datasource: {type: "prometheus", uid: "victoriametrics"}`, `editable: false`) with panels for:

- Entity availability over time (count available / unavailable / stale).
- Battery levels (low/critical counts; per-device table or low-battery list).
- Updates available (count over time).
- Automation last-triggered ages / idle automations.
- Anomaly z-scores (top |z| sensors).
- HA up (`homelab_ha_up`) status + collector run health for the HA collectors.

Panel PromQL/MetricsQL targets reference the metric names emitted in Waves B (and `homelab_collector_run_*`
for the HA collectors).

### 2. Provisioning

Confirm the dashboard is picked up by the existing provisioning (`deploy/grafana/provisioning/
dashboards/default.yaml` scans `/var/lib/grafana/dashboards`); ensure the deploy/compose mount includes
the new file. It should appear in the "Homelab" folder and embed in the Metrics screen.

### 3. Tests / validation

A validation that the JSON is well-formed + references real metric names (a lightweight check, since
Grafana JSON isn't unit-tested). 3b: confirm panels populate with real HA data on the prod rig.

### Design Decisions (resolved 2026-06-13)

**Metric-name verification (the load-bearing check).** All metric names were verified against the REAL collector source (`apps/monitor/homelab_monitor/plugins/collectors/integrations/homeassistant/ha_*.py`). All 12 card-listed metrics EXIST exactly. The collectors also emit 5 metrics the card's list omitted but the card's OVERVIEW implies (z-scores, cadence): `homelab_ha_entity_last_changed_seconds{entity_id,domain}` (staleness), `homelab_ha_binary_sensor_on{entity_id,domain,device_class}` (safety), `homelab_ha_entity_value_zscore{entity_id}` (anomaly), and two `*_parse_errors` diagnostics. The full verified registry (all Gauge): `homelab_ha_up` (no labels); `homelab_ha_entity_available{entity_id,domain}`; `homelab_ha_entity_last_changed_seconds{entity_id,domain}`; `homelab_ha_battery_level{entity_id,domain}`; `homelab_ha_update_available{entity_id,title}`; `homelab_ha_config_entry_loaded{domain,title}`; `homelab_ha_config_entry_setup_error{domain,title}`; `homelab_ha_repair_issue{domain,issue_id,severity}`; `homelab_ha_persistent_notification{notification_id}`; `homelab_ha_sensor_value{entity_id,device_class}`; `homelab_ha_automation_last_triggered_seconds{entity_id}`; `homelab_ha_script_last_triggered_seconds{entity_id}`; `homelab_ha_automation_enabled{entity_id}`; `homelab_ha_binary_sensor_on{...}`; `homelab_ha_entity_value_zscore{entity_id}`.

**Collector-run self-metrics (verified against `kernel/scheduler/scheduler.py`).** Label key is `name`; HA collectors register names `ha_up`, `ha_entity_available`, `ha_battery`, `ha_update`, `ha_cadence`, `ha_config_entry`, `ha_repairs`, `ha_persistent_notification`, `ha_anomaly_zscore`, `ha_safety_sensors`, `ha_sensor_value` тАФ so `name=~"ha_.*"` matches all 11. Real self-metrics: `homelab_collector_run_success_total` (counter), `homelab_collector_run_failure_total{name,reason}` (counter), `homelab_collector_run_duration_seconds` (summary тЖТ _sum/_count), `homelab_collector_run_last_error_age_seconds` (gauge), plus `_skipped_total` / `_shutdown_total`.

**D-DASHBOARD-MOUNT тАФ RESOLVED.** No provisioning / compose / datasource change needed. The provider (`deploy/grafana/provisioning/dashboards/default.yaml`) scans `/var/lib/grafana/dashboards` every 30s (folder "Homelab", `allowUiUpdates:false`, `disableDeletion:true`); the compose mount `../grafana/dashboards:/var/lib/grafana/dashboards:ro` already exists; the `victoriametrics` datasource (uid `victoriametrics`, type `prometheus`) is already provisioned. Just drop the JSON.

**D-DASHBOARD-PANELS тАФ RESOLVED (Option B, sectioned operational dashboard).** Mirror `deploy/grafana/dashboards/host-overview.json` panel shapes (timeseries id:1, gauge id:3, bargauge id:4, stat id:6). Structure: `uid:"home-assistant"`, `editable:false`, `schemaVersion:40`, `tags:["homelab-monitor","home-assistant"]`, `time:{from:"now-1h",to:"now"}`, `refresh:"30s"`. Every panel + every target carries `datasource:{"type":"prometheus","uid":"victoriametrics"}`. Five `row`-grouped sections:
- **Availability & Status:** HA Up (stat, `homelab_ha_up`, instant, red/green); Entities Unavailable (stat, `sum(homelab_ha_entity_available == 0)`); Availability over time (timeseries: `sum(homelab_ha_entity_available == 1)` available + `sum(homelab_ha_entity_available == 0)` unavailable); Stale Entities (stat, `count(homelab_ha_entity_last_changed_seconds > 7200)` тАФ mirrors HaEntityStale 7200s).
- **Batteries:** Low/Critical batteries (bargauge horizontal, `homelab_ha_battery_level < 20`, instant, INVERTED thresholds red@base / yellow@10 / green@20 тАФ mirrors vmalert critical<10 low 10-20); Critical count (stat, `count(homelab_ha_battery_level < 10)`).
- **Updates & Config Health:** Updates pending (stat, `count(homelab_ha_update_available == 1)`); Config errors (stat, `sum(homelab_ha_config_entry_setup_error == 1)`); Active repairs (stat, `sum(homelab_ha_repair_issue)`); Persistent notifications (stat, `sum(homelab_ha_persistent_notification)`).
- **Automation Cadence:** Idle automations (stat, `count(homelab_ha_automation_last_triggered_seconds > 86400)` тАФ mirrors HaAutomationIdle 86400s); Disabled automations (stat, `count(homelab_ha_automation_enabled == 0)`); Oldest-triggered (timeseries `topk(10, homelab_ha_automation_last_triggered_seconds)`).
- **Anomalies & Collector Health:** Top anomaly z-scores (timeseries, `topk(8, abs(homelab_ha_entity_value_zscore))`, legend `{{entity_id}}`); Collector run health (timeseries: `homelab_collector_run_last_error_age_seconds{name=~"ha_.*"}` + `rate(homelab_collector_run_failure_total{name=~"ha_.*"}[15m])`).

**Cardinality guardrail (1906 entities).** Every per-entity concern uses `count`/`sum`/`topk` тАФ NEVER a raw per-series timeseries. Battery bargauge self-bounds via `<20`; z-score via `topk(8,...)`. Never plot `homelab_ha_entity_available` un-aggregated.

**Known limitation (flagged, acceptable for a dashboard):** `count(homelab_ha_automation_last_triggered_seconds > 86400)` silently undercounts automations that have NEVER triggered (they may have no series). Acceptable; the cadence collector governs presence.

**Rejected тАФ Option A (compact single-screen at-a-glance).** Cheapest queries + densest, but DROPS automation cadence (idle automations) and last_changed staleness as explicit panels. Rejected: as the FINAL HA stage (D-DASHBOARD-LAST) the dashboard should be a COMPLETE health overview, not a subset.

**Rejected тАФ Option C (summary stats + detail tables).** `table` panels give per-row drill-down but host-overview uses NO table panel тЖТ table fieldConfig/transformations shape would be invented (contradicts "mirror host-overview shapes"); and tables de-emphasize availability-over-time trend. Rejected: weaker fit + invention risk.
