# STAGE-005-030: Metrics page tabs (System + Home Assistant Grafana embeds)

## Status: Complete

## Overview

The Metrics screen (`apps/ui/src/routes/Metrics.tsx`) currently embeds a single hardcoded Grafana dashboard (`host-overview`) in a kiosk iframe. STAGE-005-026 added a `home-assistant` Grafana dashboard that is provisioned but NOT reachable from the app's React UI (operators must switch dashboards inside the embedded Grafana). This stage wires that dashboard into the app: convert the Metrics screen to a ROUTE-BASED tabbed layout with a **System** tab (the existing `host-overview` embed) and a **Home Assistant** tab (the `home-assistant` embed), mirroring the route-based tab convention established by the Home Assistant integration panel (STAGE-005-022 `HomeAssistantLayout`).

Vertical slice: after this stage, `/metrics` redirects to `/metrics/system`; the Metrics screen shows a tab bar (System | Home Assistant); each tab renders its respective Grafana dashboard in a kiosk iframe; the HA dashboard from 026 is now discoverable directly from the app UI.

## Stage Flags

- Has Input Forms: [ ] No
- Touches secrets / production data: [ ] No

## Cross-cutting standards

- Strict TS; reuse the existing Grafana reverse-proxy (`/api/grafana/...`) and the existing kiosk-iframe pattern — do NOT add a new proxy or change the backend.
- Mirror the route-based tab convention from `HomeAssistantLayout` (layout route + index redirect + child routes + `<Outlet/>` + `<Link activeProps>` tab bar). Do NOT introduce a state-based (useState) tab toggle.
- Lazy iframes: only the active tab's iframe is mounted (each tab is its own route component, so only the active route renders — this is automatic with route-based tabs).
- Preserve the existing full-bleed full-height iframe layout and the "Open in Grafana" link per tab.
- The sidebar "Metrics" nav entry stays pointing at `/metrics` (the index redirect lands the user on the default tab).

## Concrete deliverables

### 1. MetricsLayout component + route-based tabs

A `MetricsLayout` component (mirroring `HomeAssistantLayout.tsx`): header + a tab bar (`<Link activeProps>` for System / Home Assistant) + `<Outlet/>`. Register a `metricsLayoutRoute` at `/metrics` (replacing the current single `metricsRoute`), an index route that redirects `/metrics` → `/metrics/system` (mirroring `homeAssistantIndexRoute`'s `beforeLoad`/`throw redirect`, preserving search params), and two child routes: `/metrics/system` and `/metrics/home-assistant`.

### 2. Two tab components (System + Home Assistant), each a kiosk Grafana embed

`MetricsSystemTab` renders the EXISTING `host-overview` embed: iframe `src="/api/grafana/d/host-overview/host-overview?kiosk"`, same sandbox/className/title/full-bleed layout + "Open in Grafana" link as today's `Metrics.tsx`. `MetricsHomeAssistantTab` renders the same shape but iframe `src="/api/grafana/d/home-assistant/home-assistant?kiosk"` (CONFIRM the HA dashboard's `uid` from `deploy/grafana/dashboards/home-assistant.json` during Build — the URL path uses the uid). Factor the shared iframe-embed markup into a small reusable piece if it reduces duplication (decide in Design); do NOT over-abstract.

### 3. Tests

Update/replace `apps/ui/src/routes/Metrics.test.tsx`: the old single-iframe test moves to per-tab tests (System tab asserts `host-overview` src; Home Assistant tab asserts `home-assistant` src); assert the tab bar renders both tab links; assert the `/metrics` index redirects to `/metrics/system`. Mirror how the HA tabs got their own co-located tests. Desktop + Mobile.

## Out of scope for this stage

- No new Grafana dashboard (026 already shipped the HA one; host-overview predates this).
- No backend / Grafana-proxy change (the `/api/grafana/` proxy already exists).
- No new dashboards for other integrations (future work if/when those epics add dashboards).
- No "Quick PromQL" panel (master spec §9.2 mentions it; out of scope here).

### Design Decisions (resolved 2026-06-13)

**Backend contract — VERIFIED.** The Grafana reverse-proxy (`apps/monitor/homelab_monitor/kernel/api/routers/grafana.py`, route `/grafana/{path:path}`) forwards ALL paths including dashboard `/d/...` paths and strips X-Frame-Options so iframes render. Both dashboards exist and provision (confirmed on the live rig in STAGE-005-026 Refinement). No backend change needed.

**D-METRICS-HA-UID — RESOLVED.** The HA dashboard's `uid` is `home-assistant` (title "Home Assistant"). HA tab iframe src = `/api/grafana/d/home-assistant/home-assistant?kiosk` (exact mirror of the System tab's `/api/grafana/d/host-overview/host-overview?kiosk`). Verified on the running prod rig: the proxy-prefixed kiosk URL returns 200 OK, identical behavior to the System dashboard. MUST use the full `/api/grafana/d/...` form (a bare `/d/...` 301-redirects incorrectly).

**D-METRICS-TABS-PATTERN — RESOLVED (route-based, mirror HomeAssistantLayout).** Replace the single `metricsRoute` (`/metrics` → `MetricsPage`) with: a `metricsLayoutRoute` at `/metrics` (component `MetricsLayout`), a `metricsIndexRoute` that `beforeLoad`/`throw redirect`s `/metrics` → `/metrics/system` (preserving search params, mirroring `homeAssistantIndexRoute`), and two child routes `/metrics/system` (`MetricsSystemTab`) + `/metrics/home-assistant` (`MetricsHomeAssistantTab`). `MetricsLayout` mirrors `HomeAssistantLayout.tsx`: header + a `<Link activeProps>` tab bar (TABS = [{path:'/metrics/system', label:'System'}, {path:'/metrics/home-assistant', label:'Home Assistant'}]) + `<Outlet/>`. The sidebar "Metrics" nav entry (`SidebarNav.tsx`, `{label:'Metrics', to:'/metrics'}`) stays unchanged — the index redirect lands the user on System.

**D-METRICS-LAZY — confirmed inherent.** Route-based tabs mount only the active route's component, so only the active tab's iframe exists — lazy by construction, no extra code.

**D-METRICS-EMBED-SHARE — RESOLVED (one shared embed component).** Factor the shared kiosk-iframe markup (the `<iframe sandbox className title src>` + full-bleed `-m-6 h-[calc(100vh-3.5rem)]` wrapper + the absolute-positioned "Open in Grafana" link) into ONE reusable component (e.g. `GrafanaDashboardEmbed` taking `src` + `title` props). The two tab components become thin wrappers passing their respective src. DRY without over-abstraction (one component, two one-line callers). The existing `MetricsPage` is REPLACED by this structure (it becomes redundant — its System embed moves into `MetricsSystemTab` via the shared component).

**Redundancy note:** the current `apps/ui/src/routes/Metrics.tsx` `MetricsPage` component is SUPERSEDED by this stage — its single host-overview embed becomes the System tab. Delete or repurpose it (decide in Build); do NOT leave a dead duplicate.
