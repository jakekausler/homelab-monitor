## Concrete deliverables

1. **Backup endpoint** at `apps/monitor/homelab_monitor/kernel/api/admin.py`
   - `POST /api/admin/backup` — auth-required (cookie session OR API token with scope `admin:backup:write`).
   - Steps:
     1. `sqlite3 .backup` snapshot of operational DB → `/storage/backup/homelab-monitor/sqlite-YYYYMMDD-HHMMSS.sqlite`.
     2. POST to VM `/snapshot/create` → snapshot id returned → copy snapshot files from VM volume to `/storage/backup/homelab-monitor/vm/<id>/`.
     3. Skip VL for now (added in STAGE-001-016 follow-up). Skip runbooks (none yet).
     4. Returns `{snapshot_id, sqlite_path, vm_snapshot_path, started_at, ended_at, size_bytes}`.
   - Idempotent retention: keeps last N=7 daily snapshots in each directory; older are deleted.
   - Audit log row written for every invocation.
2. **CLI** — `hm backup run` invokes the same endpoint locally; `hm backup list` reads the backup dirs and prints sizes/dates; `hm backup retention --keep N`.
3. **Disk-budget self-monitor** at `apps/monitor/homelab_monitor/plugins/collectors/builtin/self_disk.py`
   - In-process `BaseCollector` with `interval=60s`.
   - Reads `data/{vm,vl,sqlite,runbook-transcripts}` total bytes via `pathlib.Path.stat`.
   - Reads `HOMELAB_MONITOR_DISK_BUDGET_GB` from env (default 50 GB).
   - Reads policy ratios from `homelab-monitor.yaml` (default 60/30/10 per spec §6.4).
   - Emits:
     - `homelab_self_disk_used_bytes{slot="vm"|"vl"|"sqlite"|"runbook_transcripts"}` (gauge)
     - `homelab_self_disk_budget_bytes{slot}` (gauge — derived from policy ratios)
     - `homelab_self_disk_used_pct` (gauge — overall)
4. **Disk-budget alert rules** at `deploy/vmalert/metrics/self_disk.yaml`
   - `SelfDiskWarn` — `homelab_self_disk_used_pct > 70` for 5m → severity=warning.
   - `SelfDiskError` — `homelab_self_disk_used_pct > 85` for 5m → severity=error.
   - `SelfDiskCritical` — `homelab_self_disk_used_pct > 95` → severity=critical, fires immediately.
5. **Auto-shrink mechanism** at `apps/monitor/homelab_monitor/kernel/disk_budget/shrinker.py`
   - When `homelab_self_disk_used_pct > 95`, the shrinker:
     1. Calls VM's retention adjustment (dynamically reduces retention period by stepping down configured tiers; default tiers: 90d → 60d → 30d → 14d).
     2. Tightens VL byte caps (when STAGE-001-016 lands; for this stage, log a TODO with the stage reference).
     3. NEVER touches SQLite or runbook transcripts.
   - Shrink action is logged as a critical-severity audit row and emits `homelab_self_disk_shrink_total{tier}`.
6. **Test rig minimal extension** — `deploy/compose/docker-compose.test.yml`
   - Adds VM (already needed by STAGE-001-015's smoke test). This stage promotes the smoke test rig to a `tests/integration/` rig used by 016–021.
   - Adds an `integration-tests` service that runs `pytest apps/monitor/tests/integration/` and exits with the suite's code.
   - Adds the `--abort-on-container-exit --exit-code-from integration-tests` runner script `scripts/run-integration.sh`.
   - CI's existing `integration` job now runs this script (it was a placeholder previously).
7. **Tests**
   - `POST /api/admin/backup` produces both files on disk; SQLite snapshot is restorable; VM snapshot directory is non-empty.
   - Retention: when there are 8+ snapshots, oldest is removed.
   - Disk-budget collector: planted directory sizes produce expected metric values.
   - Auto-shrink: synthetic 96% trips the shrinker; assert retention call fired and audit row written.
   - Integration: full backup + restore round-trip in the test rig.

## Out of scope for this stage

- Runbook transcript backups (no runbooks exist yet; EPIC-009 owns)
- Off-site replication (the user's Synology already replicates `/storage/backup/` to Backblaze)

## Decisions in this stage

- **Auto-shrink steps down tiers, not free-form.** Predictability over cleverness; documented tiers in the README.
- **Backup retention default = 7 daily.** Configurable via `homelab-monitor.yaml`.
- **Backup never includes the master key.** `/run/secrets/master-key` is not in any backup; the user holds it in a password manager.

### Design Decisions (selected 2026-05-08)

1. **Auth scope**: Option A — Specific scope only.
   - Add `Scope.ADMIN_BACKUP_WRITE = "admin:backup:write"` to `apps/monitor/homelab_monitor/kernel/auth/scopes.py`.
   - `/api/admin/backup` accepts cookie session OR API token with this scope (via `Depends(require_user_or_token({Scope.ADMIN_BACKUP_WRITE}))`).
   - Future admin endpoints (restore, key rotate, etc.) will compose as sibling scopes.
   - Rationale: granular least-privilege; keeps audit logs intent-rich; clean future composition without auth machinery refactor.

2. **VM snapshot file copy**: Option A — Shared RO volume + `cp -al`.
   - Bind-mount VM's data directory (containing `/storage/snapshots/`) into the monitor container at `/var/vm-data/snapshots/` (read-only).
   - Monitor calls `POST http://victoriametrics:8428/snapshot/create` to get the snapshot ID.
   - Then `cp -al /var/vm-data/snapshots/<id>/. /storage/backup/homelab-monitor/vm/<id>/` (hardlink for COW efficiency, near-instant).
   - At startup, compare `os.stat().st_dev` of VM data dir vs backup dir; if different filesystems, log warning + fall back to `cp -r` (still functional, costs disk).
   - Rationale: single-host single-user homelab; strict isolation argument is weak; the compose layout already shares /storage/.

3. **Auto-shrink mechanism**: Option C — Emit metric + audit + alert; no actual action in v1.
   - When `homelab_self_disk_used_pct > 95` is detected by the collector, emit `homelab_self_disk_shrink_total{tier}` counter (incremented once per shrink decision), write a critical-severity audit log row, and rely on the SelfDiskCritical vmalert rule for operator notification.
   - DO NOT call VM's `delete_series` API (creates tombstones, can spike disk during merge).
   - DO NOT restart the VM container (requires docker-socket access; cascades during incidents).
   - Document a TODO referencing a future stage (likely EPIC-008 disk operations) for real shrink action.
   - Rationale: avoid cascading damage during disk-pressure incidents; matches the spec's accepted "log a TODO" pattern for VL byte caps; preserves the stage's "shrink decision" deliverable while deferring the action layer.

4. **Test rig integration-tests service**: Option D — Hybrid (Dockerfile.test for CI + host pytest for dev).
   - New `apps/monitor/Dockerfile.test` (multi-stage; uv sync --extra dev) — production image stays lean.
   - `deploy/compose/docker-compose.test.yml` declares an `integration-tests` service using this test image. CMD: `pytest tests/integration/ -v`.
   - `scripts/run-integration.sh` runs `docker compose -f deploy/compose/docker-compose.test.yml up --abort-on-container-exit --exit-code-from integration-tests`.
   - For local dev, the same compose file's VM service can be brought up alone (`docker compose -f deploy/compose/docker-compose.test.yml up -d victoriametrics`) and pytest run on the host against `localhost:8428` for fast iteration.
   - Both workflows documented in scripts/run-integration.sh header.
   - Rationale: CI determinism + dev iteration speed from the same compose file.

### Cross-cutting Decisions

- **Collector interval**: 60s (per stage file; spec doc said 300s but disk-pressure response time matters more than minimal collector overhead — a stat() of 4 directories is sub-millisecond).
- **Config loader**: Minimal `yaml.safe_load` helper at `apps/monitor/homelab_monitor/kernel/config.py` with explicit dataclass mapping (~30 lines). NOT a full framework. Loads from `HOMELAB_MONITOR_CONFIG` env (default `/config/homelab-monitor.yaml`).
- **Disk-budget config defaults if YAML missing**: 60% VM / 30% VL / 10% SQLite (per spec §6.4). Total budget defaults to 50 GB via `HOMELAB_MONITOR_DISK_BUDGET_GB` env (per spec).
