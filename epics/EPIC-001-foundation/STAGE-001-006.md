# STAGE-001-006: Collector protocol + base classes

## Status: Refinement

## Overview

Define the abstract contracts every collector must implement, the `CollectorContext` they receive at runtime, and the result types they emit. No scheduler, no loader yet — this stage produces the *types and base classes* with full unit-test coverage and a single working noop collector that proves the protocol can be implemented.

This is the foundational interface for the entire plugin layer. It must be stable; future plugin kinds (discoverer, enricher, channel, runbook, digest_section) follow the same shape.

## Concrete deliverables

1. **Type definitions** at `apps/monitor/homelab_monitor/kernel/plugins/types.py`
   - `class RunKind(StrEnum): ASYNC; THREAD; PROCESS`
   - `class TrustLevel(StrEnum): BUILTIN; TRUSTED; UNTRUSTED`
   - `@dataclass class CollectorConfig:` — typed pydantic-validated config; arbitrary nested fields per plugin (subclasses define their own).
   - `@dataclass class CollectorResult:` fields: `ok: bool`, `metrics_emitted: int`, `errors: list[str]`, `events: list[CollectorEvent]`, `duration_seconds: float`.
   - `class CollectorEvent(BaseModel):` — discriminated union `kind: Literal["suggestion", "alert_forward", "log_signature", "heartbeat"]` + payload.
2. **`CollectorContext`** at `apps/monitor/homelab_monitor/kernel/plugins/context.py`
   - Dataclass holding: `config: CollectorConfig`, `db: SqliteRepository`, `vm: MetricsWriter`, `vl: LogsWriter`, `http: AsyncClient`, `ssh: SshClientFactory`, `ha: HomeAssistantClient | None`, `secrets: SecretsResolver`, `log: structlog.BoundLogger`.
   - In this stage, `MetricsWriter`, `LogsWriter`, `SshClientFactory`, and `HomeAssistantClient` are **stub Protocol classes** with no implementation. STAGE-001-012 introduces a real stub MetricsWriter; STAGE-001-015 swaps in the real VM-backed one.
3. **`Collector` Protocol** at `apps/monitor/homelab_monitor/kernel/plugins/base.py`
   - ```python
     class Collector(Protocol):
         name: ClassVar[str]
         interval: ClassVar[timedelta]
         timeout: ClassVar[timedelta]
         concurrency_group: ClassVar[str]
         run_kind: ClassVar[RunKind]
         trust_level: ClassVar[TrustLevel]
         async def run(self, ctx: CollectorContext) -> CollectorResult: ...
     ```
   - Provide an abstract base class `BaseCollector` that supplies sensible defaults (`run_kind = ASYNC`, `trust_level = BUILTIN`, `concurrency_group = name`).
4. **MetricsWriter / LogsWriter Protocol stubs** at `apps/monitor/homelab_monitor/kernel/plugins/io.py`
   - `MetricsWriter`: `write_gauge(name, value, labels)`, `write_counter(...)`, `write_summary(...)`. Stub implementation `InMemoryMetricsWriter` for tests records every call into a list.
   - `LogsWriter`: `ingest(stream, line, ts=None)`. Stub `InMemoryLogsWriter` similarly.
5. **Noop collector** at `apps/monitor/homelab_monitor/plugins/collectors/builtin/noop.py`
   - Implements the protocol, does nothing successful with `metrics_emitted=0`. Used in scheduler tests later.
6. **Tests**
   - Type assertions: `isinstance` and structural typing checks (assignability via `Protocol`).
   - Noop collector test: instantiate, build a `CollectorContext` with in-memory writers, call `run`, assert `ok=True`.
   - Pydantic config validation: a malformed config dict raises `ValidationError`.
   - `CollectorEvent` discriminated union behaves correctly across all four kinds.

## Decisions in this stage

- **Use `Protocol` (PEP 544) for the public interface; provide `BaseCollector` ABC for ergonomic subclassing.** Plugins can implement either way.
- **`CollectorContext` is a dataclass with `slots=True`** — small allocation, immutable after construction, easy to pickle for `PROCESS` run-kind.
- **Plugin name regex** — `[a-z][a-z0-9_-]{2,63}`. Validated at registration.

### Locked decisions (Design — 2026-05-05)

**Collector interface:** Option (a) — `Collector` Protocol (PEP 544) + `BaseCollector` ABC. Protocol provides structural typing for cross-subprocess compatibility (STAGE-001-009); ABC provides ergonomic defaults (`concurrency_group = name`, `trust_level = BUILTIN`, no-op health check). Plugin authors typically subclass `BaseCollector`; subprocess plugins satisfy the Protocol via JSON-RPC contract.

**`trust_level` location:** On the `Collector` Protocol as `ClassVar[TrustLevel]`. `BaseCollector` defaults to `TrustLevel.BUILTIN`. STAGE-001-009 dispatches at runtime via `collector.trust_level`; manifest is the deployment-time persistence form.

**SSH/HA stubs:** `SshClientFactory` defined as a minimal `Protocol` in `kernel/plugins/io.py` with `async def open(target_id: str) -> AsyncContextManager[SshConnection]` (forward-declare `SshConnection` as another Protocol stub). Many future collectors will use it; pinning the Protocol now prevents drift. `HomeAssistantClient | None` is typed as `Optional`; scheduler injects `None` for non-HA collectors. HA stage will define the concrete class later.

**`MetricsWriter` API:** `write_gauge(name, value, labels)`, `write_counter(name, value, labels)`, `write_summary(name, value, labels)` — all single-observation, symmetric. `LogsWriter` API: `ingest(stream, line, ts=None)`. Buffering/batching is the impl's concern (real `VictoriaMetricsWriter` in STAGE-001-015), not the contract's.

**`CollectorEvent`:** Pydantic 2 discriminated union with 4 kinds (`suggestion`, `alert_forward`, `log_signature`, `heartbeat`). `Field(discriminator="kind")` gives free runtime validation when events cross the subprocess boundary in STAGE-001-009. Consistent with the existing pydantic-based `CollectorConfig`.

**`CollectorContext`:** `@dataclass(slots=True)` for immutability + pickle-safe for PROCESS run-kind. 9 fields per spec §5.2: `config`, `db`, `vm`, `vl`, `http`, `ssh`, `ha`, `secrets`, `log`.

**`CollectorResult`:** Pydantic model with fields: `ok: bool`, `metrics_emitted: int`, `errors: list[str]`, `events: list[CollectorEvent]`, `duration_seconds: float`.

**`RunKind`:** `StrEnum` with `ASYNC`, `THREAD`, `PROCESS`. **`TrustLevel`:** `StrEnum` with `BUILTIN`, `TRUSTED`, `UNTRUSTED`.

**Plugin name regex:** `[a-z][a-z0-9_-]{2,63}` (validated via pydantic field constraint on `CollectorConfig.name`).

**Module layout to create:**
```
apps/monitor/homelab_monitor/kernel/plugins/
├── __init__.py        # re-exports public symbols
├── types.py           # RunKind, TrustLevel, CollectorConfig, CollectorResult, CollectorEvent + 4 payload types
├── context.py         # CollectorContext dataclass
├── base.py            # Collector Protocol + BaseCollector ABC
├── io.py              # MetricsWriter Protocol, LogsWriter Protocol, SshClientFactory Protocol, SshConnection Protocol, InMemoryMetricsWriter, InMemoryLogsWriter

```

No new runtime deps required (pydantic, structlog, httpx all already present).
