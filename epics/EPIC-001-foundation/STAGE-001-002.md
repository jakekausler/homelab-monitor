# STAGE-001-002: Frontend skeleton

## Status: Not Started

## Overview

Establish the frontend at `apps/ui/` with React 18 + Vite + TypeScript strict, plus the testing and tooling infrastructure that mirrors the backend's strictness. Produces no application UI — produces the *project skeleton* that every subsequent UI stage builds on. After this stage, a fresh clone can run `pnpm verify` (or `npm run verify`) and get a green pipeline.

**Spec references:** §9 (UI structure), §9.4 (frontend stack details), §11 (repo layout), §12 (verify command).

## Concrete deliverables

1. **Workspace bootstrap**
   - Decide and configure workspace tool: `pnpm-workspace.yaml` if pnpm, `workspaces` field in root `package.json` if npm. **Recommend pnpm** for speed and disk efficiency. The Design phase confirms.
   - Root `package.json` declares the workspace and the `verify`, `lint`, `typecheck`, `test`, `build` scripts that fan out across workspaces.
2. **`apps/ui/` package**
   - `apps/ui/package.json` with deps: `react`, `react-dom`, `@tanstack/react-router`, `@tanstack/react-query`, `react-hook-form`, `zod`, `tailwindcss`, `@radix-ui/react-*` (the primitives we'll use first), `@tremor/react`, `recharts` (peer of Tremor), and dev: `vite`, `@vitejs/plugin-react`, `typescript`, `vitest`, `@testing-library/react`, `@testing-library/jest-dom`, `eslint`, `eslint-plugin-react`, `@typescript-eslint/*`, `prettier`, `playwright` (installed but no tests yet).
   - `vite.config.ts` with React plugin, port 5173, proxy `/api` → `http://localhost:9090` for dev.
   - `tsconfig.json` with strict mode, `noUncheckedIndexedAccess: true`, `exactOptionalPropertyTypes: true`, paths config.
   - `eslint.config.js` (flat config) extending recommended + react + typescript-eslint strict.
   - `tailwind.config.ts` with the Tremor preset and a basic theme.
   - `index.html` minimal shell.
   - `src/main.tsx` mounts a `<App />` that renders "homelab-monitor" centered with the chosen typography. No router yet.
   - `src/App.tsx` with the placeholder.
   - `src/test/setup.ts` for Vitest with `@testing-library/jest-dom`.
   - One trivial test that mounts `<App />` and asserts the heading exists.
3. **Tooling configs**
   - `prettier` config: 2-space indent, single quotes, trailing commas, no semicolons (or with — Design phase decides; spec doesn't dictate).
   - `playwright.config.ts` scaffold with `webServer: { command: ... }` that proxies a not-yet-real backend; tests directory exists but contains only a `.gitkeep`.
4. **Verify pipeline (this stage)**
   - `pnpm --filter ui run verify` runs: eslint, tsc --noEmit, vitest run with coverage, vite build.
   - Root `pnpm run verify` runs both backend (Python) and frontend pipelines so the canonical `make verify` from STAGE-001-001 ties them together (Makefile is updated to call into pnpm).

## Decisions in this stage

- pnpm vs npm vs yarn: **recommend pnpm**.
- Prettier semi yes/no: pick one and document.
- Whether to commit a single root lockfile or per-workspace: pnpm uses one root `pnpm-lock.yaml` (commit it).
