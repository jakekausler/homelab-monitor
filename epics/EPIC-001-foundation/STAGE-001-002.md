## Concrete deliverables

1. **Workspace bootstrap**
   - Decide and configure workspace tool: `pnpm-workspace.yaml` if pnpm, `workspaces` field in root `package.json` if npm. **Recommend pnpm** for speed and disk efficiency. The Design phase confirms.
   - Root `package.json` declares the workspace and the `verify`, `lint`, `typecheck`, `test`, `build` scripts that fan out across workspaces.
2. **`apps/ui/` package**
   - `apps/ui/package.json` with deps: `react`, `react-dom`, `@tanstack/react-router`, `@tanstack/react-query`, `react-hook-form`, `zod`, `tailwindcss`, `@radix-ui/react-*` (the primitives we'll use first), `@tremor/react`, `recharts` (peer of Tremor), and dev: `vite`, `@vitejs/plugin-react`, `typescript`, `vitest`, `@testing-library/react`, `@testing-library/jest-dom`, `eslint`, `eslint-plugin-react`, `@typescript-eslint/*`, `prettier`, `playwright` (installed but no tests yet).
   - `vite.config.ts` with React plugin, port 5173, proxy `/api` → `http://localhost:9090` for dev.
   - `tsconfig.json` with strict mode, `noUncheckedIndexedAccess: true`, `exactOptionalPropertyTypes: true`, paths config.
   - `eslint.config.js` (flat config) extending recommended + react + typescript-eslint strict.
   - `tailwind.config.ts` with the Tremor preset and a basic theme.
   - `index.html` minimal shell.
   - `src/main.tsx` mounts a `<App />` that renders "homelab-monitor" centered with the chosen typography. No router yet.
   - `src/App.tsx` with the placeholder.
   - `src/test/setup.ts` for Vitest with `@testing-library/jest-dom`.
   - One trivial test that mounts `<App />` and asserts the heading exists.
3. **Tooling configs**
   - `prettier` config: 2-space indent, single quotes, trailing commas, no semicolons (or with — Design phase decides; spec doesn't dictate).
   - `playwright.config.ts` scaffold with `webServer: { command: ... }` that proxies a not-yet-real backend; tests directory exists but contains only a `.gitkeep`.
4. **Verify pipeline (this stage)**
   - `pnpm --filter ui run verify` runs: eslint, tsc --noEmit, vitest run with coverage, vite build.
   - Root `pnpm run verify` runs both backend (Python) and frontend pipelines so the canonical `make verify` from STAGE-001-001 ties them together (Makefile is updated to call into pnpm).

## Design Phase

- **Approaches to present:** (a) pnpm workspaces (recommended), (b) npm workspaces, (c) yarn workspaces.
- **UI Options Presented:** placeholder text styling for the only screen this stage produces; one option centered mono, one option modern dashboard color treatment as preview of the aesthetic direction.
- **User Choice:** A=pnpm workspaces, B=Prettier `semi: false`, C=modern dashboard preview (Tremor card on dark theme background), D=single root `pnpm-lock.yaml`. Brainstormer presented options 2026-05-04 and user approved all four recommendations same day.
- **Seed Data Agreed:** N/A (no DB or test fixtures in this stage)
- **Session Notes:** Decision C requires the placeholder screen to exercise the actual Tremor + Tailwind dark-theme stack rather than rendering a bare `<h1>`. This validates the wiring (Tremor preset in `tailwind.config.ts`, dark mode default per spec §9.3) at the cost of ~10 extra lines vs a centered heading. Smoke test asserts the rendered card's heading text. Vite proxy `/api → http://localhost:9090` is forward-compatible scaffolding; STAGE-001-010 lands the FastAPI app shell that actually serves it. Root Makefile gains a frontend leg in `verify` (delegating to `pnpm --filter ui run verify`) so `make verify` runs both Python and frontend pipelines. Pre-commit config (from STAGE-001-001) is left alone in this stage — the frontend's eslint/prettier/tsc are wrapped via `pnpm verify` and the eventual STAGE-001-003 CI job; we do NOT add per-file pre-commit hooks for TS in this stage.

- **Placeholders Added:** `<App />` renders a Tremor card with `homelab-monitor • status: scaffolding • EPIC-001 STAGE-001-002`. Playwright config exists; browsers + first e2e test land in STAGE-001-014.
- **Session Notes:** Build had 4 verifier failures, all fixed:
  (1) ESLint flat config applied `recommendedTypeChecked` globally — failed parsing config files outside `src/`. Fixed by scoping type-aware preset to `files: ['src/**/*.{ts,tsx}']` and adding a separate non-type-aware block for `*.config.{js,ts,cjs,mjs}`.
  (2) `tsc --noEmit -b` (from spec) — `--noEmit` and `-b` are mutually exclusive in TypeScript. Replaced with `tsc -b`.
  (3) Initial `tsc -b` run (before `noEmit: true` added to tsconfigs) emitted `.js`/`.d.ts` next to source; coverage saw them and dropped to ~40%. Fixed: added `"noEmit": true` to both `tsconfig.app.json` and `tsconfig.node.json` (modern TS allows this with `composite: true`); cleaned up emitted files; broadened vitest exclude list.
  (4) `tailwind.config.ts` referenced `@tremor/react/dist/tremor.config.cjs` — Tremor v3 doesn't ship that preset. Replaced with the canonical Tremor v3 manual theme extension, added `@headlessui/tailwindcss` and `@tailwindcss/forms` to dev deps. Final state: `make verify` green, all 4 Python tests + 1 frontend test pass, 100% coverage on tracked files in both, `make clean && make verify` round-trip works. Vite build emits a 993KB chunk warning (Tremor + Recharts are heavy); informational, not blocking — deferred for later refinement when route-splitting lands in STAGE-001-014+.
