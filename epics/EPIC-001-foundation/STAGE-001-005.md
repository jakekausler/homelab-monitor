# STAGE-001-005: Encrypted secrets store

## Status: Complete

## Overview

Implement the encrypted-at-rest secrets store. Secrets (API tokens, webhook URLs, SMTP passwords, Unifi creds, etc.) are stored as ciphertext in the `secrets` SQLite table, encrypted with AES-GCM using a per-row nonce and a per-row key derived from a master key + per-row salt via HKDF. The master key is bootstrapped from env (`HOMELAB_MONITOR_MASTER_KEY`) or from a `/run/secrets/master-key` file. Plugins access secrets only via a narrow `SecretsResolver` interface — never the raw store.

A CLI subcommand `hm secrets ...` and a future Settings → Secrets dashboard surface the same set of operations: `set / get / list / rotate / delete / rotate-master`.

**Spec references:** §7.3 (Secrets), §3.1 (secrets store component), §6.5 (env-var bootstrap).

## Concrete deliverables

1. **Crypto module** at `apps/monitor/homelab_monitor/kernel/secrets/crypto.py`
   - `derive_key(master: bytes, salt: bytes) -> bytes` — HKDF-SHA256 with project info string, output 32 bytes.
   - `encrypt(master: bytes, plaintext: bytes) -> tuple[bytes salt, bytes nonce, bytes ciphertext]` — generates fresh 16-byte salt and 12-byte nonce, derives key, AES-GCM-256 encrypt; tag included in ciphertext.
   - `decrypt(master: bytes, salt: bytes, nonce: bytes, ciphertext: bytes) -> bytes` — derive key, decrypt; raises `SecretIntegrityError` on tag mismatch.
2. **Master key loader** at `apps/monitor/homelab_monitor/kernel/secrets/master_key.py`
   - Order: env `HOMELAB_MONITOR_MASTER_KEY` (base64-encoded 32 bytes) → file `/run/secrets/master-key` (raw 32 bytes or base64) → fail.
   - On startup, validates length and format; refuses to start with a clear error otherwise.
   - `is_loaded()` and `current_fingerprint()` (HMAC of the key with a constant info — never logs the key, but lets us detect rotation).
3. **Secrets repository** at `apps/monitor/homelab_monitor/kernel/secrets/repository.py`
   - `set(name: str, value: str, *, by: str) -> None` — writes new row; if name exists, replaces (with audit row for rotation).
   - `get(name: str) -> str | None` — decrypts.
   - `list() -> list[SecretMeta]` — returns id, name, created_at, rotated_at; **never** values.
   - `rotate(name: str, new_value: str, *, by: str) -> None` — writes new row with bumped `rotated_at`.
   - `delete(name: str, *, by: str) -> None`.
   - `rotate_master(new_master: bytes, *, by: str) -> int` — re-encrypts every row with the new master, returns count. Atomic per-row; fails fast if any decryption fails (suggests master key was wrong).
4. **CLI** — extend `hm`:
   - `hm secrets set NAME` — prompts for value (no echo); supports `--from-stdin` for scripted setup.
   - `hm secrets get NAME` — prints to stdout; refuses if `HOMELAB_MONITOR_REVEAL=1` is not set (defense against shell-history leaks).
   - `hm secrets list` — table of name, created, rotated; never values.
   - `hm secrets rotate NAME` — prompts for new value.
   - `hm secrets delete NAME` — confirm prompt unless `--yes`.
   - `hm secrets rotate-master` — prompts for new master key (or reads from `--from-file`); re-encrypts everything.
5. **Audit log integration** — every set/rotate/delete/rotate-master writes an `audit_log` row with `before` masked to just the name (no values, ever).
6. **Tests**
   - Round-trip set/get with a fixture master key.
   - Tampered ciphertext raises `SecretIntegrityError`.
   - Wrong master key raises a clean error (no panic).
   - Rotate replaces value; old value undecryptable.
   - rotate-master re-encrypts every row; old key fails on every row afterwards.
   - List never returns values.
   - **No secret ever appears in any log line** — capture stdout/stderr and the structlog test capture, assert absence.

### Locked decisions (Design — 2026-05-05)

**Crypto primitive:** AES-GCM AEAD with HKDF-SHA256 per-row key derivation. Master key + 16-byte per-row salt → HKDF → 32-byte AES-256 key. Per-row 12-byte nonce stored alongside ciphertext. Industry-standard AEAD via `cryptography>=48` (already in pyproject.toml). Matches spec §7.3 option (a).

**Repository API shape:** Async-native `AsyncSecretsRepository` is the primary API used by CLI (via `asyncio.run`), FastAPI lifespan (STAGE-001-010), and async collectors (STAGE-001-007+). Separate `SyncSecretsResolver` class builds a frozen plaintext snapshot dict at construction time, served read-only to subprocess plugins (STAGE-001-009 untrusted-tier). Snapshot can be filtered to only declared names per plugin manifest. Snapshot rebuilds on `rotate-master`.

**Master key bootstrap:** Reads from env var `HOMELAB_MONITOR_MASTER_KEY` first, then `/run/secrets/master-key` file. **Both inputs are base64-encoded** (single decode path; one error mode). Generation: `head -c 32 /dev/urandom | base64`. Refuses to start without a valid 32-byte key. Provides a `master_key_fingerprint()` helper using `HMAC-SHA256(key, info=b"homelab-monitor/master-key/fingerprint")` for rotation-detection without leaking the key.

**Audit integration:** Every mutation (`set`, `rotate`, `delete`, `rotate-master`) calls `audit_write(repo, ...)` inside the SAME `repo.transaction()` block as the data write. If audit fails, the secret op rolls back too. `before` / `after` JSON dicts contain ONLY metadata (name, never plaintext value). Audit's `who="system"` for CLI invocations until STAGE-001-011 wires user identity.

**CLI `set` source:** `--from-stdin` only — `echo -n 'value' | hm secrets set <name> --from-stdin`. No positional value argument (avoids shell history). No interactive prompt fallback. Fails with usage error if `--from-stdin` not given.

**CLI `get` reveal gate:** `HOMELAB_MONITOR_REVEAL=1` env var required. Without the env var, `hm secrets get <name>` exits 1 with `set HOMELAB_MONITOR_REVEAL=1 to reveal` to stderr. Defends against accidental disclosure when the operator runs `get` without thinking.

**HKDF `info` parameter:** Uses `secrets.id` (the UUIDv7 PK) as the HKDF info parameter, NOT `secrets.name`. Rationale: stable across renames (future rename support won't need re-encryption) and unique by construction. Stored alongside ciphertext implicitly via the row.

**Migration:** Additive `0002_secrets_columns.py` adds three columns to the existing `secrets` stub table:
- `ciphertext TEXT NOT NULL` — base64-encoded `nonce||tag||ciphertext` blob
- `kdf_salt BLOB NOT NULL` — 16 random bytes per row
- `rotated_at TEXT NULL` — ISO-8601 UTC; null until first rotation

Stub table is currently empty (no rows in fresh DB), so `NOT NULL` columns can be added without backfill. Downgrade reverses the ALTER TABLEs.
