# STAGE-001-014: UI shell + login + Overview live-tile

## Status: Refinement (Complete)

## Overview

Bring the dashboard to life. Set up the routing skeleton, the login screen, the typed API client (generated from the FastAPI OpenAPI), the global app shell (sidebar nav, top bar, theme), and the **Overview** screen with one live tile that displays the `host` collector's CPU% in real time via SSE.

After this stage, the canonical demo works end-to-end: open `http://localhost:5173` (dev) or `http://localhost:9090/ui` (containerized), log in, see the host CPU tile updating every 10s.

**Spec references:** §9 (UI structure), §9.4 (frontend stack), §15 (stage 014 acceptance).

## Concrete deliverables

1. **Routing**
   - TanStack Router configured. Routes in this stage:
     - `/login` (public)
     - `/` (protected) → redirects to `/overview`
     - `/overview` (protected)
   - Protected routes use a guard that checks `useCurrentUser()` (which calls `GET /api/auth/me`); on 401, redirects to `/login`.
2. **API client**
   - `packages/shared-types/openapi.json` is generated from the backend (Makefile target from STAGE-001-010).
   - `apps/ui/src/api/` — typed fetch client generated by `openapi-typescript-codegen` (or `orval`). Wraps `fetch` with credentials include + CSRF header injection from the cookie.
   - `apps/ui/src/api/queries.ts` — TanStack Query hooks: `useCurrentUser`, `useCollectors`, `useMetricsSnapshot`, `useAlerts`.
3. **Theme & app shell**
   - Tailwind theme tokens for the modern dashboard aesthetic; dark mode default; light mode toggleable from the user menu.
   - `<AppShell>` layout component: left sidebar (collapsible), top bar (search placeholder, notification dot, user menu), main content slot.
   - Sidebar items present but most disabled (point-of-truth list lives in spec §9.1):
     - Overview (enabled)
     - Alerts (disabled — STAGE-001-019)
     - Inventory (disabled)
     - Integrations (disabled)
     - Logs (disabled)
     - Metrics (disabled)
     - Runbooks (disabled)
     - Auto-fix history (disabled)
     - Discovery & suggestions (disabled)
     - Tool analysis (disabled)
     - Maintenance windows (disabled)
     - Self-status (disabled — but the badge in the top bar works, fed by `/api/healthz`)
     - Settings (disabled)
4. **Login screen**
   - Form: username, password, "Remember me" (extends session TTL). React Hook Form + Zod validation.
   - Calls `POST /api/auth/login`; on success, redirect to `/overview`.
   - Error states: 401 (wrong creds), 429 (rate-limited; show countdown), 500 (generic error).
5. **Overview live tile**
   - One Tremor card showing "Host CPU" with the current value (sparkline of last 60s + big number).
   - Connects to `/api/events` via `EventSource`; subscribes to `collector.tick` events, updates when `name === "host"`.
   - Backfill on mount: call `GET /api/metrics/snapshot` for initial value.
   - Empty state: "Connecting…" while EventSource is opening.
   - Error state: if EventSource fails 3 times, show a "Reconnect" button; while disconnected, the tile shows the last known value with a "stale" badge.
6. **SSE client utility** at `apps/ui/src/lib/sse.ts`
   - Reusable hook `useSSE<T>(topic: string, parser: (event) => T)` returning `{value, status: "connecting"|"open"|"error", reconnect}`.
   - Auto-reconnect with exponential backoff capped at 30s.
7. **Tests**
   - Vitest + Testing Library: login form validation, error states, redirect after success.
   - Mocked SSE in the Overview tile test verifies the tile renders the latest value and reflects "stale" badge after a connection-loss simulation.
   - Playwright (lightweight; first real test): login → see Overview → tile updates after a backend tick.
8. **Static asset serving**
   - `apps/monitor` Dockerfile (added in compose stage later) copies the built UI bundle into `/app/ui` and FastAPI's `StaticFiles` serves it at `/ui/`.
   - In dev, the UI runs on Vite at 5173 and proxies `/api` to 9090.

## Design Phase

- **Approaches to present:**
  - (a) TanStack Router + Query + openapi-typescript-codegen (recommended; spec)
  - (b) React Router 6 + tRPC-style client (deviates from FastAPI; not warranted)
- **UI Options Presented:**
  - Two visual mocks of the Overview live tile: (a) big number + sparkline below, (b) ring gauge with peripheral stats. Pick one.
  - Two login layouts: (a) centered card, (b) split-screen with marketing copy. Recommended (a) — no marketing.
- **User Choice:**
  - **Middleware sequencing: Option A** — migrate `RequestIdMiddleware`, `AccessLogMiddleware`, `AuthMiddleware` from `BaseHTTPMiddleware` to pure ASGI callables FIRST (with the 2 xfailed SSE tests as the verification harness), then build UI on top of working SSE. Removes xfail markers as part of the work; honors stage's "tile updating every 10s via SSE" acceptance criterion.
  - **OpenAPI codegen: openapi-typescript + openapi-fetch** — modern, ~5kb runtime, types-only at compile time + thin typed fetch wrapper. Integrates cleanly with TanStack Query. Deviates from spec §9.4 (which says `openapi-typescript-codegen`); deviation justified by significantly better DX and smaller bundle.
  - **UI primitives: shadcn/ui** — CLI installs components built on Radix into `apps/ui/src/components/ui/`; project owns the source. Matches spec §9.4's "Radix + Tailwind, lightly themed" intent. Note: spec mentions Tremor for charts; we use Recharts (already in deps; Tremor was removed in earlier work).
  - **Overview tile layout: big number + sparkline below** (option a; simple, scannable).
  - **Login layout: centered card** (option a; minimal; no marketing).
  - **Charting: Recharts (already in deps).** Sparkline may be hand-coded SVG path for the simple 60-point series; Recharts available for richer charts in later stages.
- **Seed Data Agreed:** the `host` collector's CPU metric (`homelab_host_cpu_percent{cpu="all"}`) is the seed; backfill from `GET /api/metrics/snapshot` on tile mount, then live updates via `EventSource` on `/api/events` filtered to `collector.tick` events where `name === "host"`.
- **Session Notes:**
  - Stage proceeds in two phases within Build: (1) backend middleware ASGI migration to unblock SSE; (2) frontend UI shell + login + Overview tile.
  - Auth middleware migration is non-trivial (~95 LOC; preserves async DB lookups for session/token resolution).
  - `/api/version` already returns `users_configured: bool` — frontend uses this to show "Welcome — please run hm user create" vs login form.
  - CSRF cookie name: `homelab_monitor_csrf` (httponly: false so JS can read it). Header name: `X-CSRF-Token`. Send on POST/PUT/PATCH/DELETE.
  - Session cookie name: `homelab_monitor_session`.
  - Refinement phase MUST prompt user for both desktop (1920×1080) AND mobile (375×667) viewport approval.

## Build Phase

- **Components Created:**
  - **Backend (Spec A — middleware ASGI migration):** RequestIdMiddleware, AccessLogMiddleware, AuthMiddleware all rewritten as pure ASGI callables (`__init__(app)` + `async __call__(scope, receive, send)`). xfail markers removed from `test_sse_http_endpoint_smoke` and `test_lifespan_e2e_sse_receives_tick`.
  - **Backend (Spec A.1 — uvicorn test fixture):** new `apps/monitor/tests/_uvicorn_fixture.py` with `UvicornTestServer` + `uvicorn_server` async fixture. The 2 SSE tests rewritten to hit the real uvicorn loopback socket. Required because `httpx.ASGITransport` ALSO buffers streaming responses (it accumulates body chunks and only returns control after the ASGI app completes — verified via httpx/_transports/asgi.py source). The middleware migration unblocks production SSE; the uvicorn fixture lets pytest verify HTTP-level streaming.
  - **Frontend foundation (Spec B Steps 0-5):** package.json deps (openapi-typescript, openapi-fetch, @hookform/resolvers ^5, lucide-react, class-variance-authority, tailwind-merge, Radix UI primitives), `cn()` util, shadcn-style UI components (button, input, label, card, dropdown-menu, tooltip), `scripts/generate-ui-types.sh` + pre-commit hook for openapi.json change-trigger, typed API client at `src/api/client.ts` with CSRF middleware + `unwrap()` for error envelope, TanStack Query hooks at `src/api/queries.ts` (useCurrentUser, useVersion, useMetricsSnapshot, useLogin, useLogout, useCollectors, useAlerts).
  - **Frontend routes + components (Spec B Steps 6-13):** code-based TanStack Router with `/login` (public) + `/overview` (protected, auth guard via `beforeLoad` calling `useCurrentUser`'s queryFn) + `/` redirect to `/overview`. AppShell with collapsible sidebar (13 nav items, Overview enabled, all others disabled with "Coming soon" tooltips), top bar (Toggle sidebar / search placeholder / notifications / user menu), theme toggle persisted to localStorage. Login screen (centered card, RHF + Zod, error states for 401/429/5xx, "Welcome — please run hm user create" variant when `users_configured: false`). Overview screen with HostCpuTile (big number + sparkline below). useSSE hook with latest-ref pattern for unstable callbacks (parser, factory) and decoupled reconnect-scheduling effect. Sparkline as raw SVG path. Theme tokens extended in `src/index.css`.
  - **Tests (Spec B Steps 14-15):** vitest coverage thresholds relaxed for UI to 80/75/80/80 (kernel stays 100%). 77 vitest tests across api/csrf, api/client, api/queries (15 hook tests), components/{AppShell,SidebarNav,TopBar,UserMenu,tiles/HostCpuTile,tiles/Sparkline}, lib/sse (10 tests including reconnect/backoff/manual reconnect/cleanup), routes/Login (welcome, validation, login flow, all error states).
- **API Endpoints Added:** none (consumes existing).
- **Placeholders Added:** disabled sidebar items render with Tooltip showing "Coming soon".
- **Session Notes:**
  - **Critical mid-build discovery (Spec A → A.1):** prior session's premise was wrong. `httpx.ASGITransport` buffers streaming responses just as `BaseHTTPMiddleware` did. Removing xfail markers wasn't enough; the 2 SSE tests required a real uvicorn server. User chose Option B (add in-process uvicorn fixture this stage). Future stages STAGE-001-015A and STAGE-001-021 use docker-compose.test.yml for HTTP-level integration; this in-process fixture is for pytest-only verification.
  - **Critical dependency upgrade:** `@hookform/resolvers` upgraded from `^3.10` to `^5.0.1` (now `5.2.2`) for Zod v4 compatibility. v3 was built for Zod v3 and silently fails to map the v4 ZodError shape, causing unhandled rejections in form validation tests.
  - **sse.ts architecture revision:** original implementation used `openRef.current()` forward-reference pattern in setTimeout callback. Refactored to use latest-ref pattern (parserRef/factoryRef synced in render body) + decoupled reconnect-scheduling into its own useEffect keyed on `[status, failureCount, open]`. This eliminated test thrashing under React 19 + fake timers and made the reconnect path deterministic.
  - **Pre-existing flake fixed:** `test_verify_session_cookie_value_tampered_session_id` and `test_verify_session_cookie_value_tampered_hmac` had a 1/16 per-test probabilistic failure (cumulative ~12% per full-suite run). The "tamper" overwrote the first hex char with `'a'`, but if the original hex already started with `'a'`, the tamper was a no-op. Fixed by selecting `'b'` when first char is `'a'`, else `'a'` — guaranteed-different.
  - **Static asset serving:** deferred to STAGE-001-015 per stage file (no backend changes required in this stage). Vite dev server proxies `/api/*` to localhost:9090.
  - **Final verification:** `make verify` exits 0 deterministically across 3 consecutive runs. Backend 870 tests at 100% kernel coverage. Frontend 77 vitest tests at 91.57% lines coverage. UI build produces 543KB JS bundle (170KB gzipped).

## Refinement Phase

- [x] Desktop Approved (full demo at 1920×1080)
- [x] Mobile Approved (Overview tile readable at 375×667; login usable)
- [x] Regression Items Added (regression.md updated)
- **Feedback Round 1:**
  - Sparkline rhythm: graph showed a bump then flatlined between 10s ticks. Root cause: series seeded with 1 element; auto-scaling Y-domain collapsed when value count was small. Fix: pre-fill 60-point baseline `Array(SERIES_CAPACITY).fill(initialValue)`.
  - `/api/metrics/snapshot` request storm (multiple per second). Root cause: TanStack Query result `snapshot` included as effect dep; refetch->re-render->refetch infinite loop. Fix: latest-ref pattern (`refetchRef`) so effect depends only on `[sse.value]`. Plus `staleTime: Infinity` and `refetchOnWindowFocus: false` on `useMetricsSnapshot`.
  - Browser-extension `[AppIntegration] NmLockState` console noise — not our app; ignored.
- **Feedback Round 2:**
  - Mobile sidebar: should be a full-screen overlay opened/closed by hamburger; persistent on desktop. Implemented: AppShell now has separate desktop sidebar (`hidden md:flex`) + mobile overlay sheet (`fixed inset-0 z-50`) with backdrop. TopBar renders two hamburgers (mobile-only + desktop-only via Tailwind responsive classes). SidebarNav accepts optional `onNavigate` callback to auto-close mobile sheet on link click.
  - Login on mobile invalidated desktop session. Root cause: `auth.py::login` called `delete_all_user_sessions(user.id)` (session-fixation defense). Changed to allow concurrent sessions. Change-password still revokes all sessions (post-incident posture). Test renamed `test_login_invalidates_prior_sessions` → `test_login_preserves_prior_sessions` and assertion inverted. `test_change_password_rotates_all_sessions` unchanged.
  - Notifications button had a yellow dot but no functionality. Per request: disabled with "Coming soon" tooltip (matches sidebar disabled-item pattern).
  - "Updated" timestamp formatted as raw ISO string. Per request: now `HH:MM:SS` in user's local TZ; date prefix `YYYY-MM-DD ` only when not today (also TZ-aware via `Date` getters).
  - Password reveal eye toggle added to Login form (Eye/EyeOff from lucide-react; toggles `type="password"|"text"`).
- **Feedback Round 3:**
  - Sparkline doesn't preserve history across page reloads. By spec, no historical metrics storage in this stage; STAGE-001-015 adds VictoriaMetrics. Confirmed STAGE-015 deliverables list now explicitly includes a "Frontend backfill" item updating `HostCpuTile.tsx` to call new `useMetricsRange` hook against `GET /api/metrics/range` and seed sparkline with real history.
