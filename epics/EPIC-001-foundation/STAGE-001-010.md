## Design Decisions (Locked 2026-05-05)

**D1 â€” Lifespan failure handling: Hybrid abort/degrade**
- Critical failures (master key bootstrap, migrations pending, loader.persist_to_db) â†’ log structured-JSON error via structlog, then `os._exit(1)` so the message is not buried by uvicorn boilerplate. Container restart policy + local-watchdog handles recovery.
- Non-critical failures (one subprocess plugin manifest invalid, one collector failing to register) â†’ log warning, exclude that collector from `loaded`, continue. Surface in `/api/healthz` as `collectors.degraded: ["foo"]`.
- `MigrationsPendingError` â†’ abort. A half-migrated DB silently serving traffic is the worst outcome.

**D2 â€” SSE broker: Per-subscriber bounded queue + replay ring buffer**
- Each subscriber gets `asyncio.Queue(maxsize=64)`. Broker fans out via `put_nowait`; subscribers whose queue fills are disconnected with HTTP 410.
- Broker retains a ring buffer of last 50 events; replays them on subscribe so brief client disconnects (wifi blips, tab refresh) recover gracefully.
- Each event carries a monotonic `event_seq: int` so a future Last-Event-ID resume can be added without protocol breakage.

**D3 â€” Schedulerâ†’Broker plumbing: EventSink Protocol**
- New module `apps/monitor/homelab_monitor/kernel/events.py` defines `class EventSink(Protocol): async def publish(self, event: CollectorEvent) -> None: ...` and a `NullEventSink` no-op for tests.
- `SchedulerConfig` gains an `event_sink: EventSink` field (default `NullEventSink()`).
- The SSE broker in `kernel/api/sse.py` implements `EventSink`. Publish is **non-throwing by contract**: broker catches its own errors, logs them, never propagates back to the scheduler tick. Documented in the Protocol docstring.
- Scheduler depends ONLY on the Protocol, never on `kernel/api/`.

**D4 â€” CollectorContext factory: Hybrid shared + per-tick rebind**
- Lifespan constructs shared resources once: `httpx.AsyncClient` with explicit `Limits(max_connections=50, max_keepalive_connections=20)` and `Timeout(10.0, connect=5.0)`; `SqliteRepository`; `InMemoryMetricsWriter` (interim until STAGE-015); `InMemoryLogsWriter` (interim until STAGE-016); stub `SshClientFactory` (raises NotImplementedError on use; STAGE-017); `ha=None` (EPIC-005).
- Per-tick rebind: fresh `BoundLogger` with `collector` + `tick_id` bindings; per-collector secrets filter via `resolver.filtered(declared_names)`.
- Secrets snapshot policy: **60-second TTL refresh**. Resolver caches decrypted values for 60s, then re-reads from DB on next access. Bounds `hm secrets rotate` propagation to ~1 minute, no restart required. Implemented as a `TtlCachingResolver` wrapper or by adding `cached_at` + `ttl_seconds` to `AsyncSecretsRepository.snapshot()`.

**D5 â€” Trigger context for scheduler runs**
- New dataclass `TriggerContext` with fields: `kind: Literal["scheduled", "retry", "manual"]`, `request_id: str | None`. Used to attach origin metadata to scheduler ticks.
- Scheduler always generates a `tick_id: str` (uuid4) for every run; this is the canonical scheduler-side correlation id.
- `request_id` (uuid4 per HTTP request) is generated by FastAPI middleware. It is **HTTP-only**; scheduler-spontaneous ticks have `request_id=None`.
- Subprocess plugins always receive `HOMELAB_TICK_ID` env var. They additionally receive `HOMELAB_TRIGGER_KIND` and `HOMELAB_REQUEST_ID` only when a `TriggerContext` is attached (e.g., from the retry endpoint).
- `structlog.contextvars.bound_contextvars(tick_id=...)` wraps each per-tick scheduler block so any logging done by the ctx_factory or collector inherits the binding.
- Spec phrasing "request_id propagated to subprocess plugins" is interpreted as: the operator-triggered case carries it; the scheduled case does not. The scheduler-side correlation id is `tick_id`.

**+ Bonus: scheduler self-metrics surfaced in `/api/healthz`**
- Closes a 5-stage operator-visibility gap (no real metrics sink until STAGE-015 lands VictoriaMetrics).
- `/api/healthz` includes: `last_tick_at: ISO-8601 UTC`, `failed_ticks_last_5m: int`, `quarantined_collectors: list[str]`. Local-watchdog can drive on real signal.
- Implemented by the healthz handler aggregating from `InMemoryMetricsWriter.recorded` (or by extending the writer with query helpers â€” to be decided in Build).

**+ Retry endpoint semantics: clear quarantine AND trigger immediate run**
- `POST /api/collectors/{name}/retry` (with `X-Auth: dev` placeholder header) does TWO things: clears quarantine via `Scheduler.clear_quarantine(name, by=user)` AND requests an immediate out-of-band run via a new `Scheduler.request_immediate_run(name, trigger=Trigger(kind="retry", request_id=req_id))` method.
- The immediate run skips the ordinary tick-interval gating but otherwise goes through the full collector pipeline (concurrency group lock, timeout, failure budget, event sink). If the next scheduled tick is imminent, the immediate run still happens; the next scheduled tick is then deferred by one interval to avoid back-to-back runs.
- This is a non-trivial scheduler API extension; the planner spec for Build phase must call it out explicitly so it isn't missed.

**+ OpenAPI export: pre-commit hook**
- Pre-commit hook fires on changes under `apps/monitor/homelab_monitor/kernel/api/**`. Runs a script `scripts/export-openapi.sh` (or equivalent) that imports `create_app()`, calls `app.openapi()`, and writes to `packages/shared-types/openapi.json`.
- The script must be idempotent (no diff = no error). The hook fails only if regeneration produces a diff that the operator hasn't committed.
- `create_app()` must support a "schema-only" mode where the lifespan is NOT executed (no DB connection, no scheduler start) â€” needed so the hook can run in the pre-commit environment without a live SQLite file. Suggested: `create_app(*, lifespan_enabled: bool = True)`.

## Acceptance Criteria

- App boots end-to-end: master key bootstrap â†’ migrations applied â†’ plugin loader registers built-in `noop` + subprocess `hello-subprocess-plugin` â†’ `loader.persist_to_db(repo)` succeeds â†’ scheduler starts with `mp_context="forkserver"` ProcessPoolExecutor (closes STAGE-007 fork() regression).
- `GET /api/healthz` returns 200 with `{ok: true, version, db: "up", scheduler: "running", last_tick_at, failed_ticks_last_5m, quarantined_collectors}`. Used by local-watchdog and Docker healthcheck.
- `GET /api/version` returns `{version, git_sha, built_at}` (build-arg-populated in Docker; placeholders in dev).
- `GET /api/collectors` returns each loaded collector with `{name, status, last_run, last_error, quarantined, next_run, run_kind}`. Both noop and hello-subprocess-plugin appear.
- `POST /api/collectors/{name}/retry` with `X-Auth: dev` (when `HOMELAB_MONITOR_DEV_AUTH=1`) clears quarantine + triggers immediate run. Without the env var or header, returns 401 with uniform error envelope.
- `GET /api/events` opens an SSE stream that delivers `collector.tick` events; replays last 50 on connect; subscriber-overflow disconnects with 410.
- All errors return uniform envelope: `{error: {code: "snake_case", message: "human", details?: {...}}}`. Status codes 400/401/403/404/409/422/500/503 covered.
- structlog produces JSON in production (`HOMELAB_MONITOR_LOG_FORMAT=json` default), pretty in dev. `httpx`/`uvicorn`/`alembic` logs piped through structlog. `request_id` and `tick_id` propagate via contextvars.
- Pre-commit hook regenerates `packages/shared-types/openapi.json` on `kernel/api/**` changes.
- 100% kernel coverage on new `kernel/api/`, `kernel/events.py`, `kernel/logging.py` modules. All ruff/pyright-strict gates pass. `make verify` green.

## Concrete deliverables

1. **App entry** at `apps/monitor/homelab_monitor/kernel/api/app.py`
   - `create_app() -> FastAPI` factory.
   - **Lifespan:** on startup â†’ bootstrap secrets master key â†’ run migrations (if `HOMELAB_MONITOR_AUTO_MIGRATE=true`) â†’ start scheduler with loaded plugins. On shutdown â†’ stop scheduler with grace.
   - Exposes routers under `/api/*` only; the future React bundle will be served from `/ui/*` (mounted via FastAPI's `StaticFiles` in a later stage when the bundle exists).
2. **Endpoints** (this stage)
   - `GET /api/healthz` â†’ `{ok: true, version, db: "up", scheduler: "running"|"stopped"}`. Used by `local-watchdog` and Docker healthchecks.
   - `GET /api/version` â†’ `{version, git_sha, built_at}` (build args populate these in the Docker image; uses placeholders in dev).
   - `GET /api/collectors` â†’ list every loaded collector with status (last_run, last_error, quarantined, next_run). Wires the scheduler API surface from STAGE-001-007/008.
   - `POST /api/collectors/{name}/retry` â†’ clears quarantine; protected by **placeholder auth header** for now (`X-Auth: dev` until STAGE-001-011 lands real auth).
3. **SSE plumbing** at `apps/monitor/homelab_monitor/kernel/api/sse.py`
   - A simple in-memory pub/sub broker; subscribers register; publishers fan out.
   - `GET /api/events` returns an SSE stream of all events the dashboard cares about. For this stage, only one event kind: `collector.tick` carrying the per-collector run summary.
4. **Structured logging** at `apps/monitor/homelab_monitor/kernel/logging.py`
   - structlog configured: JSON renderer in production (`HOMELAB_MONITOR_LOG_FORMAT=json`, default), pretty in dev.
   - Standard fields: `ts` (UTC ISO), `level`, `event`, `logger`, plus a request-scoped binding `request_id` (uuid4 per HTTP request, propagated to subprocess plugins via env when applicable).
   - `httpx`/`uvicorn`/`alembic` logs piped through structlog.
5. **Uniform error model** at `apps/monitor/homelab_monitor/kernel/api/errors.py`
   - All errors return a uniform envelope: `{error: {code: "snake_case", message: "human", details?: {...}}}`.
   - HTTP status codes: 400 invalid input, 401 unauth, 403 forbidden, 404 not found, 409 conflict, 422 validation (let pydantic provide details), 500 internal, 503 dependency_unavailable.
   - A central exception handler converts `HTTPException`, pydantic `ValidationError`, our domain exceptions (`SecretIntegrityError`, `MigrationsPending`, etc.), and unhandled exceptions into the envelope.
6. **OpenAPI generation**
   - FastAPI auto-generates `/api/openapi.json`. Add a `make openapi-export` target that writes a typed schema to `packages/shared-types/openapi.json` (and ideally to TS types via `openapi-typescript-codegen`).
7. **Tests**
   - `httpx.AsyncClient` against `create_app()`:
     - `/api/healthz` returns 200 with expected fields.
     - `/api/collectors` returns the list including the noop and hello-world plugins, each with a sane status shape.
     - `POST /api/collectors/{name}/retry` clears quarantine.
     - Error envelope is uniform across raised cases.
   - SSE: subscribe, drive a tick from a fixture, assert event delivered.
