# STAGE-001-009: Subprocess plugin runner + JSON line protocol

## Status: Complete

## Overview

Add the second plugin execution path: subprocess plugins. These are any-language scripts/binaries described by a `plugin.yaml` manifest. The runner spawns the plugin as a subprocess on schedule, reads line-delimited JSON events from stdout, captures stderr to the structured log, and interprets the exit code. This unlocks bash/Go/Rust/whatever plugins, untrusted/sandboxed plugins, and a clean path for community plugins.

After this stage, a hello-world bash plugin emits metrics that flow through the same `MetricsWriter` as in-process collectors.

**Spec references:** §5.2 (in-process contract), §5.3 (subprocess JSON protocol), §5.5 (trust tiers), §5.6 (configuration / plugin.yaml).

## Concrete deliverables

1. **Manifest schema** at `apps/monitor/homelab_monitor/kernel/plugins/manifest.py`
   - Pydantic model `PluginManifest`:
     ```yaml
     manifest: 1
     name: my-custom-probe                       # plugin name regex
     language: bash | python | binary | ...      # informational
     command: ["./run.sh"]                       # argv list, relative to manifest directory
     interval: 60s
     timeout: 30s
     concurrency_group: my-group                 # optional
     trust_level: trusted | untrusted            # untrusted gets restricted env + no DB write
     env: { FOO: "bar" }                         # static env
     secrets: ["my_token"]                       # names this plugin is allowed to receive (untrusted plugins MUST declare)
     workdir: "/plugins/my-custom-probe"         # cwd for the subprocess
     ```
   - Validation enforces `name` regex, `interval >= 5s`, `timeout < interval`, manifest version is `1`.
2. **Runner** at `apps/monitor/homelab_monitor/kernel/plugins/subprocess_runner.py`
   - Spawns the subprocess with `asyncio.create_subprocess_exec`.
   - Streams stdout line-by-line; parses each line as JSON; dispatches to the appropriate writer (metric → `MetricsWriter`, log → `LogsWriter`, event → events list, heartbeat → heartbeat handler stub for STAGE-002, result → final outcome).
   - Captures stderr concurrently into the structured log at `info` level (or `warning` if exit code non-zero).
   - Enforces `timeout`: on expiry, sends SIGTERM, waits 2s, then SIGKILL. Records as a failure.
   - Exit code: `0` → success; non-zero → failure. The plugin can also emit a `result` event with `ok=false` to mark itself failed without a non-zero exit (useful for plugins that prefer to report cleanly).
3. **Trust-tier enforcement**
   - **Untrusted plugin restrictions:**
     - Spawned with a clean env containing only the keys the manifest declares (plus a small allowlist: `PATH`, `TZ`, `HOME=/tmp/<plugin>`).
     - Working directory is the manifest's directory; no read access guaranteed elsewhere by Docker mount setup.
     - `MetricsWriter` and `LogsWriter` accept their writes; **DB writes are blocked** (the runner does not pass a DB-writable repository).
     - Secrets resolver is built from a *narrow* dict containing only the names declared in `manifest.secrets`.
   - **Trusted plugin:** receives the same context as in-process collectors except still goes through subprocess (so it's still language-agnostic and isolatable).
4. **Loader integration**
   - `loader.py` (from STAGE-001-007) extended: walks `/plugins/` for `plugin.yaml` files; registers each as a subprocess collector.
5. **Hello-world plugin** at `runbooks/_examples/hello-subprocess-plugin/`
   - `plugin.yaml` describing the plugin.
   - `run.sh` that emits a fixed `homelab_hello_world{language="bash"}` metric and a `result` event.
   - Smoke test verifies output appears in the test's `InMemoryMetricsWriter`.
6. **Tests**
   - Manifest validation: missing fields, bad regex, timeout >= interval all fail validation.
   - Bash hello-world end-to-end metric round-trip.
   - Timeout: a plugin that sleeps past timeout is killed; failure metric emitted.
   - Malformed JSON line: logged as a warning, doesn't crash the runner; subsequent valid lines still processed.
   - Non-zero exit: failure recorded; `result` events still consumed.
   - Untrusted plugin can write metrics but **cannot** write to DB; verify by attempting (the API the runner exposes for untrusted is intentionally narrower; test asserts at the type/API level).
   - Untrusted plugin's secrets resolver only returns declared names.

## Out of scope for this stage

- Plugin registry / install command (`hm plugin install ...`) — deferred
- Plugin signing / supply-chain checks — deferred
- Per-plugin resource limits beyond timeout (e.g., RLIMIT_AS, cgroups) — deferred to a hardening epic

## Decisions in this stage

- **Manifest version 1 only.** Future versions add fields; we never break v1 manifests.
- **Heartbeat events** in subprocess output are routed to a no-op handler in this stage; STAGE-002-* wires them to the real heartbeat receiver.

## Design Phase

- **Approaches to present:**
  - (a) line-delimited JSON over stdout (recommended; matches spec)
  - (b) HTTP RPC (heavier, more moving parts)
  - (c) Unix domain socket
- **UI Options Presented:** N/A
- **User Choice:** (a) line-delimited JSON over stdout
- **Seed Data Agreed:** the hello-world plugin at `runbooks/_examples/hello-subprocess-plugin/`
- **Session Notes:**
  - **D1 SubprocessCollector Protocol fit:** New `SubprocessCollector` Python class wraps the validated manifest; satisfies the existing `Collector` Protocol. Its `async def run(self, ctx)` delegates to `subprocess_runner.run_subprocess(self.manifest, ctx)`. Stored as `LoadedCollector(collector=SubprocessCollector(manifest), config=...)`. Add 4th `RunKind.SUBPROCESS` enum value for telemetry/observability/future divergence. Scheduler `_dispatch` SUBPROCESS arm reuses ASYNC pattern (`await c.run(ctx)`). Group locks + FailureBudget already RunKind-agnostic. Replaces the SCAFFOLDING comment in `_dispatch` at lines 466-470.
  - **D2 "DB writes blocked" semantics:** Architectural property of the JSON protocol — spec §5.3 enumerates 5 line types (metric/event/log/heartbeat/result), none touch the DB. "Blocked" is descriptive not enforced at runtime. Trust-tier differences are: (1) env scrubbing, (2) secrets narrowing via `SyncSecretsResolver.filtered`, (3) workdir confinement. Unknown line `type` values logged-and-discarded uniformly across both tiers.
  - **D3 Subprocess context delivery:** Hybrid. Manifest's `env` field → environment variables (non-sensitive plugin config) plus allowlist (`PATH`, `TZ`, `HOME=/tmp/<plugin>`). Secrets + runtime metadata (collector_name, deadline_unix) → single JSON object on stdin, runner writes-and-closes-EOF immediately. Plugin reads via `cat` (bash) or `sys.stdin.read()` (Python). Avoids `/proc/<pid>/environ` secret leakage. Both trust tiers get filtered secrets per manifest declarations (forces hygiene).
  - **D4 Output stream parsing:** Strict line-by-line via `asyncio.StreamReader.readline()`. Malformed JSON → structured warning log (`subprocess.plugin.malformed_json_line`) + skip + continue. Partial line at EOF (subprocess died mid-write) → warning log + discard. Unknown `type` value → info log (`subprocess.plugin.unknown_line_type`) + discard (forward-compat). Line after `result` → warning log (`subprocess.plugin.line_after_result`) + discard (protocol violation). No subprocess-emitted `result` line → synthesized `CollectorResult(ok=False, errors=["no result line emitted"])`.
  - **D5 Stderr handling:** Two concurrent `asyncio.create_task`s for stdout + stderr. Each stderr line logged at info level (`subprocess.plugin.stderr`) in real time, correlated by `collector` field. On subprocess exit with non-zero exit code, one additional warning log (`subprocess.plugin.nonzero_exit`) with exit_code + duration. Don't re-log stderr content (already in structured log).
  - **D6 Loader INSERT-on-register (closes STAGE-008's loader-INSERT gap):** Two-phase init. `loader.register()` stays SYNC (no STAGE-007 API churn). New async method `persist_to_db(repo: SqliteRepository) -> None` walks `_loaded` and runs `INSERT OR IGNORE INTO collectors (id, name, config, created_at) VALUES (...)` for each. Idempotent. STAGE-001-010's lifespan calls `await loader.persist_to_db(repo)` after `loader.load_all()`. Tests can call directly OR continue using STAGE-008's `_seed_collectors` helper.
  - **D7 hello-world bash plugin:** New directory `runbooks/_examples/hello-subprocess-plugin/`. `plugin.yaml`: manifest=1, name=hello-subprocess, language=bash, command=["./run.sh"], interval=60s, timeout=10s, trust_level=trusted, env={}, secrets=[], no workdir (defaults to manifest dir). `run.sh`: drains stdin (`cat >/dev/null`), emits one `homelab_hello_world{language="bash"}` counter metric + one `{"type":"result","ok":true,"summary":"hello world"}` line. Smoke test in `tests/test_subprocess_runner.py` verifies metrics.recorded contains the metric.
  - **Subprocess process group + signal handling:** Spawn with `start_new_session=True` (or `preexec_fn=os.setsid` on POSIX) so SIGTERM/SIGKILL targets the entire process group. Bash scripts that fork helpers won't leak children. Required for the spec-mandated SIGTERM → wait 2s → SIGKILL timeout flow.
  - **Self-metrics emission:** Parent-side runner emits the existing 5 scheduler-owned self-metrics (`homelab_collector_run_*`) via `ctx.vm` — the subprocess does NOT. The plugin's `result` line drives success/failure classification; runner-measured duration drives `homelab_collector_run_duration_seconds`. Confirms scheduler self-metrics emission is uniform across in-process and subprocess execution.
  - **Manifest validation in Pydantic:** `SubprocessManifest(BaseModel)` with constraints: name regex (matches PLUGIN_NAME_PATTERN), `interval >= 5s`, `timeout < interval`, `manifest == 1`. Reject invalid manifests at load time with clear errors. Per-manifest failures during directory scans logged and skipped (one bad plugin doesn't break the others).
  - **Forward-compat verified:**
    - STAGE-001-010 FastAPI lifespan: calls `loader.load_all()` then `await loader.persist_to_db(repo)` then `scheduler.start()`.
    - STAGE-001-013 alert dispatch: subprocess heartbeat events route to no-op handler in this stage; STAGE-002-* wires real heartbeat receiver.
    - Future trust tier hardening (cgroups, RLIMIT, Docker mount restrictions) deferred to a hardening epic.
