# STAGE-001-009: Subprocess plugin runner + JSON line protocol

## Status: Not Started

## Overview

Add the second plugin execution path: subprocess plugins. These are any-language scripts/binaries described by a `plugin.yaml` manifest. The runner spawns the plugin as a subprocess on schedule, reads line-delimited JSON events from stdout, captures stderr to the structured log, and interprets the exit code. This unlocks bash/Go/Rust/whatever plugins, untrusted/sandboxed plugins, and a clean path for community plugins.

After this stage, a hello-world bash plugin emits metrics that flow through the same `MetricsWriter` as in-process collectors.

**Spec references:** §5.2 (in-process contract), §5.3 (subprocess JSON protocol), §5.5 (trust tiers), §5.6 (configuration / plugin.yaml).

## Concrete deliverables

1. **Manifest schema** at `apps/monitor/homelab_monitor/kernel/plugins/manifest.py`
   - Pydantic model `PluginManifest`:
     ```yaml
     manifest: 1
     name: my-custom-probe                       # plugin name regex
     language: bash | python | binary | ...      # informational
     command: ["./run.sh"]                       # argv list, relative to manifest directory
     interval: 60s
     timeout: 30s
     concurrency_group: my-group                 # optional
     trust_level: trusted | untrusted            # untrusted gets restricted env + no DB write
     env: { FOO: "bar" }                         # static env
     secrets: ["my_token"]                       # names this plugin is allowed to receive (untrusted plugins MUST declare)
     workdir: "/plugins/my-custom-probe"         # cwd for the subprocess
     ```
   - Validation enforces `name` regex, `interval >= 5s`, `timeout < interval`, manifest version is `1`.
2. **Runner** at `apps/monitor/homelab_monitor/kernel/plugins/subprocess_runner.py`
   - Spawns the subprocess with `asyncio.create_subprocess_exec`.
   - Streams stdout line-by-line; parses each line as JSON; dispatches to the appropriate writer (metric → `MetricsWriter`, log → `LogsWriter`, event → events list, heartbeat → heartbeat handler stub for STAGE-002, result → final outcome).
   - Captures stderr concurrently into the structured log at `info` level (or `warning` if exit code non-zero).
   - Enforces `timeout`: on expiry, sends SIGTERM, waits 2s, then SIGKILL. Records as a failure.
   - Exit code: `0` → success; non-zero → failure. The plugin can also emit a `result` event with `ok=false` to mark itself failed without a non-zero exit (useful for plugins that prefer to report cleanly).
3. **Trust-tier enforcement**
   - **Untrusted plugin restrictions:**
     - Spawned with a clean env containing only the keys the manifest declares (plus a small allowlist: `PATH`, `TZ`, `HOME=/tmp/<plugin>`).
     - Working directory is the manifest's directory; no read access guaranteed elsewhere by Docker mount setup.
     - `MetricsWriter` and `LogsWriter` accept their writes; **DB writes are blocked** (the runner does not pass a DB-writable repository).
     - Secrets resolver is built from a *narrow* dict containing only the names declared in `manifest.secrets`.
   - **Trusted plugin:** receives the same context as in-process collectors except still goes through subprocess (so it's still language-agnostic and isolatable).
4. **Loader integration**
   - `loader.py` (from STAGE-001-007) extended: walks `/plugins/` for `plugin.yaml` files; registers each as a subprocess collector.
5. **Hello-world plugin** at `runbooks/_examples/hello-subprocess-plugin/`
   - `plugin.yaml` describing the plugin.
   - `run.sh` that emits a fixed `homelab_hello_world{language="bash"}` metric and a `result` event.
   - Smoke test verifies output appears in the test's `InMemoryMetricsWriter`.
6. **Tests**
   - Manifest validation: missing fields, bad regex, timeout >= interval all fail validation.
   - Bash hello-world end-to-end metric round-trip.
   - Timeout: a plugin that sleeps past timeout is killed; failure metric emitted.
   - Malformed JSON line: logged as a warning, doesn't crash the runner; subsequent valid lines still processed.
   - Non-zero exit: failure recorded; `result` events still consumed.
   - Untrusted plugin can write metrics but **cannot** write to DB; verify by attempting (the API the runner exposes for untrusted is intentionally narrower; test asserts at the type/API level).
   - Untrusted plugin's secrets resolver only returns declared names.
