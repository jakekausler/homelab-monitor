## Overview

Stand up the operational database. Add SQLite via SQLAlchemy Core (NOT the ORM â€” see spec Â§6.1) behind a narrow `SqliteRepository` facade, configure Alembic for migrations, and produce the first migration that creates the *initial* set of tables: `users`, `sessions`, `audit_log`, `api_tokens`, **plus minimal-schema stubs** for the tables future stages will use so additive migrations stay focused: `targets`, `collectors`, `crons`, `heartbeats_state`, `alerts`, `alert_outcomes`, `runbooks`, `runbook_runs`, `secrets`, `channels`, `routing_rules`, `digest_configs`, `maintenance_windows`, `suggestions`, `tool_scorecards`.

**"Minimal-schema stub" rule:** the initial migration creates each future table with ONLY its identity, name/key columns, and `created_at`. Behavioral or feature-specific columns are added by the stage that implements that behavior. Examples: `collectors` has `id, name, config, created_at` here; `quarantined_at`, `quarantine_reason`, `consecutive_failures` are added by STAGE-001-008's `0002_collector_quarantine.py` when the quarantine logic lands. This way every additive migration ships alongside the code that uses it, while still avoiding "table doesn't exist yet" headaches in tests that span subsystems.

## Schema notes

- **All timestamps stored as ISO-8601 UTC strings** (`TEXT` columns). Rationale: sqlite has no native timestamptz; storing as ISO strings avoids the timezone-loss problem with the `TIMESTAMP` type in many SQLite setups. Repository helpers convert to/from `datetime` objects.
- **Primary keys** â€” UUIDv7 strings everywhere except `users.id` (INTEGER autoincrement is fine for a tiny user table). UUIDv7 chosen so primary keys carry creation-time ordering (helpful for the audit log and runbook runs).
- **Indexes** â€” at least the indexes named in spec Â§6.1 (alerts.fingerprint, alerts(source_tool, opened_at), runbook_runs(runbook_id, started_at), targets(kind, name), crons(host, command)).
- **Foreign keys** declared on all relations; `PRAGMA foreign_keys = ON` enforced on every connection.
