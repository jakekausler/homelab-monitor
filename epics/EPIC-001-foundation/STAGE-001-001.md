Establish the Python backend at `apps/monitor/` with strict typing, formatting, linting, testing, and a working `make verify` target. ## Decision: typing tool

The spec leaves "mypy or pyright" deferred. **Decide pyright** in this stage's Design phase unless a strong reason emerges. Rationale: pyright is faster, integrates better with VS Code, and aligns with the TS-strict frontend mindset.

## Decision: package manager

Choose between `uv`, `poetry`, and `pip + pip-tools`. **Recommend `uv`** for speed, simplicity, and lockfile reproducibility. The Design phase confirms.
